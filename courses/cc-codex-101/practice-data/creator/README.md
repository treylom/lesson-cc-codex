# practice-data / creator 안내 (cc-codex-101)

이 폴더는 Claude Code + Codex 실습용 데이터다. 업종 트랙 = **1인 콘텐츠·지식 사업자(크리에이터)**.
페르소나 '지우'는 온라인 강의·뉴스레터·컨설팅을 혼자 운영하는 1인 사업자다. 비개발자도 부담 없이 다루도록 **데이터 6종(CSV 3 + 문서 3)**으로 구성했다.

> ⚠️ 모든 수치·이름·날짜는 실습용 가상 데이터다. 실제 개인정보·매출이 아니다.

## 구성 파일

### CSV 데이터 (3개)
- `course_sales_monthly.csv` — 강의별·월별 매출 데이터(2025-07 ~ 2026-03). 강의명·플랫폼·수강신청수·환불·객단가·매출·순매출.
- `newsletter_stats.csv` — 뉴스레터 호별 발송·오픈율·클릭율·구독 증감(18호 ~ 41호).
- `student_inquiries.csv` — 수강생·고객 문의 내역. 채널·카테고리·강의명·처리상태·소요시간·우선순위.

### 문서 데이터 (3개)
- `content_plan.md` — 2분기 콘텐츠 기획 노트(채널 전략·아이디어 백로그·배운 점·고민).
- `research_memo.md` — AI 도구·1인 사업 시장 리서치 메모(시장·도구 비교·인터뷰·가격 구조).
- `consulting_schedule.md` — 3월 컨설팅·강의 운영·발행 일정 노트.

### 기타
- `README.md` — 현재 문서.

## 이 폴더로 해볼 수 있는 것
1. 폴더 탐색 질문하기 ("어떤 파일이 있고 뭐부터 보면 좋나요?").
2. CSV 열 구조·패턴 파악하기 (어느 강의가 가장 잘 팔렸나, 환불이 잦은 강의는?).
3. 뉴스레터에서 반응 좋은 콘텐츠 유형 찾기 (오픈율·클릭율 패턴).
4. 문의 로그에서 반복되는 고객 문제 정리하기.
5. 숫자(매출·구독)와 현장 맥락(기획·리서치 메모)을 연결해 보고서 초안 만들기.

## 추천 질문 예시
- "이 폴더에 어떤 파일이 있고, 무엇부터 보면 좋나요?"
- "course_sales_monthly.csv 에서 매출 상위 강의 3개와 그 흐름을 정리해주세요."
- "newsletter_stats.csv 에서 오픈율·클릭율이 높았던 호의 공통점을 찾아주세요."
- "student_inquiries.csv 에서 가장 많이 들어온 문의 유형 3가지를 뽑아주세요."
- "매출 데이터와 content_plan.md 를 연결해 다음 분기 제안 보고서 초안을 써주세요."

## 실습 팁
- 처음부터 완벽한 분석을 하려 하지 말 것 — 파일 역할 파악 → 질문 범위 좁히기 → 한 번에 하나씩.
- 문서(기획·리서치·일정 노트)는 숫자의 *이유*를 설명하는 맥락 자료로 보면 좋다 (예: AI 강의가 왜 잘 팔렸나는 research_memo에 단서가 있다).
- 보고서는 "관찰 → 해석 → 제안" 순서로 정리하면 쉽다.
- 같은 질문을 Claude Code와 Codex 양쪽에 던져보고 답이 어떻게 다른지 비교해 보는 것도 좋은 실습이다.
- AI는 비서·짝꿍이지 시험 감독이 아니다 — 결과가 마음에 안 들면 다시 시키면 된다.
