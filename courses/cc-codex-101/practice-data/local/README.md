# practice-data — local 트랙 (동네 가게/로컬)

이 폴더는 Claude Code(CC) + Codex 실습용 **샘플 데이터**다. 비개발자도 부담 없이 다루도록, 동네에서 흔히 보는 **작은 가게 한 곳의 실제 운영 흐름**을 본떠 만들었다.

## 가게 설정 — "골목카페 무지개"

- **업종**: 서울 망원동의 작은 동네 카페. 사장 "지우"가 혼자(+알바 1명) 운영한다.
- **특징**: 커피·음료를 팔면서, 주 2~3회 **원데이 공방 클래스**(드립백 만들기, 캔들, 라떼아트)도 연다. 그래서 매출이 **음료 + 클래스(수강)** 두 갈래다.
- 1인 사업자라 "혼자 다 한다" — 매출 정리·재고 발주·예약 관리·리뷰 확인·이벤트 기획까지. 이 데이터로 그 한 바퀴를 AI와 함께 돌려본다.
- ⚠️ 전부 **연습용 가짜 데이터**다. 실제 가게·고객 정보가 아니다.

## 들어 있는 파일 (6개)

### 표 데이터 (CSV 4개)
- `daily_sales_2026-05.csv` — 5월 한 달 **일별 매출**(음료/디저트/클래스/방문객/날씨 등). 패턴(요일·날씨·이벤트 효과) 찾기 좋음.
- `menu_sales.csv` — **메뉴별 판매량·매출·원가**. 무엇이 잘 팔리고, 무엇이 남는 장사인지 보기 좋음.
- `class_reservations.csv` — **공방 클래스 예약/수강 신청** 내역. 정원·결제·노쇼(no-show)·유입경로.
- `inventory_orders.csv` — **재고·발주** 내역. 무엇이 떨어져 가는지, 발주 타이밍을 보는 자료.

### 문서 데이터 (md 2개)
- `customer_reviews.md` — 네이버·인스타·방문 후기 등 **고객 리뷰 모음**. 칭찬·불만·반복되는 요청을 뽑기 좋음.
- `promo_ideas.md` — 사장이 메모해 둔 **이벤트·프로모션 기획 노트**. 매출 데이터와 묶어 우선순위를 정해보기 좋음.

## 이 폴더로 해볼 수 있는 것 (CC·Codex 공통)
1. **폴더 탐색** — "여기 어떤 파일이 있고 뭐부터 보면 좋아?" 라고 물어보기.
2. **표 읽고 요약** — 일별 매출에서 잘 되는 요일/날씨/메뉴 찾기.
3. **문서에서 패턴 뽑기** — 리뷰에서 반복되는 불만·요청 정리하기.
4. **숫자 + 맥락 연결** — "비 오는 날 매출이 왜 낮지?" 를 매출표 + 리뷰로 같이 보기.
5. **보고서/할 일 초안** — "이번 달 한 장 요약 보고서" 또는 "다음 주 발주 목록" 만들기.

## 추천 질문 예시 (그대로 복사해서 써보세요)
- "이 폴더에 어떤 파일이 있고, 사장 입장에서 뭐부터 보면 좋을지 알려줘."
- "daily_sales_2026-05.csv 에서 눈에 띄는 패턴 2~3가지를 비개발자도 알기 쉽게 정리해줘."
- "menu_sales.csv 에서 '많이 팔리지만 남는 게 적은 메뉴'와 '적게 팔려도 마진 좋은 메뉴'를 골라줘."
- "class_reservations.csv 에서 노쇼(예약하고 안 온 사람) 비율과, 노쇼가 많은 유입경로를 찾아줘."
- "customer_reviews.md 에서 손님들이 반복해서 말하는 칭찬 1개, 불만 1개를 뽑고, 불만에 대한 개선안을 제안해줘."
- "매출표와 리뷰를 함께 보고, 다음 달에 시도해볼 만한 것 3가지를 우선순위와 함께 제안해줘."

## 실습 팁
- 처음부터 완벽한 분석을 노리지 말 것 — **① 파일 역할 파악 → ② 질문 좁히기 → ③ 한 가지씩** 물어보기.
- 숫자(CSV)는 "무슨 일이 있었나", 문서(md)는 "왜 그랬나"를 설명한다. 둘을 **연결**하면 보고서가 된다.
- 보고서는 **관찰 → 해석 → 제안** 순서로 정리하면 쉽다.
- 같은 질문을 **CC와 Codex 양쪽에** 해보고, 답이 어떻게 다른지 비교해보면 좋다(모듈 2).
