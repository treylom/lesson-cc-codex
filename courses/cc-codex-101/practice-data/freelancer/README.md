# practice-data 안내 — 프리랜서/1인 컨설턴트 트랙 (cc-codex-101)

이 폴더는 Claude Code + Codex 실습용 샘플 데이터다. 비개발자도 부담 없이 다루도록 만든 **가짜(샘플) 데이터**라서, 마음껏 열어보고 AI에게 질문해도 된다.

## 시나리오
주인공은 **지우** — 혼자 일하는 1인 컨설턴트/프리랜서다. 디자인·마케팅·개발 외주를 받아 여러 고객사와 동시에 프로젝트를 굴린다. 견적도 직접 내고, 입금도 직접 챙기고, 고객 연락도 혼자 받는다. "혼자 다 하는" 사람이라 일이 마케팅·정산·고객관리·리서치·일정으로 마구 섞인다. 지우는 이 흩어진 일들을 **Claude Code와 Codex를 운영 파트너(짝꿍)** 삼아 정리하려 한다.

## 구성 파일 (6개)

### CSV 데이터 (3개) — 표 형태, 엑셀처럼 행/열로 된 자료
- `projects.csv` — 진행했거나 진행 중인 외주 프로젝트·견적·상태 목록 (20행).
- `settlements.csv` — 견적 대비 실제 입금·세금·미수금 정산 내역 (24행).
- `client_messages.csv` — 고객과 주고받은 연락(카톡·이메일·전화) 로그 (26행).

### 문서 데이터 (3개) — 글로 된 자료
- `proposal_notes.md` — 제안서·포트폴리오용 메모와 단가/조건 기준.
- `weekly_notes.md` — 지우가 주마다 적은 업무 일지(8주치).
- `README.md` — 지금 이 문서.

## 파일끼리 어떻게 연결되나
세 CSV는 **프로젝트 ID(P001~)** 로 서로 이어진다.
- `projects.csv` 에서 어떤 일을 얼마에 맡았는지 본다.
- `settlements.csv` 에서 그 일의 돈이 실제로 들어왔는지(미수금은 없는지) 본다.
- `client_messages.csv` 에서 그 고객과 무슨 대화가 오갔는지 본다.
- `weekly_notes.md` 는 숫자 뒤의 *사정*(왜 늦었나, 왜 추가비를 못 받았나)을 설명한다.

## 이 폴더로 해볼 수 있는 것 (cc101 / codex101 실습)
1. **폴더 탐색** — "여기 무슨 파일들이 있고 뭐부터 보면 돼?"
2. **패턴 찾기** — "미수금이 가장 많은 고객 3곳은?", "견적보다 일이 늘어난(추가 작업) 프로젝트 찾아줘"
3. **요약** — "이번 분기에 어떤 업종 외주가 제일 돈이 됐어?"
4. **검색** — "수정 요청이 5번 넘은 프로젝트만 골라줘"
5. **보고서 초안** — "이 데이터로 '나의 상반기 사업 리뷰' 초안 만들어줘"
6. **숫자와 맥락 잇기** — settlements 의 미수금을 weekly_notes 의 사정과 연결해 원인 정리

## 추천 질문 예시
- "이 폴더에 어떤 파일이 있고, 1인 사업자 입장에서 뭐부터 보면 좋을까?"
- "projects.csv 에서 눈에 띄는 패턴 2~3가지를 찾아줘."
- "settlements.csv 기준으로 아직 못 받은 돈(미수금)이 총 얼마이고, 어느 고객 건이야?"
- "client_messages.csv 에서 자주 반복되는 고객 요청 유형을 정리해줘."
- "weekly_notes.md 를 읽고, 내가 반복해서 겪는 어려움 3가지를 짚어줘."
- "세 CSV를 P001 같은 프로젝트 ID로 연결해서, '문제가 됐던 프로젝트' 보고서 초안을 써줘."

## 실습 팁
- 처음부터 완벽한 분석을 노리지 말고 **파일 역할 파악 → 질문 범위 좁히기** 순서로.
- CSV는 "숫자", 문서(md)는 그 숫자의 "이유"라고 생각하면 둘을 연결하기 쉽다.
- 보고서는 **관찰 → 해석 → 다음에 할 일** 3단으로 정리하면 쉽다.
- AI는 비서·짝꿍이지 시험 감독이 아니다 — 틀린 답이 나오면 다시, 더 구체적으로 물어보면 된다.
- ⚠️ 모두 연습용 가짜 데이터다. 실제 고객 정보·계약서는 절대 이 폴더에 넣지 말 것.
