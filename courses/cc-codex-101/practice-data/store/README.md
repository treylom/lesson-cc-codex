# practice-data (store 트랙) — 온라인 스토어 셀러

이 폴더는 Claude Code + Codex 실습용 **가상 데이터**다. 학습 시나리오 속 주인공 **지우**는 혼자 스마트스토어·쿠팡에서 생활/수납 용품을 파는 **1인 온라인 셀러**다. 마케팅·고객응대·재고·정산을 전부 혼자 한다. 이 데이터로 "AI와 함께 가게 운영을 끝내는" 연습을 한다.

> 모든 수치·이름·주문번호는 실습용으로 지어낸 것이다. 실제 정보가 아니다.

## 구성 파일 (6종)

### 표 데이터 (CSV 3개)
- `channel_sales_monthly.csv` — 최근 12개월(2025-07 ~ 2026-06) 채널별(스마트스토어·쿠팡·자사몰) 월매출·주문수·광고비·반품수.
- `product_sales_inventory.csv` — 상품 15종의 판매가·원가·판매수·재고·재고회전·리뷰평점.
- `customer_inquiries.csv` — 6월 고객 문의 내역(채널·상품·문의유형·처리상태·소요시간).

### 문서 데이터 (md 2개)
- `product_reviews.md` — 주요 상품 리뷰·고객 피드백(긍정/부정 키워드 정리).
- `ad_promotion_log.md` — 광고 운영·기획전 성과·효율 나쁜 상품 검토.

### 기타
- `README.md` — 현재 문서.

## 이 폴더로 해볼 수 있는 것
1. 폴더에 무슨 파일이 있는지 AI에게 묻고 첫 길잡이 받기.
2. CSV 열 구조·매출 패턴 파악하기 (계절성·채널별 차이).
3. 리뷰·문의에서 반복되는 불만 키워드 찾기.
4. 매출 숫자와 광고/리뷰 맥락을 연결해 "왜?"에 답하기.
5. 재고·광고 액션을 정리한 보고서 초안 만들기.

## 추천 질문 예시
- "이 폴더에 어떤 파일이 있고, 뭐부터 보면 좋을까요?"
- "channel_sales_monthly.csv 에서 눈에 띄는 패턴 2~3가지를 찾아주세요."
- "재고가 부족한 상품과 과다한 상품을 각각 정리해주세요." (product_sales_inventory.csv)
- "6월 고객 문의에서 가장 많은 유형이 뭔지 세어주세요."
- "평점이 낮은 P014 식탁매트가 왜 안 팔리는지 리뷰로 설명해주세요."
- "매출 CSV와 광고 로그를 함께 보고 11월 매출이 왜 튀었는지 설명해주세요."

## 실습 팁
- 처음부터 완벽한 분석을 노리지 말 것 — **파일 역할 파악 → 질문 좁히기** 순서.
- CSV는 숫자, 리뷰·광고 로그는 그 숫자의 *이유*다. 둘을 연결하면 보고서가 설득력 있어진다.
- 보고서는 "관찰 → 해석 → 제안" 순으로 정리하면 비개발자도 쉽게 만든다.
- 같은 질문을 Claude Code와 Codex 양쪽에 던져보고 답이 어떻게 다른지 비교해보자.
