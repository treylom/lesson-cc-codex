# TOOL-GUIDE-CC — Claude Code 빠른 참조 (치트시트)

> **지우의 상황**: 1인 콘텐츠 사업자. 온라인 강의·뉴스레터·컨설팅을 혼자 운영한다. 터미널은 처음이지만 엑셀·메신저·문서는 익숙하다. Claude Code(이하 CC)를 막 설치했거나 설치하려는 상태.

>
> **읽는 법**: 처음엔 0~B절만 쭉 읽어도 충분합니다. 외울 필요 없어요. 나머지는 막힐 때 그 절만 펼쳐 보세요.
>
> **기본 약속**: "Claude Code는 시험 감독이 아니라 짝꿍이다. 명령어를 외우는 게 목표가 아니라, AI와 함께 내 사업 일을 한 사이클 끝내는 것이 목표다." 실수해도 다시 시키면 됩니다.
>
> **대원칙(forward-utility = 앞으로 계속 쓰는 것만 담는다)**: 여기 담은 기능은 전부 **이후 파트에서 다시 쓰입니다.** Part 2(Codex), Part 3(지식관리·옵시디언), Part 4+(여러 AI가 함께 일하는 멀티에이전트)로 그대로 이어집니다. "있으니까 한번 보여주는" 기능은 일부러 뺐습니다. 각 항목 끝의 🎯 표시가 "이게 어느 실습/파트에서 다시 쓰이나"를 한 줄로 알려줍니다.

## 0. 30초 큰 그림 — Claude Code가 대체 뭐죠?

명령어를 외우기 전에, 머릿속 그림 하나만 그려 둡시다.

```
[ 당신(지우) ]  ←→  [ Claude Code ]  ←→  [ 당신 컴퓨터의 폴더·파일 ]
   질문/지시          읽고·고치고·만든다       매출 CSV, 회의록, 강의 원고...
```

- **Claude Code = "내 폴더 안에서 일하는 AI 비서".** ChatGPT 같은 채팅창과 가장 큰 차이는 **당신 컴퓨터의 실제 파일을 직접 읽고, 분석하고, 새 파일을 만들어준다**는 점입니다. 복사-붙여넣기 왕복이 없어요.
- **터미널(terminal) = AI 비서와 대화하는 창구.** 검은 화면이 무섭게 느껴질 수 있지만, 사실은 "AI에게 말 거는 채팅방"입니다. Mac은 `터미널` 앱, Windows는 `PowerShell` 또는 WSL(우분투) 창이에요.
- **입력 방식은 딱 두 가지 (이게 핵심!)**:
  1. **터미널 명령어** — `claude`, `claude --help`처럼 **AI를 켜고 끄거나 옵션을 거는** 명령. 터미널 일반 프롬프트(`$`)에서 칩니다.
  2. **슬래시(/) 명령어** — `claude`를 켜서 **AI와 대화 중일 때** `/compact`, `/init`처럼 `/`로 시작하는 지시. AI 채팅창 **안에서** 칩니다.

> 💡 **자주 헷갈리는 지점 미리 짚기**: `/compact`를 터미널에서 `claude /compact`라고 치는 게 **아니라**, `claude`를 먼저 켜서 대화하는 상태에서 `/compact`를 칩니다. 이거 하나만 구분해도 절반은 끝입니다. (C절·I절에서 다시 강조.)

### 안심하고 시작하세요

- **명령어를 잘못 쳐도 컴퓨터는 안 망가집니다.** 대부분 "그런 명령 없는데요?" 같은 안내만 뜨고 끝이에요.
- **파일을 읽힌다고 인터넷에 올라가는 게 아닙니다.** 내 컴퓨터 안, 내 세션 안에서만 봅니다.
- **AI가 엉뚱한 답을 해도 괜찮습니다.** 다시 시키면 돼요. AI는 비서이지 시험 감독이 아닙니다.
- **한 번에 하나씩**만 하면 됩니다. 이 문서도 위에서 아래로 한 칸씩 따라오면 돼요.

🎯 이 "AI ↔ 내 파일" 그림은 Part 2 Codex에서도, Part 3 옵시디언에서도 **똑같이** 적용됩니다. 도구만 바뀔 뿐 뼈대는 그대로예요.

## A. 처음 5분 — 맨바닥에서 설치·로그인·첫 실행

> 새 컴퓨터를 가정합니다. 아무것도 안 깔려 있어도 괜찮아요. 순서대로만 따라오세요.

### A0. 준비물 — Node.js (한 번만, 이미 있으면 건너뛰기)

CC는 Node.js(노드제이에스, 자바스크립트 실행기)라는 토대 위에서 돕니다. 설치돼 있는지 먼저 확인하세요.

```bash
node --version
```

- 숫자(예: `v20.11.0`)가 나오면 → 준비 완료, A1로.
- `command not found`가 나오면 → Node.js를 먼저 설치해야 합니다.
  - **Mac**: [nodejs.org](https://nodejs.org)에서 LTS 버전 설치, 또는 Homebrew 사용자는 `brew install node`
  - **Windows**: [nodejs.org](https://nodejs.org)에서 LTS 설치 파일(.msi) 실행 (또는 WSL/우분투 안에서 설치)

**무엇인가**: Node.js = CC라는 프로그램이 돌아가기 위한 "엔진". 자동차로 치면 휘발유 같은 토대입니다. 한 번만 깔면 됩니다.

### A1. Claude Code 설치 (한 번만)

```bash
npm install -g @anthropic-ai/claude-code
# 또는 (Mac, Homebrew 사용자)
brew install --cask claude-code
```

**무엇인가**: "Claude Code"라는 프로그램을 내 컴퓨터에 내려받아 까는 것. 앱스토어에서 앱 하나 까는 거랑 똑같다고 보면 됩니다. `npm install -g`의 `-g`는 "이 컴퓨터 어디서나 쓸 수 있게 전역(global)으로 깔아라"라는 뜻이에요.

**설치 확인**:

```bash
claude --version
```

- `2.1.183 (Claude Code)`처럼 버전 숫자가 뜨면 성공!
- `command not found`가 뜨면: 터미널을 완전히 껐다 다시 켜 보세요(설치 후 새 창에서 인식되는 경우가 많아요). 그래도 안 되면 `npm`(=Node.js)이 깔려 있는지 A0 확인.

🎯 **cc-01(설치와 첫 실행)** 실습에서 처음부터 함께 깝니다. 여기서 막히면 cc-01 트러블슈팅 절을 보세요. Part 2 `codex-01`도 **똑같은 흐름**(`npm install -g @openai/codex`)이라, 한 번 해 보면 Codex 설치는 복습이 됩니다.

### A2. 로그인

```bash
claude
# 첫 실행이면 로그인 안내가 뜹니다. (또는 세션 안에서) /login
```

**무엇인가**: CC에게 "나는 누구다"를 알려주는 과정. Claude.ai에 가입할 때 쓰던 계정으로 로그인합니다. 보통 **브라우저 창이 자동으로 열려** 로그인·승인하면, "터미널로 돌아가세요" 안내가 나오고 끝.

> 💡 **안심 포인트**: 로그인한다고 내 파일이 어디 올라가는 게 아니에요. "이 사람이 쓸 권한이 있구나"만 확인하는 겁니다.

> 💡 **요금제 한마디**: CC는 ① **구독제(Pro/Max)** 와 ② **API 키**(쓴 만큼 과금) 두 방식이 있어요. 1인 사업자는 보통 매달 정액인 **구독제**가 마음 편합니다. 실습을 넉넉히 돌리려면 사용량 제한이 적은 상위 요금제가 편해요. 정확한 비용 관리는 cc-06에서.

🎯 한 번 로그인하면 이후 모든 파트에서 유지됩니다. Codex(`codex login`)도 "브라우저로 계정 승인"이라는 같은 패턴.

### A3. 첫 대화 — "와우!" 모먼트

두 가지 방법이 있어요. 둘 다 해보세요.

**방법 1 — 한 번에 물어보고 끝내기 (one-shot)**

```bash
claude -p "안녕! 나는 1인 콘텐츠 사업자야. 다음 달 온라인 강의 계획을 잡아줄 수 있어?"
```

`-p`(print, 출력)는 "딱 한 번 답하고 종료"하는 방식입니다. 빠른 질문에 좋아요.

**방법 2 — 대화방을 열어 두고 계속 이야기하기 (권장)**

```bash
claude
```

입력창(`>`)이 뜨면, 그냥 **한국어로** 말을 걸면 됩니다(영어도, 완벽한 문법도 필요 없어요):

```
> 안녕! 나는 1인 콘텐츠 사업자야. 온라인 강의·뉴스레터를 혼자 운영해.
  이 폴더의 파일들을 보고, 지금 할 수 있는 가장 현실적인 일 3가지를 알려줄래?
```

**체험 포인트**: CC가 답하는 동안 **화면에 글자가 한 줄씩 주르륵 떨어집니다.** 이게 "AI가 생각하며 일하는 모습"이에요. 답이 끝나면 "그럼 첫 강의 제목 3개만 뽑아줘"처럼 **대화를 계속** 이어갈 수 있습니다.

**대화방에서 나오기**: `/exit` 입력 또는 `Ctrl + C`를 두 번.

🎯 **cc-02(에이전트 루프 체험)** — 지금 본 "내가 묻고 → AI가 도구를 써서 일하고 → 결과를 돌려주는" 한 바퀴를 **에이전트 루프(agent loop)** 라고 부릅니다. 이 흐름이 모든 실습의 기본 동작이며, Part 4 멀티에이전트에서 "여러 AI에게 일을 거는" 방식의 원형입니다.

## B. 매일 쓰는 핵심 5 — 이것만 알면 80%

> CC는 터미널에서 `claude`로 **세션을 켠 다음, 그 안에서 한국어로 말하거나 `/`로 시작하는 슬래시 명령을 입력**합니다. 엑셀을 켠 뒤 그 안에서 작업하는 것과 같아요.
>
> 각 항목: **이럴 때 써요 / 예시 / 🎯 어디서 또 나와요(forward-utility)**

### 1️⃣ 폴더 안에서 `claude` 켜고 파일 읽히기

```bash
cd ~/Documents/내사업      # 내 자료가 있는 폴더로 이동
claude                      # 그 폴더를 무대로 대화 시작
```

대화방 안에서 말로 시키면 됩니다. 파일명을 문장에 적거나 `@`로 콕 집어주면 CC가 알아서 읽어요:

```
> @sales_data.csv 이 매출 데이터에서 눈에 띄는 패턴 3가지를,
  비개발자도 이해하게, 실제 숫자를 인용해서 설명해줄래?
```

**이럴 때 써요**: 엑셀·CSV·텍스트·마크다운·회의록을 CC에게 읽혀 요약·분석. `claude`는 **지금 터미널이 있는 폴더**를 무대로 삼으니, 자료가 있는 폴더로 먼저 이동한 뒤 켜는 게 좋아요.

**1인 사업자 실전 상황(지우)**:
- 월매출 CSV → "어느 채널(상품)이 효자였어?"
- 고객 문의 모음 → "제일 많이 나온 불만 3개만 뽑아줄래?"
- 회의록 여러 개 → "반복되는 운영 이슈가 뭐야?"

> 💡 **할루시네이션(AI가 지어내기) 방지 팁**: AI가 추측만 하는 것 같으면 "**CSV의 실제 값**으로 근거를 보여줘"라고 다시 요청하세요.

🎯 cc-03(파일 다루기)·cc-04(좋은 프롬프트)에서 집중 실습. Part 2 Codex도 **똑같이** 파일명을 적으면 읽어요. **Part 3 옵시디언에서 노트 더미를 통째로 분석하는 것의 첫 단추**이고, Part 4에선 AI 팀에게 "이 파일 읽고 분석해"를 시키는 기본 메커니즘이 바로 이거예요.

### 2️⃣ Plan(계획) 모드 — 먼저 계획부터, 실행은 그다음

복잡한 일은 곧장 시키지 말고, **계획을 먼저 보여달라고** 하는 게 안전해요.

- 대화방에서 키보드 **`Shift + Tab`** 을 누르면 모드가 `일반 → 자동승인 → Plan`으로 바뀝니다. 화면 위에 `plan mode` 표시가 뜨면 켜진 거예요. (켤 때부터 옵션으로 주려면 `claude --permission-mode plan`.)
- 이 상태에서 일을 시키면 CC가 **"①이렇게 ②저렇게 하겠습니다"** 계획만 먼저 보여줍니다. 내가 "좋아, 진행해" 하면 그제서야 실행해요.

```
(Shift+Tab → plan mode 표시 확인 후)
> 다음 달 온라인 강의 3부작 구성을 짜 줘. 콘셉트→목차→회차별 핵심→예시 순서로,
  먼저 계획만 보여주고 내가 OK 하면 진행해.
```

**왜 좋아요?**: 1인 사업자한테 제일 중요한 건 시간이에요. "막 다 해줘" 했다가 결과가 산으로 가면 처음부터 다시예요. **계획 보고 → 내가 확인 → 그다음 실행**이 결국 제일 빠릅니다. 인테리어 업체한테 "알아서 다 해주세요"가 아니라 **도면 먼저 보고 OK** 하는 것과 같아요.

🎯 cc-04(워크플로우 + Plan)에서 자세히. Codex에도 같은 개념의 계획 모드가 있어요(Part 2). Part 4 멀티에이전트에선 "팀 전체가 먼저 플랜을 짠다"의 기초가 됩니다.

### 3️⃣ CLAUDE.md — 내 사업 규칙서 (한 번 쓰면 계속 적용)

대화방에서:

```
> /init
```

**이게 뭔가요?**: `/init`을 치면 CC가 **지금 폴더를 둘러보고 `CLAUDE.md`라는 규칙서 파일 초안을 자동으로** 만들어줍니다. 이건 "내 프로젝트를 할 땐 이 규칙을 지켜줘"를 적어두는 메모예요. 한 번 적어두면, 그 폴더에서 `claude`를 켤 때마다 CC가 **자동으로 읽고 따릅니다.** 신입 비서에게 주는 "업무 매뉴얼"인 셈이죠.

쉬운 예시 (지우의 콘텐츠 사업용):

```markdown
# 내 콘텐츠 사업 규칙서

## 내 역할 / Claude의 역할
나는 온라인 강의·뉴스레터를 운영하는 1인 사업자다. 나는 비개발자다.
Claude는 매출 분석 · 고객 피드백 정리 · 강의 스크립트 검수를 돕는다.

## 보고 스타일
- 결론을 맨 위에, 그다음 근거. 표와 짧은 불릿 위주.
- 어려운 영어/기술 용어는 첫 등장 시 괄호로 쉬운 풀이.

## 금지
- 실제 고객·수강생 실명 노출 금지
- 확인 없이 기존 강의 원고를 통째로 바꾸지 말 것 (먼저 제안만)

## 완료 기준
- 보고서는 "초급자도 한눈에 이해되는가"를 통과해야 완료
```

**효과**: 이 파일이 있으면, 다음부터는 "이번 달 매출 정리해줘"만 해도 CC가 알아서 **결론 먼저·표 위주·실명 가림·쉬운 말로** 정리해줍니다. 매번 "나는 비개발자야, 쉽게 설명해줘"를 반복할 필요가 없어요. **같은 말을 두 번 안 하게 해주는 비서 수첩**입니다.

> 💡 **적용 범위 살짝만**: 폴더 안 `CLAUDE.md` = 그 사업 폴더에서만 적용(프로젝트 규칙). 홈 전체 규칙 = 모든 곳에 공통(예: "항상 한국어로"). 계층·템플릿 상세는 cc-05에서.

🎯 **이게 강의 전체에서 가장 중요한 다리예요** → cc-05(전체)에서 배우고:
- **Part 2 Codex**: 똑같은 역할의 규칙서를 `AGENTS.md`라는 이름으로 만듭니다(이름만 다르고 역할은 같음).
- **Part 3 지식관리·옵시디언**: "내 노트를 이렇게 정리해줘" 규칙도 같은 방식으로 문서화.
- **Part 4+ 멀티에이전트**: 여러 AI 팀원이 **이 규칙서를 공통으로 읽고** 같은 기준으로 움직입니다.

### 4️⃣ `/compact` — 길어진 대화 정리하기 (느려짐·비용 방지)

```
> /compact
```

**이게 뭔가요?**: 하루 종일 "매출 분석 좀… 아 고객 피드백도… 강의 기획도…" 길게 대화하면 CC가 기억할 게 너무 많아져요. `/compact`는 그 긴 대화를 **핵심만 남기고 요약**합니다. 화면은 깔끔해지지만 **하던 작업 맥락(지금 무슨 작업 중인지, 결론이 뭐였는지)은 그대로 유지**됩니다.

**왜 필요해요?**: 대화가 길수록 ① 답이 느려지고 ② 비용이 올라가요. 적당할 때 한 번씩 정리하면 다시 가뿐해집니다.

**언제 누르면 좋아요?**:
- 큰 작업 하나가 끝났을 때 ("매출 분석 끝! 정리하고 마케팅 가자")
- **30~45분마다**, 또는 중간 목표 달성 후 한 번씩
- 화면 아래 컨텍스트 게이지가 가득 차기 **전에** 미리 (현재 상태는 `/context`로 배터리 % 보듯 확인)

🎯 cc-06(세션 관리 + 비용)에서. Codex에도 똑같은 `/compact`(Part 2). Part 4에서 여러 AI가 동시에 일할 때 각자의 기억을 관리하는 단계로 확장됩니다.

### 5️⃣ `/clear` — 완전히 새 출발 (주제가 바뀔 때)

```
> /clear
```

`/compact`가 "요약해서 이어가기"라면, `/clear`는 **"이전 대화를 통째로 비우고 백지에서 시작"**입니다. **무관한 다른 일로 넘어갈 때** 쓰세요 (예: "매출 분석"을 끝내고 전혀 다른 "강의 원고 교정"을 시작).

> ⚠️ **`/clear`는 되돌릴 수 없어요.** 중요한 결과는 미리 파일로 저장한 뒤에 쓰세요. **헷갈리면 `/compact`가 더 안전**합니다(맥락을 잃지 않으니까). `/compact`=정리하되 기억 유지 / `/clear`=기억 삭제.

🎯 cc-06. 작업 간 컨텍스트를 깨끗이 나누는 습관은 모든 파트 공통.

## C. 슬래시(/) 명령 빠른 표 — 대화 "중"에 내리는 지시

> ⚠️ 슬래시 명령은 **`claude`를 켜서 대화방에 들어간 뒤**, 그 안에서 칩니다. 대화방에서 `/`만 쳐 보면 쓸 수 있는 목록이 자동으로 떠요. 골라서 엔터.

| 명령 | 한 줄 용도 | 언제 배워요 | 🎯 다시 쓰는 곳 |
|------|-----------|------------|----------------|
| `/help` | 쓸 수 있는 명령·스킬 목록 보기 | 언제든 | 전 파트 |
| `/init` | CLAUDE.md 규칙서 초안 자동 생성 | cc-05 | Codex `/init` 동일 |
| `/context` | 지금 기억(컨텍스트)이 몇 % 찼나 확인 | cc-06 | 비용·세션 관리 전반 |
| `/compact` | 긴 대화 요약(기억 유지) | cc-06 | Codex·멀티에이전트 |
| `/clear` | 대화 전부 비우고 새 출발 | cc-06 | 작업 전환 시 |
| `/model` | 쓰는 모델 바꾸기(깊게 vs 빠르게) | cc-04·cc-06 | **Codex `/model`로 직결** |
| `/resume` | 과거 대화 목록에서 골라 이어가기 | cc-06 | 장기 프로젝트 |
| `/exit` | 대화방 닫고 터미널로 나가기 | 언제든 | — |

> 💡 **같은 기능, 두 입구**: 같은 일을 "켤 때 옵션으로"(터미널, B/A절) 줄 수도, "쓰던 중에 슬래시로"(C절) 바꿀 수도 있어요. 예) 모델은 켤 때 `claude --model opus`, 도중엔 `/model`. **어제 대화 이어 하기**는 켤 때 `claude -c`(또는 `--continue`), 도중엔 `/resume`. 둘 다 정답입니다.
>
> 💡 **모델(두뇌) 고르기**: 깊은 모델(예: opus)은 똑똑하지만 느리고 비싸고, 가벼운 모델(예: sonnet)은 빠르고 싸고 충분할 때가 많아요. "회의록 3줄 요약" 같은 단순 작업은 가벼운 모델, "3개월 매출·피드백 다 읽고 다음 분기 전략" 같은 복잡 작업은 깊은 모델. 정확한 모델 목록은 `/model`로 확인. 이건 Codex의 `/model`·effort(추론 강도)와 **정확히 같은 사고방식**이에요.

> 표의 명령·플래그는 CC 버전에 따라 조금씩 다를 수 있어요. 내 환경에서 실제 가능한 목록은 `/help`가 가장 정확합니다.

🎯 "대화 중 지시"라는 개념은 Codex(슬래시 또는 `$` 명령)에서도 똑같이 씁니다 — 전 파트 공통.

## D. 터미널 파일·폴더 기본기 — 길 찾기 (딱 4개)

1인 사업자라면 매출 CSV, 고객 피드백, 강의 스크립트, 마케팅 메모가 흩어져 있을 거예요. CC로 다루기 전, "지금 어디에 있고 뭐가 있는지"만 알면 됩니다.

```bash
pwd                          # 지금 내가 어느 폴더에 있나
ls                           # 이 폴더에 뭐가 있나 (목록 보기)  /  ls -la = 숨김파일까지
cd Documents/내사업           # 그 폴더로 이동
cd ..                        # 한 단계 위로 (뒤로 가기)  /  cd ~ = 홈으로 리셋
claude                       # 그 폴더를 무대로 CC 켜기
```

- **쉬운 기억법**: `pwd`=현 위치 / `ls`=둘러보기 / `cd`=이동(윈도우 탐색기 더블클릭) / `claude`=일꾼 부르기. 파인더·탐색기를 글자로 한다고 생각하면 됩니다. 길 잃으면 `cd ~`가 리셋 버튼이에요.
- **폴더 통째로 맡기기**: 폴더로 들어간 뒤 대화방에서 "이 폴더의 매출 CSV와 회의록을 전부 읽고, 지난 3개월 핵심 이슈 3개를 뽑아줄래?"

🎯 cc-03. 이 4개는 **Part 2 Codex, Part 3 옵시디언 vault(저장소) 다루기에서 그대로** 씁니다. CLI 기본기는 전 파트의 공용 발판.

## E. 좋은 프롬프트 5원칙 — "좋은 질문 = 좋은 답"

> **프롬프트(prompt) = AI에게 하는 질문·지시.** "잘 쓴다" = 화려한 말이 아니라 **의도가 명확**하다는 뜻. **친한 동료에게 일 부탁하듯** 하면 됩니다.

| # | 원칙 | 한 줄 |
|---|------|-------|
| ① | **역할을 먼저 정해준다** | "너는 내 온라인 강의 사업의 데이터 분석가야" → 톤·깊이·형식이 내 상황에 맞춰진다 |
| ② | **원하는 결과 모양을 못 박는다** | "표로 3가지만" "200자 이내로" "이메일 초안 형태로" → 다시 시키는 횟수가 확 준다 |
| ③ | **한 번에 한 덩어리** | 매출+마케팅+고객답장을 한 번에 ❌ → 하나씩. 결과가 또렷하고 고치기 쉽다 |
| ④ | **대상 파일·기준을 준다** | "sales_data.csv를 읽고" "초급자도 이해할 수준으로" (기준은 CLAUDE.md에 박아두면 매번 안 써도 됨 — E절 연결!) |
| ⑤ | **틀리면 그냥 다시 시킨다** | 첫 답이 70점이어도 정상. "이 부분 너무 길어, 절반으로 줄여줘"처럼 대화로 다듬는다 |

**예시 — ❌ 막연한 부탁 vs ✅ 또렷한 부탁**

```
❌  판매 데이터 분석해 줘.

✅  너는 내 온라인 강의 사업의 데이터 분석가야.
   sales_data.csv 를 읽고, 비개발자도 이해되게:
   1) 가장 성장한 채널
   2) 가장 매출이 떨어진 달
   3) 다음 달 한 줄 예측
   이렇게 3가지만 표로 짧게 정리해 줘.
```

🎯 cc-04(좋은 프롬프트)에서 집중. 이 습관은 **Part 2·3·4 전부에서 매 순간** 쓰는 평생 스킬입니다. Codex의 outcome-first 프롬프트(목표+성공 기준+제약)와 사실상 같아서, Part 2에서 "아, 같은 얘기네" 하고 그대로 전이됩니다. Part 4 멀티에이전트 지시문의 기본기이기도 해요.

## F. 세션 관리 — 대화가 길어졌을 때 다시 (체크리스트)

B-4·B-5에서 `/compact`·`/clear`를 봤어요. 실전 루틴으로 정리합니다.

**왜 정리가 필요한가**: CC는 대화 기록 전체를 **컨텍스트(context, 배경 정보 그릇)** 로 기억하며 일합니다. 길어지면 그릇이 차서 ① 느려지고 ② 비용이 오르고 ③ 옛날 얘기가 섞여 엉뚱해질 수 있어요.

**언제 정리하나 (체크리스트)**:
- [ ] 하나의 큰 작업이 끝났을 때 → `/compact` ("매출 분석 끝 → 정리")
- [ ] 주제가 완전히 바뀔 때 → `/clear`
- [ ] 답이 눈에 띄게 느려졌을 때 → `/compact`
- [ ] 30~45분마다, 또는 장기 프로젝트는 하루 1~2회 정리 습관
- [ ] 게이지가 차기 전에 미리 → `/context`로 % 확인

🎯 cc-06(세션 관리 + 비용·토큰). Codex도 동일한 `/compact`. Part 3·4의 며칠짜리 장기 리서치·여러 AI 동시 작업에서 각자의 기억을 관리하는 필수 습관입니다.

## G. 막혔을 때 — 미니 트러블슈팅

| 증상 | 먼저 해볼 것 |
|---|---|
| `command not found: claude` | 터미널 껐다 켜기 → 그래도 안 되면 A1 설치 재실행 |
| `command not found: npm` | Node.js가 아직 없는 거예요 → A0 (cc-01에서 함께 깝니다) |
| 로그인이 안 됨 / 자꾸 물어봄 | `Ctrl+C`로 껐다 다시 `claude`, 세션에서 `/login`, 구독 상태 확인 |
| AI가 내 파일을 못 찾음 | `ls`로 파일이 그 폴더에 진짜 있는지 확인 → `cd`로 맞는 폴더로 이동 후 다시 `claude` |
| 답이 점점 느려지고 이상해짐 | `/compact`로 압축, 무관한 일이면 `/clear` |
| 답이 너무 길고 장황함 | `/model`로 가벼운 모델, 또는 프롬프트에 "200자 이내" 같은 제약 추가 |
| AI가 엉뚱한 방향으로 감 | `Esc`로 중단 → Plan 모드(`Shift+Tab`)로 계획부터 보고 진행. 또는 "그건 아니고 ○○로 다시 해줘" |

> **마음가짐**: 어떤 명령을 잘못 쳐도 컴퓨터는 안 망가집니다. CC는 기본적으로 **파일을 바꾸기 전에 물어보는** 안전장치가 있어요. 틀려도 괜찮으니 일단 해보세요.

🎯 더 자세한 트러블슈팅은 부록 **cc-11(워크스페이스 종합·트러블슈팅)** 에 모여 있어요. 지금은 위 7개만 알면 충분합니다.

## H. 한 장 요약 — 외워두기 (헷갈릴 때)

```bash
# 한 줄로 빠르게
claude -p "지시문"               # 한 번 답하고 종료
claude                          # 대화방 열기
claude -c                       # 어제/방금 대화 이어 하기
claude --help                   # 도움말(막히면 이것부터)

# 작업 폴더에서 파일 읽히기
cd ~/Documents/내사업
claude
> @sales_data.csv 채널별 매출 추이 정리해줘

# 대화 중 자주 쓰는 슬래시
/init      ← CLAUDE.md 규칙서 만들기
/compact   ← 긴 대화 요약(이어가기)
/clear     ← 새 작업으로 리셋
/context   ← 기억 몇 % 찼나 확인
/model     ← 두뇌(모델) 바꾸기
/help      ← 쓸 수 있는 명령 보기

[역할] + [읽을 파일] + [원하는 결과·형식]
```

### 절대 헷갈리지 말 것 (3줄)
1. **`claude --무엇`** = 터미널에서 AI를 **켤 때** 거는 옵션 (A·B절)
2. **`/무엇`** = AI 대화 **중에** 내리는 지시 (C절)
3. **파일은 말로 부탁**하거나 **`@파일명`** 으로 집어주면 AI가 알아서 읽는다 (B·D절)

## I. 도구 간 다리 — 지금 배우는 게 어디로 이어지나 (forward-utility 지도)

이 강의의 설계는 **"앞에서 익힌 걸 뒤에서 다시 쓴다"**입니다. CC에서 배운 개념이 어떻게 건너가는지 한 장으로:

| CC에서 배우는 것 (Part 1) | → Codex (Part 2) | → 지식관리·옵시디언 (Part 3) | → 멀티에이전트 (Part 4+) |
|---|---|---|---|
| `claude` 세션·말 걸기 | `codex` 세션·말 걸기 (같은 흐름) | 노트 vault 위에서 분석 | 여러 AI에게 동시에 일 걸기 |
| **CLAUDE.md 규칙서** | **AGENTS.md** (이름만 다름) | "노트 정리 규칙"도 문서로 | 팀원 전부가 규칙서 공유 |
| Plan(계획) 모드 | 계획 모드 (동일 개념) | 정리 작업 단계 설계 | 팀 전체 플랜 |
| `/compact`·`/clear` 세션 관리 | `/compact` (동일) | 긴 정리 세션 관리 | AI별 기억 관리 |
| `/model` 모델·깊이 조절 | `/model`·effort (동일 개념) | — | 작업별 모델 배분 |
| 파일 읽히기·좋은 프롬프트 | 동일 | 노트 통째 분석의 첫 단추 | 에이전트가 받는 기본 지시 |
| **codex plugin** (이후 소개) | **CC 안에서 Codex 같이 쓰기** → Part 2로 직결 | — | 두 도구 혼용 워크플로우 |

- **가장 굵은 다리 = `CLAUDE.md ↔ AGENTS.md`.** "프로젝트 규칙을 파일로 박아 두면 매번 자동 적용"이라는 한 가지 아이디어가, 도구가 바뀌고(Codex) 영역이 바뀌어도(옵시디언) 팀이 늘어도(멀티에이전트) 그대로 통합니다. **여기만 확실히 잡으면 나머지가 술술 풀립니다.**
- **codex plugin 다리**: CC 안에서 Codex를 쉽게 함께 쓰는 공식 플러그인은 Part 1 후반에서 소개되어 **그대로 Part 2 Codex 트랙으로 이어집니다.** (정확한 설치·기능은 시연 직전 공식 docs로 확인.)

## J. 마지막 조언 — 치트시트 너머 (지우에게)

처음엔 CC의 답이 완벽하지 않을 수 있어요. 그런데 며칠 쓰다 보면 이런 작은 깨달음이 쌓입니다.
- "아, 이렇게 물으니 더 정확하네" (→ E절 프롬프트 감각)
- "이 규칙은 CLAUDE.md에 박아 두자" (→ E절)
- "큰 작업 끝나면 `/compact` 하는 습관이 편하네" (→ B·F절)

이런 깨달음이 모여 **당신만의 Claude Code 사용 패턴**이 됩니다. 그게 이 강의의 진짜 목표예요.

> **당신의 목표는 "터미널 고수"가 아닙니다.** **"AI 짝꿍과 함께 내 사업을 한 단계 끌어올리는 사람"** 입니다. 명령어는 거들 뿐이에요. AI는 시험 감독이 아니라 비서이자 짝꿍이라, 실수해도 다시 시키면 됩니다.

자, 이제 슬라이드의 **`cc-01`(설치와 첫 실행)** 부터 따라해 봅시다. 헷갈리면 이 문서를 다시 펼치면 됩니다.
