# cc-codex-101 코스 컨텍스트

## 학습 시나리오
당신은 중견기업 **운영기획팀의 비개발자 실무자 박서연**이다. 엑셀·메신저·문서 작업엔 익숙하지만 터미널과 코드는 낯설다. 팀장이 "AI 도구로 업무를 좀 더 빠르게 해보라"고 했고, 마침 **Claude Code(CC)**와 **Codex** 두 가지를 배우게 됐다. 이 코스의 목표는 둘 중 무엇이 더 좋은지 가리는 게 아니라, **두 도구로 실제 업무 한 사이클(자료 읽기 → 분석 → 보고서 초안)을 직접 끝내보고, 언제 무엇을 쓰면 좋을지 감을 잡는 것**이다.

## 현재 미션
1. `practice-data/` 의 매출 데이터와 회의록을 AI에게 읽히고 요약시킨다.
2. Claude Code로 폴더 탐색·파일 분석·보고서 초안을 만든다(모듈 1).
3. Codex를 설치하고 같은 종류의 작업을 해보며, **effort·모델 선택에 따라 응답이 어떻게 달라지는지** 체감한다(모듈 2).
4. "CC와 Codex는 무엇이 같고 무엇이 다른가"를 한 문장으로 말할 수 있게 된다.

## 성공 기준
- `practice-data/sales_data.csv` 에서 눈에 띄는 패턴 2개 이상을 AI와 함께 찾는다.
- `practice-data/meeting_notes.md` 에서 반복되는 운영 이슈를 정리한다.
- Claude Code로 보고서 초안 1개를 만든다.
- Codex를 설치·실행하고, `/model`·effort 를 바꿔 **같은 요청의 응답 차이**를 직접 본다.
- CC ↔ Codex 의 공통점·차이점을 스스로 1문장으로 요약한다.

## 학습자 프로필
- 비개발자 또는 개발 경험 거의 없는 실무자.
- 터미널·명령어가 낯설고 살짝 무서울 수 있음.
- 엑셀·메신저·문서 작성은 익숙함.
- 목표 = "코드를 잘 짜는 사람"이 아니라 "**AI와 함께 실무를 끝내는 사람**".

## 코스 진행 원칙
- 모든 설명은 비개발자 눈높이로. 전문 용어는 첫 등장 시 쉬운 비유와 함께.
- 한 번에 하나씩만 실행. 완벽함보다 "직접 해봤다"에 초점.
- AI는 비서이자 짝꿍이지 시험 감독이 아니다 — 실수해도 다시 시키면 된다.
- ⚠️ **실습은 이미 배운 범위만.** 아직 안 배운(미리 세팅된) 에이전트/기능으로 건너뛰지 않는다.

## 실습 데이터 설명
- `practice-data/sales_data.csv`: 2025년 12개월 채널별 매출 데이터.
- `practice-data/meeting_notes.md`: 운영·고객지원·분기리뷰 회의록 3건.
- `practice-data/README.md`: 데이터 의미와 실습 팁.

## 모듈 구성
- **모듈 1 (Claude Code 실습)**: 1-1 첫 대화·역할 부여(와우 모먼트) / 1-2 폴더 탐색·파일 분석·보고서 초안.
- **모듈 2 (Codex 실습 + 비교)**: 2-1 Codex 설치·첫 실행·`/model`·effort 응답 차이 / 2-2 AGENTS.md·MCP·샌드박스 + CC↔Codex 같은 것·다른 것.

## 코스의 마지막 장면
박서연은 (1) 데이터를 AI에게 읽히고 질문하고, (2) Claude Code로 보고서 초안을 만들고, (3) Codex를 설치·조절(effort·모델)해 같은 일을 해보고, (4) "두 도구는 뼈대가 같고 제품·모델만 다르다"를 스스로 설명할 수 있게 된다.
이 코스의 목적은 "도구를 안다"가 아니라 **"두 AI 도구로 실제 업무를 한 사이클 끝낸다"** 이다.
