# cc-08: 플러그인 · 스킬 · codex plugin — CC의 능력을 확장하고 두 도구를 한 자리에서

> **레슨 목표**: Claude Code에 플러그인과 스킬을 추가해 기능을 확장하고, 자주 하는 반복 작업을 한 줄 명령으로 줄이며, codex plugin으로 CC 안에서 Codex까지 함께 불러 두 AI 파트너를 한 무대에서 부린다.
> **예상 시간**: 70분 (개념 도입 10분 + 실습 1 플러그인 15분 + 실습 2 스킬 만들기 25분 + 실습 3 codex plugin 20분)
> **사전 준비**: 터미널 1개 · Claude Code 설치·로그인 완료(cc-01~cc-07) · `practice-data/creator/` 폴더 접근 가능
> **진입 커맨드**: `/cc-08`

---

## 📚 학습 목표 — 4가지 와우 모먼트

1. **"앱처럼 기능이 추가된다"** — 플러그인 한 줄로 CC가 못 하던 일(심층 리서치, 기획서 자동 생성)을 하게 만든다.
2. **"자주 하는 일을 레시피로 저장한다"** — SKILL.md를 만들면 `/monthly-review` 한 줄로 지우의 매출·뉴스레터 요약이 자동으로.
3. **"CC 안에서 Codex까지 부른다"** — codex plugin으로 창을 옮기지 않고 두 AI 파트너를 한 자리에서 쓴다.
4. **"같은 데이터, 두 엔진 비교"** — `course_sales_monthly.csv` 분석을 CC와 Codex에 각각 시켜보며 "언제 뭘 쓸지" 직접 체감한다.

## 🎯 개념 도입 — 비유로 이해하는 플러그인·스킬·codex plugin

### 플러그인 — "CC에 앱을 추가한다"

스마트폰을 처음 샀을 때를 떠올려보자. 전화·문자는 기본으로 되지만, 카카오톡을 깔아야 채팅이 되고, 네이버 지도를 깔아야 길 찾기가 된다. **Claude Code도 똑같다.** 기본 CC는 파일 읽기·분석·보고서 초안까지는 잘 하지만, "7단계 심층 리서치", "인터뷰 내용으로 기획서 4종 자동 생성" 같은 전문 기능은 **플러그인(plugin)**을 설치해야 생긴다.

CC 안에서 `/plugin` 명령을 열면 **공식 마켓플레이스(Discover 탭)**가 나온다. Notion·Slack·GitHub 연결부터 비개발자에게 유용한 리서치 에이전트까지 이미 만들어진 플러그인들이 줄 서 있다.

| 플러그인 예시 | 어떤 일을 해주나 |
|---|---|
| `deep-research` | 7단계 멀티에이전트 심층 리서치 |
| `show-me-the-prd` | 인터뷰 내용 → 4종 기획서 자동 생성 |
| `바르다 깃선생` | 비개발자용 Git 온보딩 |
| `docs-guide` | 공식 문서 기반 정확한 답변 |

플러그인을 설치하면 CC 대화창 안에서 `/deep-research`, `/show-me-the-prd` 같은 **슬래시 명령어**를 쓸 수 있게 된다. 앱처럼 한 번 깔아두면 그 다음부터는 쭉 쓸 수 있다.

### 스킬(Skill) — "자주 하는 작업을 레시피로 저장한다"

레시피 앱을 쓰면 "파스타 만들기"를 검색해서 따라 만들 수 있다. 그런데 내 입맛에 맞게 조금 달리 만드는 방법을 따로 저장해두면, 다음에는 "내 파스타 레시피"를 바로 꺼내 쓸 수 있다. **스킬(Skill)은 "나만의 작업 레시피"다.** `SKILL.md`라는 파일에 작업 단계를 적어두면, 그 스킬 이름을 `/monthly-review`처럼 슬래시로 부를 때마다 같은 과정을 정확히 반복해준다.

지우의 경우를 생각해보자. 매달 초에 `course_sales_monthly.csv`를 열어 강의별 매출을 정리하고, `newsletter_stats.csv`와 비교해 "이번 달 잘된 것·아쉬운 것"을 정리하는 게 반복 업무다. 이 과정을 `/monthly-review` 스킬로 만들어두면, 매달 그 한 줄만 치면 된다.

| | 일반 대화 | 스킬 |
|---|---|---|
| 실행 방법 | 매번 긴 설명을 다시 씀 | `/스킬이름` 한 줄 |
| 결과 일관성 | 매번 조금씩 다름 | 항상 같은 방식으로 |
| 팀·미래의 나와 공유 | 말로 설명해야 함 | 파일로 저장 → 재사용 |

### codex plugin — "작업실에 두 번째 짝꿍을 초대한다"

지우에게는 CC와 Codex 두 AI 파트너가 있다. 보통은 CC 창, Codex 창을 따로 열어야 하는데, **codex plugin**을 설치하면 CC 대화창 안에서 "Codex야, 이것 좀 봐줘"라고 부를 수 있게 된다. 작업실(CC)에 두 번째 짝꿍(Codex)을 초대하는 것이다.

> 💡 **한 줄 정리**: 플러그인 = 앱스토어에서 받은 앱 / 스킬 = 내가 직접 만든 단축키 레시피 / codex plugin = 작업실에 두 번째 짝꿍 초대.

### 플러그인 vs 스킬 한눈 비교

| 구분 | 플러그인(Plugin) | 스킬(Skill / SKILL.md) |
|------|-----------------|------------------------|
| 만든 곳 | 다른 사람이 만든 기능 꾸러미 | **내가** 직접 만드는 맞춤 레시피 |
| 설치 방법 | `/plugin marketplace add` → `/plugin install` | `.claude/skills/` 폴더에 `SKILL.md` 파일 만들기 |
| 호출 방법 | `/플러그인-스킬이름` | `/내가-정한-이름` |
| 언제 쓰나 | 공개된 전문 기능을 쓸 때 | 반복 업무를 나만의 명령으로 묶을 때 |
| 수정 가능 여부 | 배포자가 수정 | 내가 자유롭게 수정 |

## 🔧 실습 1 — 플러그인 설치와 첫 사용

> 목표: 마켓플레이스에서 플러그인을 설치하고, 한 줄 명령으로 지우의 데이터를 분석해보기.

### 1단계: 작업 폴더로 이동하고 CC 열기

터미널을 열고 지우의 실습 폴더로 이동한 뒤 Claude Code를 시작한다.

```bash
# 먼저 실습 폴더가 어디 있는지 찾는다 (cc-03·cc-07에서 배운 방법)
find ~ -name "course_sales_monthly.csv" 2>/dev/null
```

위에서 나온 경로의 폴더가 `practice-data/creator`입니다. 아래 경로는 **예시**이니 찾은 실제 경로로 바꿔 입력한 뒤 Claude Code를 시작한다.

```bash
cd <위에서 찾은 practice-data/creator 폴더 경로>
claude
```

CC가 켜지면 `>` 프롬프트가 뜬다. 이제 대화창 안에서만 움직인다.

**기대 결과**: `>` 입력창이 나타난다.

**관찰 포인트**: 폴더를 먼저 이동하고 `claude`를 켜면, CC가 그 폴더를 "무대"로 삼아 `course_sales_monthly.csv`, `newsletter_stats.csv`, `student_inquiries.csv`, `content_plan.md`, `research_memo.md`, `consulting_schedule.md` 등 파일을 자동으로 인식한다.

---

### 2단계: 현재 설치된 플러그인 확인하기

CC 대화창(CC 안)에서 아래를 입력한다.

```text
/plugin list
```

**기대 결과**: 현재 설치된 플러그인 목록이 뜬다. 아직 아무것도 설치하지 않았다면 목록이 비어 있거나 기본 플러그인만 보인다.

```
현재 설치된 플러그인:
(없음 — 아직 기본 상태입니다)

설치 가능한 플러그인을 보려면 /plugin discover 를 입력하세요.
```

**관찰 포인트**: `/plugin`이라는 명령이 터미널 프롬프트(`$`)가 아니라 CC 대화창 **안**에서 입력된다는 점에 주목하자. `claude`를 켜고 그 안에서 쓰는 슬래시 명령과, 터미널에서 쓰는 `claude --옵션`은 전혀 다른 입력 위치다.

---

### 3단계: 마켓플레이스에서 플러그인 둘러보기

```text
/plugin discover
```

**기대 결과**: 설치 가능한 플러그인 목록이 나열된다. 어떤 것들이 있는지 메뉴를 보듯 훑어본다.

```
사용 가능한 플러그인:
- notion         : Notion 페이지 읽기·쓰기
- github         : GitHub 이슈·PR 연결
- deep-research  : 7단계 멀티에이전트 리서치
- docs-guide     : 공식 문서 기반 정확한 답변
...
```

**관찰 포인트**: "이런 게 있구나" 정도로 훑으면 충분하다. 어떤 게 내 일에 유용할지 메모해두면 좋다.

---

### 4단계: 커뮤니티 플러그인 저장소 등록하기

지우에게 바로 유용한 플러그인 8개가 담긴 커뮤니티 저장소(gptaku_plugins)를 마켓플레이스에 등록한다. 처음 한 번만 하면 된다. 이 명령은 "플러그인을 설치"하는 게 아니라, 나중에 설치할 수 있도록 "어디서 받을지 주소를 등록"하는 것이다.

```text
/plugin marketplace add https://github.com/fivetaku/gptaku_plugins.git
```

**기대 결과**:
```
저장소 등록 완료: fivetaku/gptaku_plugins
포함된 플러그인: docs-guide, 바르다 깃선생, 바선생, deep-research, 품앗이, show-me-the-prd, 끼리끼리, 스킬러들의 수다
```

**관찰 포인트**: 앱스토어에서 계정을 연결하는 것과 같다. 이제부터 gptaku_plugins 안의 여러 플러그인을 골라 설치할 수 있다.

> ⚠️ **네트워크 오류가 났다면**: 인터넷 연결을 확인하거나, CC 대화창을 `/exit`로 나갔다가 다시 들어와 시도해보자.

---

### 5단계: 플러그인 설치하기

지우의 강의 사업에 바로 쓸 수 있는 플러그인 두 개를 설치한다.

```text
/plugin install show-me-the-prd
```

```text
/plugin install deep-research
```

**기대 결과**:
```
show-me-the-prd 설치 완료.
사용법: /show-me-the-prd 로 시작하거나, "기획서 만들어줘"라고 말하면 자동 활성화됩니다.
```

**관찰 포인트**: `show-me-the-prd`는 인터뷰 내용을 입력하면 기획서 4종으로 자동 변환해주는 플러그인이다. `deep-research`는 7단계 멀티에이전트 심층 리서치 플러그인이다. "AI 도구 시장에서 비개발자용 강의 경쟁 현황 조사해줘" 같은 복잡한 리서치를 자동으로 여러 단계로 나눠 진행한다.

---

### 6단계: 설치된 플러그인 목록 재확인

```text
/plugin list
```

**기대 결과**: 방금 설치한 플러그인들이 목록에 보인다.

```
설치된 플러그인:
- show-me-the-prd (fivetaku/gptaku_plugins) — 인터뷰 → 4종 기획서 자동 생성
- deep-research   (fivetaku/gptaku_plugins) — 7단계 멀티에이전트 리서치
```

**체크포인트**: 두 플러그인이 모두 목록에 보이면 성공이다. 스마트폰에 앱이 추가된 것과 같다.

---

### 7단계: deep-research로 시장 조사 해보기

지우의 다음 강의 기획을 위해, "비개발자 1인 사업자 대상 AI 도구 강의 시장"을 간단히 조사해보자.

```text
/deep-research

주제: 비개발자 1인 사업자가 AI 도구 강의를 선택할 때 가장 중요하게 보는 기준
범위: 국내 강의 플랫폼(클래스101, 탈잉, 자사사이트), 2025년 이후
목표: 내 다음 강의 기획에 참고할 핵심 인사이트 3가지
```

**기대 결과**: deep-research가 여러 단계로 나눠 조사를 진행하고, 구조화된 리서치 보고서를 출력한다. 단계별로 "지금 이 부분을 조사하고 있습니다" 상태가 보인다.

**관찰 포인트**: `/deep-research`를 치기 전에는 이런 구조적 리서치가 CC만으로는 훨씬 더 복잡한 프롬프트가 필요했다. 플러그인 하나로 CC가 "새로운 능력을 배운" 효과가 난다. 플러그인은 한 번 깔면 이후 CC를 켤 때마다 자동으로 활성화된다는 점도 기억하자.

> 💡 **다른 트랙이라면**: 스토어 트랙은 "경쟁사 스마트스토어 상위 노출 전략", 프리랜서는 "국내 마케팅 외주 단가 시장 현황", 로컬은 "동네 카페 SNS 마케팅 성공 사례"로 바꿔 쓰면 된다.

---

### 8단계: /help로 사용 가능한 슬래시 명령 전체 보기

```text
/help
```

**기대 결과**: 현재 CC에서 쓸 수 있는 슬래시 명령과 설치된 플러그인 전체 목록이 나온다. 방금 설치한 `/deep-research`, `/show-me-the-prd`가 포함되어 있으면 성공이다.

```
사용 가능한 명령 목록:
  /help              — 이 도움말 보기
  /exit              — CC 종료
  /compact           — 대화 요약 (컨텍스트 절약)
  /clear             — 대화 초기화

플러그인 명령:
  /deep-research     — 7단계 멀티에이전트 심층 리서치 (fivetaku/gptaku_plugins)
  /show-me-the-prd   — 인터뷰 → 4종 기획서 자동 생성 (fivetaku/gptaku_plugins)

내 스킬:
  (아직 없음 — 실습 2에서 만들 예정)
```

**관찰 포인트**: `/help`는 언제든 쓸 수 있는 "메뉴판"이다. "어떤 명령이 있더라?" 할 때 이것만 치면 된다. 실습 2에서 스킬을 만들고 나면 이 목록 아래에 `/monthly-review` 같은 항목이 추가된다.

## 🎬 실습 2 — 나만의 스킬(SKILL.md) 만들기

> 목표: 지우의 매월 반복 업무를 SKILL.md로 정의하고, 한 줄 명령으로 실행해보기.

스킬은 플러그인처럼 남이 만든 걸 쓰는 게 아니라, **내가 직접 만드는 작업 레시피**다. 지우가 매달 초에 하는 "강의 매출 + 뉴스레터 성과 합산 리뷰"를 스킬로 만들어보자.

### 1단계: 실제 데이터 현황 먼저 파악하기

스킬 레시피를 쓰기 전에, "이 파일에서 어떤 분석이 가능한지"를 직접 확인한다. 좋은 레시피를 쓰려면 재료를 먼저 봐야 하는 것과 같다.

```text
course_sales_monthly.csv를 읽어줘. 어떤 열이 있고, 어느 기간 데이터가 있는지 간단히 알려줘.
```

**기대 결과**:
```
파일: course_sales_monthly.csv
기간: 2025-07 ~ 2026-03 (9개월)
열 구성:
- 월, 강의명, 카테고리, 플랫폼
- 수강신청수, 환불수, 객단가(원)
- 매출(만원), 쿠폰사용수, 순매출(만원)

포함된 강의:
- 노션으로 1인 사업 시스템 만들기 (생산성)
- 초보 뉴스레터 7일 완성 (글쓰기)
- 챗GPT로 콘텐츠 10배 빠르게 (AI활용)
- 1인 지식사업 수익화 로드맵 (비즈니스)
- 크리에이터 세금 기초 (비즈니스)
- 클로드코드로 업무 자동화 입문 (AI활용, 2026-02 신규)
```

**관찰 포인트**: CC가 파일을 스스로 열어 구조를 파악했다. 복사-붙여넣기 없이 AI가 직접 파일을 읽은 것이다.

### 2단계: 실제 데이터로 패턴 먼저 찾아보기

스킬 레시피를 쓰기 전에, "이 파일에서 어떤 분석이 유용한지"를 직접 확인한다.

```text
course_sales_monthly.csv를 읽고, 아래 3가지를 정리해줘:
1. 전체 기간 중 순매출이 가장 높았던 달 Top 3
2. 강의별 평균 순매출 (비개발자도 이해하게 표로, 전체 강의 포함)
3. 최근 3개월(2026-01, 02, 03) 흐름 — 어떤 강의가 뜨고 있나

실제 CSV 수치를 인용해서 설명해줘.
```

**기대 결과** (실제 데이터 기반):
```

   | 강의명 | 평균 순매출(만원) | 집계 기간 |
   |---|---|---|
   | 클로드코드로 업무 자동화 입문 | 875만 | 2026-02~03 (2개월) |
   | 1인 지식사업 수익화 로드맵 | 885만 | 2025-10~2026-03 (6개월) |
   | 챗GPT로 콘텐츠 10배 빠르게 | 908만 | 2025-08~2026-03 (8개월) |
   | 노션으로 1인 사업 시스템 만들기 | 384만 | 2025-07~2026-01 (6개월) |
   | 초보 뉴스레터 7일 완성 | 317만 | 2025-07~2025-12 (5개월) |
   | 크리에이터 세금 기초 | 96만 | 2025-07, 09 (2개월) |

3. 최근 흐름: 2026-02 첫 등장한 '클로드코드 입문'이 2026-03에

```

**체크포인트**: 숫자가 CSV 파일과 일치하는지 한 줄만 확인해본다. (예: 2026-03 클로드코드 순매출이 1,242만원인지.) AI 답이 맞으면 다음 단계로, 다르면 "CSV의 실제 값을 다시 확인해줘"라고 요청한다.

> 💡 **표를 보며 궁금할 수 있는 것**: "크리에이터 세금 기초는 왜 평균이 그렇게 낮아?" — CSV를 보면 이 강의는 2025-07(순매출 83만)과 2025-09(108만)에만 2개월 등장한다. 소규모 탈잉 강의라 다른 강의에 비해 규모가 작다.

```text
# 월간 리뷰 스킬 — 지우의 사업 요약

## 역할
나는 1인 콘텐츠 사업자 지우의 데이터 분석 짝꿍이다.
이 스킬이 실행되면 다음 단계를 순서대로 진행한다.

## 단계 1: 매출 데이터 읽기
course_sales_monthly.csv를 열어 아래를 정리한다.
- 강의별 순매출 합계 (전체 기간 + 최근 3개월)
- 월별 총 순매출 추이
- 환불수가 가장 많은 강의

## 단계 2: 뉴스레터 성과 읽기
newsletter_stats.csv를 열어 아래를 정리한다.
- 오픈율 상위 3호와 제목
- 클릭율 상위 3호와 제목
- 최근 3개월 누적 구독자 증감

## 단계 3: 교차 분석
매출이 높았던 달과 뉴스레터 오픈율/클릭율이 높았던 호를 비교해 연결점을 찾는다.

## 단계 4: 리포트 작성
결과를 아래 형식으로 작성한다.
- 제목: [월간 리뷰] YYYY-MM
- 섹션 1: 강의 매출 요약 (표)
- 섹션 2: 뉴스레터 성과 요약 (표)
- 섹션 3: 이번 달 눈에 띄는 것 2가지 (불릿)
- 섹션 4: 다음 달 제안 한 줄

## 출력 방식
- 비개발자가 한눈에 읽히도록 표와 불릿 위주
- 어려운 용어 없이, 구체적 숫자 인용
- 파일로 저장: monthly-review-YYYY-MM.md
```

**기대 결과**: CC가 `.claude/skills/monthly-review.md` 파일을 생성했다는 안내가 뜬다.

**관찰 포인트**: 파일 맨 위의 `name`, `description` 부분이 핵심이다. `description`에 적힌 내용을 CC가 읽고 "이 스킬이 어떤 상황에 쓰이는지" 자동으로 감지한다. 잘 적어두면, "이번 달 강의 정리 좀 해줘"처럼 말만 해도 CC가 `/monthly-review`를 추천해줄 수 있다.

### codex plugin 개념 심화 — 왜 한 자리에서 쓰는가?

CC와 Codex는 같은 뼈대(파일 읽기·규칙서·슬래시 명령)를 가진 형제 도구지만, 만든 회사(Anthropic vs OpenAI)와 내부 AI 모델이 다르다. 이 차이가 실무에서 어떻게 드러나는지 아래 표로 비교해보자.

| 항목 | CC (Claude Code) | Codex |
|------|-----------------|-------|
| 만든 회사 | Anthropic | OpenAI |
| 내부 모델 | Claude (claude-opus, sonnet 등) | OpenAI 모델 (gpt-5.5 등) |
| 강점 느낌 | 긴 글·보고서·전략적 판단 | 코드 수정·데이터 처리·구조화 |
| 비유 | 글 잘 쓰는 전략가 | 꼼꼼한 데이터 분석가 |

보통은 CC 창과 Codex 창을 따로 열어 일하는데, **codex plugin을 설치하면 CC 대화창 안에서 바로 Codex를 불러** 같은 파일·같은 맥락에서 분석을 이어받길 수 있다. 작업실(CC)에 두 번째 짝꿍(Codex)을 초대하는 것이다.

> 💡 "우열 가리기"가 아니라 "각도 차이 활용"이 핵심이다. CC는 기회를, Codex는 리스크를 잡는 식으로 역할을 나누면 두 도구가 서로 보완재가 된다.

### 1단계: codex plugin 설치 확인 및 설치

CC 대화창 안에서 플러그인 목록을 먼저 확인한다. codex plugin이 이미 설치돼 있을 수도 있다.

```text
/plugin list
```

목록에 `codex` 또는 `codex-plugin`이 없다면 마켓플레이스에서 검색한다.

```text
/plugin discover codex
```

검색 결과에서 OpenAI의 **`codex`** 플러그인을 설치한다.

```text
/plugin install codex
```

**기대 결과**: `codex-plugin` 설치 완료 메시지가 뜬다.

```
codex 플러그인 설치 완료.
새 명령 추가: /codex:review (코드·데이터 리뷰), /codex:rescue (작업 위임·구조), /codex:setup (준비 확인) 등.
```

**관찰 포인트**: codex plugin이 설치되면 `/help` 목록에 새 항목이 추가된다. 설치 후 `/help`를 다시 쳐서 확인해보자.

### 2단계: CC 혼자 먼저 분석하고 결과 파일로 저장하기

### 3단계: 같은 파일을 Codex에게도 맡겨 비교하기

codex plugin이 설치되면 **`/codex:rescue <맡길 일>`** 로 Codex에게 작업을 통째로 위임하거나(막혔을 때 특히 유용), **`/codex:review`** 로 변경한 내용을 Codex가 리뷰하게 할 수 있다. 아래처럼 같은 데이터를 Codex에게 다른 관점으로 맡겨보자.

```text
Codex에게 아래 분석을 맡겨줘 (codex plugin 활성화 상태):

파일: course_sales_monthly.csv

CC와 다른 관점으로 분석해줘:
1) 강의별 환불률(환불수 / 수강신청수) 계산 후 순위
2) 환불률이 높은 강의의 공통점
3) 환불 줄이기 위한 실행 가능한 제안 1가지

표와 불릿으로 간결하게. 비개발자가 읽기 쉬운 한국어로.
결과를 codex-analysis-2026-03.md 파일로 저장해줘.
```

**기대 결과**:
```
[Codex 분석]

| 강의명 | 수강신청수 | 환불수 | 환불률 |
```

### 4단계: 두 분석 결과를 비교 문서로 정리하기

CC가 생성한 `cc-analysis-2026-03.md`와 Codex가 생성한 `codex-analysis-2026-03.md`를 나란히 읽고, 비교 요약 문서를 만든다.

```text
cc-analysis-2026-03.md 와 codex-analysis-2026-03.md 두 파일을 읽고, 아래 형식으로 비교 요약 문서를 만들어줘. 파일명: analysis-comparison-2026-03.md

형식:
1. CC 분석 요점 (3줄 이내)
2. Codex 분석 요점 (3줄 이내)

4. 두 분석의 차이점
5. 지우가 다음 달 실행할 1가지 (CC + Codex 종합 판단)
```

**기대 결과**:
```
[CC vs Codex 분석 비교] 2026-03

- 챗GPT 강의 누적 순매출 7,260만원으로 압도적 1위

▶ 지우의 다음 달 실행 1가지
```

### 5단계: 설치 실패 또는 오류 대응

codex plugin 설치나 실행 중 오류가 날 수 있다. 이 단계에서는 가장 흔한 오류와 대응법을 직접 확인한다.

**오류 1 — "codex plugin을 찾을 수 없습니다"**

```text
/plugin discover
```

로 전체 목록을 확인하고, "codex"가 포함된 항목의 정확한 이름을 찾아 다시 설치한다.

**오류 2 — "Codex 로그인이 필요합니다"**

CC와 별도로 Codex 자체에 로그인돼 있어야 한다. 터미널 탭을 새로 열어 아래를 실행한다.

```bash
codex --version
codex login
```

로그인 후 CC 대화창으로 돌아와 다시 시도한다.

**오류 3 — plugin 설치는 됐는데 Codex 호출이 안 된다**

```text
/plugin list
```

로 `codex-plugin` 상태를 확인한다. "활성화됨(active)" 상태인지 체크하고, 아니라면 `/plugin enable codex-plugin`을 시도한다.

**오류 4 — codex plugin이 마켓플레이스에 아예 없다**

💡 **설치가 안 되면** — 네트워크·로그인·버전 문제로 설치가 막힐 수 있다. 이 경우 오늘 실습은 "CC 단독으로 분석 → 결과 파일 저장"(2단계·4단계)만 완성해도 충분하다. 핵심 개념(두 도구의 역할 차이)은 이 과정에서 이미 체감했다.

**관찰 포인트**: 오류가 난다고 실패가 아니다. codex plugin은 아직 발전 중인 기능이므로, "시도해보고 안 되면 대안으로 간다"는 태도가 중요하다. 실습의 진짜 목적은 "두 도구의 각도 차이 체감"이며 그것은 2단계~4단계에서 이미 달성됐다.

### 6단계: 최종 체험 정리 — "언제 뭘 쓸지" 한 문장 만들기

실습이 끝나면 아래 문장을 스스로 완성해본다.

```text
오늘 내가 느낀 CC와 Codex의 차이를 한 문장으로 정리해줘.
내가 말할게: "CC는 ___, Codex는 ___ 느낌이었다."
```

**기대 결과** (예시 — 정답은 없다):
```
"CC는 전체 그림을 보고 다음 수를 제안하는 전략가, Codex는 숫자를 꼼꼼히 파고드는 리스크 분석가 느낌이었다."
```

어떤 답이든 맞다. 이 한 문장이 나중에 "언제 뭘 쓸지" 판단하는 자기만의 기준이 된다. 파트2 Codex 트랙에서도 이 감각을 바탕으로 두 도구를 더 깊이 배운다.

| 상황 | 원인 | 해결 |
|------|------|------|
| `/plugin list` 명령이 안 먹힌다 | CC 대화창이 아닌 터미널에서 쳤다 | `claude`를 켜고 대화창 **안**에서 입력한다 |

| codex plugin 검색해도 없다 | 아직 공개 배포 전이거나 저장소 미등록 | CC 단독으로 2단계 분석만 완료 후 Codex는 별도 탭에서 수동 비교 |

| CC와 Codex 분석 결과가 너무 비슷하다 | 프롬프트가 동일하면 비슷한 답이 나올 수 있다 | 의도적으로 다른 관점 지시 ("CC는 기회 관점으로", "Codex는 리스크 관점으로") |
| 스킬 실행이 중간에 멈춘다 | 컨텍스트(기억)가 꽉 찼을 가능성 | `/compact`로 정리 후 `/monthly-review` 재실행 |

## 💡 핵심 규칙 / 개념 정리

**1. 플러그인 = 앱스토어, 스킬 = 나만의 레시피**
- 플러그인: 남이 만든 기능 꾸러미. `/plugin marketplace add` → `/plugin install` → 자동 활성화.
- 스킬: 내가 만든 반복 작업 레시피. `.claude/skills/SKILL.md` 파일 하나로 완성.
- 둘 다 `/help`로 현재 쓸 수 있는 명령 전체를 볼 수 있다.

**2. SKILL.md의 핵심 3요소**
- `name`: 스킬 호출 이름 (`name: monthly-review` → `/monthly-review`로 실행)
- `description`: CC가 이 스킬을 언제 쓸지 자동 감지하는 기준. 자세히 적을수록 자동 추천 정확도가 높아진다.
- 단계 목록: "단계 1 → 단계 2 → ..." 순서로 CC가 실행할 작업 흐름.

**3. 반복 업무를 스킬로 만드는 기준**
- 같은 과정을 **달에 2번 이상** 반복한다면 → 스킬 후보.
- 단계가 **3개 이상**이라면 → 스킬로 묶을 가치 있음.
- "매번 CC에게 같은 프롬프트를 치고 있다면" → 그 프롬프트를 SKILL.md에 넣어라.

**4. codex plugin = 두 도구를 한 자리에서**
- CC 대화창 안에서 Codex를 불러 같은 파일·맥락으로 분석을 이어받는다.
- 두 도구는 같은 뼈대(파일 읽기·슬래시 명령)를 쓰지만 모델·만든 회사가 달라 관점이 조금 다르다.
- "우열"이 아니라 "각도 차이"를 활용하는 것이 핵심. CC는 기회를, Codex는 리스크를 잡는 식으로 역할을 나누면 좋다.
- 🔎 검증대기: 정확한 설치·호출 방식은 공식 문서 실측 후 확정.

**5. forward arc — 이 레슨이 다음에 어떻게 쓰이나**
- 지금 만든 `/monthly-review` 스킬은 Part 2 Codex 트랙에서 `AGENTS.md`로 비슷하게 적용된다.
- Part 3(옵시디언·LLM Wiki)에서는 스킬 결과물을 vault에 자동 저장하는 흐름으로 확장된다.
- Part 4+(멀티에이전트)에서는 여러 AI 팀원이 각자 담당 스킬을 가지고 협업하는 기반이 된다.
