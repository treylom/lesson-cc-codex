# cc-04: 워크플로우 + Plan + 좋은 프롬프트 — "계획하고 요청하면, AI는 제대로 일한다"

> **레슨 목표**: Plan 모드(계획 먼저 보기)와 좋은 프롬프트 공식을 익혀, 지우의 실제 강의 매출·뉴스레터·수강생 문의 데이터에서 의미 있는 결과물을 뽑아낸다.
> **예상 시간**: 65분 (개념 도입 10분 + 실습 1 — 워크플로우·Plan 모드 체험 20분 + 실습 2 — 실제 데이터 분석 25분 + 실습 3(선택) — 보고서 초안 만들기 10분)
> **사전 준비**: Claude Code가 켜진 터미널 1개 + `practice-data/creator/` 폴더에 접근 가능한 상태 (cc-03 완료 권장)
> **진입 커맨드**: `/cc-04`

## 📚 학습 목표 — 4가지 도달점

1. **"어떻게 시키느냐"가 결과를 바꾼다** — 막연한 질문과 잘 만든 프롬프트(AI에게 하는 지시·질문)의 차이를 같은 데이터로 직접 비교해서 체감한다.
2. **Plan 모드를 써보고 "왜 계획부터인가"를 이해한다** — 복잡한 일은 먼저 계획을 보고 OK 한 다음 실행하면 훨씬 안전하다는 것을 직접 경험한다.
3. **지우의 실제 데이터에서 패턴을 찾는다** — `course_sales_monthly.csv`·`newsletter_stats.csv`·`student_inquiries.csv`를 AI에게 읽혀 실제 수치를 인용한 인사이트를 받아낸다.
4. **숫자와 맥락을 연결해 보고서 초안을 만들어본다** — 데이터(매출·뉴스레터)와 기획 메모(`content_plan.md`)를 이어서 "이번 분기 다음에 뭘 할까"에 답하는 한 페이지 초안을 완성한다.

## 🎯 개념 도입 (10분)

### "좋은 요청 = 좋은 결과"를 비유로 이해하기

식당에 들어갔다고 생각해보세요.

**손님 A**: "밥 주세요."

**손님 B**: "저는 매운 걸 못 먹어요. 따뜻하고 부드러운 걸로, 혼자니까 소자로 주세요."

두 주문에 대한 식당의 반응은 다릅니다. A는 직원이 "어떤 걸로요?"를 다시 물어봐야 하고, B는 그냥 주문을 넣으면 됩니다. **Claude Code(이하 CC)도 똑같습니다.** 내가 어떻게 요청하느냐에 따라 AI가 주는 결과물의 품질이 크게 달라집니다.

좋은 프롬프트(AI에게 하는 질문·지시)의 공식은 간단합니다.

```
[역할 또는 맥락] + [읽을 파일] + [원하는 결과와 형식]
```

예시로 비교해봅시다.

```
❌ "매출 분석해줘."

✅ "너는 내 온라인 강의 사업 데이터 분석가야.
   course_sales_monthly.csv를 읽고,
   강의별 순매출 합산 1~5위를 표로 정리해줘.
   실제 CSV에 있는 숫자를 꼭 인용해줘."
```

첫 번째는 AI가 "어디 매출인지, 어떤 형식으로 원하는지"를 모르기 때문에 뭉뚱그린 답을 합니다. 두 번째는 AI가 **정확히 어떤 파일을, 어떤 형식으로, 어떤 관점으로** 봐야 할지 알기 때문에 훨씬 구체적인 답이 나옵니다.

### Plan 모드 — "인테리어 업체에 도면 먼저 받기"

1인 사업자 지우한테 제일 아까운 건 시간입니다. 그런데 "다 알아서 해줘"라고 시켰다가 방향이 엉뚱하면 처음부터 다시 시켜야 합니다. 이게 훨씬 더 시간 낭비예요.

**Plan 모드**는 이 문제를 해결합니다. 공사를 시작하기 전에 **도면(계획서)을 먼저 보고 OK 하면 그다음에 공사를 시작하는 것**과 같아요.

| 일반 모드 | Plan 모드 |
|-----------|-----------|
| 내가 시키면 CC가 바로 실행 | 먼저 계획서만 보여주고 "OK?"를 기다림 |
| 중간에 방향이 틀려도 이미 실행된 것들이 있음 | 계획 단계에서 방향을 바로잡고 나서 실행 |
| 단순하고 빠른 일에 적합 | 복잡하거나 여러 단계가 얽힌 일에 적합 |

**Plan 모드 켜는 법:**
- 대화창(`>`)에서 키보드의 **`Shift + Tab`** 을 두 번 누릅니다.
- 한 번 누르면 `auto-approve` 모드, 두 번 누르면 `plan` 모드.
- 화면 상단에 `plan mode`라는 표시가 뜨면 켜진 것입니다.
- 이 상태에서 일을 시키면, CC는 "1단계: ○○, 2단계: ○○, 이렇게 진행하겠습니다"라고 계획만 먼저 보여줍니다.
- 계획이 마음에 들면 "좋아, 진행해줘"라고 입력하면 그제서야 실행합니다.

> 💡 **비개발자 안심**: Plan 모드에서는 CC가 실제로 파일을 수정하거나 새 파일을 만들지 않습니다. 말 그대로 계획서만 보여줍니다. "혹시 잘못되면 어쩌지?"라는 걱정 없이 편하게 먼저 확인할 수 있어요.

## 🔧 실습 1 — 워크플로우 흐름과 Plan 모드 체험 (20분)

> **목표**: 일반 모드와 Plan 모드의 차이를 같은 요청으로 직접 비교해보고, 좋은 프롬프트가 왜 결과를 바꾸는지를 체감한다.

### 1단계: 실습 폴더로 이동하고 CC 켜기

터미널을 열고 실습 폴더로 이동합니다. `<경로>`는 자신의 실제 폴더 경로입니다.

```bash
cd <lesson-cc-codex 폴더>/courses/cc-codex-101/practice-data/creator
claude
```

CC가 켜지면 `>` 입력창이 나타납니다. 이제 이 폴더 안의 파일들을 CC가 볼 수 있는 상태입니다.

**기대 결과**: `>` 입력 대기 상태가 나타남.

**체크포인트**:
- 폴더를 `creator`로 먼저 이동한 다음 `claude`를 켜야 CC가 이 폴더의 파일들을 무대로 삼습니다.
- 만약 `claude`가 다른 폴더에서 켜졌다면 `/exit`로 나온 뒤, `cd` 명령으로 `creator` 폴더로 이동 후 다시 `claude`를 입력하세요.

---

### 2단계: 상황 설명하고 할 수 있는 일 파악하기

대화창에 아래를 입력해봅니다.

```text
안녕하세요! 저는 온라인 강의와 뉴스레터를 운영하는 1인 사업자 지우예요.
이 폴더에 내 사업 데이터가 있는데, 지금 이 파일들로 할 수 있는 현실적인 일 3가지를 알려주세요.
파일 이름도 같이 말해주세요.
```

**기대 결과**:
```
안녕하세요! 폴더를 살펴봤습니다. 파일들이 있네요.

지금 할 수 있는 일 3가지:

1. 강의 매출 분석 (course_sales_monthly.csv)
   - 어느 강의가 가장 잘 팔렸는지, 매출 추이가 어떤지 파악

2. 수강생 문의 패턴 정리 (student_inquiries.csv)
   - 가장 많이 들어온 문의 유형, 처리에 시간이 많이 드는 케이스 파악

3. 콘텐츠 기획 연결 (content_plan.md + 숫자 데이터)
   - 매출과 뉴스레터 데이터를 연결해 다음 분기 전략 제안

무엇부터 시작할까요?
```

**관찰 포인트**:
- 폴더를 직접 탐색해서 **실제 파일 이름을 읽어서** 제안합니다. 내가 파일 목록을 복사해서 붙여넣지 않았는데도요.
- 이것이 "AI가 내 파일을 직접 본다"는 것의 의미입니다. 채팅 AI와 다른 핵심입니다.

---

### 3단계: 막연한 요청과 구체적 요청 비교 (첫 번째 차이 체감)

먼저 막연한 요청부터 해봅니다.

```text
매출 데이터 분석해줘.
```

**기대 결과**: CC가 파일 하나를 열어 적당한 분석을 하거나, "어떤 파일을 봐야 할까요?" 같은 역질문을 합니다. 구체적인 인사이트보다는 일반적인 안내에 그칩니다.

**관찰 포인트**: 결과가 원하던 것과 조금 다를 겁니다. 이게 막연한 요청의 한계입니다.

이번에는 같은 주제를 구체적으로 요청합니다.

```text
너는 내 온라인 강의 사업의 데이터 분석가야.
course_sales_monthly.csv를 읽고,
강의별로 2025-07부터 2026-03까지의 순매출(만원) 합산을 계산해서
1위부터 5위까지 표로 정리해줘.
실제 CSV에 있는 숫자를 인용해야 해.
```

**기대 결과**:
```
강의별 순매출 합산 (2025-07 ~ 2026-03)

| 순위 | 강의명 | 순매출 합산(만원) |
|------|-------|----------------|
| 1위 | 챗GPT로 콘텐츠 10배 빠르게 | 7,260 |
| 2위 | 1인 지식사업 수익화 로드맵 | 5,312 |
| 3위 | 클로드코드로 업무 자동화 입문 | 1,750 |
| 4위 | 노션으로 1인 사업 시스템 만들기 | 1,920 |
| 5위 | 초보 뉴스레터 7일 완성 | 1,268 |
```

**관찰 포인트**:
- 첫 번째 막연한 요청보다 훨씬 구체적인 표가 나옵니다.
- AI가 CSV의 실제 숫자를 읽어서 계산해줍니다.
- 숫자가 이상하게 느껴지면 "CSV의 실제 값을 확인해서 다시 계산해줘"라고 한 번 더 요청해보세요. 이게 할루시네이션(AI가 숫자를 지어내는 현상)을 잡는 기본 기술입니다.

---

### 4단계: Plan 모드 켜고 복잡한 일 시켜보기

이번에는 복잡한 요청을 Plan 모드로 해봅니다.

**먼저 Plan 모드 켜기**: 키보드에서 `Shift + Tab`을 두 번 눌러보세요. 화면 상단에 `plan mode` 표시가 뜨면 됩니다.

Plan 모드가 켜진 상태에서 입력합니다.

```text
내 사업 데이터를 종합 분석해서 이번 달 보고서를 만들고 싶어.
course_sales_monthly.csv, newsletter_stats.csv, student_inquiries.csv
세 파일을 읽고, 각 파일에서 핵심 인사이트 2개씩 뽑은 뒤
"관찰 → 해석 → 제안" 구조로 보고서 초안을 파일로 저장해줘.
```

**기대 결과**: CC가 바로 실행하는 게 아니라, 계획서를 먼저 보여줍니다.

```
계획을 먼저 보여드릴게요:

1단계: course_sales_monthly.csv 읽기
 → 강의별 순매출, 월별 추이, 성장 구간 파악
 → 핵심 인사이트 2개 추출

2단계: newsletter_stats.csv 읽기
 → 오픈율·클릭율 높은 호 특성, 구독자 증감 추이
 → 핵심 인사이트 2개 추출

3단계: student_inquiries.csv 읽기
 → 카테고리별 문의 빈도, 미해결 건 확인
 → 핵심 인사이트 2개 추출

4단계: "관찰 → 해석 → 제안" 구조 보고서 초안 작성
5단계: 보고서를 파일로 저장

이렇게 진행해도 될까요?
```

**체크포인트**:
- 계획이 마음에 들면 "좋아, 진행해줘"라고 입력하면 5단계를 차례로 실행합니다.
- 계획이 이상하면 "3단계를 빼줘" 혹은 "2단계와 3단계 순서를 바꿔줘"처럼 조정할 수 있습니다. 이게 Plan 모드의 핵심 장점입니다.

> 💡 **왜 Plan 모드를 쓰나**: 파일을 여러 개 읽고 → 분석하고 → 파일로 저장하는 복잡한 작업일수록, 중간에 방향이 틀리면 되돌리기 어렵습니다. Plan 모드로 먼저 확인하면 시간 낭비를 막을 수 있어요.

---

### 5단계: 결과 확인하고 "다시 시키기" 연습

보고서가 만들어지면 바로 "다시 시키기"를 연습합니다.

```text
고마워. 그런데 보고서가 너무 길어.
핵심만 남기고, 제목은 "2026년 3월 사업 현황 요약"으로 바꿔서
전체를 10줄 이내로 다시 써줄래?
```

**기대 결과**: 훨씬 짧고 핵심만 담긴 버전이 나옵니다.

**관찰 포인트**: AI는 비서이지 시험 감독이 아닙니다. 첫 결과가 마음에 안 들어도 "이 부분이 문제야, 이렇게 고쳐줘"라고 말하면 됩니다. 몇 번이고 다시 시켜도 짜증 내지 않아요.

## 🎬 실습 2 — 지우의 실제 데이터로 패턴 찾기 (25분)

> **목표**: `course_sales_monthly.csv`, `newsletter_stats.csv`, `student_inquiries.csv` 세 파일을 순서대로 AI에게 읽혀, 각 파일에서 의미 있는 패턴을 찾아낸다. 모든 인사이트는 실제 CSV 수치를 인용해야 한다.

### 6단계: 강의 매출 데이터에서 패턴 찾기

아래 프롬프트를 입력합니다.

```text
이번엔 course_sales_monthly.csv를 집중해서 분석해줘.
아래 3가지를 알고 싶어:

1. 카테고리별(AI활용·생산성·글쓰기·비즈니스)로 묶어서
   어느 카테고리가 전체 순매출에서 몇 퍼센트를 차지하는지 표로
2. "챗GPT로 콘텐츠 10배 빠르게" 강의의 월별 수강신청수 추이
   (2025-08부터 2026-03까지, 왜 이 강의가 잘 되는지 단서도)
3. "클로드코드로 업무 자동화 입문" 강의의
   첫 달(2026-02) 대비 두 번째 달(2026-03) 순매출 성장률

꼭 CSV의 실제 숫자를 인용해서 설명해줘.
```

**기대 결과** (실제 데이터 기반):

```
1. 카테고리별 순매출 비중 (2025-07 ~ 2026-03)

| 카테고리 | 순매출 합산(만원) | 비중 |
```

| 보고서 방향이 원하는 것과 다르다 | 프롬프트에서 형식·구조 지정이 부족했음 | Plan 모드로 다시 시작. "구조: 1.○○ 2.○○ 3.○○" 형식으로 구체적으로 지정 |
| CC가 파일 저장 허락을 자꾸 물어본다 | 기본 보안 설정(파일 쓰기 전 확인) | "응" 또는 "yes" 입력으로 허락. 매번 물어보는 게 싫으면 이후 cc-05(CLAUDE.md)에서 조정 가능 |
| 결과가 한 번에 좋지 않다 | 정상. AI 첫 답은 70점 수준 | "이 부분을 더 구체적으로", "숫자를 더 강조해서" 같은 피드백으로 이어서 대화 |

## 💡 핵심 규칙 정리

### 규칙 1: 프롬프트(AI에게 하는 지시) 공식

```
[역할 또는 맥락] + [읽을 파일] + [원하는 결과와 형식]

예:
"너는 내 강의 사업 데이터 분석가야.(역할)
course_sales_monthly.csv를 읽고(파일)
강의별 순매출 합산을 표로 정리해줘.(결과 형식)"
```

이 공식만 지켜도 AI의 첫 답이 눈에 띄게 좋아집니다.

---

### 규칙 2: Plan 모드는 복잡한 일에만 쓰면 충분

| 이런 일은 그냥 시켜도 됨 | 이런 일은 Plan 모드 추천 |
|----------------------|----------------------|
| 파일 하나를 요약해줘 | 여러 파일을 읽고 분석해서 보고서까지 만들어줘 |
| 이 문장 다듬어줘 | 기획서 전체 구조를 잡고 목차부터 만들어줘 |
| 이 CSV에서 최고값 찾아줘 | 세 개 데이터를 연결해서 다음 분기 전략 세워줘 |

단순한 일에는 오히려 Plan 모드가 번거롭습니다. 복잡하고 여러 단계가 얽힌 일에만 쓰면 됩니다.

---

### 규칙 3: "다시 시키기"가 기술이다

```
1회: AI 첫 답 받기 (70점)
2회: "이 부분을 더 짧게" / "숫자를 꼭 인용해줘" (85점)
3회: "표 형식으로 다시" / "결론을 맨 위로" (95점)
```

AI는 비서이지 시험 감독이 아닙니다. 첫 답이 완벽하지 않아도 되고, 다시 시키는 것이 오히려 더 좋은 결과를 만드는 기술입니다.

---

### 규칙 4: 할루시네이션(AI가 숫자를 지어내기)을 막는 법

```
❌ "이 CSV에서 어떤 패턴이 있나요?"
✅ "course_sales_monthly.csv의 실제 값을 기반으로,
   각 행의 숫자를 인용해서 패턴을 설명해줘."
```

"실제 값을 인용해줘" 한 줄이 AI가 데이터를 추측해서 만들어내는 것을 막아줍니다. 숫자가 의심스러울 때는 "CSV에서 직접 확인해서 다시 말해줘"라고 재확인하세요.
