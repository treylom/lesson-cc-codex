# 1-2: 에이전트 루프와 첫 와우 — "AI가 스스로 읽고, 손을 움직이고, 답한다"

> **레슨 목표**: Claude Code가 챗봇과 다른 이유 — **에이전트 루프(읽기 → 도구 사용 → 응답)** — 를 직접 눈으로 보고, 내 사업 데이터로 첫 "와우" 순간을 만든다.
> **예상 시간**: 40분 (개념 도입 7분 + 실습 1 / 2 각 13분 + 실습 3·마무리 7분)
> **사전 준비**: 터미널 1개 + Claude Code 설치·로그인 완료(/cc-01 수료) + 이 코스 폴더(`practice-data/creator/` 포함)
> **진입 커맨드**: `/cc-02`

---

## 📚 학습 목표 (4가지 와우 모먼트)

1. **AI가 "스스로 움직인다"는 걸 본다** — 한 번 시키면 Claude Code가 알아서 *폴더를 열고 → 파일을 읽고 → 정리해서 답한다*. 이게 챗봇과 다른 에이전트 루프다.
2. **내 진짜 데이터로 답이 나온다** — 지어낸 예시가 아니라 `course_sales_monthly.csv`의 실제 매출 숫자로 답이 돌아오는 순간(첫 와우).
3. **AI가 "권한을 묻는" 이유를 이해한다** — 파일을 읽거나 명령을 실행하기 전에 왜 한 번 멈춰서 물어보는지(Permissions), 어떻게 답하면 되는지.
4. **AI가 기억하는 범위(Context)를 감 잡는다** — 한 대화 안에서 앞서 읽은 내용을 이어서 쓰는 걸 체감하고, 언제 새로 시작하면 좋은지 첫 감각을 얻는다.

## 🎯 개념 도입 — 에이전트 루프란? (7분)

### 비유: 챗봇은 "전화 상담원", Claude Code는 "내 책상에 앉은 동료"

챗봇(예: 일반 채팅 AI)에게 "우리 강의 매출 어때요?"라고 물으면, 그건 마치 **전화 상담원**과 같다. 상담원은 내 책상 서�NB을 직접 못 연다. 내가 숫자를 읽어줘야 답한다. 자료를 못 보니, 자칫 "보통 이맘때는 매출이 오릅니다" 같은 *일반론*이나 지어낸 숫자를 말하기 쉽다.

Claude Code는 다르다. **내 컴퓨터 폴더 앞에 직접 앉은 동료**다. 내가 "매출 어때요?"라고 하면 이 동료는:

1. 책상 위 폴더를 **직접 연다** (`practice-data/creator/` 탐색)
2. 매출 파일을 **직접 읽는다** (`course_sales_monthly.csv` 열어보기)
3. 읽은 *진짜 숫자*로 **정리해서 말한다**

이 "**열고 → 읽고 → 답하는**" 한 바퀴가 바로 **에이전트 루프(agentic loop)** 다. 'agentic'은 '스스로 알아서 행동하는'이라는 뜻이고, 'loop'는 '한 바퀴 도는 과정'이다.

### 에이전트 루프 한눈에 보기

```
내 지시   →   ① 상황 파악      →   ② 행동(도구 사용)   →   ③ 결과 확인   →   답변
("매출 어때요?")  (어떤 파일 있지?)    (CSV 파일 읽기)        (숫자 맞나?)     (정리해서 보고)
                                                              │
                                                  부족하면 다시 ①로 (반복)
```

핵심은 ②번 **"행동"** 이다. 챗봇은 ②가 없어서 *말만* 한다. Claude Code는 ②에서 실제로 파일을 읽고, 폴더를 뒤지고, 필요하면 계산도 한다. 그래서 답이 "내 데이터 기반"이 된다.

### 이 동료가 움직이는 5가지 부품 (Sec07 핵심 개념)

에이전트 루프가 돌아갈 때 안에서 작동하는 5가지를 알아두면, 앞으로 어떤 일이 일어나도 당황하지 않는다. 지금은 *이름만 익히는* 수준으로 충분하다 — 뒤 레슨에서 하나씩 깊게 다룬다.

| 부품 | 쉬운 뜻 | 이번 레슨에서 |
|------|---------|---------------|
| **Context (컨텍스트)** | AI가 *지금 대화에서 기억하는 범위*. 노트 한 권 같은 것 — 무한정은 아님 | 앞서 읽은 매출 내용을 이어서 쓰는 걸 체감 (실습 2) |
| **Tools (도구)** | AI가 쓰는 *손* — 파일 읽기·폴더 탐색·검색·실행 | CSV·문서를 직접 읽는 행동으로 직접 봄 (실습 1·2) |
| **Permissions (권한)** | 행동하기 전 *허락을 묻는 안전장치* (허용/물어봄/거부) | "이 파일 읽어도 돼요?" 묻는 순간을 만남 (실습 1) |
| **Memory (메모리)** | `CLAUDE.md`라는 파일로 *프로젝트에 영구 기억* 남기기 | 이름만 익힘 — `/cc-05`에서 직접 만듦 |
| **Session (세션)** | 시작부터 끝까지의 *한 작업 단위* | 새 대화 vs 이어가기 감각 (실습 2) |

> 💡 한 줄 요약: **Claude Code = 내 폴더를 직접 보는 동료 + "읽고 → 도구로 움직이고 → 답하는" 루프.** 이 한 문장만 챙기면 오늘 목표의 절반은 끝났다.

### 본문은 creator 트랙으로

이 레슨은 기본 트랙인 **creator(콘텐츠·지식 사업자 '지우')** 데이터로 진행한다. 다른 트랙(🛒 store 스마트스토어 / 💼 freelancer 외주 / ☕ local 카페·공방·학원)을 고른 분은 **자기 폴더(`practice-data/<자기트랙>/`)의 똑같은 종류 파일로 동일한 방식**으로 따라 하면 된다 — 파일 이름만 다를 뿐 흐름은 똑같다.

## 🔧 실습 1 — 에이전트 루프를 눈으로 보기 (13분)

목표: "한 번 시켰는데 AI가 *스스로 여러 번 움직이는*" 장면을 직접 목격한다.

### 1단계: 올바른 폴더에서 Claude Code 켜기

AI가 내 자료를 보려면, 내 자료가 있는 폴더에서 켜야 한다. (전화 상담원을 *내 책상 앞에* 앉히는 것과 같다.)

```text
# 터미널에서: 지금 어디에 있는지 확인
pwd

# creator 실습 폴더로 이동 (코스 폴더 안)
cd practice-data/creator

# 안에 뭐가 있는지 슬쩍 보기
ls

# Claude Code 켜기
claude
```

**기대 결과**: `ls`를 치면 `course_sales_monthly.csv`, `newsletter_stats.csv`, `student_inquiries.csv`, `content_plan.md`, `research_memo.md`, `consulting_schedule.md`, `README.md` 7개가 보인다. `claude`를 치면 입력 대기 상태(`>` 또는 입력창)가 뜬다.

**관찰 포인트**: 파일이 안 보이면 폴더 위치가 틀린 것이다. `pwd` 결과 끝이 `.../practice-data/creator`인지 확인.

---

### 2단계: "이 폴더에 뭐가 있어요?" — 첫 루프 돌리기

아래를 **그대로** 입력한다(복사-붙여넣기 OK).

```text
안녕하세요! 저는 1인 사업자 지우예요. 온라인 강의를 만들어 팔고, 뉴스레터도 보내고, 가끔 컨설팅도 합니다.
지금 이 폴더에 어떤 파일이 있고, 각각 뭐 하는 파일인지 정리해주세요. README.md부터 읽어주세요.
```

**기대 결과** (예시):
```
폴더를 살펴봤습니다. 7개 파일이 있네요.

[데이터(CSV) 3개]
- course_sales_monthly.csv : 강의별·월별 매출 (2025-07 ~ 2026-03)
- newsletter_stats.csv     : 뉴스레터 18호~41호 발송·오픈율·구독 증감
- student_inquiries.csv    : 수강생 문의 내역 (채널·카테고리·처리상태)

[문서(메모) 3개]
- content_plan.md      : 2분기 콘텐츠 기획 노트
- research_memo.md     : AI 도구·1인 사업 시장 리서치 메모
- consulting_schedule.md : 3월 컨설팅·운영 일정

[안내]
- README.md : 데이터 의미와 실습 팁
```

**관찰 포인트 — 여기가 핵심이다**: 답이 나오기 *전에* 화면을 보면, Claude Code가 한 번의 지시로 **여러 번 손을 움직인 것**이 보인다.

- "Reading README.md" (또는 비슷한 표시) → 파일을 **읽는 행동(Tool)**
- 폴더 목록을 훑는 행동
- 그 결과를 모아서 정리하는 답변

내가 "파일 하나하나 열어봐"라고 시키지 않았는데도 *스스로* README를 먼저 읽고, 폴더를 탐색하고, 정리했다. **이게 에이전트 루프다.** 챗봇이라면 "폴더 내용을 붙여넣어 주세요"라고 *나에게* 되물었을 것이다.

---

### 3단계: 권한 묻기(Permissions)를 만나기

이번엔 파일 *내용*을 직접 읽게 시켜본다.

```text
course_sales_monthly.csv 파일을 직접 열어서, 맨 위 몇 줄이 어떻게 생겼는지 그대로 보여주세요.
```

**기대 결과**: Claude Code가 파일을 읽기 전에 (설정에 따라) **"이 파일을 읽어도 될까요?"** 같은 허락을 묻는 화면이 뜰 수 있다. 보통 `Yes / Yes, and don't ask again / No` 같은 선택지가 나온다. `Yes`를 고르면 읽고, 아래처럼 실제 첫 줄들을 보여준다.

```
월,강의명,카테고리,플랫폼,수강신청수,환불수,객단가(원),매출(만원),쿠폰사용수,순매출(만원)
2025-07,노션으로 1인 사업 시스템 만들기,생산성,자사사이트,42,3,89000,374,8,341
2025-07,초보 뉴스레터 7일 완성,글쓰기,클래스101,67,5,59000,395,12,355
...
```

**관찰/체크포인트**:
- AI가 멈춰서 물어보는 건 고장이 아니라 **안전장치(Permissions)** 다. "네 폴더를 함부로 안 건드린다"는 약속이다. 우선순위는 거부(Deny) > 물어봄(Ask) > 허용(Allow) 순서로 작동한다.
- 읽기만 하는 작업은 안심하고 `Yes` 해도 된다 — 파일을 *바꾸는* 게 아니라 *보는* 것뿐이다.
- 매번 묻는 게 번거로우면 "Yes, and don't ask again"으로 그 종류는 자동 허용할 수 있다. (지금은 그냥 `Yes`로 충분.)

### 4단계: 컬럼(열) 구조를 사람 말로 풀어달라기

```text
방금 본 course_sales_monthly.csv의 열(컬럼)이 각각 무슨 뜻인지, 엑셀 처음 보는 사람도 알게 한 줄씩 설명해주세요.
```

**기대 결과** (예시):
```
- 월        : 매출이 집계된 달 (예: 2025-07)
- 강의명     : 판매한 강의 이름
- 카테고리   : 강의 분야 (생산성 / 글쓰기 / 비즈니스 / AI활용)
- 플랫폼     : 판매처 (자사사이트 / 클래스101 / 탈잉)
- 수강신청수 : 그 달에 신청한 사람 수
- 환불수     : 그중 환불한 사람 수
- 객단가(원) : 강의 1개 가격
- 매출(만원) : 그 달 총 매출
- 순매출(만원): 환불·쿠폰 빼고 실제로 남은 매출
```

**관찰 포인트**: AI가 다시 파일을 읽지 않고도 *방금 읽은 내용*을 기억해서 답한다. 이게 **Context(컨텍스트)** — 한 대화 안에서 앞 내용을 이어 쓰는 것이다.

---

### 5단계: "추측 말고 진짜 숫자로" — 할루시네이션 차단 연습

AI가 가끔 그럴듯하게 *지어낼* 수 있다. 이걸 막는 습관을 지금 들인다.

```text
이 데이터에서 가장 매출이 큰 강의 하나만 골라주세요.
단, 반드시 CSV에 실제로 있는 행의 숫자를 인용해서 근거를 보여주세요. 추측은 하지 마세요.
```

**기대 결과** (예시):
```
가장 큰 단일 월 매출은 "챗GPT로 콘텐츠 10배 빠르게"입니다.
근거: 2026-01 행 — 수강신청 187명, 매출 1,477만원 (CSV 20번째 줄).
이 강의는 2025-08 814만원에서 시작해 꾸준히 올랐습니다.
```

**체크포인트**: 답에 **실제 줄·실제 숫자**(1,477만원 등)가 인용돼 있는지 본다. 만약 두루뭉술하게 "약 1,400만원대"처럼만 말하면 → "CSV에 적힌 **정확한 값**을 그대로 보여주세요"라고 한 번 더 시킨다. *이 한 번 더 시키기*가 비개발자가 AI를 믿고 쓰는 핵심 안전 습관이다.

---

### 6단계: "다시 해줘"의 힘 — 피드백 루프 체감

```text
방금 설명이 조금 길어요. 사장님(나 자신)한테 보고하듯, 한 문장으로 요약해주세요.
```

**기대 결과** (예시):
```
"챗GPT로 콘텐츠 10배 빠르게" 강의가 2026년 1월 1,477만원으로 우리 강의 중 단일 월 매출 1위입니다.
```

**관찰 포인트**: 한 번에 완벽하지 않아도 된다. "더 짧게 / 더 쉽게 / 표로 / 다시"라고 말하면 AI가 같은 자료로 모양만 바꿔 답한다. 에이전트 루프는 *대화하며 다듬는* 도구다.

## 🎬 실습 2 — 내 사업 데이터로 첫 와우 만들기 (13분)

목표: 숫자 파일과 메모 파일을 *연결*해, "혼자였으면 30분 걸릴 정리"를 AI와 3분에 끝낸다.

### 1단계: 매출 흐름 3가지 패턴 뽑기

```text
course_sales_monthly.csv 전체를 읽고, 1인 사업자가 의사결정에 쓸 만한 "눈에 띄는 패턴" 3가지를 찾아주세요.
각 패턴마다 CSV의 실제 강의명·월·숫자를 인용해주세요. 비개발자도 이해되게요.
```

**기대 결과** (실제 데이터 기반 예시):
```
1. AI활용 강의가 매출 견인차
   - "챗GPT로 콘텐츠 10배 빠르게": 2025-08 814만원 → 2026-01 1,477만원으로 성장
   - 가장 꾸준히 팔리는 효자 강의

2. 비싼 강의가 객단가로 매출 보완
   - "1인 지식사업 수익화 로드맵": 객단가 149,000원 (가장 높음)
   - 2026-01 89명 신청 → 매출 1,326만원

3. 신규 강의의 빠른 안착
   - "클로드코드로 업무 자동화 입문": 2026-02 출시 46명(547만원)
     → 2026-03 118명(1,404만원)으로 두 달 만에 상위권
```

**체크포인트**: 답의 숫자(814만원, 1,477만원, 149,000원, 547만원, 1,404만원)가 실제 CSV 값과 맞는지 한두 개만 눈으로 대조해 본다. 맞으면 — **이게 첫 와우다.** 내가 엑셀로 피벗 안 돌리고, AI가 직접 읽어서 정리했다.

---

### 2단계: "왜 그랬을까?"의 단서를 메모에서 찾기 (파일 연결)

숫자만으로는 *이유*를 모른다. AI에게 메모 파일을 읽혀 단서를 잇게 한다. (앞서 본 매출 내용을 *기억한 채로* 다음 파일을 읽는 — Context가 이어지는 장면이다.)

```text
방금 본 매출 흐름의 "이유"가 될 만한 단서를, research_memo.md와 content_plan.md에서 찾아 연결해주세요.
"매출 관찰 → 메모 속 단서" 형태로 짝지어주세요.
```

**기대 결과** (예시):
```
- 관찰: AI활용 강의가 매출 1위
  단서: research_memo "매출 기여 1위 = AI활용 카테고리" / content_plan
        "AI·자동화 키워드 콘텐츠가 오픈율·클릭율이 확연히 높다"

- 관찰: 신규 클로드코드 강의가 빠르게 안착
  단서: research_memo "신규 클로드코드 강의(119,000원)는 출시 두 달 만에 단일 강의 매출 상위권"

- 관찰: 비즈니스 강의는 객단가로 보완
  단서: research_memo "객단가 최고 = 1인 지식사업 수익화 로드맵(149,000원)"
```

**관찰 포인트**: AI가 *두 개의 다른 파일*을 읽어 한 이야기로 엮었다. 숫자(CSV) + 맥락(메모)을 잇는 이 작업이 "의사결정 가능한 인사이트"가 되는 지점이다.

### 3단계: 고객 문의에서 반복되는 신호 찾기

```text
student_inquiries.csv를 읽고, 가장 자주 들어오는 문의 카테고리 3가지와, 그게 무슨 뜻인지 정리해주세요.
처리에 시간이 오래 걸린 문의가 있으면 따로 짚어주세요.
```

**기대 결과** (예시):
```
가장 잦은 문의:
1. 실습 관련 (가장 많음) — 터미널 설치 막힘, WSL 설치 어려움,
   AGENTS.md/CLAUDE.md 위치를 모름 → 신규 "클로드코드" 강의에 집중
2. 환불 — "생각했던 내용과 달라서" 사유가 반복
3. 강의문의(인스타DM) — "비개발자도 따라갈 수 있나요" 질문 다수

오래 걸린 문의:
- "윈도우 WSL 설치가 어려워요"(3.0시간), "터미널 설치 에러"(2.0시간)
  → 설치 단계 진입장벽이 큼
```

**체크포인트**: 실제 CSV에 '실습' 카테고리(터미널·WSL·AGENTS.md·CLAUDE.md 관련)가 가장 많고, '환불' 사유로 "생각과 다름"이 반복된다. AI가 *데이터에 실제로 있는* 신호를 짚었는지 확인한다. content_plan.md의 "환불 사유 중 '생각과 다름'이 반복됨"과도 맞아떨어진다 — 숫자가 메모를 증명한다.

### 4단계: 세 파일을 한 문단으로 잇기

```text
지금까지 본 매출(course_sales_monthly.csv), 시장 단서(research_memo.md), 고객 문의(student_inquiries.csv)를
연결해서, "지금 내 사업에 무슨 일이 일어나고 있나"를 한 문단(5~7줄)으로 설명해주세요.
```

**기대 결과** (예시):
```
2026년 들어 매출의 중심이 AI활용 강의로 확실히 옮겨갔다. 효자 강의 "챗GPT로 콘텐츠
10배"가 1월 1,477만원으로 정점을 찍었고, 신규 "클로드코드로 업무 자동화 입문"이 출시
두 달 만에 1,404만원으로 따라붙으며 라인업 확장 신호를 보인다. 다만 신규 강의 수강생의
문의가 '터미널·WSL 설치'처럼 진입 단계에 몰려 있어(최장 3시간 소요), 초반 이탈과 "생각과
다름" 환불 위험이 함께 커졌다. 즉 성장은 분명하지만, 그 성장을 받쳐줄 '설치·온보딩 안내'를
자동화하지 않으면 문의·환불이 매출 발목을 잡을 수 있다.
```

💡 **이 단계가 중요한 이유**: 숫자만 보면 "잘 되네"에서 끝나고, 문의만 보면 "왜 이리 막히지"에서 끝난다. 둘을 *연결*하면 "성장 중이니 온보딩을 자동화하자"는 **행동 가능한 결론**이 나온다. 이게 에이전트 루프가 한 사업 사이클(읽기→분석)을 끝내는 모습이다.

3. 효자 강의(AI활용) 라인업 한 개 더 기획

## 🔍 흔한 오류와 해결법 (트러블슈팅)

| 상황 | 원인 | 해결 |
|------|------|------|
| "파일을 못 찾겠다"고 함 | Claude Code를 엉뚱한 폴더에서 켬 | 종료 후 `cd practice-data/creator` → `pwd`로 확인 → 다시 `claude` |

| 응답이 멈춤·이상함 | 일시 오류 | `Esc`로 중단 후 다시 시도, 안 되면 종료 후 `claude` 재실행 |

## 💡 핵심 개념 정리

### 1. 에이전트 루프 = "읽고 → 도구로 움직이고 → 답한다"
챗봇은 *말만* 하지만 Claude Code는 *행동*한다. 한 번 시키면 스스로 폴더를 열고 파일을 읽고 정리한다. 부족하면 다시 한 바퀴 돈다.

### 2. 도구(Tools)와 권한(Permissions)은 한 쌍
AI의 '손'이 도구(파일 읽기·탐색·실행)고, 그 손을 쓰기 전 한 번 묻는 게 권한이다. 묻는 건 고장이 아니라 *내 폴더를 함부로 안 건드린다는 약속*이다.

### 3. "추측 말고 실제 값으로" 한마디가 신뢰를 만든다
AI는 그럴듯하게 지어낼 수 있다. "CSV 실제 숫자로 근거 보여주세요"를 습관처럼 붙이면 비개발자도 안심하고 결과를 쓸 수 있다.

### 4. 한 번에 완벽 ❌, 대화로 다듬기 ✅
"더 짧게 / 표로 / 다시"는 실패가 아니라 정상 사용법이다. 에이전트 루프는 대화하며 가까워지는 도구다.

## 📊 이번 레슨 사이클

```
   내 지시 ("이 폴더 매출 어때요?")
        │
        ▼
 ① 상황 파악  ── Claude Code가 폴더를 스스로 훑음 (Context)
        │
        ▼
 ② 행동      ── README·CSV·메모를 직접 읽음 (Tools)
        │        └─ 읽기 전 "읽어도 돼요?" 한 번 물음 (Permissions)
        ▼
 ③ 결과 확인  ── 실제 숫자가 맞는지 정리 (할루시네이션 차단)
        │
        ▼
   답변  ── "챗GPT 강의 1월 1,477만원 1위" 같은 내 데이터 기반 보고
        │
        ▼
   피드백 ("한 문장으로!") → 필요하면 ①로 다시 (루프 반복)
```

이 한 바퀴가 **에이전트 루프**이고, 오늘 그걸 내 사업 데이터로 직접 돌려봤다.
