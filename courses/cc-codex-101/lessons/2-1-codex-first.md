# 2-1: Codex 첫 실습 — 설치·첫 실행 + effort·모델로 답이 달라진다

## 학습 목표
- Codex(OpenAI의 에이전트형 코딩 도구)를 설치·로그인·실행할 수 있다.
- Claude Code에서 한 것과 같은 종류의 일을 Codex로 해본다.
- **effort(추론 강도)와 모델 선택에 따라 같은 요청의 답이 어떻게 달라지는지** 직접 본다(이 레슨의 핵심).

## 사전 준비
- 모듈 1(Claude Code 기초) 완료.
- 인터넷 + 터미널. (Codex 설치를 이 레슨에서 함께 한다.)
- ChatGPT 계정(로그인용). 없으면 강사 안내.
- `practice-data/` 폴더(모듈 1과 동일).

## 진행

### 도입 (4분)
- Codex = "코드를 읽고·고치고·테스트하고·설명하는" 코딩 에이전트. Claude Code와 **같은 종류의 도구**, 만든 회사(OpenAI)·모델만 다르다.
- 오늘은 ① 설치·첫 실행 ② 같은 일 시켜보기 ③ **effort·모델 바꿔 답 차이 보기**.
- ⚠️ 모델 이름·명령은 버전에 따라 바뀔 수 있다 — 화면에 보이는 실제 옵션 기준으로 진행.

### 실습 1 — 설치·로그인·첫 실행 (12분)
따라 하기:
1. 설치(택1, 안내에 맞춰 하나만).
```bash
npm install -g @openai/codex
```
```bash
brew install codex
```
2. 로그인(ChatGPT 계정 권장).
```bash
codex login
```
3. 실습 폴더로 이동해 실행.
```bash
codex --cd /실습폴더경로
```
4. 첫 인사 + 폴더 파악 요청(모듈 1에서 배운 방식 그대로).
```text
저는 비개발자 운영기획 실무자예요. practice-data 폴더에 뭐가 있고 무엇부터 보면 좋은지 알려주세요.
```

기대 결과: Codex가 폴더를 읽고 제안을 준다 → "Claude Code랑 비슷하네"라는 감각.

### 실습 2 — effort(추론 강도) 차이 보기 🎬 (12분)
> 핵심 시연: **같은 요청**을 effort 만 바꿔 두 번 → 답의 깊이·길이 차이 관찰.

따라 하기:
1. 먼저 낮은 effort로. 세션에서 `/model` 로 effort 를 `low` 로 맞추거나(메뉴에서 선택), 빠른 모드로 둔다. 그리고:
```text
practice-data/meeting_notes.md 를 읽고 반복되는 운영 문제를 정리해줘.
```
→ 답을 보고 **길이·상세함**을 기억해 둔다(보통 빠르고 짧음).
2. 이제 effort 를 `high` 로 올린다(`/model` 로 reasoning effort 변경).
```text
(같은 요청) practice-data/meeting_notes.md 를 읽고 반복되는 운영 문제를 정리해줘.
```
→ 보통 더 신중하게: 먼저 파일을 짚고, 근거를 들고, 더 구조화된 답.
3. 두 답을 나란히 비교한다.
```text
방금 두 번의 답이 어떻게 달랐는지, effort 가 무슨 역할을 하는지 한 줄로 설명해줘.
```

관찰 포인트(강사 멘트):
- **effort ↑ = 더 신중·검증·길다 / ↓ = 더 빠르다.**
- 단순 작업엔 낮게, 복잡하거나 위험한 작업엔 높게.

### 실습 3 — 모델 선택 차이 보기 🎬 (10분)
따라 하기:
1. `/model` 로 빠른·가벼운 모델(예: `gpt-5.4-mini` 류)로 바꾼다. 단순 작업:
```text
practice-data/README.md 를 3줄로 요약해줘.
```
→ 빠르고 충분.
2. `/model` 로 상위 모델(예: `gpt-5.5`)로 바꾼다. 더 복잡한 작업:
```text
sales_data.csv 와 meeting_notes.md 를 묶어서, 7~8월에 무슨 일이 있었는지 근거와 함께 분석해줘.
```
→ 더 깊은 추론.
3. 정리한다.
```text
어떤 작업에 가벼운 모델, 어떤 작업에 상위 모델이 맞는지 한 문장으로 알려줘.
```

포인트: **작업 난이도에 모델·effort 를 맞춘다 = 속도·비용·품질 균형.** (Claude Code도 모델·effort 선택은 같은 개념.)

⚠️ **이미 배운 범위만**: 이 레슨은 설치·첫 실행·effort·모델까지만. AGENTS.md·MCP 같은 세팅은 다음 레슨에서 배운다 — 지금 미리 세팅된 상태로 건너뛰지 않는다.

## 트러블슈팅
- **설치 실패**: 권한/네트워크 확인. `npm` 안 되면 `brew`(mac) 또는 강사 안내 설치 경로로.
- **로그인 안 됨**: `codex login` 재시도, ChatGPT 구독/계정 확인.
- **모델이 안 보임**: 구독 등급 확인 + CLI 업데이트(`npm update -g @openai/codex`) + 보이는 기본 모델로 진행.
- **답이 느리다**: effort 를 낮추거나 작업을 쪼갠다(실습 2에서 배운 대로).
- **effort 메뉴를 못 찾음**: `/model` 입력 → 모델·reasoning effort 옵션 확인. 없으면 보이는 옵션으로만 비교.
