# codex-fc-6 (강의 클립): CC↔Codex 같은 것·다른 것
> 대응 자습 레슨: `/lesson-cc-codex codex-11` · 강의 클립 codex-fc-6 · 예상 12~15분
> 이 문서 = 강의 촬영·진행용 런시트(이론 짧게 → 터미널 라이브). 깊은 자습은 위 레슨으로.

## 🎬 이 클립 한 줄 목표
CC를 익혔으면 Codex는 90%가 따라온다는 것을 나란히 실행해서 눈으로 확인한다.

## 🗣️ 이론 (강사 설명 · 2~4분)
- CC와 Codex는 만든 회사(Anthropic vs OpenAI)만 다를 뿐, 설치·로컬 파일 접근·세션 명령·MCP 연결·긴 대화 정리 — 뼈대가 같다.
- 비유: 삼성 스마트폰에서 LG로 바꾼 것. 전화·문자·앱 — 다 되고, 앱 배치만 약간 다르다. 한 달이면 완전히 익숙해진다.
- 핵심 차이 딱 3가지만 외우면 된다. ① 규칙서 이름(`CLAUDE.md` vs `AGENTS.md`) ② 스킬 호출 기호(`/스킬이름` vs `$스킬이름`) ③ 샌드박스 표기(승인 게이트 vs `read-only` / `workspace-write` / `danger-full-access` 3단계 명칭).
- 모델 이름도 다르다. CC = Claude 계열, Codex = `gpt-5.5` / `gpt-5.4-mini`.
- 세션 관리 명령(`/compact`, `/model`, `/plan`)은 Codex에서도 `/`를 그대로 쓴다. `$`는 스킬(반복 작업 묶음)을 부를 때만.

## ⌨️ 터미널 라이브 시연 (8~10분)

### 스텝 1 — 데이터 폴더에서 CC 켜기
- **시연**: `cd ~/practice-data/creator` 후 `claude`
- **보여줄 것**: 입력창(`>`)이 뜨면 준비 완료. CC는 현재 폴더를 무대로 작동한다.
- **학습자에게**: "직접 같은 명령을 입력해보세요. `>` 가 뜨면 성공입니다."

### 스텝 2 — CSV 파일 읽고 분석 요청
- **시연**:
```
나는 온라인 강의를 운영하는 1인 사업자 지우야.
@course_sales_monthly.csv 를 읽고 순매출이 가장 높았던 강의 TOP 3을 알려줘.
2025-07부터 2026-03까지 합산 기준으로. 결과는 표로 정리해줘.
```
- **보여줄 것**: CC가 실제 CSV 수치를 읽어 표로 정리한다. 챗GPT와 달리 파일 안의 숫자를 직접 근거로 쓴다는 점이 핵심.
- **학습자에게**: "여러분 트랙 데이터로 같은 질문을 해보세요."

### 스텝 3 — 새 터미널에서 Codex 켜기 (샌드박스 포함)
- **시연**: `codex --cd ~/practice-data/creator --sandbox read-only "course_sales_monthly.csv 에서 순매출 TOP 3 강의를 알려줘. 2025-07부터 2026-03까지 합산 기준으로."`
- **보여줄 것**: CC와 거의 같은 답이 나온다. `--cd`로 폴더 지정하는 것이 CC의 `cd → claude`와 같은 역할. `--sandbox read-only`는 파일을 보기만 하고 아무것도 바꾸지 않는 가장 안전한 설정.
- **학습자에게**: "직접 입력해보고 CC 결과와 숫자가 같은지 비교해보세요."

### 스텝 4 — `/status`로 샌드박스 모드 확인
- **시연**: Codex 세션 입력창에 `/status`
- **보여줄 것**: 모델(`gpt-5.5`), effort(`medium`), 샌드박스 모드(`read-only`) 한눈에 표시. CC의 `/context`와 같은 역할, 명령 이름만 다르다.
- **학습자에게**: "직접 `/status` 입력해서 모드를 확인해보세요."

### 스텝 5 — 가벼운 모델로 바꿔보기
- **시연**: `codex --cd ~/practice-data/creator -m gpt-5.4-mini "course_sales_monthly.csv 에서 2026-03 한 달 데이터만 보고 강의별 순매출을 정리해줘. 빠르게만."`
- **보여줄 것**: `gpt-5.4-mini`(빠르고 가벼움) vs `gpt-5.5`(깊고 정확). 작업 난이도에 맞춰 모델을 고르는 것이 두 도구 공통 원리.
- **학습자에게**: "직접 -m gpt-5.4-mini 로 바꿔서 응답이 어떻게 달라지는지 비교해보세요."

### 스텝 6 — 규칙서 이름 차이 확인 (`AGENTS.md` vs `CLAUDE.md`)
- **시연**: Codex 세션 안에서 `/init` 입력 → `AGENTS.md` 뼈대 생성 확인. CC 창에서 `cat CLAUDE.md`로 나란히 비교.
- **보여줄 것**: 두 파일 구조가 거의 같다. 프로젝트 소개·규칙·완료 기준 — 이름만 다르고 역할이 같다.
- **학습자에게**: "두 파일을 나란히 보고 어디가 같고 어디가 다른지 직접 찾아보세요."

## ✅ 마무리 (1분)
CC와 Codex는 뼈대가 같고 브랜드·모델·기호 세 가지만 다르다. 하나를 익혔으면 나머지 90%는 따라온다. 더 깊은 실습(세 파일 연결 분석, 비교 보고서 초안 만들기, 스킬 호출 체험)은 `/lesson-cc-codex codex-11`에서. 다음 클립(km-01)에서는 오늘 분석한 데이터를 검색되는 지식으로 만드는 작업을 시작한다.

## 📌 강사 노트
- **자주 막히는 지점 1**: `command not found: codex` — Codex 설치 후 터미널을 닫았다 다시 열어야 인식된다. `npm install -g @openai/codex` 재실행으로도 해결.
- **자주 막히는 지점 2**: `gpt-5.5` 모델이 없다는 오류 — `npm update -g @openai/codex`로 업데이트하거나, `gpt-5.4-mini`로 진행하고 나중에 구독 상태 확인 권유. `codex login` 재로그인으로 구독 상태 갱신되기도 함.
- 수강생이 "CC와 Codex 답이 많이 다른데요"라고 하면: 숫자(파일 실수치)는 같아야 정상, 표현만 다를 수 있다고 안내. 다르다면 "실제 파일 수치로 계산해줘"라고 다시 요청하게 유도.
