# fc-03 (강의 클립): 파일 다루기 + 명령어
> 대응 자습 레슨: `/lesson-cc-codex cc-03` · 강의 클립 1.3.1 · 예상 12~15분
> 이 문서 = 강의 촬영·진행용 런시트(이론 짧게 → 터미널 라이브). 깊은 자습은 위 레슨으로.

## 🎬 이 클립 한 줄 목표
내 사업 폴더로 정확히 찾아 들어가(`cd`), 원하는 파일을 콕 집어 Claude Code에게 읽혀(`@파일명`), 대화 흐름을 가볍게 굴리는(`/cost` · `/compact`) 손동작 세트를 한 번씩 직접 해본다.

## 🗣️ 이론 (강사 설명 · 2~4분)

- **"AI가 내 자료를 못 찾는다"의 90%는 엉뚱한 폴더에서 켠 것** — Claude Code는 "터미널이 지금 서 있는 폴더"를 작업 무대로 삼는다. 자료 폴더 안에서 켜는 것이 전부.
- **길 찾기는 세 글자면 충분하다**: `pwd`(여기 어디지?) → `ls`(뭐가 있지?) → `cd`(저기로 가자). 탐색기에서 폴더를 더블클릭하는 것을 글자로 하는 것.
- **비유 — 새 직원 첫날 출근**: "몇 층이죠?"(`pwd`) → "이 방에 뭐가 있나요?"(`ls`) → "자료실은 옆 방이에요, 거기 가요"(`cd`) → "이 매출 보고서 좀 봐줘요"(`@파일명`). 이 네 손동작이 전부.
- **명령은 딱 두 입구**: "켜기 전 터미널(`$`)"에서는 `claude`처럼 AI를 켜고 끄는 명령, "대화 중 채팅창"에서는 `/compact` · `/cost` · `/clear`처럼 슬래시(/) 명령. 헷갈리면 엑셀을 먼저 켜고(`claude`) 그 안에서 작업하는 것으로 기억.
- **안심**: 명령을 잘못 쳐도 컴퓨터는 안 망가진다. 파일을 읽혀도 인터넷에 올라가지 않는다. 길을 잃으면 `cd ~`로 무조건 홈으로 리셋.

## ⌨️ 터미널 라이브 시연 (8~10분)

**[스텝 1] 지금 어디 있는지 확인하고, 실습 폴더로 이동**

- **시연**:
  ```
  pwd
  ls
  cd ~/practice-data/creator
  pwd
  ls
  ```
- **보여줄 것**: `pwd`가 현재 경로를 한 줄로 출력. `cd` 후 `pwd`/`ls`로 실습 폴더 안의 `course_sales_monthly.csv` · `newsletter_stats.csv` · `student_inquiries.csv` 등 7개 파일이 보임.
- **학습자에게**: "직접 `cd ~/practice-data/creator` 치고, `ls`로 파일 목록이 뜨는지 확인해 보세요."

---

**[스텝 2] 자료 폴더 안에서 Claude Code 켜기**

- **시연**:
  ```
  claude
  ```
- **보여줄 것**: 입력창(`>`)이 뜨고 대화 대기 상태가 됨. "폴더 안에서 켰다"는 것이 핵심.
- **학습자에게**: "직접 `claude`를 입력해 대화방에 들어가 보세요."

---

**[스텝 3] `@파일명`으로 매출 CSV 콕 집어 읽히기**

- **시연**:
  ```
  @course_sales_monthly.csv 이 매출 데이터에서 눈에 띄는 점 3가지를,
  비개발자도 이해되게, 실제 숫자를 인용해서 짧게 설명해줘.
  ```
- **보여줄 것**: `@`를 치면 파일 목록이 자동으로 뜨는 것 → 파일을 복사·붙여넣기 없이 그 파일만 정확히 읽혀서 실제 수치가 담긴 분석 3가지가 나옴.
- **학습자에게**: "직접 `@`를 쳐서 파일 목록이 뜨면, `course_sales_monthly.csv`를 골라 같은 질문을 해보세요."

---

**[스텝 4] 비용 확인 — `/cost`**

- **시연**:
  ```
  /cost
  ```
- **보여줄 것**: 이번 세션에서 쓴 토큰(대화 단위 사용량)과 대략의 비용 요약이 표시됨.
- **학습자에게**: "직접 `/cost`를 쳐서 이번 대화가 얼마나 됐는지 확인해 보세요."

---

**[스텝 5] 대화 정리 — `/compact`**

- **시연**:
  ```
  /compact
  ```
- **보여줄 것**: 긴 대화가 핵심만 남기고 요약됨. 화면은 깔끔해지지만 맥락은 유지. 직후 `/cost`로 다시 보면 가벼워진 것을 확인 가능.
- **학습자에게**: "직접 `/compact`를 쳐보고, `/cost`로 전후를 비교해 보세요."

---

**[강사 팁 — 시연 도중 슬쩍 언급]**
- `Esc`(또는 `Ctrl+C` 한 번): AI가 엉뚱한 방향으로 가면 즉시 중단.
- `/clear`: 완전히 다른 일로 갈아탈 때 대화를 백지로(되돌릴 수 없으니 중요한 결과는 먼저 저장).
- `/compact`는 정리하되 기억 유지, `/clear`는 기억까지 삭제. 헷갈리면 `/compact`가 더 안전.

## ✅ 마무리 (1분)

오늘 손에 익힌 건 딱 두 가지다. **`cd`로 자료 폴더 안에 서고, `@`로 파일을 정확히 쥐여준다.** 이 두 동작이 Claude Code와 일하는 모든 실습의 기본 골격이고, 이후 Codex 실습에서도 같은 감각 그대로 쓴다. 더 깊은 내용(에러 상황·세션 이어 하기·비용 관리 심화)은 `/lesson-cc-codex cc-03`에서 확인. 다음 클립(1.4)에서는 복잡한 일을 안전하게 시키는 **Plan 모드(`Shift+Tab`)와 좋은 프롬프트 5원칙**으로 넘어간다.

## 📌 강사 노트

- **자주 막히는 지점 1 — "파일을 못 찾는다고 해요"**: 십중팔구 자료 폴더가 아닌 엉뚱한 폴더에서 `claude`를 켠 것. `Ctrl+C`로 나오게 하고 → `pwd`로 위치 확인 → `cd ~/practice-data/creator` → `ls`로 파일 보이면 다시 `claude`. "켜기 전에 `ls`로 파일이 보이는지 먼저 확인" 습관 강조.
- **자주 막히는 지점 2 — "/compact를 터미널에서 쳤더니 안 돼요"**: 슬래시 명령을 대화방 밖(켜기 전)에서 입력한 것. `claude`로 대화방에 먼저 들어간 뒤 그 안에서 `/compact`를 치도록 안내. "켜기 전 / 대화 중 두 칸" 그림을 다시 한 번 보여주면 효과적.
