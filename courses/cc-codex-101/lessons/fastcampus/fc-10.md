# fc-10 (강의 클립): 모델·effort·응답 차이
> 대응 자습 레슨: `/lesson-cc-codex codex-03` · 강의 클립 1.2.4 · 예상 12~15분
> 이 문서 = 강의 촬영·진행용 런시트(이론 짧게 → 터미널 라이브). 깊은 자습은 위 레슨으로.

## 🎬 이 클립 한 줄 목표

같은 부탁을 모델과 effort(추론 강도)를 바꿔가며 던져보고, 답의 깊이·속도가 달라지는 것을 눈으로 확인한다.

## 🗣️ 이론 (강사 설명 · 2~4분)

- **effort = "얼마나 깊이 생각할지"** 다이얼. `low → medium → high → xhigh` 순서로 꼼꼼함이 올라간다.
- **model = "어떤 엔진을 부를지"** 선택. 가벼운 엔진(`gpt-5.4-mini`)은 빠르고, 무거운 엔진(`gpt-5.5`)은 깊다.
- 비유: 아르바이트생에게 "빠르게 숫자만 봐줘" vs "꼼꼼히 읽고 패턴까지 찾아줘"라고 시키는 차이.
- **무조건 최고급을 쓸 필요 없다.** 단순 확인엔 가벼운 모델·낮은 effort = 빠르고 저렴. 중요한 결정엔 강한 모델·높은 effort = 신중하고 정확.
- 이 감각은 Claude Code에서도 동일하게 통한다. 도구 이름만 다를 뿐 원리는 같다.

### 스텝 3 — 가벼운 모델로 빠른 답 받기

- **시연**: `/model gpt-5.4-mini` 입력 후 아래 질문
  ```
  course_sales_monthly.csv 파일에서 2026년 3월 강의별 순매출을 빠르게 알려줘.
  ```
- **보여줄 것**: 몇 초 안에 강의별 숫자만 깔끔하게 나옴. 이유·패턴 분석은 없음.
- **학습자에게**: "직접 같은 질문을 해보세요. 얼마나 빨리 나오는지 시간을 봐보세요"

---

### 스텝 4 — 무거운 모델로 같은 주제 깊이 분석

- **시연**: `/model gpt-5.5` 입력 후 아래 질문
  ```
  course_sales_monthly.csv 에서 2026년 3월 강의별 순매출 흐름을 정리해 줘.
  어떤 강의가 성장 중이고, 어떤 강의가 줄고 있는지도 봐줘.
  특히 2월 출시한 클로드코드 강의가 기존 강의들과 비교해 어떤지 눈에 띄는 패턴이 있으면 알려줘.
  ```
- **보여줄 것**: 전월 대비 성장률, 패턴 해석, 흐름 분석까지 포함된 긴 답변. 시간은 10~30초 더 걸림.
- **학습자에게**: "스텝 3 답변과 이번 답변을 비교해 보세요. 무엇이 달라졌나요?"

---

### 스텝 5 — 멀티파일 교차 분석 (xhigh effort 체험)

- **시연**: `/model` 메뉴에서 effort를 `xhigh`로 설정 후
  ```
  newsletter_stats.csv 에서 클릭율이 높았던 호의 공통점을 찾아줘.
  그리고 course_sales_monthly.csv 의 강의 흐름과 연결해서,
  4월에 뉴스레터로 어떤 강의를 어떤 메시지로 홍보하면 좋을지 제안해 줘.
  ```
- **보여줄 것**: 두 파일을 동시에 읽고 교차 분석한 뒤 제목 후보까지 제안. `/status`로 토큰 사용량이 늘어난 것도 함께 확인.
- **학습자에게**: "xhigh를 언제 쓰면 좋을지 스스로 판단해 보세요"

## ✅ 마무리 (1분)

같은 부탁도 모델(`gpt-5.4-mini` vs `gpt-5.5`)과 effort(`low`→`xhigh`)에 따라 답의 속도·깊이·비용이 달라진다. 단순 확인은 가볍게, 중요한 결정은 꼼꼼하게 — 이 감각 하나가 Codex를 잘 쓰는 핵심이다.

더 깊은 내용과 전체 실습은 `/lesson-cc-codex codex-03`에서 확인하세요.

다음 클립(1.2.5)에서는 **Codex의 안전장치** — "Codex가 내 파일을 함부로 바꾸지 않도록 권한을 내가 조절하는 방법"을 다룹니다.
