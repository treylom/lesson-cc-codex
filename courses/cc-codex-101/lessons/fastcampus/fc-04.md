# fc-04 (강의 클립): 워크플로우 + Plan + 좋은 프롬프트

> 대응 자습 레슨: `/lesson-cc-codex cc-04` · 강의 클립 1.3.1/1.3.3 · 예상 12~15분
> 이 문서 = 강의 촬영·진행용 런시트(이론 짧게 → 터미널 라이브). 깊은 자습은 위 레슨으로.

---

## 🎬 이 클립 한 줄 목표

"어떻게 요청하느냐"가 AI 결과물을 바꾼다는 것을 같은 데이터로 직접 비교하고, Plan 모드로 복잡한 일을 안전하게 시키는 법을 터미널에서 익힌다.

---

## 🗣️ 이론 (강사 설명 · 2~4분)

- **AI는 주문을 받는 식당 직원이다.** "밥 주세요"(막연)와 "매운 거 못 먹어요, 소자로 주세요"(구체)는 완전히 다른 결과를 낳는다. Claude Code도 마찬가지다.
- **좋은 프롬프트(AI 지시)의 공식은 세 덩어리다.** `[역할 또는 맥락] + [읽을 파일] + [원하는 결과와 형식]`
- **Plan 모드 = 인테리어 도면 먼저 받기.** 공사를 시작하기 전에 계획서를 보고 OK하면 그때 실행한다. 중간에 방향이 틀려도 되돌릴 게 없다.
- **Plan 모드에서는 파일을 절대 수정하지 않는다.** 계획서만 보여준다. "혹시 잘못되면?" 걱정 없이 먼저 확인할 수 있다.
- **다시 시키는 것이 기술이다.** 첫 답은 70점이어도 된다. "이 부분 짧게 해줘" 한 마디로 85→95점까지 올릴 수 있다.

---

## ⌨️ 터미널 라이브 시연 (8~10분)

### 스텝 1 — 실습 폴더로 이동하고 Claude Code 켜기

- **시연**:
  ```bash
  cd ~/practice-data/creator
  claude
  ```
- **보여줄 것**: `>` 입력창이 뜨면 성공. 이 폴더 안 파일들이 Claude Code의 작업 공간이 된다.
- **학습자에게**: "직접 터미널을 열고 `cd ~/practice-data/creator`, `claude` 순서로 입력해 보세요."

---

### 스텝 2 — 막연한 요청 vs 구체적 요청 비교

먼저 막연하게 시켜본다.

- **시연** (대화창에 입력):
  ```text
  매출 데이터 분석해줘.
  ```
- **보여줄 것**: 뭉뚱그린 안내 또는 "어떤 파일인가요?" 역질문. 구체적인 숫자가 없다.

바로 이어서 같은 주제를 구체적으로.

- **시연**:
  ```text
  너는 내 온라인 강의 사업의 데이터 분석가야.
  course_sales_monthly.csv를 읽고,
  강의별로 2025-07부터 2026-03까지의 순매출(만원) 합산을 계산해서
  1위부터 5위까지 표로 정리해줘.
  실제 CSV에 있는 숫자를 인용해야 해.
  ```
- **보여줄 것**: 강의별 순매출 합산 표가 실제 CSV 숫자로 나온다. 막연한 요청과 결과가 눈에 띄게 다르다.
- **학습자에게**: "직접 위 프롬프트를 그대로 붙여넣고 결과를 확인해 보세요. 숫자가 이상하면 '실제 CSV 값을 확인해서 다시 계산해줘'라고 한 번 더 요청해 보세요."

---

### 스텝 3 — Plan 모드 켜고 복잡한 일 시키기

- **시연**: 대화창 안에서 키보드 `Shift + Tab`을 **두 번** 누른다. 화면 상단에 `plan mode` 표시 확인.

  ```text
  내 사업 데이터를 종합 분석해서 이번 달 보고서를 만들고 싶어.
  course_sales_monthly.csv, newsletter_stats.csv, student_inquiries.csv
  세 파일을 읽고, 각 파일에서 핵심 인사이트 2개씩 뽑은 뒤
  "관찰 → 해석 → 제안" 구조로 보고서 초안을 파일로 저장해줘.
  ```
- **보여줄 것**: Claude Code가 바로 실행하는 게 아니라 5단계 계획서를 먼저 보여준다. "이렇게 진행해도 될까요?"
- **학습자에게**: "직접 Plan 모드를 켜고 같은 요청을 입력해 보세요. 계획이 마음에 들면 '좋아, 진행해줘'를 입력하고, 마음에 안 들면 '3단계를 빼줘'처럼 조정해 보세요."

---

### 스텝 4 — 결과에 피드백 주기 ("다시 시키기")

- **시연** (보고서가 나온 뒤):
  ```text
  고마워. 그런데 보고서가 너무 길어.
  핵심만 남기고, 제목은 "2026년 3월 사업 현황 요약"으로 바꿔서
  전체를 10줄 이내로 다시 써줄래?
  ```
- **보여줄 것**: 훨씬 짧고 핵심만 담긴 버전이 나온다.
- **학습자에게**: "직접 '이 부분을 더 짧게' 또는 '표 형식으로 다시' 같은 피드백을 입력해 보세요. AI는 짜증 내지 않는다는 것을 확인해 보세요."

---

### 스텝 5 — 보고서 파일로 저장하기

- **시연**:
  ```text
  이 보고서를 "report_2026Q1_draft.md"라는 파일명으로
  현재 폴더에 저장해줘.
  ```
- **보여줄 것**: `practice-data/creator/report_2026Q1_draft.md` 파일이 실제로 생성된다. 폴더를 열어 파일이 있는지 확인.
- **학습자에게**: "직접 파일 이름을 정해서 저장 요청을 해보세요. Claude Code가 저장 허락을 물어보면 '응'이라고 입력하면 됩니다."

---

## ✅ 마무리 (1분)

- 이번 클립의 핵심: **"역할 + 파일 + 형식" 공식**으로 요청하면 AI의 첫 답이 눈에 띄게 좋아진다. **Plan 모드(Shift + Tab 두 번)**는 복잡한 여러 단계 작업에서 방향을 먼저 확인하는 안전장치다.
- 더 깊이 익히고 싶으면 → `/lesson-cc-codex cc-04`에서 뉴스레터·수강생 문의 데이터 분석, 숫자와 기획 메모 연결하기까지 이어서 해볼 수 있다.
- 다음 클립에서는: 이번에 "항상 비개발자 눈높이로", "실제 숫자를 꼭 인용해줘"를 매번 반복한 것 — 그걸 `CLAUDE.md` 규칙서 파일 한 번에 박아두면 다음부터는 말 안 해도 AI가 알아서 그 규칙대로 일하는 방법을 익힌다.

---

## 📌 강사 노트

- **자주 막히는 지점 1 — Plan 모드가 안 켜진다**: `Shift + Tab`을 Claude Code 대화창(`>` 프롬프트) 바깥에서 눌렀거나 한 번만 눌렀을 때 생긴다. 대화창 안으로 커서를 옮기고 두 번 눌러 `plan mode` 표시를 확인하게 안내한다.
- **자주 막히는 지점 2 — "파일을 찾을 수 없다"**: `claude`를 다른 폴더에서 켰을 때 발생한다. `/exit`로 나간 뒤 `cd ~/practice-data/creator`로 이동하고 다시 `claude`를 입력하게 안내한다.
