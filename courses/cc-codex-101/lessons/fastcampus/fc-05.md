# fc-05 (강의 클립): 나만의 규칙서 CLAUDE.md
> 대응 자습 레슨: `/lesson-cc-codex cc-05` · 강의 클립 1.3.x · 예상 12~15분
> 이 문서 = 강의 촬영·진행용 런시트(이론 짧게 → 터미널 라이브). 깊은 자습은 위 레슨으로.

## 🎬 이 클립 한 줄 목표

Claude Code에게 매번 나를 소개하는 대신, CLAUDE.md 규칙서 한 장을 만들어 "이제부터 자동으로 내 방식대로 일하게" 만든다.

## 🗣️ 이론 (강사 설명 · 2~4분)

- **AI는 세션(대화 한 회)이 끝나면 기억이 초기화된다.** 어제 "저는 1인 사업자예요, 친근하게 써주세요"를 말했어도, 오늘 켜면 다시 처음부터 설명해야 한다. 결함이 아니라 원래 그런 구조다.
- **비유**: 아주 능력 있는 비서인데 출근할 때마다 기억이 리셋된다. CLAUDE.md = 비서 책상 위에 올려둔 업무 매뉴얼. 켜자마자 자동으로 읽어서 오늘도 어제처럼 일한다.
- **파일 위치에 따라 적용 범위가 다르다.** 작업 폴더 안에 두면 그 폴더에서만, `~/.claude/CLAUDE.md`에 두면 내 컴퓨터 전체에서 적용된다. 오늘은 작업 폴더 안에 먼저 만든다.
- **규칙서에 꼭 넣을 5가지**: 나는 누구인가 / 말투·스타일 / 보고서 형식 / 절대 하지 말 것 / 자주 쓰는 파일.
- **규칙서는 처음부터 완벽할 필요 없다.** 5가지 뼈대로 시작하고, 쓰다가 불편한 게 생기면 그때 추가한다. AI에게 직접 수정을 시킬 수도 있다.

## ⌨️ 터미널 라이브 시연 (8~10분)

---

**[스텝 1] 규칙서 없을 때 — 기본 상태 체감**

- **시연**:
  ```bash
  cd ~/practice-data/creator
  claude
  ```
  CC가 켜지면 입력:
  ```text
  course_sales_monthly.csv 파일을 읽고, 주목할 만한 인사이트를 알려주세요.
  ```
- **보여줄 것**: 말투가 딱딱하거나 요약 없이 긴 글, 또는 영어·한국어가 섞인 결과가 나온다. AI가 지우가 누구인지 전혀 모르는 상태의 응답.
- **학습자에게**: "결과를 보면서 느껴보세요. 내 스타일이 아니다 싶은 느낌이 드나요? 그게 규칙서가 필요한 이유입니다."

---

**[스텝 2] CC 종료 후 CLAUDE.md 파일 만들기**

- **시연**: `/exit` (또는 Ctrl+C) 로 CC 종료. 텍스트 에디터로 `~/practice-data/creator/CLAUDE.md` 파일을 새로 만들고 아래 뼈대를 붙여 넣는다.
  ```text
  # 지우의 사업 운영 규칙서 (CLAUDE.md)

  ## 나는 누구인가
  저는 1인 콘텐츠·지식 사업자 지우입니다. 온라인 강의 제작·판매,
  주간 뉴스레터 발송, 1:1 컨설팅을 혼자 운영합니다.

  ## 말투·스타일
  - 친근하고 따뜻한 말투로 이야기하듯 써주세요 ("~해요" 체)
  - 전문용어가 나오면 바로 뒤에 괄호로 쉬운 풀이를 붙여주세요

  ## 보고서·분석 형식
  1. 세 줄 요약 — 핵심을 30초에 잡을 수 있게
  2. 관찰 — 파일에서 실제로 보이는 것 (수치 직접 인용 필수)
  3. 해석 — 왜 그렇게 됐을지 가설
  4. 제안 — 다음에 뭘 해볼지 (1~3가지)

  ## 절대 하지 말 것
  - 데이터에 없는 수치를 지어내거나 추정값을 사실처럼 쓰지 마세요

  ## 자주 쓰는 파일
  - course_sales_monthly.csv — 강의별·월별 매출
  - newsletter_stats.csv — 뉴스레터 성과
  - student_inquiries.csv — 수강생·고객 문의 내역
  ```
  저장 후 터미널에서 확인:
  ```bash
  ls -la CLAUDE.md
  ```
- **보여줄 것**: 파일 이름과 크기가 나오면 성공.
- **학습자에게**: "파일이 `practice-data/creator/` 안에 있는지 확인해보세요. 다른 폴더에 있으면 CC가 읽지 못합니다."

---

**[스텝 3] 규칙서 있을 때 — 같은 질문 다시**

- **시연**:
  ```bash
  claude
  ```
  CC 시작 시 `✓ Loaded CLAUDE.md` 메시지 확인. 아까와 똑같은 질문 입력:
  ```text
  course_sales_monthly.csv 파일을 읽고, 주목할 만한 인사이트를 알려주세요.
  ```
- **보여줄 것**: "세 줄 요약 → 관찰(실제 수치 인용) → 해석 → 제안" 구조, "~해요" 말투로 정리된 응답. 규칙서에 적어둔 것이 자동 적용된 상태.
- **학습자에게**: "아까 결과와 지금 결과를 나란히 보세요. 말투·구조·수치 인용 방식이 어떻게 달라졌나요?"

---

**[스텝 4] AI에게 직접 규칙서 수정 시키기**

- **시연**: CC가 켜진 채로 입력:
  ```text
  CLAUDE.md 파일에 아래 내용을 추가해줘.

  ## 강의별 별칭 (내가 부르는 이름)
  - "클로드코드 강의" = "클로드코드로 업무 자동화 입문"
  - "챗GPT 강의" = "챗GPT로 콘텐츠 10배 빠르게"
  앞으로 내가 별칭으로 물어봐도 이 강의를 말하는 거야.
  ```
- **보여줄 것**: CC가 CLAUDE.md를 직접 수정하고 "추가했어요" 확인 메시지를 출력한다.
- **학습자에게**: "AI가 규칙서를 직접 고쳐줬어요. 일하면서 새로 배운 것을 바로 규칙서에 담을 수 있습니다. `/memory` 명령으로도 빠르게 확인·편집할 수 있어요."

---

**[스텝 5] `/init` — AI가 초안을 대신 써주는 방법 (빠른 시연)**

- **시연**:
  ```bash
  /init
  ```
- **보여줄 것**: CC가 현재 폴더를 분석해 CLAUDE.md 초안을 자동 생성한다.
- **학습자에게**: "직접 쓰기 막막하다면 `/init`으로 초안을 받은 다음, 내 사업 맥락에 맞게 다듬는 것도 방법입니다."

## ✅ 마무리 (1분)

CLAUDE.md 한 장이 AI를 "범용 도구"에서 "내 사업 파트너"로 바꿉니다. 한 번 적어두면 매 세션마다 자동으로 내 방식대로 일합니다. 처음부터 완벽할 필요 없고, 쓰다가 추가하면 됩니다.

더 깊은 실습(뉴스레터 분석·문의 분류·세 파일 연결 보고서)은 `/lesson-cc-codex cc-05` 로.

다음 클립 **fc-06**에서는 "오래 쓸수록 AI가 이상해지는 이유와 해결법" — 세션 관리·맥락 오염·비용 조절을 다룹니다.

## 📌 강사 노트

- **자주 막히는 지점 1 — "Loaded CLAUDE.md 메시지가 안 뜬다"**: 파일이 현재 작업 폴더에 없는 경우가 대부분. `ls -la CLAUDE.md`로 위치를 먼저 확인하게 안내. 파일명 대소문자 오타(`claude.md`, `CLAUDES.md`)도 빈번하므로 정확히 `CLAUDE.md`(대문자 6글자)임을 강조.
- **자주 막히는 지점 2 — "규칙서를 수정했는데 이전 규칙이 남아있다"**: 세션을 새로 시작하지 않아서 생기는 문제. `/exit` 후 다시 `claude`로 켜면 최신 규칙서를 읽는다.
