# fc-06 (강의 클립): 세션관리 + 맥락오염 + 비용

> 대응 자습 레슨: `/lesson-cc-codex cc-06` · 강의 클립 1.2.3 / 1.3.2 · 예상 12~15분
> 이 문서 = 강의 촬영·진행용 런시트(이론 짧게 → 터미널 라이브). 깊은 자습은 위 레슨으로.

---

## 🎬 이 클립 한 줄 목표

AI와 긴 대화를 할 때 "메모지가 가득 차는" 문제를 `/context`·`/compact`·`/clear`로 직접 정리하고, 그 차이를 손으로 체감한다.

---

## 🗣️ 이론 (강사 설명 · 2~4분)

- **AI는 대화를 "메모지 한 장"에 전부 적어서 들고 일한다.** 우리가 말한 것, AI가 답한 것, 읽은 파일 내용이 전부 그 메모지에 누적되고, AI는 답할 때마다 처음부터 다시 읽는다. 이 메모지가 "컨텍스트(context, 배경 정보 그릇)"다.

- **메모지가 차면 세 가지 문제가 생긴다.** ① 느려짐(매번 긴 종이를 다 읽어야 해서) ② 맥락 오염(옛 주제가 섞여 엉뚱한 답을 함) ③ 비용 증가(종량제라면 글자가 길수록 과금). "맥락 오염"은 어려운 말이 아니다 — 한 메모지에 장보기·회의·일기를 다 적어두면 나중에 뭐가 뭔지 모르게 되는 것과 같다.

- **정리 도구 3+1.** `/context`는 "배터리 잔량 보기"(메모지 몇 % 찼나), `/compact`는 "메모지를 요약해 깨끗한 새 장으로 옮겨 적기"(핵심 유지), `/clear`는 "메모지 버리고 완전 백지"(되돌리기 불가). 이 세 가지는 이미 켜진 대화방 안에서 치는 슬래시 명령이다. 추가로 `claude --continue`(직전 대화 이어 켜기)·`claude --resume`(여러 세션 중 골라 이어가기)는 터미널에서 대화방을 켤 때 붙이는 옵션이다.

- **핵심 한 줄:** `/compact`는 "정리하되 기억 유지", `/clear`는 "기억 삭제하고 새 출발". 이 둘의 차이만 손으로 체감하면 오늘 절반은 끝난다.

- **비용 감각 한 줄:** 구독제(Pro/Max)는 정액이라 메모지가 길어도 추가 요금이 없다. 종량제(API 키)는 메모지 길이만큼 과금된다. 지우 같은 1인 사업자는 구독제가 마음 편하다.

## ⌨️ 터미널 라이브 시연 (8~10분)

### 스텝 1 — 실습 폴더에서 Claude Code 켜고 잔량 확인

- **시연:**
  ```
  cd ~/practice-data/creator
  claude
  ```
  그리고 대화방 안에서:
  ```
  /context
  ```
- **보여줄 것:** 화면 아래 컨텍스트 게이지(예: `Context: 2%`)가 뜨고, `/context`를 치면 시스템·CLAUDE.md·대화 항목별 토큰 수와 전체 몇 %가 찼는지 표로 나온다. 갓 켰으니 한 자릿수 %다.
- **학습자에게:** "직접 `/context` 입력해서 몇 %가 뜨는지 확인해 보세요."

---

### 스텝 2 — 파일 분석으로 메모지 채우기

- **시연:**
  ```
  @course_sales_monthly.csv 이 파일은 내 온라인 강의 월별 매출이야.
  '매출(만원)' 컬럼을 강의별로 전부 더해서(2025-07~2026-03 누적),
  누적 매출 상위 3개를 표로 정리하고,
  각 강의가 어떤 흐름(상승/하락/등장)인지 한 줄씩 붙여줘.
  ```
- **보여줄 것:** AI가 표로 답을 내린다. 이 시점에 `/context`를 한 번 더 치면 % 숫자가 올라 있다(예: 12~20%대). "일을 할수록 메모지가 찬다"는 걸 숫자로 보여준다.
- **학습자에게:** "답 받은 직후 `/context` 다시 쳐서 아까보다 숫자가 올랐는지 확인해 보세요."

---

### 스텝 3 — `/compact`로 정리하고 기억이 유지되는지 확인

- **시연:**
  ```
  /compact
  ```
  그 직후:
  ```
  방금 정리한 매출 상위 3개 강의 중 1등이 뭐였지? 한 줄로만 답해줘.
  ```
- **보여줄 것:** `/compact` 후 `/context`를 치면 % 가 확 줄어 있다. 그런데 1등 강의를 물어보면 AI가 척척 답한다 — 자잘한 대화 과정은 버리고 핵심 요약은 유지되어 있기 때문이다.
- **학습자에게:** "직접 `/compact` 해보고, 그 직후 1등 강의를 물어보세요. 기억하는지 확인하는 게 포인트입니다."

---

### 스텝 4 — 맥락 오염 만들기: 같은 세션에 다른 주제 섞기

- **시연 (오염 만들기):**
  ```
  @student_inquiries.csv 이건 수강생 문의 내역이야.
  '카테고리' 컬럼을 기준으로 몇 건씩 들어왔는지 세어주고, 가장 많은 3개를 알려줘.
  ```
  답을 받은 후, 일부러 두루뭉술하게:
  ```
  그래서 환불이 제일 많은 거 뭐였지?
  ```
- **보여줄 것:** AI가 "매출 데이터의 환불인지, 문의 데이터의 환불 카테고리인지" 모르겠다며 되묻거나 엉뚱한 쪽을 골라 답한다. 이게 맥락 오염이다 — 한 메모지에 두 주제가 뒤섞여서 AI가 "지금 무슨 환불 얘기지?" 헷갈리는 상태.
- **학습자에게:** "직접 이렇게 물어보세요. AI가 어떻게 헷갈리는지 직접 보는 게 핵심입니다."

---

### 스텝 5 — `/clear` 전에 저장하고, `/clear`로 백지 만들기

- **시연 (저장 먼저):**
  ```
  지금까지 정리한 '문의 카테고리 상위 3개'를
  inquiry_summary.md 라는 파일로 저장해줘.
  ```
  확인 후:
  ```
  /clear
  ```
  그 직후:
  ```
  방금 정리한 문의 카테고리 1등이 뭐였지?
  ```
- **보여줄 것:** `/clear` 직후 AI는 "앞선 대화 내용이 없어서 어떤 문의를 말씀하시는지 모르겠습니다"라고 답한다. `/compact`와 달리 기억이 완전히 사라졌다. `/context`를 치면 % 가 처음처럼 바닥이다.
- **학습자에게:** "직접 `/clear` 해보고, 같은 질문을 던져보세요. `/compact`와 어떻게 다른지 체감이 옵니다. 그리고 앞으로는 `/clear` 전에 꼭 저장부터 하는 습관 기억해 주세요."

---

## ✅ 마무리 (1분)

오늘 핵심은 딱 하나다 — **"같은 일 계속 = `/compact`(기억 유지), 다른 일로 전환 = (저장 후) `/clear`(백지 시작)."** `/context`로 메모지 잔량을 수시로 확인하고, 30~45분마다 또는 큰 작업 하나 끝날 때 `/compact` 누르는 게 기본 습관이다. 더 깊은 내용(이어가기 `claude --continue`·`claude --resume`, 비용 절약 5습관, 트러블슈팅)은 자습 레슨 `/lesson-cc-codex cc-06`에서 확인할 수 있다. 다음 클립에서는 자주 하는 작업을 "스킬"로 묶어 한 번에 부르고, Claude Code 안에서 Codex를 같이 쓰는 방법을 배운다.

## 📌 강사 노트

- **자주 막히는 지점 1 — "슬래시 명령이 안 먹혀요":** 터미널에서 `claude /compact`처럼 치는 경우가 많다. 먼저 `claude`로 대화방을 연 **뒤**, 그 입력창(`>` 프롬프트)에서 `/compact`를 치는 것임을 강조한다. "대화방 안에서 치는 명령"이라는 점을 반복해서 짚는다.

- **자주 막히는 지점 2 — "`/clear` 했더니 중요한 결과가 사라졌어요":** `/clear`는 되돌리기가 안 된다. 이미 발생했다면 "괜찮아요, 파일 분석을 다시 시키면 됩니다"로 안심시키고, **다음부터는 `/clear` 전에 파일로 저장**하는 순서를 다시 한 번 보여준다. 이게 실습 스텝 5에서 '저장 먼저'를 강조하는 이유다.
