# codex-fc-5 (강의 클립): AGENTS.md + config.toml — 규칙서

> 대응 자습 레슨: `/lesson-cc-codex codex-06` · 강의 클립 codex-fc-5 · 예상 12~15분
> 이 문서 = 강의 촬영·진행용 런시트(이론 짧게 → 터미널 라이브). 깊은 자습은 위 레슨으로.

## 🎬 이 클립 한 줄 목표

AGENTS.md 규칙서와 config.toml을 직접 만들어, Codex가 "지우의 사업 방식"을 처음부터 알고 일하게 만든다.

---

## 🗣️ 이론 (강사 설명 · 2~3분)

- **AGENTS.md = 새 알바생 업무 매뉴얼**: Codex를 켤 때마다 "저는 비개발자예요, 한국어로 해주세요"를 반복하는 대신, 규칙서 한 장에 적어두면 처음부터 그대로 일한다.
- **cc-05의 CLAUDE.md와 이름만 다른 쌍둥이**: 역할·규칙·보고 형식·금지사항 — 적는 내용도, "폴더에 파일을 두면 Codex가 자동으로 읽는다"는 작동 방식도 같다. 파일 이름 하나만 다르다.
- **우선순위**: 여러 위치에 둘 수 있고, 가장 가까운(프로젝트 폴더) 것이 이긴다. 지금은 "작업 폴더에 하나만 두면 된다"만 기억하면 충분.
- **config.toml = 단골 식당 저장 주문**: 매번 모델과 effort를 손으로 고르는 대신, 파일 한 장에 `model = "gpt-5.5"` 식으로 적어두면 다음부터 자동 적용된다.
- **비유로 묶기**: AGENTS.md는 "어떻게 일할지 업무 매뉴얼", config.toml은 "어떤 도구로 일할지 기본 주문". 두 파일이 있으면 Codex는 더 이상 첫날 신입이 아니다.

## ⌨️ 터미널 라이브 시연 (8~10분)

### 스텝 1 — 규칙서 없이 먼저 시켜보기 (비교 기준)

- **시연**:
  ```bash
  codex --cd ~/practice-data/creator
  ```
  Codex 세션이 켜지면 입력:
  ```text
  student_inquiries.csv를 읽고, 이번 달 문의에서 가장 많이 들어온 문의 유형 3가지를 정리해 줘
  ```
- **보여줄 것**: 영어가 섞이거나 제안이 없는 일반적인 답이 나온다. 이 답을 스크린샷으로 찍어 두거나 기억해 둔다.
- **학습자에게**: "결과를 보고 기억해 두세요. 잠시 뒤 같은 질문을 다시 던질 거예요."

세션 종료: `Ctrl+C` 또는 `/exit`

---

### 스텝 2 — 세션 안에서 `/init`으로 AGENTS.md 뼈대 자동 생성

- **시연**:
  ```bash
  codex --cd ~/practice-data/creator
  ```
  Codex 세션 **안의 입력창**에서:
  ```text
  /init
  ```
- **보여줄 것**: Codex가 폴더를 잠깐 살펴본 뒤 AGENTS.md 초안을 만들어 준다. 파일명 목록과 기본 규칙 골격이 자동으로 생성된다.
- **학습자에게**: "이건 Codex 세션 안에서 치는 슬래시 명령이에요. 터미널 셸 명령이 아닙니다. `/init`을 입력하면 Codex가 폴더를 보고 초안을 잡아줘요."

---

### 스텝 3 — 텍스트 편집기로 지우의 규칙 추가 후 재시작

- **시연**: 텍스트 편집기로 `~/practice-data/creator/AGENTS.md`를 열어 아래 내용을 붙여넣는다:
  ```markdown
  # AGENTS.md — 지우의 콘텐츠 사업 운영 규칙서

  ## 내가 누구인가
  나는 온라인 강의·뉴스레터·컨설팅을 운영하는 1인 사업자 지우다. 코딩은 모른다.

  ## 반드시 지킬 규칙
  - 모든 답변은 비개발자가 읽기 쉬운 한국어로.
  - 파일을 삭제하거나 덮어쓰기 전에는 반드시 먼저 물어볼 것.

  ## 보고서 형식
  분석 결과는 항상 이 순서로:
  1. 관찰 — 데이터에서 발견한 것
  2. 해석 — 그게 왜 일어났는지
  3. 제안 — 다음에 무엇을 하면 좋은지
  ```
  저장 후 Codex를 새로 시작:
  ```bash
  codex --cd ~/practice-data/creator
  ```
- **보여줄 것**: 재시작 후 세션 안에서 `내가 누구야?`라고 물으면, Codex가 지우의 사업 맥락으로 정확히 답한다.
- **학습자에게**: "파일을 고친 뒤에는 꼭 Codex를 새로 시작하세요. 세션을 시작할 때 AGENTS.md를 읽기 때문입니다."

---

### 스텝 4 — 같은 질문 다시 던져 차이 확인

- **시연**: 스텝 1과 완전히 똑같은 질문 입력:
  ```text
  student_inquiries.csv를 읽고, 이번 달 문의에서 가장 많이 들어온 문의 유형 3가지를 정리해 줘
  ```
- **보여줄 것**: 이번에는 한국어, 실제 수치 인용, "관찰 → 해석 → 제안" 형식의 답이 나온다. 스텝 1 결과와 나란히 비교한다.
- **학습자에게**: "스텝 1 결과와 비교해 보세요. 같은 질문인데 답이 달라졌습니다. 이게 규칙서의 힘이에요."

---

### 스텝 5 — config.toml로 기본 설정 고정

- **시연**: 텍스트 편집기로 `~/.codex/config.toml` 파일을 새로 만들고:
  ```toml
  # config.toml — 지우의 Codex 기본 설정
  model = "gpt-5.5"
  model_reasoning_effort = "medium"
  approval_policy = "on-request"
  ```
  저장 후 Codex 재시작:
  ```bash
  codex --cd ~/practice-data/creator
  ```
  세션 안에서:
  ```text
  /status
  ```
- **보여줄 것**: `현재 모델: gpt-5.5 / Effort: medium / 승인 정책: on-request` 가 표시된다.
- **학습자에게**: "이제 매번 `-m gpt-5.5`를 손으로 치지 않아도 됩니다. `~/.codex/config.toml`에 한 번만 적어두면 다음부터 자동 적용돼요. 직접 `/status`를 입력해서 확인해 보세요."

## ✅ 마무리 (1분)

AGENTS.md에 내 사업 방식을 적고, config.toml에 기본 설정을 고정해 두면 — Codex는 매번 "처음 만난 낯선 비서"가 아니라 내 방식을 아는 파트너가 된다. 더 깊은 실습(우선순위 체험, 교차 분석, 파일 저장 승인 게이트)은 `/lesson-cc-codex codex-06`에서. 다음 클립에서는 Codex와 Claude Code를 나란히 놓고 "같은 것과 다른 것"을 직접 비교해 본다.

## 📌 강사 노트

**자주 막히는 지점 1 — AGENTS.md를 만들었는데 Codex가 무시하는 것 같다**
원인 대부분은 파일 위치 오류 또는 재시작 누락. 체크 포인트:
- 파일이 `~/practice-data/creator/AGENTS.md`에 있는가?
- 파일 이름이 `AGENTS.md`(대문자)인가? `agents.md`(소문자)면 인식 안 됨.
- AGENTS.md를 수정한 뒤 Codex 세션을 새로 시작했는가?

**자주 막히는 지점 2 — `/init`을 입력했는데 아무것도 안 생긴다**
`/init`은 터미널 셸이 아니라 Codex 세션 **안의 입력창**에서 입력해야 한다. 그래도 안 되면 직접 텍스트 편집기로 `AGENTS.md` 파일을 만들어도 무방 — 효과는 같다.
