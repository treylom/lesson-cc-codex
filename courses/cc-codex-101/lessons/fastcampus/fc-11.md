# fc-11 (강의 클립): 승인 + 샌드박스 3모드

> 대응 자습 레슨: `/lesson-cc-codex codex-05` · 강의 클립 fc-11 · 예상 12~15분
> 이 문서 = 강의 촬영·진행용 런시트(이론 짧게 → 터미널 라이브). 깊은 자습은 위 레슨으로.

## 🎬 이 클립 한 줄 목표

Codex의 샌드박스 3모드(read-only / workspace-write / danger-full-access)와 승인 정책(on-request)을 직접 모드를 바꿔가며 시연해, "AI에게 얼마나 열쇠를 줄지 내가 직접 정한다"는 감각을 체험한다.

---

## 🗣️ 이론 (강사 설명 · 2~4분)

- **비유: 새 비서에게 열쇠를 얼마나 줄까.** 첫날엔 "봐도 돼, 하지만 만지지 마"(read-only) → 업무가 익숙해지면 "사무실 안에서 자유롭게, 바깥 연락은 먼저 물어봐"(workspace-write) → 완전히 신뢰하면 "인터넷까지 다 써도 돼"(danger-full-access). Codex 샌드박스가 정확히 이 3단계다.
- **기본값은 이미 안전하다.** 플래그 없이 실행하면 workspace-write 모드가 기본값이며, 인터넷은 이미 차단된 상태다. "기본값이 위험한 모드 아닐까?"는 걱정하지 않아도 된다.
- **"danger"라는 이름은 경고 표시.** 겁주는 게 아니라 "인터넷까지 열리니 충분히 이해하고 써라"는 의미다. 약통의 "전문가 상담 후 복용"과 같다.
- **승인 정책(on-request)은 "필요할 때만 물어보기".** 읽기처럼 안전한 작업은 알아서 진행하고, 모델이 "확인이 필요하다"고 판단할 때만 멈춘다. 매번 묻는 게 아니다.
- **자동 보호되는 것 3가지**: `.git` / `.codex` / `.agents` 폴더는 항상 읽기 전용 보호, workspace-write에서 네트워크 기본 차단, OS(Mac Seatbelt)가 한 겹 더 보호.

## ⌨️ 터미널 라이브 시연 (8~10분)

### 스텝 1 — 실습 폴더로 이동 후 read-only 탐색

- **시연**:
  ```bash
  cd ~/practice-data/creator
  codex --sandbox read-only --ask-for-approval on-request "이 폴더에 어떤 파일들이 있는지 둘러보고, 지우라는 1인 사업자가 어떤 사업을 하는지 요약해 줘"
  ```
- **보여줄 것**: Codex가 파일 목록을 나열하고 사업 현황을 요약해 준다. 새 파일은 생기지 않는다.
- **학습자에게**: "직접 자기 폴더에서 같은 명령 쳐보세요. 파일이 전혀 바뀌지 않는 걸 확인하세요."

---

### 스텝 2 — read-only 모드에서 파일 수정 요청 (거부 확인)

- **시연**:
  ```
  course_sales_monthly.csv 파일을 열어서 2026년 3월 데이터 아래에 "분석 완료"라고 한 줄 추가해 줘
  ```
- **보여줄 것**: Codex가 "read-only 샌드박스 모드라 수정할 수 없습니다. `--sandbox workspace-write`로 재실행해 주세요."라고 정중히 거부한다.
- **학습자에게**: "직접 수정 요청을 넣어보세요. 거부 메시지가 뜨는지 확인하세요."

---

### 스텝 3 — workspace-write 모드로 파일 생성

- **시연**:
  ```bash
  codex -m gpt-5.5 --sandbox workspace-write "course_sales_monthly.csv 파일을 읽고, 매출 상위 강의 3개를 골라 한 줄씩 요약해줘. 결과를 sales_summary.txt 파일로 저장해줘."
  ```
- **보여줄 것**: 분석 결과가 나오고 `sales_summary.txt` 파일이 실제로 생성된다. `ls` 명령으로 파일이 새로 생긴 것을 확인한다.
- **학습자에게**: "직접 실행 후 `ls`로 파일이 생겼는지 확인해 보세요. read-only 때는 없던 파일이에요."

---

### 스텝 4 — on-request 승인 정책: "n"으로 멈추기

- **시연**:
  ```bash
  codex --ask-for-approval on-request "student_inquiries.csv에서 '실습' 카테고리 문의만 뽑아서 같은 파일에 정리해줘."
  ```
- **보여줄 것**: Codex가 "student_inquiries.csv 파일을 수정하려 합니다. 진행할까요? (y/n)" 확인을 요청한다. `n`을 눌러 중단한 후, 파일이 그대로인 걸 확인한다.
- **학습자에게**: "직접 `n`을 눌러보세요. 파일이 바뀌지 않는 걸 확인하면 됩니다."

---

### 스텝 5 — /permissions로 현재 상태 확인

- **시연**:
  ```bash
  codex "안녕"
  ```
  세션이 열리면:
  ```
  /permissions
  ```
- **보여줄 것**: 현재 샌드박스 모드, 승인 정책, 네트워크 차단 여부, 보호된 경로가 출력된다. "네트워크 접근: 차단됨"을 강조해서 보여준다.
- **학습자에게**: "직접 `/permissions` 쳐보세요. '네트워크 차단됨'이 보이면 workspace-write가 제대로 작동하고 있는 겁니다."

## ✅ 마무리 (1분)

- **한 줄 정리**: read-only는 "보기만", workspace-write는 "내 폴더 안 자유롭게"(기본값), danger-full-access는 "인터넷까지 전부"(검증 후만) — 내가 상황에 맞게 고르는 것이 핵심이다.
- **더 깊이·전체** = `/lesson-cc-codex codex-05`
- **다음 클립 예고**: 다음 시간엔 `AGENTS.md`와 `config.toml` — Codex에게 "지우 방식대로 일해"라고 한 번만 써두면, 매번 설명하지 않아도 알아서 맞춰주는 규칙서를 만든다.

## 📌 강사 노트

- **자주 막히는 지점 1 — "y/n이 안 뜨는데요?"**: on-request는 모델이 안전하다고 판단한 읽기 작업에선 확인 없이 진행한다. 정상 동작이다. "더 자주 확인받고 싶으면 `--ask-for-approval untrusted`를 써보세요"라고 안내.
- **자주 막히는 지점 2 — "파일이 어디 생겼는지 모르겠어요"**: `/status`로 작업 디렉터리를 확인하거나, `ls` 명령으로 현재 폴더를 바로 확인하라고 안내. `cd ~/practice-data/creator`를 처음에 했으면 그 폴더 안에 생성된다.
