> 대응 자습 레슨: `/lesson-cc-codex codex-02` · 강의 클립 1.2.4 · 예상 12~15분

## 🎬 이 클립 한 줄 목표

"폴더를 건네고, 계획을 확인하고, 파일을 집어주면 — Codex가 혼자 읽고 분석하고 파일까지 만들어준다"는 흐름을 터미널에서 직접 체험한다.

---

## 🗣️ 이론 (강사 설명 · 2~4분)

- **Codex 일상 조작은 딱 세 패턴**: ① 시작할 때 `codex --cd ~/폴더이름 "지시문"` → ② 대화 중에 `/슬래시명령` → ③ 스킬 부를 때 `/skills 또는 $스킬이름`
- **비유 — 작업실에 상주하는 비서**: 예전엔 파일을 열어 복사·붙여넣기를 반복했다. Codex는 `--cd 폴더이름`으로 "이 서랍에서 일해"라고 지시하면 직접 파일을 열어 읽고 분석한다. CD는 "change directory(폴더 이동)"의 줄임말이다.
- **슬래시(`/`) vs 달러(`$`) 차이**: 슬래시는 세션 중 설정·관리용(리모컨 버튼 같은 것), 달러는 미리 만든 반복 작업 묶음을 부르는 기호. CC에서 `/스킬이름`이었던 것이 Codex에서는 `/skills 또는 $스킬이름`으로 부른다. 슬래시 세션 명령은 두 도구가 같다.
- **자주 쓸 슬래시 명령 4개**: `/plan`(계획 먼저 보기), `/status`(현재 모델·토큰·샌드박스 상태 확인), `/compact`(대화 정리 — 작업 끝난 뒤에만), `@파일명`(특정 파일 지목 — 기호는 달러가 아님)
- **`/compact` 주의**: 한창 분석 중에 누르면 흐름이 끊긴다. 한 작업이 완전히 끝난 다음에만 쓴다.

---

## ⌨️ 터미널 라이브 시연 (8~10분)

### 스텝 1 — 폴더를 건네며 Codex 시작하기
- **시연**: `codex --cd ~/practice-data/creator "이 폴더에 어떤 파일이 있고, 각 파일이 무엇인지 한 줄씩 설명해 줘"`
- **보여줄 것**: Codex가 폴더를 직접 열어 파일 목록(course_sales_monthly.csv, newsletter_stats.csv, student_inquiries.csv 등)과 각 파일 역할을 나열하고, "분석 추천 순서"까지 제안한다.
- **학습자에게**: "직접 자기 트랙 폴더(`practice-data/store/` 등)로 바꿔서 실행해 보세요"

---

### 스텝 2 — `/plan`으로 계획 먼저 확인하기
- **시연**: 세션 안에서 `/plan` 입력 후, `course_sales_monthly.csv 분석해서 매출이 가장 많이 오른 강의와 가장 많이 떨어진 강의를 찾고, 이유를 한 줄씩 붙인 요약 파일 만들어줘` 입력
- **보여줄 것**: Codex가 결과를 바로 내지 않고 "1단계: 파일 읽기 → 2단계: 집계 → ... → 5단계: 파일 저장" 계획을 보여주고 "계속할까요? (yes/no)"로 기다린다.
- **학습자에게**: "yes 입력해서 계획대로 실행해 보세요. 방향이 틀렸으면 yes 전에 수정 요청하면 됩니다"

---

### 스텝 3 — `@파일명`으로 특정 파일 집어내기
- **시연**: `@newsletter_stats.csv 이 파일에서 오픈율이 가장 높았던 호 Top 3와, 가장 낮았던 호 Bottom 2를 뽑아줘. 각각 어떤 제목이었는지도 함께 알려줘.`
- **보여줄 것**: `@`를 붙이자 그 파일만 집중해서 읽고 실제 수치(62.0%, 61.1% 등)를 인용해 답한다. "막연히 봐줘"보다 답이 훨씬 정확하다.
- **학습자에게**: "AI 답이 파일과 조금 다르면 파일을 직접 열어 확인해 보세요. 동률 항목이 있으면 '같은 수치인 호가 더 있나요?'라고 다시 물으면 됩니다"

---

### 스텝 4 — `/status`로 현재 세션 상태 확인
- **시연**: `/status`
- **보여줄 것**: 현재 모델(gpt-5.5), 추론 강도(effort: medium), 토큰 사용량, 샌드박스 모드(workspace-write), 승인 정책(on-request)이 표시된다.
- **학습자에게**: "gpt-5.5와 effort를 직접 바꾸는 법은 다음 클립에서 배웁니다. 지금은 '현재 설정을 확인하는 명령'으로만 기억하세요"

---

### 스텝 5 — 두 파일 교차분석 + 결과 파일 저장
- **시연**: `@newsletter_stats.csv @content_plan.md 뉴스레터에서 클릭율이 20% 이상 나온 호들의 공통점을 찾아줘. content_plan.md의 "이번 분기 배운 점" 섹션 내용과 비교해서, 기획 노트의 관찰이 실제 데이터로 확인되는지 알려줘.`
- **보여줄 것**: 두 파일을 함께 읽어 교차분석한다. 기획 메모의 관찰(정성)이 데이터(정량)로 뒷받침되는지 확인하는 흐름을 강조한다. 이어서 `방금 분석한 내용을 practice-data/creator 폴더 안에 "2026-03-분석-보고서.md" 라는 이름으로 저장해줘.` 입력 → Codex가 저장 전에 확인을 요청하면 yes 입력 → `ls ~/practice-data/creator/`로 파일 생성 확인.
- **학습자에게**: "파일을 저장하기 전에 확인하는 게 Codex의 안전 동작입니다. yes/no로 내가 결정합니다"

## ✅ 마무리 (1분)

`codex --cd`로 폴더를 건네고, `/plan`으로 계획을 먼저 확인하고, `@파일명`으로 정확히 집어주면 Codex가 직접 읽고 분석하고 파일까지 만들어 준다. 슬래시(`/`) 명령은 세션 관리, 달러(`$`)와 `/skills`는 반복 작업 묶음 호출이라는 차이만 기억하면 충분하다.

더 깊은 내용과 전체 실습 → `/lesson-cc-codex codex-02`

다음 클립(1.2.5 — 모델·effort·응답 차이)에서는 `/status`에서 봤던 `gpt-5.5`와 `effort: medium`을 직접 바꿔서 응답이 어떻게 달라지는지 체험한다.
