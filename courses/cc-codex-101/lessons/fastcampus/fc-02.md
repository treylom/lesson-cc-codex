## 🎬 이 클립 한 줄 목표

"Claude Code가 챗봇과 다른 이유 — 에이전트 루프(읽기 → 도구 사용 → 응답) — 를 지우의 강의 매출 데이터로 직접 눈으로 확인한다."

---

## 🗣️ 이론 (강사 설명 · 2~4분)

- **챗봇 vs Claude Code**: 챗봇은 "전화 상담원" — 내가 숫자를 읽어줘야 한다. Claude Code는 "내 책상에 앉은 동료" — 폴더를 직접 열고, 파일을 직접 읽고, 진짜 숫자로 답한다.
- **에이전트 루프**: 한 번 시키면 AI가 ① 상황 파악 → ② 파일 읽기(도구 사용) → ③ 결과 정리 → 답변의 한 바퀴를 스스로 돈다. 부족하면 다시 ①로 돌아와 반복한다.
- **도구(Tools)**: AI의 '손' — 파일 읽기·폴더 탐색이 이 손에 해당한다. 한 번 시켰는데 AI가 여러 번 움직이는 장면이 바로 도구가 작동하는 것이다.
- **권한(Permissions)**: AI가 파일을 읽기 전에 "읽어도 될까요?" 하고 한 번 멈추는 것 — 고장이 아니라 "내 폴더를 함부로 안 건드린다"는 안전장치다.
- **Context(컨텍스트)**: 한 대화 안에서 AI가 앞서 읽은 내용을 기억해 이어서 쓴다 — 여러 파일을 연결해 인사이트를 만들 수 있는 이유가 바로 이것이다.

---

## ⌨️ 터미널 라이브 시연 (8~10분)

### 스텝 1 — 올바른 폴더에서 Claude Code 켜기

- **시연**:
  ```bash
  pwd
  cd ~/practice-data/creator
  ls
  claude
  ```
- **보여줄 것**: `ls` 결과에 `course_sales_monthly.csv`, `newsletter_stats.csv`, `student_inquiries.csv`, `content_plan.md`, `research_memo.md`, `consulting_schedule.md`, `README.md` 7개가 보인다. `claude` 입력 후 입력 대기 상태(`>`)가 뜬다.
- **학습자에게**: "직접 `cd ~/practice-data/creator` 입력하고, `ls` 로 파일 7개가 보이는지 확인해 보세요. 그다음 `claude` 입력."

---

### 스텝 2 — 첫 루프: 폴더 파악 시키기

- **시연**: Claude Code 입력창에 아래 입력
  ```
  안녕하세요! 저는 1인 사업자 지우예요. 온라인 강의를 만들어 팔고, 뉴스레터도 보내고, 가끔 컨설팅도 합니다.
  지금 이 폴더에 어떤 파일이 있고, 각각 뭐 하는 파일인지 정리해주세요. README.md부터 읽어주세요.
  ```
- **보여줄 것**: 답이 나오기 전에 화면에 "Reading README.md"처럼 AI가 스스로 파일을 여는 동작이 표시된다. 이후 7개 파일 목록과 각 파일 역할이 정리되어 나온다.
- **학습자에게**: "직접 같은 문장을 입력해 보세요. 답이 나오기 전, AI가 파일을 스스로 여는 장면을 눈으로 확인하세요. 이게 에이전트 루프입니다."

---

### 스텝 3 — 권한(Permissions) 안전장치 만나기

- **시연**:
  ```
  course_sales_monthly.csv 파일을 직접 열어서, 맨 위 몇 줄이 어떻게 생겼는지 그대로 보여주세요.
  ```
- **보여줄 것**: 설정에 따라 "이 파일을 읽어도 될까요? Yes / Yes, and don't ask again / No" 선택지가 뜬다. `Yes` 선택 후 CSV 첫 몇 줄(월·강의명·매출 등 컬럼)이 화면에 표시된다.
- **학습자에게**: "허락 창이 뜨면 `Yes`를 선택해 보세요. 파일을 보는 것뿐이라 안전합니다."

---

### 스텝 4 — 진짜 숫자로 첫 와우 만들기 (핵심)

- **시연**:
  ```
  course_sales_monthly.csv 전체를 읽고, 1인 사업자가 의사결정에 쓸 만한 "눈에 띄는 패턴" 3가지를 찾아주세요.
  각 패턴마다 CSV의 실제 강의명·월·숫자를 인용해주세요. 비개발자도 이해되게요.
  ```
- **보여줄 것**: AI가 CSV를 읽고, 실제 강의명·월·매출 숫자(예: "챗GPT로 콘텐츠 10배 빠르게 2026-01 1,477만원")를 인용하며 패턴 3가지를 정리해 준다.
- **학습자에게**: "직접 입력해 보세요. 나오는 숫자가 실제 CSV 값과 맞는지 파일을 열어 한두 개만 눈으로 대조해 보세요. 맞다면 — 이게 첫 와우입니다."

---

### 스텝 5 — "추측 말고 실제 값으로" 습관 만들기

- **시연**:
  ```
  이 데이터에서 가장 매출이 큰 강의 하나만 골라주세요.
  단, 반드시 CSV에 실제로 있는 행의 숫자를 인용해서 근거를 보여주세요. 추측은 하지 마세요.
  ```
  (답에 실제 줄·숫자가 나오면 OK. 두루뭉술하면 추가 요청:)
  ```
  CSV에 적힌 정확한 값을 그대로 보여주세요.
  ```
- **보여줄 것**: 실제 강의명 + 정확한 매출 숫자 + "CSV 몇 번째 줄" 같은 근거가 나온다.
- **학습자에게**: "직접 입력해 보세요. '추측은 하지 마세요'를 붙이는 게 AI를 믿고 쓰는 핵심 습관입니다."

---

## ✅ 마무리 (1분)

Claude Code는 챗봇처럼 말만 하지 않는다 — 내 폴더를 직접 열고, 읽고, 진짜 숫자로 답하는 동료다. 그 한 바퀴가 에이전트 루프다.

더 깊은 내용(Context 이어가기, 여러 파일 연결 인사이트, 보고서 초안 만들기)은 자습 레슨 `/lesson-cc-codex cc-02` 에서 이어집니다.

다음 클립(1.2.2 또는 cc-03)에서는 "원하는 파일을 콕 집어 시키는 법 — 파일 지목과 `/` 명령어"를 배웁니다.

## 📌 강사 노트

- **자주 막히는 지점 1**: `claude` 입력했는데 "파일을 못 찾겠다"고 함 → 폴더 위치 문제. `pwd` 결과가 `practice-data/creator`로 끝나는지 확인 후 재실행.
- **자주 막히는 지점 2**: 숫자가 실제 CSV와 다르게 나옴(할루시네이션) → "CSV에 **실제로 있는 행의 값**을 그대로 인용해주세요"로 한 번 더 요청. 이 한 번 더 시키기가 비개발자가 AI를 안심하고 쓰는 핵심 안전 습관임을 강조.
