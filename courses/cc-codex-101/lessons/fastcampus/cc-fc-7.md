# cc-fc-7 (강의 클립): 플러그인 + 스킬·슬래시 + codex plugin
> 대응 자습 레슨: `/lesson-cc-codex cc-08` · 강의 클립 1.3.2 · 예상 12~15분
> 이 문서 = 강의 촬영·진행용 런시트(이론 짧게 → 터미널 라이브). 깊은 자습은 위 레슨으로.

## 🎬 이 클립 한 줄 목표

플러그인으로 CC에 새 기능을 추가하고, 나만의 SKILL.md 레시피를 만들어 반복 업무를 한 줄 명령으로 끝내며, codex plugin 개념을 이해한다.

---

## 🗣️ 이론 (강사 설명 · 2~4분)

- **플러그인 = 앱스토어에서 받은 앱**: 스마트폰에 카카오톡을 깔아야 채팅이 되듯, CC에도 `/plugin install`로 플러그인을 깔아야 심층 리서치·기획서 자동 생성 같은 전문 기능이 생긴다. 한 번 깔면 CC를 켤 때마다 자동 활성화.

- **스킬(SKILL.md) = 내가 직접 만든 단축키 레시피**: `.claude/skills/` 폴더에 SKILL.md 파일 하나를 만들면, 매달 반복하는 "매출 정리 + 뉴스레터 합산"을 `/monthly-review` 한 줄로 끝낼 수 있다. 매번 긴 프롬프트를 치던 걸 레시피로 저장하는 것.

- **둘의 차이 한 줄**: 플러그인은 남이 만든 기능 꾸러미, 스킬은 내가 만드는 맞춤 레시피.

- **슬래시 명령 = 둘 다 같은 방식으로 호출**: 플러그인도, 스킬도, 설치·생성 후에는 `/명령어` 형식으로 CC 대화창 안에서 실행한다. `/help`로 지금 쓸 수 있는 명령 전체를 언제든 볼 수 있다.

- **codex plugin = 작업실에 두 번째 짝꿍 초대**: CC와 Codex는 만든 회사(Anthropic vs OpenAI)·모델이 달라 분석 각도가 조금 다르다. codex plugin을 설치하면 CC 대화창 안에서 Codex를 불러 같은 파일을 다른 관점으로 분석할 수 있다.

---

## ⌨️ 터미널 라이브 시연 (8~10분)

### 스텝 1 — CC 열고 플러그인 저장소 등록·설치

터미널에서 실습 폴더로 이동 후 CC를 켠다.

- **시연**:
```bash
cd ~/practice-data/<업종>
claude
```

CC 대화창 안에서:
```text
/plugin marketplace add https://github.com/fivetaku/gptaku_plugins.git
```
```text
/plugin install deep-research
```

- **보여줄 것**: 저장소 등록 완료 메시지 → `deep-research` 설치 완료 메시지. 앱스토어에서 앱을 고르고 설치하는 것과 같은 흐름.

- **학습자에게**: "직접 `/plugin list`를 쳐서 방금 설치한 플러그인이 목록에 보이는지 확인해보세요."

---

### 스텝 2 — 플러그인 슬래시 명령 실행 + /help 확인

- **시연** (CC 대화창 안):
```text
/help
```

- **보여줄 것**: 기본 명령 목록 아래 "플러그인 명령"에 `/deep-research`가 보인다. `/help`가 언제든 쓸 수 있는 "메뉴판"임을 확인.

- **학습자에게**: "직접 `/deep-research`를 실행해 간단한 리서치 주제를 하나 넣어보세요."

---

### 스텝 3 — 데이터 파악 후 SKILL.md 만들기

먼저 CC에게 파일 구조를 파악하게 한 뒤, 스킬 파일을 생성한다.

- **시연** (CC 대화창 안):
```text
course_sales_monthly.csv를 읽어줘. 어떤 열이 있고, 어느 기간 데이터가 있는지 간단히 알려줘.
```

이어서:
```text
아래 내용으로 .claude/skills/monthly-review.md 파일을 만들어줘.

---
name: monthly-review
description: 지우의 월간 강의 매출과 뉴스레터 성과를 합산해 요약 리포트를 만든다. course_sales_monthly.csv와 newsletter_stats.csv를 읽고, 강의별 순매출 순위·전월 대비 증감·뉴스레터 오픈율 상위 3호·클릭율 상위 3호를 정리한 뒤 다음 달 한 줄 제안까지 작성한다.
---

# 월간 리뷰 스킬

## 단계 1: 매출 데이터 읽기
course_sales_monthly.csv를 열어 강의별 순매출 합계, 월별 총 순매출 추이, 환불수가 많은 강의를 정리한다.

## 단계 2: 뉴스레터 성과 읽기
newsletter_stats.csv를 열어 오픈율 상위 3호, 클릭율 상위 3호, 최근 3개월 구독자 증감을 정리한다.

## 단계 3: 교차 분석
매출 높은 달과 뉴스레터 오픈율·클릭율 높은 호를 비교해 연결점을 찾는다.

## 단계 4: 리포트 작성
[월간 리뷰] YYYY-MM 형식으로, 표와 불릿 위주로, 다음 달 한 줄 제안까지 작성한다.
파일 저장: monthly-review-YYYY-MM.md
```

- **보여줄 것**: CC가 `.claude/skills/monthly-review.md` 파일을 생성했다는 안내. SKILL.md 맨 위의 `name`·`description` 필드가 핵심임을 강조.

- **학습자에게**: "직접 같은 방법으로 본인 업종의 반복 업무를 스킬로 만들어보세요. 파일 이름은 자유롭게."

---

### 스텝 4 — 스킬 등록 확인 후 실행

- **시연** (CC 대화창 안):
```text
/help
```
```text
/monthly-review
```

- **보여줄 것**: `/help` 목록 "내 스킬"에 `/monthly-review`가 보인다 → `/monthly-review` 실행 시 단계별로 자동 진행되며 리포트가 완성된다. CSV 두 개를 읽고, 교차 분석하고, 보고서까지 한 줄 명령 한 번으로.

- **학습자에게**: "직접 `/monthly-review` 또는 본인이 만든 스킬을 실행해보세요."

---

### 스텝 5 — codex plugin 개념 체험 (간략 시연)

- **시연** (CC 대화창 안):
```text
/plugin install codex
```
```text
/help
```

- **보여줄 것**: 설치 후 `/codex:review`, `/codex:rescue` 명령이 목록에 추가된다. 설치가 안 될 경우에도 당황하지 말 것 — "codex plugin은 현재 발전 중인 기능"임을 안내하고, CC 단독으로 분석한 파일을 Codex 창에서 같은 질문으로 비교하는 방법도 유효함을 설명.

- **학습자에게**: "직접 설치를 시도해보고, 설치되면 `/codex:rescue 이 데이터 분석 맡겨줘`를 한 번 실행해보세요."

## 📌 강사 노트

**자주 막히는 지점 1 — "/plugin list가 안 먹혀요"**
> CC 대화창 안에서 입력했는지 확인. 터미널 `$` 프롬프트에서 치면 안 된다. `claude`를 켜고 `>` 프롬프트가 뜬 상태에서 입력해야 슬래시 명령이 작동한다.

**자주 막히는 지점 2 — "스킬 만들었는데 /help에 안 보여요"**
> `.claude/skills/` 폴더 안에 저장했는지 확인. CC에게 "monthly-review 스킬이 어느 폴더에 저장됐는지 확인해줘"라고 바로 물으면 CC가 직접 찾아준다. 파일 안의 `name:` 값이 실행 명령과 정확히 일치해야 한다.
