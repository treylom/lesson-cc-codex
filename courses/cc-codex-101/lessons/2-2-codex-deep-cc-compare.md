# 2-2: Codex 심화 + Claude Code ↔ Codex 비교

## 학습 목표
- AGENTS.md(프로젝트 규칙서)·MCP(외부 도구 연결)·샌드박스(안전 모드)가 무엇인지 비개발자 눈높이로 이해한다.
- 안전하게(샌드박스·승인) AI에게 일을 맡기는 감각을 익힌다.
- **Claude Code와 Codex의 공통점·차이점**을 스스로 1문장으로 말할 수 있다.

## 사전 준비
- 2-1 완료(Codex 설치·실행·effort·모델 경험).
- Codex 실행 상태, `practice-data/` 폴더.

## 진행

### 도입 (3분)
- 지금까지 = 말 걸고, effort·모델 조절. 오늘 = AI에게 **규칙을 주고, 도구를 붙이고, 안전하게** 일 시키기.
- 세 가지 쉬운 비유:
  - **AGENTS.md** = "AI에게 주는 업무 매뉴얼"(우리 팀은 이렇게 일해요).
  - **MCP** = "AI에게 끼우는 외부 도구 어댑터"(공식 문서·브라우저 등 연결).
  - **샌드박스** = "실수해도 안전한 연습실"(허락 없이 위험한 변경 못 하게).

### 실습 1 — AGENTS.md (10분)
따라 하기:
1. 규칙서가 뭔지 Codex에게 직접 물어본다.
```text
AGENTS.md 가 뭔지, 비개발자에게 한 문단으로 쉽게 설명해줘. Claude Code의 CLAUDE.md와 비슷한지도 알려줘.
```
2. 간단한 규칙서를 만들어 본다(있으면 `/init`, 없으면 직접 요청).
```text
이 폴더용 AGENTS.md 초안을 만들어줘. 내용: "나는 비개발자다 / 항상 비개발자 눈높이로 설명 / 위험한 변경은 먼저 물어봐 / 보고서는 관찰-해석-제안 순서".
```
3. 확인한다.
```text
방금 만든 AGENTS.md 를 보여주고, 이게 앞으로 네 행동을 어떻게 바꾸는지 한 줄로 말해줘.
```

포인트: 규칙서를 한 번 써두면 매번 다시 설명할 필요가 없다(재현성). **CLAUDE.md(CC) ≈ AGENTS.md(Codex)**.

### 실습 2 — 샌드박스·승인 (8분)
따라 하기:
1. 현재 안전 모드를 확인한다.
```text
지금 너는 어떤 샌드박스 모드야? 파일을 고치거나 인터넷에 접근할 수 있어?
```
2. 안전 개념을 체감한다.
```text
read-only(읽기 전용)·auto(쓰기 가능)·full-access의 차이를 비개발자에게 설명해줘. 비개발자는 보통 어떤 모드로 시작하면 좋아?
```

포인트: 위험한 변경(파일 삭제 등)은 **승인을 요구**하도록 되어 있다 → 안심하고 연습. (Claude Code도 승인 게이트가 있다.)

### 실습 3 — MCP 맛보기 (개념 위주, 8분)
> ⚠️ 실제 MCP 서버 추가는 환경 따라 다르므로, 이 레슨은 **개념 이해 + 1회 확인**까지.

따라 하기:
1. MCP가 뭔지 묻는다.
```text
MCP(Model Context Protocol)가 뭔지, "AI에게 외부 도구를 끼운다"는 비유로 설명해줘. 어떤 도구를 끼우면 비개발자에게 유용할까?
```
2. (강사 시연/선택) 연결된 MCP 확인.
```text
지금 연결된 MCP 도구가 있는지 확인해줘.
```

포인트: 공식 문서 MCP를 끼우면 "모르면 공식 문서에 물어보는" AI가 된다(추측↓). 깊은 설정은 다음 단계 과정에서.

### 실습 4 — CC ↔ Codex 비교 정리 (10분)
따라 하기:
1. 직접 비교하게 한다.
```text
지금까지 배운 걸로, Claude Code와 Codex의 공통점 3가지와 차이점 3가지를 표로 정리해줘.
```
2. 한 문장으로 압축한다.
```text
"Claude Code를 익혔으면 Codex는 ___" 형식으로, 두 도구 관계를 한 문장으로 정리해줘.
```

정리(강사 보강):
| 축 | 공통(같다) | 차이(다르다) |
|---|---|---|
| 규칙서 | 프로젝트 규칙 파일로 재현 | CLAUDE.md(CC) / AGENTS.md(Codex) |
| 안전 | 샌드박스 + 승인 게이트 | 모드 이름·표기만 차이 |
| 도구확장 | MCP 공통 | 연결 방식 세부 차이 |
| 추론조절 | effort·모델 선택 | 모델 라인업(Claude vs gpt-5.x) |
- 한 문장: **"CC를 익히면 Codex는 90% 전이된다 — 규칙서·샌드박스·MCP·effort는 같고, 제품·모델만 다르다."**

### 마무리 (3분)
- 오늘: 규칙서·샌드박스·MCP + 두 도구 비교.
- 코스 전체 정리: 데이터 읽기 → 분석 → 보고서(CC) → 같은 일을 Codex로 + 조절·안전 → 둘 비교.
- 한 줄 정리: **"도구는 둘이지만 뼈대는 하나 — 작업과 상황에 맞춰 고른다."**

## 트러블슈팅
- **AGENTS.md 가 적용 안 되는 듯**: 파일 위치(폴더 루트·`.codex/`)·정확한 파일명 확인.
- **모드/권한이 헷갈림**: "지금 모드와 내가 할 수 있는 것"을 다시 물어 현재 상태 확인.
- **MCP 추가가 막힘**: v1에서는 개념 이해까지만 — 추가는 다음 과정/강사 안내로.
- **비교가 추상적**: "내가 오늘 실제로 해본 것"만으로 비교해달라고 요청.
