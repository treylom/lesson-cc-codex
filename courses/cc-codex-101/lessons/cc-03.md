# cc-03: 파일 다루기 + 명령어 — "내 자료를 AI 손에 정확히 쥐여주는 법"

> **레슨 목표**: 흩어진 사업 자료가 있는 폴더로 정확히 찾아 들어가고, 원하는 파일을 콕 집어 Claude Code(CC)에게 읽힌 뒤, 매일 쓰는 명령 몇 개로 대화를 가볍게 굴린다(요약·비용 확인 포함).
> **예상 시간**: 45분 (도입 7분 + 실습 1 / 2 각 15분 + 실습 3·마무리 8분)
> **사전 준비**: 터미널 1개 + Claude Code 설치·로그인 완료(cc-01) + 에이전트 루프 체험(cc-02) + 이 코스 폴더(`practice-data/creator/`)
> **진입 커맨드**: 슬라이드에서 지금 `/cc-03` 을 입력하세요.

---

## 📚 학습 목표 (4가지 와우 모먼트)

1. **길을 잃지 않는다** — `pwd`·`ls`·`cd` 세 글자로 "지금 어디 있고, 뭐가 있고, 어디로 갈지"를 안다. 파인더·탐색기를 글자로 한다고 보면 끝.
2. **파일을 콕 집어 쥐여준다** — `@파일명` 한 방으로 매출 CSV·문의 로그를 정확히 그 파일만 읽힌다. ChatGPT처럼 내용을 복사-붙여넣기할 필요가 사라진다.
3. **명령은 딱 두 종류뿐** — 켤 때 거는 **터미널 명령**(`claude`)과 대화 중 내리는 **슬래시 명령**(`/compact`·`/cost`), 그리고 손가락 단축키 몇 개. 이 구분 하나면 80%가 풀린다.
4. **지우의 진짜 자료로 끝낸다** — 실습용 가짜 폴더가 아니라 지우의 강의 매출·뉴스레터·문의 데이터를 직접 읽혀, 파일 한 개에서 "오, 이건 진짜 쓸모 있는데?" 소리가 나오는 순간을 만든다.

---

## 🎯 개념 도입 (7분)

### 한 줄로: cc-03은 "AI에게 자료를 건네는 손동작"을 배우는 시간

cc-01에서 우리는 AI를 켰고, cc-02에서 "내가 묻고 → AI가 일하고 → 결과를 돌려주는" 한 바퀴(에이전트 루프, agent loop)를 봤어요. 그런데 한 가지가 빠져 있었죠. **"그래서 내 매출 파일을 어떻게 보여주지?"**

지우의 사업 폴더를 떠올려 봅시다. 강의 매출 CSV, 뉴스레터 성과표, 고객 문의 로그, 콘텐츠 기획 노트… 폴더 여기저기 흩어져 있어요. 새로 온 똑똑한 비서(=Claude Code)에게 일을 시키려면, 먼저 **"이 서랍 안에 자료가 있어, 이 파일을 봐줘"** 라고 정확히 손으로 가리켜 줘야 합니다. cc-03은 바로 그 **손동작**을 배우는 시간이에요.

### 비유: 사무실에 새 직원이 출근한 첫날

새 직원에게 일을 맡기는 과정을 그대로 따라가 봅시다.

1. **"우리 사무실이 몇 층이죠?"** → 지금 내가 어느 폴더에 있는지 확인 = `pwd`
2. **"이 방 책상에 뭐가 있나요?"** → 이 폴더에 무슨 파일이 있는지 둘러보기 = `ls`
3. **"자료실은 옆 방이에요, 그리로 가요"** → 자료가 있는 폴더로 이동 = `cd`
4. **"이 매출 보고서 좀 봐줘요"** → 특정 파일을 콕 집어 건네기 = `@파일명`

터미널(terminal, AI에게 말 거는 검은 채팅창)이 무섭게 느껴졌다면, 사실은 이 네 가지 손동작이 전부예요. 외울 것도 없습니다. **첫 글자만 기억하면 됩니다.**

### 명령은 딱 두 입구 — 이거 하나만 구분하면 절반은 끝

가장 많이 헷갈리는 지점을 먼저 못 박고 갑시다. CC에 거는 명령은 **두 곳에서, 두 종류**로 들어갑니다.

| 구분 | 어디서 치나 | 무엇을 하나 | 예시 |
|------|------------|------------|------|
| **터미널 명령** | AI를 켜기 **전**, 일반 프롬프트(`$`)에서 | AI를 **켜고·끄고·옵션**을 건다 | `claude`, `claude --help` |
| **슬래시(/) 명령** | AI와 **대화하는 중**, 채팅창 안에서 | 대화 흐름에 **지시**를 내린다 | `/compact`, `/clear`, `/cost`, `/help` |

> 💡 **딱 한 문장만 기억하세요**: `/compact`를 터미널에서 `claude /compact`로 치는 게 **아니라**, `claude`로 먼저 대화방에 들어간 다음 그 안에서 `/compact`를 칩니다. 엑셀을 켠(=`claude`) 다음 그 안에서 작업(=`/명령`)하는 것과 똑같아요.

여기에 손가락으로 누르는 **키보드 단축키**(`Ctrl+C`로 멈추기 등) 몇 개가 곁들여집니다. 그래서 "명령 3종류"라고 부르지만, 실제로 머릿속에 둘 그림은 **켜기 전 / 대화 중** 두 칸이면 충분해요.

### 안심하고 시작하세요

- **명령을 잘못 쳐도 컴퓨터는 안 망가집니다.** 대부분 "그런 명령 없는데요?" 같은 안내만 뜨고 끝이에요.
- **파일을 읽힌다고 인터넷에 올라가는 게 아닙니다.** 내 컴퓨터 안, 내 세션(session, 지금 켜둔 대화 한 판) 안에서만 봅니다.
- **길을 잃으면 리셋 버튼이 있어요.** `cd ~`(물결표)를 치면 무조건 내 홈 폴더로 돌아옵니다.

## 🔧 실습 1 — 길 찾기 네 글자와 첫 파일 읽히기 (15분)

> 목표: 터미널에서 내 사업 폴더로 정확히 들어가, 안에 뭐가 있는지 보고, 파일 하나를 CC에게 읽힌다. **한 번에 하나씩** 따라오세요.

### 1단계: 지금 내가 어디 있는지 확인 (`pwd`)

터미널을 열고, AI를 켜기 **전에** 다음을 칩니다.

```text
pwd
```

**기대 결과**: 지금 서 있는 폴더의 전체 경로가 한 줄로 나옵니다.

```
/Users/jiu/Documents
```

**관찰 포인트**: `pwd`는 "print working directory(지금 작업 폴더를 찍어줘)"의 약자예요. 길을 잃었을 때 가장 먼저 누르는 **"여기 어디지?"** 버튼입니다. 결과가 내 자료 폴더가 아니어도 괜찮아요 — 다음 단계에서 이동하면 됩니다.

---

### 2단계: 이 폴더에 뭐가 있나 둘러보기 (`ls`)

```text
ls
```

**기대 결과**: 현재 폴더 안의 파일·폴더 목록이 쭉 나옵니다.

```
Documents   Downloads   Desktop   내사업
```

옵션을 붙여 더 자세히 볼 수도 있어요.

```text
ls -la
```

`-la`를 붙이면 숨김 파일까지, 한 줄에 하나씩 자세히 보입니다. **관찰 포인트**: `ls`는 "list(목록)"의 약자, 윈도우 탐색기에서 폴더를 열어 안을 들여다보는 것과 같아요. 지금은 그냥 `ls`만으로 충분합니다.

---

### 3단계: 자료가 있는 폴더로 이동 (`cd`)

이 코스 폴더 안의 creator 실습 데이터로 들어가 봅시다.

```text
cd practice-data/creator
```

이동한 뒤 진짜 들어왔는지 확인하는 습관이 좋아요.

```text
pwd
ls
```

**기대 결과**:

```
/Users/jiu/.../cc-codex-101/practice-data/creator

README.md                course_sales_monthly.csv   newsletter_stats.csv
student_inquiries.csv    content_plan.md            research_memo.md
consulting_schedule.md
```

**관찰 포인트**:
- `cd`는 "change directory(폴더 바꿔)" = 탐색기에서 폴더를 더블클릭해 들어가는 것.
- `cd ..`(점 두 개)는 한 칸 위로(뒤로 가기), `cd ~`(물결표)는 홈으로 리셋.
- **이동 → `pwd`/`ls`로 확인**, 이 두 박자가 길 안 잃는 비결입니다.

> 💡 **자주 나는 실수**: "AI가 내 파일을 못 찾아요"의 90%는 **엉뚱한 폴더에서 CC를 켠 것**입니다. 켜기 전에 `ls`로 그 파일이 진짜 이 폴더에 보이는지부터 확인하세요.

---

### 4단계: 이 폴더를 무대로 CC 켜기 (`claude`)

자료가 있는 폴더 안에 선 채로 AI를 켭니다. **CC는 "지금 터미널이 있는 폴더"를 무대로 삼기 때문**에, 자료 폴더 안에서 켜는 게 핵심이에요.

```text
claude
```

**기대 결과**: 입력창(`>`)이 뜨고 대화 대기 상태가 됩니다.

> 💡 **곁다리 한 줄(지금은 안 써요)**: 나중에 어제 하던 대화를 그대로 이어서 켜고 싶으면 `claude -c`(continue, 이어 하기)라는 옵션도 있어요. **세션을 이어 가는 건 cc-06에서 자세히** 다루니, 지금은 "그런 게 있구나" 정도만 알고 넘어가면 됩니다. 이번 레슨은 매번 새로 `claude`로 켜면 충분해요.

---

### 5단계: 첫 한국어 부탁 — "이 폴더에 뭐가 있어?"

대화방 안에서, 그냥 한국어로 말을 겁니다(복사-붙여넣기 OK).

```text
나는 온라인 강의·뉴스레터·컨설팅을 혼자 하는 1인 사업자 '지우'야.
이 폴더에 어떤 파일이 있고, 각각 뭘 담고 있는지, 무엇부터 보면 좋은지
README.md부터 읽고 비개발자도 알기 쉽게 정리해줘.
```

**기대 결과** (요지):

```
이 폴더에는 파일 7개가 있어요.
- course_sales_monthly.csv : 강의별·월별 매출 (2025-07 ~ 2026-03)
- newsletter_stats.csv     : 뉴스레터 18호~41호 오픈율·클릭율·구독 증감
- student_inquiries.csv    : 수강생 문의 로그 (채널·카테고리·처리상태)
- content_plan.md          : 2분기 콘텐츠 기획 노트
- research_memo.md         : AI 도구·시장 리서치 메모
- consulting_schedule.md   : 3월 컨설팅·운영 일정
- README.md                : 데이터 안내

추천: 숫자 흐름이 궁금하면 매출 CSV부터, 콘텐츠 반응이 궁금하면 뉴스레터부터요.
```

**관찰 포인트**: 한국어로 물었는데 한국어로 답하고, **폴더를 실제로 읽어** 파일 역할까지 짚어줍니다. cc-02에서 본 에이전트 루프가, 이번엔 "내 진짜 폴더" 위에서 돌고 있는 거예요.

---

### 6단계: `@파일명`으로 콕 집어 읽히기

이제 핵심 손동작입니다. 파일 하나를 **`@`** 로 정확히 가리켜 읽혀 봅시다. 대화방에서 `@`를 치면 폴더 안 파일 목록이 자동으로 떠서 골라 넣을 수 있어요.

```text
@course_sales_monthly.csv 이 매출 데이터에서 눈에 띄는 점 3가지를,
비개발자도 이해되게, 실제 숫자를 인용해서 짧게 설명해줘.
```

**기대 결과** (실제 데이터 기반 예시):

```
1. 'AI활용' 카테고리가 매출의 중심이에요.
   '챗GPT로 콘텐츠 10배 빠르게'(79,000원)가 2026-01에 수강신청 187명,
   매출 1,477만원으로 단일 강의 최고치를 찍었어요.

2. 신규 강의가 두 달 만에 급성장했어요.
   '클로드코드로 업무 자동화 입문'(119,000원)이 2026-02 수강신청 46명에서
   2026-03 수강신청 118명, 매출 1,404만원으로 단숨에 상위권에 올라왔어요.

3. 객단가 높은 강의가 매출을 받쳐줘요.
   '1인 지식사업 수익화 로드맵'(149,000원)은 2026-01 순매출 1,167만원으로,
   수강 수는 적어도 단가로 기여가 큽니다.
```

> 💡 **할루시네이션(AI가 그럴듯하게 지어내기) 방지 팁**: 숫자가 의심되면 "**CSV의 실제 값**으로 다시 확인해서 보여줘"라고 한 번 더 요청하세요. 이게 비개발자가 AI를 안전하게 쓰는 가장 기본 기술입니다.

**관찰 포인트**: 파일을 복사해 붙이지 않았는데도 CC가 그 파일만 읽었습니다. **`@`는 "이 서류 좀 봐줘" 하고 책상에 정확히 올려주는 손동작**이에요.

## 🎬 실습 2 — 지우의 진짜 자료로 한 파일 깊게 파기 (15분)

> 목표: 파일 하나를 여러 각도로 읽혀 "관찰 → 해석"까지 가본다. 이번엔 **뉴스레터 성과표**(`newsletter_stats.csv`)와 **문의 로그**(`student_inquiries.csv`)를 쓰고, 마지막에 **비용 확인(`/cost`)** 과 **대화 정리(`/compact`)** 까지 손에 익힌다.

### 1단계: 뉴스레터에서 반응 좋은 호 찾기

```text
@newsletter_stats.csv 오픈율과 클릭율이 특히 높았던 호 3개를 뽑고,
각 호의 제목·오픈율·클릭율을 표로 정리해줘. 실제 수치 그대로.
```

**기대 결과**:

```
| 호수 | 제목                                      | 오픈율 | 클릭율 |
|------|-------------------------------------------|--------|--------|
| 33호 | AI에게 일을 시키는 사람 vs 못 시키는 사람       | 60.0% | 24.7% |
| 36호 | 터미널이 무서웠던 내가 AI 비서를 갖게 되기까지   | 61.0% | 25.8% |
| 40호 | 사전등록 마감 임박 (오늘 밤 12시)             | 58.0% | 30.3% |
```

**관찰 포인트**: 단순히 "잘 된 호"가 아니라 **수치를 표로** 받았습니다. 실습 1 6단계에서 본 `@` 손동작 위에, "표로 / 3개만" 같은 결과 모양을 얹은 것뿐이에요.

---

### 2단계: 패턴 한 줄로 요약시키기

```text
방금 뽑은 3개 호의 공통점이 뭔지, 반대로 반응이 낮았던 호와 비교해서
한 문단으로 설명해줘.
```

**기대 결과**:

```
반응이 높은 33·36·40호는 모두 'AI·자동화' 또는 '마감 임박' 같은
구체적·실용적·긴급성 주제예요. 특히 40호의 "오늘 밤 12시" 카피는
클릭율 30.3%로 최고치였어요. 반대로 34호(설 연휴 글 모음)·38호(외로움)
같은 감성 에세이류는 오픈율 42~44%로 평균 이하였습니다.
→ 정보형·긴급성 콘텐츠가 클릭을 끌어낸다는 신호예요.
```

**관찰 포인트**: 이건 `content_plan.md`의 "이번 분기 배운 점"과도 맞아떨어집니다(거기에도 "33·36·39호 클릭율 24%+", "40호 긴급성 카피 효과"가 적혀 있어요). 나중에(실습 3) 숫자 파일과 메모 파일을 **연결**할 때 이 관찰이 재료가 돼요.

---

### 3단계: 문의 로그에서 반복 이슈 뽑기

이번엔 다른 파일로 갈아탑니다.

```text
@student_inquiries.csv 카테고리별로 문의가 몇 건씩 들어왔는지 세고,
가장 많은 3개 유형을 건수와 함께 정리해줘.
```

**기대 결과** (예시):

```
1. '실습' 관련 문의가 가장 많아요 (설치·WSL·CLAUDE.md·AGENTS.md·Codex 로그인 등).
   특히 '클로드코드로 업무 자동화 입문' 수강생의 설치/실습 문의가 반복돼요.
   예) "터미널 설치하다 막혔어요 맥인데 명령어 입력하면 에러가 나요"(S004),
       "윈도우인데 WSL 설치가 너무 어려워요 도와주세요"(S013).
2. '강의문의' (비개발자도 따라갈 수 있나·뭐부터 들어야 하나·평생 소장인가 등).
3. '결제'·'환불' (강의가 안 열림·이중 결제·할인코드 미적용 / "생각했던 내용과 달라서" 환불).
```

**관찰 포인트**: AI가 "실습 문의가 많다"라고만 하면 멈추고 **"실제 문의 문구를 인용해줘"** 라고 시키세요. 위처럼 S004·S013 같은 실제 행을 끌어오면 신뢰할 수 있는 분석입니다.

---

### 4단계: 운영 개선 한 줄 제안받기

```text
방금 정리한 반복 문의를 보고, 지우가 시간을 가장 아낄 수 있는
운영 개선 1가지를 비개발자 눈높이로 제안해줘.
```

**기대 결과**:

```
'윈도우 WSL 설치'(S013, 처리 3시간)와 '터미널 설치 막힘'(S004, 처리 2시간)
같은 설치 문의가 반복되고 처리에 2~3시간씩 걸려요. → 설치 과정을 화면
녹화한 안내 영상을 하나 만들어 결제 직후 자동으로 보내면, 같은 문의를
사람이 매번 답하는 시간을 크게 줄일 수 있어요.
```

**관찰 포인트**: 이 제안은 `consulting_schedule.md`의 운영 메모("윈도우(WSL) 설치 문의가 반복됨 → 설치 가이드 영상을 따로 만들어 자동 안내로 돌리면 문의가 줄 듯")와 정확히 일치합니다. AI가 데이터에서 같은 결론에 도달한 거예요.

---

### 5단계: 비용 확인하기 — `/cost`

비개발자가 AI 도구를 쓸 때 가장 궁금한 것 중 하나가 **"이거 돈 얼마나 나가는 거지?"** 입니다. 여기까지 두 파일을 분석하느라 대화를 꽤 했으니, 대화방 안에서 이번 세션이 얼마나 들었는지 확인해 봅시다.

```text
/cost
```

**기대 결과**: 이번 세션에서 지금까지 쓴 양과 대략의 비용·사용량이 한눈에 표시됩니다(요금제·버전에 따라 화면 형태는 조금 다를 수 있어요).

```
이번 세션 사용량 / 비용 요약이 표시됩니다.
(예: 입력·출력 토큰 합계, 대략적인 누적 비용 등)
```

**관찰 포인트**: `/cost`는 "지금까지 얼마 썼지?"를 보는 **계기판**이에요. 대화가 길어질수록 비용은 올라갑니다. 그래서 다음 단계의 `/compact`(대화 요약)가 **비용을 줄이는 실전 습관**으로 이어져요 — 길게 쌓인 대화를 핵심만 남겨 가볍게 만들기 때문이죠. (비용·세션 관리의 깊은 내용은 cc-06에서 본격적으로 다룹니다.)

---

### 6단계: 명령으로 대화 정리하기 — `/compact`

큰 작업이 한 단락 끝났으니, 대화방 안에서:

```text
/compact
```

**기대 결과**: 긴 대화가 **핵심만 남기고 요약**됩니다. 화면은 깔끔해지지만 "지금 무슨 작업 중이었는지, 결론이 뭐였는지"는 그대로 유지돼요. 직후 `/cost`를 다시 한 번 눌러 보면, 대화가 가벼워진 만큼 이어지는 응답이 더 효율적이 되는 걸 느낄 수 있어요.

**관찰 포인트**: `/compact`는 cc-06에서 깊게 다루지만, **여기서 손에 익혀두면 좋은 슬래시 명령 첫 번째**예요. 대화가 길수록 답이 느려지고 비용이 오르니(방금 `/cost`로 확인했죠?), 큰 작업 끝마다 한 번씩 누르는 습관을 들이세요.

---

### 7단계: 자기 트랙도 같은 방식 (1줄 안내)

지금 본문은 **creator(콘텐츠·지식)** 트랙 데이터로 진행했어요. 만약 자기 사업이 다르다면 — store(스마트스토어)·freelancer(외주)·local(카페·공방·학원) — `practice-data/store/`, `practice-data/freelancer/`, `practice-data/local/` 폴더로 `cd` 해서 들어간 뒤, **똑같은 `@파일명` 손동작으로** 자기 트랙 데이터를 읽히면 됩니다. 파일 이름만 다를 뿐 손동작은 100% 같아요.

## 📝 실습 3 — 한 파일 미니 분석 메모 만들기 (선택·시간 남으면)

> 목표: 지금까지 읽힌 내용으로 **나에게 보내는 짧은 분석 메모 한 장**을 받아본다. (보고서 작성 본격 실습은 다음 모듈로 이어집니다.)

### 1단계: 관찰 → 해석 구조로 정리시키기

```text
지금까지 본 매출 CSV와 뉴스레터·문의 내용을 합쳐,
'이번 분기 내 사업에서 눈여겨볼 점' 메모를 만들어줘.
형식은 [관찰] 3줄 → [해석] 2줄 → [다음에 해볼 것] 2줄.
비개발자 눈높이로, 실제 수치를 한 번씩 인용해서.
```

**기대 결과**:

```
[관찰]
- 'AI활용' 강의가 매출 중심 (챗GPT 강의 2026-01 매출 1,477만원).
- 신규 클로드코드 강의가 두 달 만에 1,404만원으로 급성장.
- AI·긴급성 뉴스레터(33·36·40호)가 클릭율 24~30%로 최고.

[해석]
- 시장 관심이 'AI로 일하기'에 쏠려 있고, 우리 라인업이 그 흐름에 정확히 올라타 있다.
- 반대로 감성 에세이류(34·38호)·세금 기초 강의는 반응이 약하다.

[다음에 해볼 것]
- 클로드코드 강의 2차 모집 검토 (성장세 유효).
- WSL 설치 안내 영상 제작 → 반복 문의 자동화.
```

**관찰 포인트**: 한 폴더의 여러 파일을 읽혀 **"숫자(매출)와 맥락(뉴스레터·문의)을 잇는"** 첫 경험입니다. 이게 cc-04 이후 본격 보고서 작성의 씨앗이 돼요.

---

### 2단계: 메모를 한 단계 더 — 근거 붙이기

방금 받은 메모는 좋지만, 나중에 다시 봐도 믿으려면 **"왜 그렇게 말했는지"** 한 줄씩 근거가 붙으면 더 좋아요.

```text
방금 메모의 [관찰] 3줄 각각에, 어느 파일·어느 수치에서 나온 건지
괄호로 출처를 한 줄씩 붙여줘. 예: (course_sales_monthly.csv, 2026-01).
```

**기대 결과**:

```
[관찰]
- 'AI활용' 강의가 매출 중심 (course_sales_monthly.csv, 챗GPT 2026-01 매출 1,477만원).
- 신규 클로드코드 강의 두 달 만에 급성장 (course_sales_monthly.csv, 2026-02→03: 547→1,404만원).
- AI·긴급성 뉴스레터가 클릭율 최고 (newsletter_stats.csv, 33/36/40호 24.7~30.3%).
```

**관찰 포인트**: "출처를 붙여줘"는 비개발자가 AI 분석을 **검증 가능하게** 만드는 강력한 한마디예요. 숫자가 의심되면 그 파일을 `@`로 다시 열어 직접 대조하면 됩니다.

---

### 3단계: 단축키로 깔끔하게 마무리

대화를 마칠 땐 다음 단축키·명령이 손에 익으면 편해요.

- `Esc`(또는 `Ctrl+C` 한 번): AI가 엉뚱한 방향으로 가면 **즉시 중단**.
- `Shift+Enter`: 한 메시지 안에서 **여러 줄**로 길게 쓰고 싶을 때 줄바꿈.
- `/clear`: 완전히 다른 일로 갈아탈 때 대화를 **백지로**(되돌릴 수 없으니 중요한 결과는 먼저 저장).
- `/exit` 또는 `Ctrl+C` 두 번: 대화방 닫고 터미널로 나가기.

> ⚠️ `/clear`와 `/compact`를 헷갈리지 마세요. **`/compact`=정리하되 기억 유지**, **`/clear`=기억까지 삭제**. 헷갈리면 더 안전한 `/compact`를 쓰세요.

## 🔍 흔한 오류와 해결법 (트러블슈팅)

| 상황 | 원인 | 해결책 |
|------|------|--------|
| "파일을 못 찾는다"고 함 | 엉뚱한 폴더에서 `claude`를 켬 | `Ctrl+C`로 나와서 `pwd`로 위치 확인 → `cd`로 자료 폴더 이동 → `ls`로 파일 보이는지 확인 후 다시 `claude` |
| `cd` 했는데 "no such file or directory" | 폴더 이름 오타 또는 위치 착오 | `ls`로 실제 폴더 이름을 보고 그대로 입력. 띄어쓰기·한글 이름은 따옴표로 감싸기 |
| `@`를 쳐도 파일 목록이 안 뜸 | CC를 켠 폴더에 그 파일이 없음 | 대화 밖에서 `ls`로 확인. 파일이 다른 폴더면 거기서 다시 `claude` |
| AI가 지어낸 숫자 제시 | 할루시네이션(모델 한계) | "**CSV의 실제 값**을 확인해서 다시 말해줘" 요청 |
| 답이 점점 느려지고 이상해짐 | 대화가 길어 컨텍스트가 참 | `/cost`로 사용량 확인 → `/compact`로 요약, 완전히 다른 일이면 `/clear` |
| 비용이 걱정됨 | 사용량을 본 적이 없음 | 대화 중 `/cost`로 이번 세션 사용량·비용 확인. 길어졌으면 `/compact` |
| `/compact`를 터미널에서 쳤더니 안 됨 | 슬래시 명령을 켜기 전에 입력 | `claude`로 대화방에 **먼저 들어간 뒤** 그 안에서 `/compact` |
| AI가 엉뚱한 방향으로 감 | 지시가 모호하거나 과함 | `Esc`로 중단 → "그건 아니고 ○○로 다시 해줘"로 재요청 |

> **마음가짐**: 어떤 명령을 잘못 쳐도 컴퓨터는 안 망가집니다. CC는 파일을 바꾸기 전에 먼저 물어보는 안전장치가 있어요. 틀려도 괜찮으니 일단 해보세요.

## 💡 핵심 규칙·개념 정리

### 규칙 1: 길 찾기는 세 글자면 충분하다

```
pwd  = 여기 어디지?   (현 위치)
ls   = 뭐가 있지?     (둘러보기)
cd   = 저기로 가자     (이동) /  cd .. = 뒤로 /  cd ~ = 홈으로 리셋
```
→ **이동 후 항상 `pwd`/`ls`로 확인**하는 두 박자가 길 안 잃는 비결.

### 규칙 2: 파일은 `@`로 콕 집어 쥐여준다

```
❌  (CSV 내용을 통째로 복사해서 붙여넣기)
✅  @course_sales_monthly.csv 매출 패턴 3가지만 표로 정리해줘
```
→ `@`는 "이 서류 좀 봐줘" 하고 책상에 정확히 올려주는 손동작.

### 규칙 3: 명령은 "켜기 전 / 대화 중" 두 칸으로 나눠 기억한다

```
켜기 전 (터미널 $) : claude / claude --help
                     claude -c (이어 하기 — 세션 이어가기는 cc-06에서 자세히)
대화 중 (채팅창 /) : /compact / /clear / /cost / /help
손가락 단축키      : Ctrl+C(멈춤·나가기) / Esc(중단) / Shift+Enter(줄바꿈)
```

### 초보자 명령 TOP 5 (책상에 붙여두기 — 가장 자주 쓰는 다섯)

> 처음 한 달은 이 다섯 개만 손에 익혀도 충분합니다.

1. **`/compact`** — 긴 대화 요약 (가장 중요)
2. **`Ctrl+C`** — 작업 취소·나가기
3. **`/clear`** — 새로 시작 (되돌릴 수 없음 주의)
4. **`Shift+Enter`** — 여러 줄 입력
5. **`/cost`** — 비용·사용량 확인

### 이 레슨의 핵심 손동작 (TOP 5와 별개 — "파일 다루기" 두 가지)

위 TOP 5는 어느 레슨에서나 매일 쓰는 명령이고, **이번 cc-03에서 새로 손에 익힌 건 아래 두 가지**예요.

- **`cd` (+ `pwd`/`ls`)** — 자료가 있는 폴더로 정확히 들어가기.
- **`@파일명`** — 그 폴더의 파일을 콕 집어 AI에게 읽히기.
