# 1-1: Claude Code 첫 대화 — "말을 걸면 일이 시작된다"

## 학습 목표
- Claude Code를 처음 실행하는 흐름을 이해한다.
- 한국어 자연어로 요청해도 일이 시작된다는 감각을 얻는다.
- "명령어를 다 몰라도 된다"는 첫 자신감을 만든다.
- 역할·배경을 주면 답이 좋아진다는 것을 체감한다(첫 와우 모먼트).

## 사전 준비
- Claude Code가 설치·실행 가능한 환경(터미널 1개).
- 현재 위치가 실습용 폴더(이 코스 폴더 또는 작업 폴더)인지 확인.
- "실수해도 괜찮다"는 마음. 오늘은 *대화 성공* 하나면 충분.

## 진행

### 도입 (5분)
- 오늘 목표는 거창하지 않다. **AI와 첫 대화**만 성공하면 된다.
- 비유: 새 동료에게 처음 말을 거는 시간. 중요한 건 문법이 아니라 **요청의 의도**.
- 박서연(비개발자)은 개발자처럼 말하려 애쓸 필요 없다 — 평소 업무 메신저처럼 말하면 된다.
  - 예: "이 폴더가 뭔지 알려줘" / "내가 여기서 뭘 하면 좋을지 알려줘"
- Claude Code = "엑셀을 대신 클릭해주는 도구"가 아니라 "**같이 생각해주는 실무 파트너**".
- 오늘의 와우 모먼트: **말을 걸었더니 컴퓨터 안의 일을 같이 풀어주기 시작한다.**

체크 포인트: 입력창 위치를 찾았나 / 현재 폴더가 어디인지 아나 / 한국어로 물어도 된다는 걸 이해했나.

### 실습 1 — 첫 인사 (10분)
**목표**: Claude Code를 열고 첫 인사 + 현재 폴더 기준 도움 요청.

따라 하기:
1. 터미널에서 Claude Code를 실행한다.
2. 아래처럼 아주 짧게 입력한다(완벽할 필요 없음).
```text
안녕하세요! 저는 비개발자인데, 이 폴더에서 무엇을 할 수 있는지 알려주세요.
```
3. 답이 나오면 바로 이어서 묻는다.
```text
저는 운영기획팀 실무자예요. 오늘 어떤 실습을 하면 좋을까요?
```
4. 제안 중 가장 쉬워 보이는 것을 골라 다시 묻는다.
```text
좋아요. 가장 쉬운 것부터 한 단계씩 안내해주세요.
```

기대 결과: 현재 폴더·상황 기반 제안이 나오고, 한 번으로 끝나지 않고 대화가 이어진다. → "다음 질문을 던지면 흐름이 계속된다"는 감각.

관찰 포인트: 답이 너무 길면 "더 짧게", 용어가 어려우면 "비개발자 말투로 바꿔줘"라고 요청할 수 있다. **수강생은 답을 다듬게 만드는 사람**이기도 하다.

### 실습 2 — 역할 부여 와우 모먼트 (15분)
**목표**: 역할·배경·목표를 주면 답이 더 구체적이 됨을 체감.

따라 하기:
1. 아래 문장을 그대로 입력한다.
```text
저는 운영기획팀의 비개발자 실무자예요. 매출 데이터와 회의록으로 첫 분석 보고서를 만들어야 해요. 오늘 Claude Code로 할 수 있는 가장 현실적인 첫 작업 3가지를 알려주세요.
```
2. 답에서 가장 끌리는 항목 하나를 고른다.
3. 이어서 요청한다.
```text
좋아요. 그 작업을 제가 따라할 수 있게 10분 단위로 나눠서 안내해주세요.
```
4. 이해 안 되는 문장이 있으면 바로 다시 묻는다.
```text
이 단계가 어려워요. 중학생도 이해할 수 있게 다시 말해주세요.
```

기대 결과: 역할·상황·목표를 주자 답이 더 구체적이 됨. Claude Code가 단순 채팅이 아니라 **업무 코치**처럼 보이기 시작. → "좋은 질문 = 배경을 더 준 질문".

미니 회고: 그냥 "뭐 할 수 있어?"와 무엇이 달랐나 / "운영기획팀 실무자" 역할이 붙자 더 현실적이었나.

### 마무리 (5분)
- 오늘 핵심은 명령어가 아니라 **대화 방식**.
- 세 가지만 기억: ① 한국어로 말해도 된다 ② 상황을 알려주면 답이 좋아진다 ③ 답이 별로면 다시 시킬 수 있다.
- 한 줄 정리: **"좋은 협업은 좋은 첫 질문에서 시작된다."**
- 다음 시간엔 "이 폴더에 뭐가 있어?"로 실제 파일(매출·회의록)을 다룬다.

## 트러블슈팅
- **무엇을 입력할지 막막**: 자기소개 + 목표 + 원하는 도움 3가지만 넣기. (예: "비개발자예요 / 폴더를 이해하고 싶어요 / 한 단계씩 설명해주세요")
- **답이 너무 길다**: "핵심만 5줄로 줄여주세요."
- **용어가 어렵다**: "비개발자 눈높이로 다시 설명해주세요."
- **잘 하고 있는지 모르겠다**: "제가 잘 따라오고 있는지 체크리스트로 알려주세요."
