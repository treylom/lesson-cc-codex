# codex-08: 비대화형 자동화 · codex exec · CI/CD — "자는 동안에도 일이 끝나 있다"

> **레슨 목표**: Codex를 대화 없이 플래그 한 줄로 실행하는 '비대화형(non-interactive)' 방식을 익히고, 반복 업무를 자동화하는 첫 경험을 만든다.
> **예상 시간**: 60분 (개념 도입 10분 + 실습 1 20분 + 실습 2 20분 + 실습 3 선택 10분)
> **사전 준비**: Codex 설치·로그인 완료(codex-01) / AGENTS.md 경험(codex-06) / `practice-data/creator/` 폴더 열어두기 / 터미널 1개
> **진입 커맨드**: `/codex-08`

---

## 📚 학습 목표 — 4가지 와우 모먼트

1. **"대화 없이 일이 끝난다"를 직접 체감한다** — Codex를 켜고 말을 주고받는 대신, 플래그 한 줄로 Codex가 파일을 읽고 분석 결과 파일까지 만드는 장면을 본다.
2. **`codex exec`와 `--sandbox read-only` 패턴을 손에 익힌다** — 안전 모드와 자동 실행 모드를 구분해서 세 번 써보면 자연스러워진다.
3. **"언제 대화형, 언제 비대화형인가"를 스스로 판단한다** — 탐색할 때는 대화, 같은 일을 반복할 때는 자동(비대화형)이라는 구분을 감각으로 잡는다.
4. **CI/CD가 무엇인지 한 줄로 설명한다** — "자동화 파이프라인"이 비대화형 실행의 연장선이라는 것을 코딩 지식 없이 이해한다.

## 🎯 개념 도입 — "대화형 vs 비대화형", 왜 다를까?

### 일상 비유로 시작

지금까지 배운 방식은 이렇습니다.

```
지우 → "이 파일 분석해줘"
Codex → 분석 중… → "이렇게 됩니다"
지우 → "더 자세히 보여줘"
Codex → "알겠습니다"
```

이걸 **대화형(interactive)** 이라고 합니다. 사람이 앞에 앉아서 계속 반응을 주고받는 방식입니다. 그런데 지우는 이런 생각이 들기 시작했습니다.

> *"매달 말에 하는 매출 정리, Codex에게 시키는 방법은 이미 알아. 그런데 그때마다 터미널 앞에 앉아서 똑같은 말을 해야 한다는 게… 비효율 아닌가?"*

맞습니다. 그래서 있는 것이 **비대화형(non-interactive) 실행**입니다.

> 📬 **비유**: 대화형은 **전화 통화**입니다. 주고받으면서 진행합니다. 비대화형은 **문자 메시지 + 자동 처리**입니다. "이거 해줘"라고 보내두면, 상대가 알아서 처리해서 결과를 돌려줍니다. 기다리지 않아도 됩니다.

또 다른 비유: 매일 카운터에 가서 "아이스 아메리카노, 얼음 많이"를 말하는 것(대화형) vs 커피머신에 "아침 7시에 아이스 아메리카노 뽑아줘"를 설정해 두는 것(비대화형). 자는 동안에도 커피가 만들어집니다.

### Codex 비대화형 실행이란?

Codex를 **대화 없이 명령 한 줄로 바로 실행**하는 방법이 있습니다.

```bash
# 대화형 — Codex가 채팅창을 열고 기다림
codex "이 파일 분석해줘"

# 대화형(read-only 구경 모드) — 대화창이 열리되 파일을 읽기만 함
codex --sandbox read-only "course_sales_monthly.csv 분석해서 요약해줘"

# 비대화형(자동 실행) — 파일 생성까지 자동으로
codex exec --sandbox workspace-write "course_sales_monthly.csv 분석해서 report.md로 저장해줘"
```

뜯어보면:
- `--sandbox read-only` = "파일을 읽기만 하고 아무것도 쓰지 마라"는 안전장치. `codex exec`와 함께 써야 비대화형으로 작동합니다.
- `codex exec --sandbox workspace-write` = "작업 폴더 안에서 파일을 만들어도 된다"는 허가증. 결과 파일이 필요할 때 사용.

차이를 한 줄로 정리하면:

| 구분 | 대화형 | 비대화형(`codex exec`) |
|------|--------|---------------------|
| 방식 | 채팅처럼 주고받음 | 명령 한 줄로 끝 |
| 사람 필요 여부 | 앞에 앉아서 봐야 함 | 자리 비워도 됨 |
| 결과 | 대화창에서 확인 | 파일로 남음 |
| 언제 유리 | 복잡한 판단, 방향 수정, 처음 보는 데이터 | 반복 작업, 결과물 형식이 정해진 일, 자동화 |

### CI/CD가 뭔가요? — 비개발자 버전

이 레슨의 제목에 "CI/CD"라는 말이 있습니다. 개발자 세계에서 많이 쓰는 용어인데, 본질은 단순합니다.

- **CI(지속적 통합, Continuous Integration)** = "코드를 올릴 때마다 자동으로 검사해줘"
- **CD(지속적 배포, Continuous Delivery/Deployment)** = "검사 통과하면 자동으로 배포해줘"

개발자 세계의 개념이지만, 1인 사업자 지우에게 적용하면 이렇게 됩니다.

> "매달 1일이 되면 알아서 매출 정리 보고서가 만들어져 있다."
> "매주 월요일 아침, 지난주 뉴스레터 성과 요약이 폴더에 저장돼 있다."
> "새 문의가 쌓이면 분류·정리가 자동으로 된다."

이게 지우 버전의 CI/CD입니다. `codex exec`는 그 자동화 파이프라인을 만드는 가장 첫 번째 벽돌입니다.

> 💡 이 레슨은 **"부록"** 입니다. 당장 매일 써야 하는 기능이 아니라, "이런 것도 가능하다"를 체험하고 나중에 자동화를 생각할 때 꺼낼 수 있는 도구입니다. 완벽히 익히지 않아도 괜찮습니다.

## 🔧 실습 1 — 비대화형 첫 체험: 강의 매출 파일을 대화 없이 분석

**목표**: `course_sales_monthly.csv`를 대화 없이 분석하고, 결과를 파일로 받아봅니다.

### 1단계: 터미널에서 practice-data/creator 폴더 확인

터미널을 열고 실습 폴더로 이동합니다.

```bash
cd ~/practice-data/creator
```

확인을 위해 아래를 쳐보세요.

```bash
ls
```

**기대 결과**:
```
README.md

newsletter_stats.csv
research_memo.md

```

> **체크**: 이 7개 파일이 보여야 합니다. 안 보인다면 경로를 다시 확인하세요.

---

### 2단계: 비대화형 플래그의 생김새 눈으로 읽기

실제로 실행하기 전에, 명령의 구조를 먼저 눈으로 읽어봅니다. 외울 필요는 없고 "이런 구조구나" 하고 감만 잡으세요.

```
codex exec  --sandbox read-only  "지시문"
     ↑              ↑                  ↑
  비대화형         읽기만 해          무엇을 할지
  자동 실행       (안전모드)           설명
```

한글로 읽으면: "읽기만 하는 안전 모드로, 대화 없이 혼자 끝내라."

> **참고**: `codex --sandbox read-only "..."` (exec 없이)는 대화창이 열리는 대화형으로 작동합니다. 비대화형으로 실행하려면 반드시 `codex exec`를 앞에 붙여야 합니다.

두 모드의 차이:

| 방식 | 파일 읽기 | 파일 쓰기 | 사용 시점 |
|--------|----------|----------|----------|
| `codex exec --sandbox read-only` | ✅ | ❌ | 처음 탐색, 안전하게 확인할 때 |
| `codex exec --sandbox workspace-write` | ✅ | ✅ | 결과 파일을 만들어야 할 때 |

---

### 3단계: 가장 안전한 모드로 첫 실행

`codex exec --sandbox read-only`를 쓰면 Codex가 대화창 없이 파일을 읽기만 하고 아무것도 고치거나 만들지 않습니다. 처음엔 이 안전 모드로 시작합니다.

```text
codex exec --sandbox read-only "이 폴더에 있는 course_sales_monthly.csv 파일을 읽고, 전체 강의 목록과 9개월(2025-07~2026-03) 동안 각 강의의 총 순매출을 정리해줘. 한국어로."
```

**기대 결과 (예시)**:
```
course_sales_monthly.csv를 분석했습니다.

강의별 총 순매출 (2025-07 ~ 2026-03):
- 챗GPT로 콘텐츠 10배 빠르게: 약 7,260만원 (AI활용, 자사사이트)
- 1인 지식사업 수익화 로드맵: 약 5,312만원 (비즈니스, 자사사이트)
- 노션으로 1인 사업 시스템 만들기: 약 1,920만원 (생산성, 자사사이트)
- 초보 뉴스레터 7일 완성: 약 1,268만원 (글쓰기, 클래스101)
- 클로드코드로 업무 자동화 입문: 1,750만원 (AI활용, 2개월 합산)
- 크리에이터 세금 기초: 약 191만원 (비즈니스, 탈잉)
```

> **관찰**: 대화창이 열리지 않았습니다. Codex가 파일을 알아서 읽고 결과만 터미널에 출력했습니다. `codex exec`가 붙어야 이게 가능합니다 — 이게 비대화형의 핵심입니다.

**체크포인트**: 결과가 한국어로 나왔나요? 강의 목록이 실제 CSV 내용과 맞게 나왔나요?

---

### 4단계: 결과를 파일로 저장하게 시키기

이번엔 결과를 **파일로** 저장하게 해봅니다. `codex exec --sandbox workspace-write`를 사용합니다.

> ⚠️ `workspace-write`는 Codex가 작업 폴더 안에서 파일을 만들 수 있는 모드입니다. 기존 CSV 파일은 수정하지 않고, 새 결과 파일만 만듭니다.

```text
codex exec --sandbox workspace-write "course_sales_monthly.csv를 읽어서 다음 내용을 담은 요약 파일을 만들어줘:
1. 전체 기간(2025-07 ~ 2026-03) 순매출 Top 3 강의
2. 가장 최근 달(2026-03)의 강의별 순매출 비교
3. 한 줄 제안: 지금 가장 투자할 만한 강의는?
파일명: sales_summary.md, 한국어로, 비개발자가 읽기 쉽게."
```

**기대 결과** — `sales_summary.md` 파일이 생성되며, 내용은 대략:

```markdown
# 강의 매출 요약

## 순매출 Top 3 강의 (2025-07 ~ 2026-03)
1. 챗GPT로 콘텐츠 10배 빠르게 — 누적 순매출 약 7,260만원
2. 1인 지식사업 수익화 로드맵 — 누적 순매출 약 5,312만원
3. 클로드코드로 업무 자동화 입문 — 약 1,750만원 (단 2개월)

## 2026년 3월 강의별 순매출
- 클로드코드로 업무 자동화 입문: 1,242만원 (신규 강의 1위)
- 1인 지식사업 수익화 로드맵: 895만원
- 챗GPT로 콘텐츠 10배 빠르게: 823만원

## 제안
클로드코드 강의가 출시 2개월 만에 월 1,242만원으로 1위에 올랐습니다.
AI 활용 키워드에 대한 시장 수요가 데이터로 확인됐으므로,
다음 강의도 AI 활용 계열로 이어가는 것이 현실적입니다.
```

> **관찰**: 지우가 아무것도 하지 않았는데 Codex가 CSV를 읽고, 분석하고, 새 파일까지 만들었습니다. 대화창이 없었습니다.

### 5단계: 파일이 실제로 생겼는지 확인

```bash
ls -la sales_summary.md
```

**기대 결과**: `sales_summary.md`가 목록에 나타납니다.

```bash
cat sales_summary.md
```

### 6단계: `@파일명`으로 정확히 지목하기

Codex는 `@파일명`으로 파일을 정확히 지목할 수 있습니다. 폴더에 파일이 여러 개 있을 때 헷갈리지 않게 합니다.

```text
codex exec --sandbox read-only "@course_sales_monthly.csv 에서 2026년 데이터만 보고, 클로드코드 강의(클로드코드로 업무 자동화 입문)가 처음 나온 달이 언제이고, 그 다음 달에 얼마나 성장했는지 알려줘."
```

**기대 결과**:
```
course_sales_monthly.csv 확인했습니다.

"클로드코드로 업무 자동화 입문" 강의:
- 2026-02: 첫 등장, 수강신청 46명, 순매출 508만원
- 2026-03: 수강신청 118명 (전월 대비 +156%), 순매출 1,242만원 (+144%)

한 달 만에 수강생이 2.5배 이상, 매출이 2.4배 이상 늘어난 급성장 강의입니다.
```

> **관찰 포인트**: 실제 CSV 데이터를 보면 클로드코드 강의는 2026-02에 첫 등장(순매출 508만원), 2026-03에 1,242만원으로 폭발적 성장했습니다. Codex가 이 흐름을 정확히 잡아냈는지 확인해보세요.

## 🎬 실습 2 — 실제 사업 데이터 연결 분석: 세 파일 통합 보고서

**목표**: `course_sales_monthly.csv`, `newsletter_stats.csv`, `student_inquiries.csv` 세 파일을 동시에 분석해 "3월 사업 현황 보고서"를 자동으로 만들어봅니다. 지우가 매달 손으로 하던 일을 비대화형으로 자동화하는 경험입니다.

---

### 1단계: 세 파일의 연결 구조 파악

분석을 시키기 전에, Codex에게 파일 구조를 파악하게 합니다.

```text
codex exec --sandbox read-only "이 폴더에 있는 course_sales_monthly.csv, newsletter_stats.csv, student_inquiries.csv 세 파일의 구조를 각각 한 줄씩 설명하고, 세 파일이 어떻게 연결될 수 있는지 알려줘."
```

**기대 결과 (예시)**:
```
세 파일 설명:

1. course_sales_monthly.csv
   - 강의별·월별 매출 데이터 (2025-07 ~ 2026-03, 7개 강의)
   - 주요 열: 강의명, 플랫폼, 수강신청수, 환불수, 순매출

2. newsletter_stats.csv
   - 뉴스레터 18호~41호 성과 데이터
   - 주요 열: 호수, 발송일, 오픈율, 클릭율, 구독 증감

3. student_inquiries.csv
   - 수강생·문의 내역 (2026년 3월, 24건)
   - 주요 열: 카테고리, 강의명, 처리상태, 소요시간, 우선순위

연결 포인트:
- newsletter → course_sales: 특정 호 발송 후 그달 강의 매출 변화 추적 가능
- student_inquiries → course: 문의 빈도가 높은 강의 = 지원 비용도 높음
- 세 파일 종합: "어떤 콘텐츠를 어떻게 홍보하면 → 구독자가 늘고 → 강의가 팔리나"
```

> **관찰**: 세 파일의 관계를 Codex가 파악합니다. 이 이해가 다음 단계 분석의 기반이 됩니다.

### 2단계: 뉴스레터와 강의 매출의 연관 관계 분석

### 5단계: CI/CD의 큰 그림 맛보기 (이론만)

⚠️ 이 단계는 **직접 하지 않습니다** — 아직 배우지 않은 범위(GitHub Actions 등)이기 때문입니다. 하지만 "이런 세계가 있다"는 것만 확인합니다.

지금 만든 `analyze_monthly.sh` 스크립트는 이론상 이런 곳에 연결할 수 있습니다:

```yaml
# GitHub Actions에서 자동 실행 예시 — 참고용, 실습 아님
name: 월간 사업 분석 자동화
on:
  schedule:
    - cron: '0 9 1 * *'   # 매달 1일 오전 9시
jobs:
  analyze:
    runs-on: ubuntu-latest
    steps:
      - name: Codex 분석 실행
        run: ./analyze_monthly.sh
```

이것이 "매달 1일 오전 9시, 자동으로 보고서가 만들어지는" 세계입니다. 지우가 이 코스를 끝내고 GitHub(cc-07, 이미 배운 내용)와 연결하면 이 자동화가 현실이 됩니다.

## 🔍 흔한 오류와 해결법

| 상황 | 원인 | 해결 방법 |
|------|------|-----------|
| `codex: command not found` | Codex가 설치됐지만 경로를 못 찾음 | 터미널을 완전히 닫았다 다시 열기. 안 되면 `npm update -g @openai/codex` 시도 |
| `codex exec: command not found` | Codex 버전이 낮거나 업데이트 필요 | `npm update -g @openai/codex` 후 재시도. `codex --help`로 exec 서브명령 확인 |
| Codex가 파일을 못 찾는다고 함 | 현재 폴더가 practice-data/creator가 아님 | `pwd`로 위치 확인 → `cd .../practice-data/creator` 다시 이동 |
| Codex가 중간에 멈추고 질문함 | 승인이 필요한 작업 감지 | `codex exec --sandbox workspace-write` 형식 사용 또는 질문에 yes 입력 |
| 파일이 생성되지 않음 | `--sandbox read-only`는 파일 쓰기 불가 | `codex exec --sandbox workspace-write`로 바꾸기 |
| 분석 결과의 수치가 이상함 | AI가 추측으로 채웠을 가능성 | `@파일명`으로 정확히 지목해서 다시 시도. AGENTS.md에 "추측 금지, 파일 실값만" 규칙 추가 |
| 결과 파일 내용이 너무 짧음 | 지시문이 모호하거나 형식 미지정 | 지시문에 "파일명과 담을 내용 항목"을 명시 |
| `permission denied` 오류 | 쉘 스크립트에 실행 권한이 없음 | `chmod +x 파일명.sh` 한 줄이면 해결 |
| 영어로 답이 나옴 | Codex가 기본 영어로 응답 | 요청문 끝에 "한국어로 답해줘" 한 마디 추가 |
| 모델 관련 에러 (`gpt-5.5` 없음) | 구독·계정에 따라 사용 가능 모델이 다를 수 있음 | `gpt-5.4-mini`로 변경: `codex -m gpt-5.4-mini "..."` 또는 재로그인·구독 확인 |
| 같은 명령인데 결과가 매번 다름 | AI의 확률적 생성 특성 | AGENTS.md에 형식 규칙 강화, 결과 검토 습관 |

## 💡 핵심 규칙 / 개념 정리

### 비대화형 실행의 3가지 원칙

**원칙 1: 지시를 완전하게 써라**
대화형이면 "더 자세히" 하고 다시 요청할 수 있습니다. 비대화형은 한 번 실행하면 끝입니다. 처음부터 원하는 것을 모두 담아야 합니다.

```text
❌ 짧은 버전: codex exec --sandbox workspace-write "분석해줘"
✅ 완전한 버전: codex exec --sandbox workspace-write "course_sales_monthly.csv를 읽어서 강의별 순매출을 정리하고 결과를 report.md로 저장해줘. 한국어로."
```

**원칙 2: 결과를 파일로 남겨라**
터미널 출력은 창을 닫으면 사라집니다. "~로 저장해줘"를 항상 붙이세요.

```text
✅ "분석 결과를 analysis.md로 저장해줘"
```

**원칙 3: `@파일명`으로 정확히 지목해라**
폴더에 파일이 여러 개 있으면 Codex가 어느 파일을 봐야 하는지 헷갈릴 수 있습니다. `@파일명`으로 정확히 지목하면 답이 더 정확해집니다.

```text
✅ codex exec --sandbox read-only "@course_sales_monthly.csv 에서 3월 데이터만 보여줘"
```

### 대화형 vs 비대화형 — 언제 무엇을?

| 상황 | 추천 방식 |
|------|----------|
| 처음 보는 데이터, 방향이 불확실할 때 | 대화형 — 주고받으며 탐색 |
| 매달 반복하는 정해진 분석 | 비대화형 — `codex exec` |
| "이게 맞나?" 판단이 필요할 때 | 대화형 — Codex와 상의 |
| 파일을 만들어 남겨야 할 때 | 비대화형 — 결과 파일 지정 |
| 자동화, 스크립트로 만들어야 할 때 | 비대화형 — 쉘 스크립트 |

### 비대화형 실행 명령 요약

```bash
# 기본 형태 (대화형 — 비교용)
codex "지시문"

# 비대화형 안전 모드 (읽기만, 파일 생성 불가)
codex exec --sandbox read-only "지시문"

# 비대화형 파일 수정·생성 허용 (비대화형 표준)
codex exec --sandbox workspace-write "지시문 → 결과를 파일로 저장해줘"

# 모델 지정 (gpt-5.5 없을 때는 gpt-5.4-mini로)
codex -m gpt-5.4-mini "지시문"

# 파일 정확히 지목 (비대화형)
codex exec --sandbox read-only "@파일명 지시문"
```

### AGENTS.md와의 연결

비대화형 실행도 AGENTS.md의 규칙을 읽습니다. 한 번 써두면 매번 명령에서 반복하지 않아도 됩니다. codex-06에서 배운 것이 여기서 빛을 발합니다.
