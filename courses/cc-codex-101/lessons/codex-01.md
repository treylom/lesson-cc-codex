2. **두 번째 AI 파트너가 내 폴더 안에서 일한다** — 채팅창에 복붙하는 게 아니라, Codex가 `practice-data/creator/` 폴더를 직접 열어 내 강의 매출·문의 파일을 읽는다. 〔와우: AI가 알아서 파일을 찾아 읽고 요약해 준다.〕

4. **위험한 일은 먼저 묻는다(follow-through)** — Codex가 되돌릴 수 있는 일은 알아서 하고, 파일을 지우거나 덮어쓰는 위험한 일은 "이거 해도 될까요?" 하고 멈춰 묻는다는 안전감.

## 🎯 개념 도입 (7분)

### Codex는 뭔가요? — "Claude Code의 형제 도구"

cc-01에서 만난 Claude Code를 떠올려 보세요. "내 컴퓨터 폴더 안에서 일하는 AI 비서"였죠. 채팅창에 파일을 복사해 붙여넣는 대신, AI가 폴더를 직접 열어 파일을 읽고·고치고·새 파일을 만들어 줬습니다.

**Codex는 정확히 같은 결의 도구입니다.** 만든 회사만 다릅니다.

- **Claude Code** = Anthropic(Claude를 만든 회사)이 만든 터미널 AI 비서
- **Codex** = OpenAI(ChatGPT를 만든 회사)가 만든 터미널 AI 비서

> 💡 **비유 — 같은 직군의 두 비서**
> CC와 Codex는 "다른 회사에서 온 두 명의 비서"라고 생각하면 됩니다. 한 명은 Anthropic 출신(CC), 한 명은 OpenAI 출신(Codex). 둘 다 일하는 방식(파일 읽고 → 분석하고 → 결과 만들기)은 똑같고, 출신 회사·이름표·말버릇 몇 개만 다릅니다. 같은 종류의 두 가전 브랜드(둘 다 냉장고인데 한쪽은 삼성, 한쪽은 LG)처럼, 큰 사용법은 거의 같고 버튼 모양 몇 개만 다른 셈이에요. 한 명을 부려 봤으면 다른 명은 첫날부터 손발이 맞습니다.

### 그런데 왜 두 번째 도구까지 깔아요?

지우의 마음속 질문: *"CC 하나면 충분한 거 아닌가? 굳이 또 깔아야 해?"*

답은 이 코스의 큰 그림에 있습니다. 이 코스는 **"CC vs Codex, 뭐가 더 좋냐"를 가리는 게 아닙니다.** 두 도구를 **둘 다 내 사업의 운영 파트너로** 만드는 게 목표예요. 같은 부탁을 두 비서에게 던져 보고 "이 일은 이 친구가 낫네"를 아는 것 — 그게 1인 사업자에게 진짜 힘이 됩니다.

지우의 동료가 했던 말을 기억하세요. *"큰 작업은 한쪽으로 하다가, 특정 부분은 다른 AI 시각으로도 한 번 봐줘 하고 맡겨."* 혼자 일하지만, 같은 일을 두 시각으로 검산해 줄 짝꿍이 둘이 되는 셈이죠.

> 🎬 **이번 클립의 위치**: 이건 Part 1의 cc-01(Claude Code 설치)과 **짝을 이루는 클립**입니다. cc-01에서 첫 비서를 들였다면, codex-01에서 두 번째 비서를 들입니다. 같은 동작을 한 번 더 하는 거라, cc-01을 했다면 오늘은 사실상 복습 + α예요.

### "CC를 익혔으면 Codex는 90% 그대로다"

이 한 줄이 이번 레슨 전체의 마음가짐입니다. 새로 외울 건 사실 몇 개뿐이에요.

| 무엇을 | Claude Code (cc-01에서 함) | Codex (지금 codex-01) | 한 줄 |
|---|---|---|---|
| 만든 곳 | Anthropic | OpenAI | 회사만 다름 |
| 설치 | `npm install -g @anthropic-ai/claude-code` | `npm install -g @openai/codex` | 둘 다 npm 한 줄 |
| 로그인 | `claude login` (Anthropic 계정) | `codex login` (ChatGPT 계정) | 계정만 다름 |
| 첫 실행 | `claude "..."` | `codex "..."` | 똑같이 따옴표 안에 질문 |
| 규칙서 | `CLAUDE.md` | `AGENTS.md` | 이름만 다른 같은 파일 |
| 작업 폴더 지정 | 그 폴더로 `cd` 후 실행 | `codex --cd <폴더>` 또는 `cd` 후 실행 | 폴더를 무대로 삼는 건 같음 |
| 설치 확인 | `claude --version` | `codex --version` | 버전 숫자가 뜨면 성공 |

> 💡 **용어 풀이 — npm**: npm(엔피엠)은 "프로그램을 인터넷에서 내려받아 깔아 주는 설치 도우미"예요. 앱스토어에서 앱 하나 까는 것과 같다고 보면 됩니다. cc-01에서 Claude Code를 깔 때 이미 한 번 써봤습니다.
> 💡 **용어 풀이 — 터미널**: 검은 명령어 창. 무섭게 느껴지지만 사실은 "AI에게 말 거는 채팅방"입니다. Mac은 `터미널` 앱, Windows는 WSL(윈도우 안의 리눅스) 창이에요.

### 이번 레슨에서 새로 만나는 것 딱 2개

대부분은 cc-01 복습입니다. 진짜 새로운 건 두 가지뿐이에요.

1. **로그인이 ChatGPT 계정** — `codex login`을 치면 브라우저가 열리고, 평소 ChatGPT 쓸 때 그 계정으로 승인하면 끝.
2. **샌드박스(안전 울타리) 개념** — Codex는 "어디까지 허락할지"를 또렷하게 보여줍니다. 이번 레슨에서는 **가장 안전한 read-only(보기만) 모드**로 첫발을 떼고, 자세한 3모드는 codex-05에서 배웁니다.

> 💡 **용어 풀이 — 샌드박스(sandbox)**: "모래놀이판"이라는 뜻이에요. 아이가 정해진 울타리 안에서만 노는 안전한 공간처럼, Codex가 내 컴퓨터를 함부로 헤집지 않도록 정해진 범위 안에서만 일하게 하는 안전장치입니다.

## 🔧 실습 1 — 맨바닥에서 설치·로그인하기 (13분)

> 새 컴퓨터를 가정합니다. 아무것도 안 깔려 있어도 괜찮아요. cc-01에서 Node.js를 이미 깔았다면 1단계는 "확인만 하고 통과"입니다. 한 단계 입력 → 결과 확인 → 다음 단계. 한 번에 하나씩 갑니다.

### 1단계 — 토대(Node.js) 확인하기

Codex도 CC처럼 **Node.js**(노드제이에스, 자바스크립트를 돌리는 엔진)라는 토대 위에서 돕니다. 이미 cc-01에서 깔았을 가능성이 높지만, **있는지 확인만** 하고 갑니다.

**왜 이걸 먼저 치나요?** Codex를 깔 `npm`이라는 설치 도구가 Node.js 안에 들어 있거든요. 토대가 있어야 그 위에 집을 올립니다. 토대가 없으면 설치가 도중에 멈춰서 괜히 당황하게 되니, 먼저 확인하고 가는 게 마음 편합니다.

```text
node --version
```

**기대 결과 (예시 출력)**:

```text
v20.11.0
```

**관찰·체크포인트**:
- ✅ `v` + 숫자가 떴다 → 토대 준비 완료, 2단계로.
- ❌ `command not found`가 떴다 → Node.js가 아직 없는 거예요. [nodejs.org](https://nodejs.org)에서 LTS 버전을 깔거나(Mac은 `brew install node`), cc-01의 Node.js 설치 절로 돌아가 먼저 깔고 오세요. (Windows는 nodejs.org LTS 설치 파일 실행 또는 WSL 안에서 설치.)
- CC를 cc-01에서 깔았다면 이 단계는 대개 "이미 통과"일 겁니다. 숫자가 뜨면 안심하고 넘어가세요.

---

### 2단계 — Codex 설치하기 (한 번만)

이제 Codex라는 프로그램을 내 컴퓨터에 깝니다. cc-01에서 Claude Code를 깔 때 `@anthropic-ai/claude-code`였죠? Codex는 `@openai/codex`입니다. **회사 이름만 바뀐 셈**이에요. 세 가지 길 중 하나를 고르면 됩니다. cc-01에서 npm으로 깔았다면 길 1이 제일 친숙합니다.

```text
# 길 1) npm 으로 (Claude Code를 npm으로 깔았으면 이게 제일 익숙)
npm install -g @openai/codex

# 길 2) Homebrew 로 (Mac에서 brew를 쓰는 사람)
brew install --cask codex

# 길 3) 설치 스크립트로 (한 줄 복붙)
curl -fsSL https://chatgpt.com/codex/install.sh | sh
```

> 💡 **보안 팁**: `curl ... | sh`는 받은 스크립트를 바로 실행해요. 여기 주소(`chatgpt.com`)는 OpenAI 공식이라 안전하지만, 출처가 불확실한 `| sh` 명령은 일반적으로 조심해야 합니다(스크립트 내용을 먼저 확인). 처음이라면 길 1(`npm install -g @openai/codex`)이나 길 2(Homebrew)를 권합니다.

**무엇을 하는 건가요?** "Codex"라는 프로그램을 내 컴퓨터에 다운로드·설치하는 것. 앱스토어에서 앱 하나 까는 거랑 똑같다고 보면 됩니다. `-g`는 "이 컴퓨터 어디서나 쓸 수 있게 전역(global)으로 깔아라"라는 뜻이에요 — cc-01에서 Claude Code 깔 때 봤던 그 `-g` 맞습니다.

> 🪟 **Windows 사용자라면**: Codex는 WSL(윈도우 안의 리눅스 환경) 위에서 가장 잘 돌아갑니다. cc-01에서 Claude Code를 WSL 터미널에서 썼다면, **같은 WSL 터미널**에 그대로 위 명령을 입력하세요.

**기대 결과 (예시 출력)**: 설치 진행 메시지가 주르륵 흐르다가, 마지막에 이런 줄이 나오면 성공이에요.

```text
added 1 package in 6s
```

**관찰·체크포인트**: CC 깔 때(`@anthropic-ai/claude-code`)와 비교해 보세요. 패키지 이름의 **회사 부분만** `@openai/codex`로 바뀌었죠? "회사 이름만 갈아끼웠다" — 이 감각을 오늘 내내 느끼게 됩니다.

### 3단계 — 설치가 됐는지 확인하기

깔았으면 진짜 깔렸는지 한 줄로 확인합니다. cc-01에서 `claude --version`을 쳤던 것과 똑같아요.

```text
codex --version
```

**기대 결과 (예시 출력)**:

```text
codex-cli 0.xx.x   〔정확한 버전 숫자는 설치 시점마다 다릅니다 — 숫자만 뜨면 성공〕
```

**관찰·체크포인트**:
- ✅ 버전 숫자가 떴다 → 설치 성공! 4단계로.
- 😟 `command not found: codex`가 떴다 → **당황하지 마세요.** 거의 항상 "터미널이 방금 깐 프로그램을 아직 못 찾는" 일시적 문제입니다. **터미널 창을 완전히 닫았다가 다시 열고** `codex --version`을 한 번 더 쳐보세요. 대부분 해결됩니다. (cc-01에서 `claude`가 안 잡혔을 때와 똑같은 처방이에요.)

---

### 4단계 — 로그인하기 (ChatGPT 계정)

이제 Codex에게 "나 누구야"를 알려줍니다. 여기가 cc-01과 다른 **첫 번째 새로운 지점**이에요. Claude Code는 Anthropic 계정이었지만, **Codex는 ChatGPT 계정**으로 로그인합니다(평소 ChatGPT 쓸 때 그 계정).

```text
codex login
```

**기대 결과 (화면 흐름)**:

```text
codex login 입력
   ↓
브라우저(인터넷 창)가 자동으로 열림
   ↓
ChatGPT 로그인 화면 → 평소 쓰던 계정으로 로그인
   ↓
"승인(Authorize)" 버튼 누르기
   ↓
"터미널로 돌아가세요" 안내 → 터미널로 돌아오면 준비 완료
```

**관찰·체크포인트**:
- ✅ 터미널에 "로그인 완료" 류의 안내가 뜨면 끝. 한 번 로그인하면 이후 계속 유지돼요.
- 💡 **안심 포인트**: 로그인한다고 내 파일이 인터넷 어딘가로 올라가는 게 아니에요. "이 사람이 쓸 권한이 있구나"만 확인하는 겁니다. 내 폴더의 파일은 여전히 내 컴퓨터 안에서만 봅니다. cc-01에서 Claude 로그인할 때와 같은 안전 원리예요.
- 브라우저가 자동으로 안 열리면, 터미널에 표시된 주소(URL)를 직접 복사해 브라우저 주소창에 붙여넣으면 됩니다.

> 📌 **cc-01과 나란히 보기**: CC가 `claude login`(Anthropic 계정)이었다면, Codex는 `codex login`(ChatGPT 계정). 둘 다 "브라우저로 계정 승인"이라는 **같은 패턴**입니다 — 명령 모양도, 흐름도 거의 같아요.

> 💡 **대안(지금은 몰라도 됨)**: 회사·자동화 환경에서는 API 키(`export OPENAI_API_KEY=...`)로 로그인하기도 합니다. 지우처럼 혼자 쓰는 1인 사업자는 `codex login`(ChatGPT 계정) 한 줄이면 충분해요.

## 🎬 실습 2 — 첫 실행: 내 강의 폴더를 Codex에게 보여주기 (17분)

설치·로그인이 끝났으니, 이제 지우의 진짜 사업 폴더(`practice-data/creator/`)를 Codex에게 보여주고 첫 일을 시켜봅니다. 지우의 실제 자료 `course_sales_monthly.csv`(강의별 월매출)와 `student_inquiries.csv`(수강생 문의)를 읽혀서, 손으로 한 시간 걸릴 정리를 몇 분 만에 받아 봅니다.

> 🗣️ **트랙 안내**: 본문은 기본 트랙인 ✍️ **creator(콘텐츠·지식 크리에이터)** 폴더로 진행합니다. 스토어·프리랜서·로컬 트랙을 골랐다면, 아래 `creator`를 자기 트랙 폴더(`store`/`freelancer`/`local`)로 바꾸고, 같은 이름의 파일을 읽히면 **똑같이** 따라할 수 있습니다. 배우는 원리는 트랙이 달라도 같아요.

### 5단계 — 안전하게 첫발 떼기 (read-only로 폴더 둘러보기)

처음 만지는 도구는 **가장 안전한 모드**로 시작하는 게 좋습니다. `--sandbox read-only`는 "보기만 하고 절대 안 고치는" 모드예요. 새 비서에게 처음엔 "서류는 보기만 해, 손대진 말고"라고 부탁하는 것과 같습니다.

폴더 안의 파일을 대상으로 일하게 하려면 `--cd`로 폴더를 지정합니다(cc-01에서 그 폴더로 `cd` 한 다음 `claude`를 켰던 것과 같은 역할).

```text
codex --cd practice-data/creator --sandbox read-only "이 폴더에 어떤 파일이 있는지 둘러보고, 각 파일이 뭔지 한 줄씩 설명해 줘. 나는 비개발자야. README.md부터 읽어줘."
```

명령을 뜯어보면:
- `codex` → Codex 실행
- `--cd practice-data/creator` → "이 폴더 안의 파일을 대상으로 일해" (CC의 작업 폴더 지정과 같은 역할)
- `--sandbox read-only` → "**보기만 해. 절대 고치지 마.**" (가장 안전한 안전 모드)
- `"이 폴더에…"` → 지우의 지시문(=프롬프트, AI에게 하는 부탁)

**기대 결과 (예시 출력)**:

```text
이 폴더를 둘러봤어요. 1인 콘텐츠 사업자의 자료가 모여 있네요. 6개 파일이 있어요:

CSV 데이터 (숫자 자료)
- course_sales_monthly.csv : 강의별·월별 매출 (2025-07 ~ 2026-03)
- newsletter_stats.csv     : 뉴스레터 호별 오픈율·클릭율·구독 증감 (18호~41호)
- student_inquiries.csv    : 수강생 문의 내역 (채널·카테고리·처리상태)

문서 데이터 (줄글 자료)
- content_plan.md          : 2분기 콘텐츠 기획 노트
- research_memo.md         : AI 도구·1인 사업 시장 리서치 메모
- consulting_schedule.md   : 3월 컨설팅·운영 일정 노트

무엇부터 자세히 볼까요?
```

**관찰·체크포인트**:
- ✅ Codex가 답만 주는 게 아니라 "이 파일을 읽어볼게요 → 이건 매출 데이터네요" 식으로 **스스로 파일을 열어 읽는다**는 점을 보세요. 복사·붙여넣기를 한 번도 안 했는데요. 이게 채팅창과의 결정적 차이, "터미널 AI"의 와우 모먼트예요.
- ✅ `--sandbox read-only`라서 Codex는 파일을 **보기만** 했지, 단 한 글자도 고치지 않았습니다. "둘러보기"만 한 거예요. 안심하고 시도하세요.
- 📌 cc-01에서 Claude Code에게 "이 폴더에 뭐가 있어?"라고 물었던 것과 **거의 같은 경험**이라는 걸 느꼈을 거예요. 옵션 몇 개만 붙은 정도죠.

> 💡 **학습자 안심** — "내 파일을 망가뜨리면 어쩌죠?"
> 망가지지 않습니다. 첫째, 방금처럼 `--sandbox read-only`로 켜면 Codex는 **보기만** 합니다. 둘째, 기본 설정(workspace-write)이라도 Codex는 되돌릴 수 없는 위험한 작업(파일 삭제 등) 전에는 "이거 해도 될까요?" 하고 먼저 물어봐요(이걸 follow-through, '끝까지 처리하되 위험할 땐 묻기'라고 합니다).

### 6단계 — 진짜 데이터로 첫 분석 (효자 강의 찾기)

이제 진짜 사업 질문입니다. "어느 강의가 제일 잘 팔렸나?" 특정 파일을 콕 집고 싶을 때는 `@`를 씁니다(Part 1(cc-03)에서 배운 `@파일명`과 똑같은 문법). 결과 모양까지 또렷하게 부탁해요(좋은 프롬프트 원칙 — cc-01에서 배운 그대로).

```text
codex --cd practice-data/creator --sandbox read-only "너는 내 온라인 강의 사업의 데이터 분석가야. @course_sales_monthly.csv 를 읽고, 매출이 가장 큰 강의 TOP 3를 CSV의 실제 숫자를 인용해서 표로 정리해 줘. 비개발자도 이해되게."
```

**기대 결과 (실제 데이터 기반 예시)**:

```text
매출 기준 효자 강의 TOP 3 (2025-07 ~ 2026-03):

| 순위 | 강의명 | 근거 (실제 숫자) |
|----|------|------|
| 1 | 챗GPT로 콘텐츠 10배 빠르게 | 객단가 79,000원으로 낮은데도 수강 수가 압도적. 2026-01에 187명·매출 1,477만원으로 단일 강의 월매출 최고치 |
| 2 | 1인 지식사업 수익화 로드맵 | 객단가 149,000원으로 가장 비쌈. 2026-01에 89명·1,326만원 — 적은 인원으로 큰 매출 |
| 3 | 클로드코드로 업무 자동화 입문 | 신규 강의인데 2026-03에 118명·1,404만원으로 급성장 |

한 줄 코멘트: 'AI활용' 카테고리 강의가 매출을 끌고 있어요.
```

⚠️ **중요한 체크포인트 (할루시네이션 방지)**:
- ✅ Codex가 **CSV의 실제 숫자**를 근거로 댔는지 보세요. 두루뭉술하게 "잘 팔렸어요"가 아니라 숫자가 붙어야 믿을 수 있어요. 예를 들어 위 답의 "187명·1,477만원"은 CSV에 실제로 있는 값(2026-01 챗GPT 강의 행)이어야 합니다.
- 💡 Codex가 추측만 하는 것 같으면 멈추고 **"CSV의 실제 값으로 근거를 보여줘"**라고 다시 요청하세요. 숫자가 파일에 정말 있는지 한번 같이 확인해 보면, AI가 지어낸 게 아니라는 걸 직접 검증하는 좋은 연습이 됩니다.

> 💡 **용어 풀이 — 할루시네이션(hallucination)**: AI가 없는 사실을 그럴듯하게 지어내는 현상이에요. "실제 값으로 근거를 보여줘"라고 요구하는 습관 하나로 대부분 막을 수 있습니다.

### 8단계 — 다른 종류의 자료도 읽혀보기 (문의 로그에서 반복 이슈 찾기)

숫자(매출)만이 아니라 글 자료(문의 내역)도 Codex가 읽고 정리해 줍니다. 지우가 매달 손으로 분류하던 일이죠.

```text
codex --cd practice-data/creator --sandbox read-only "@student_inquiries.csv 를 읽고, 가장 자주 들어온 문의 카테고리 3가지와 각각 어떤 내용이 반복되는지 표로 짧게 정리해 줘."
```

**기대 결과 (실제 데이터 기반 예시)**:

```text
3월 문의 카테고리 TOP 3 (총 24건 기준):

| 카테고리 | 자주 나온 내용 | 메모 |

눈에 띄는 점: '실습' 문의가 가장 많고, 그중 상당수가
```

### 9단계 — 숫자와 맥락 연결하기 (분석의 핵심)

cc-01에서 배운 "숫자(매출)와 맥락(메모)을 연결하면 진짜 인사이트가 나온다"를 Codex로도 해봅니다. 이번엔 두 파일을 동시에 읽힙니다 — 매출 숫자(`course_sales_monthly.csv`)와 현장 맥락(`research_memo.md`).

```text
codex --cd practice-data/creator --sandbox read-only "@course_sales_monthly.csv 와 @research_memo.md 를 함께 읽고, 'AI활용 카테고리 강의가 왜 잘 팔리는지'를 매출 숫자와 리서치 메모의 단서를 연결해 5줄 이내로 설명해 줘."
```

**기대 결과 (실제 데이터 기반 예시)**:

```text
AI활용 카테고리('챗GPT로 콘텐츠 10배', '클로드코드로 업무 자동화 입문')가
매출 기여 1위입니다. 리서치 메모에 따르면 비개발자 대상 AI 도구 시장이
2025년 후반부터 커지는데, 경쟁 강의는 대부분 개발자용이라 '비개발자·1인 사업자'
자리가 비어 있었습니다. 지우의 강의가 바로 그 빈자리를 차지한 것이죠.
즉 '낮은 객단가 + 빈 시장 + 강한 수요'가 겹쳐 수강 수가 폭발했습니다.
```

💡 **이 단계가 중요한 이유**:
- 숫자만 보면 "AI 강의가 잘 팔린다"까지만 알 수 있어요.
- 리서치 메모를 더하면 **"왜" 잘 팔리는지**가 보입니다 (빈 시장 + 비개발자 포지션).
- 둘을 연결해야 "다음에 뭘 할까"(의사결정)로 이어집니다. 이게 "관찰 → 해석 → 제안" 흐름이고, 분석의 진짜 가치예요.

> 📌 **`@`로 두 파일을 한 번에**: 6단계에서 한 파일을 `@`로 콕 집었다면, 여기선 `@course_sales_monthly.csv 와 @research_memo.md` 처럼 **두 파일을 나란히** 가리켰습니다. Part 1에서 배운 `@파일명` 문법 그대로, 개수만 늘린 거예요.

### 10단계 — Claude Code에도 같은 질문 던져 비교하기 (선택)

오늘의 마지막 한 수이자 이 코스의 핵심 정신입니다. 여유가 있으면, **9단계에서 Codex에게 한 똑같은 질문을 cc-01에서 켰던 Claude Code에도** 던져보세요. 답이 어떻게 다른지(말투, 정리 방식, 길이) 눈으로 비교하는 게 이 코스의 핵심 재미입니다.

```text
# Claude Code 쪽에서 (Part 1에서 배운 방식)
cd practice-data/creator
claude
> @course_sales_monthly.csv 와 @research_memo.md 를 함께 읽고, 'AI활용 카테고리 강의가 왜 잘 팔리는지'를 실제 숫자로 설명해 줘.
```

**관찰·체크포인트**:
- ✅ 두 도구가 **같은 데이터에서 비슷한 핵심**(챗GPT 강의의 수강 수, 클로드코드 강의의 성장, 빈 시장 포지션)을 짚는다면, 그만큼 결과를 믿어도 된다는 신호예요. 혼자라서 검산해 줄 사람이 없던 지우에게 **두 AI가 서로 검산해 주는** 셈입니다.
- ✅ 미세한 차이(어느 쪽이 더 표를 잘 쓰나, 더 친절하나)는 느껴만 두세요. 우열을 가리는 게 아니라 **둘 다 쓰는 감**을 잡는 거예요. 자세한 비교는 codex-11(종합 비교)에서 정리합니다.

> ⚠️ **코스 불변 규칙**: 모델·effort를 바꿔보는 건 아직입니다(codex-03에서 배워요). 지금은 기본 설정으로 "설치→로그인→첫 실행→첫 분석"까지만. 한 번에 하나씩.

## 📝 실습 3 — 첫 결과물 파일로 남기기 (선택·시간 남으면)

지금까지는 read-only(보기만)였어요. 이번엔 Codex가 **새 파일을 하나 만들어** 보게 합니다. 단, 파일을 만들려면 "보기만(read-only)"이 아니라 **쓰기가 허용된 기본 설정(workspace-write)**이 필요해요. 그래서 이번엔 `--sandbox read-only`를 빼고 실행합니다.

> `--sandbox read-only`를 빼면 Codex는 **기본 설정(workspace-write)** 으로 동작합니다 — "내 폴더 안에서 읽고·고치고·만들 수 있고, 위험한 일은 먼저 묻는" 딱 적당한 모드예요. 자세한 샌드박스 모드 차이는 codex-05에서 배웁니다.

### 11단계 — 한 줄 요약 메모 만들기

```text
codex --cd practice-data/creator "지금까지 본 매출·문의·리서치 내용을 바탕으로, '이번 달 핵심 3줄 + 다음에 해볼 일 1가지'를 정리한 짧은 메모를 codex_first_summary.md 라는 새 파일로 만들어 줘. 기존 파일은 절대 건드리지 말고 새 파일만 만들어. 비개발자도 읽기 쉽게."
```

**기대 결과 (화면 흐름)**:

```text
'codex_first_summary.md' 라는 새 파일을 만들어도 될까요? (이 작업은 새 파일 생성입니다)
   → yes 입력
   ↓
'codex_first_summary.md' 파일을 만들었어요.
```

만들어진 파일을 열어보면 이런 식입니다:

```text
# 이번 달 핵심 메모 (Codex 첫 실행 정리)

## 핵심 3줄
- '챗GPT로 콘텐츠 10배 빠르게'가 수강 수 1위 (2026-01 187명·1,477만원).
- 신규 '클로드코드' 강의가 두 달 만에 약 2.6배 성장 (46명 → 118명).
- 문의 1위는 '실습' — 대부분 클로드코드 설치 단계 막힘.

## 다음에 해볼 일
- 클로드코드 설치 가이드 영상을 따로 만들어 자동 안내로 돌리기 (설치 문의 줄이기).
```

**관찰·체크포인트**:
- ✅ Codex가 파일을 만들기 **전에 "괜찮나요?" 하고 먼저 물어봤죠?** 이게 안전장치(follow-through)예요 — **되돌릴 수 있는 일은 알아서, 위험한 일은 먼저 묻기.** 그래서 마음 놓고 시켜도 됩니다.
- ✅ `ls`를 쳐서 `codex_first_summary.md`가 생겼는지 눈으로 확인하세요. 내 폴더에 진짜 결과물이 남았다면 성공!
- 💡 만든 파일이 마음에 안 들면? "3줄을 더 짧게 줄여줘" 하고 다시 시키면 됩니다. AI는 시험 감독이 아니라 짝꿍이에요. 이 "결과물을 파일로 남기기"는 다음 단계들(지식 위키·자동화)에서 계속 쓰는 기본기입니다.

## 💡 핵심 규칙·개념 정리

오늘 꼭 챙겨갈 다섯 가지입니다.

1. **Codex = OpenAI가 만든, CC의 형제 도구.** 내 폴더의 파일을 직접 읽고·고치고·설명하는 터미널 AI. 설치·로그인·첫 실행 흐름이 cc-01과 90% 같다. 새로 외울 건 사실상 "계정은 ChatGPT, 샌드박스라는 안전 개념이 있다" 정도뿐.
2. **설치 4단계**: `node --version`(토대 확인) → `npm install -g @openai/codex`(설치) → `codex --version`(확인) → `codex login`(ChatGPT 계정 로그인).
3. **첫 실행 공식**: `codex --cd <폴더> "지시문"`. `--cd`로 일감 폴더를 가리키고, 따옴표 안에 우리말로 부탁한다. `@파일명`으로 특정 파일을 콕 집는다.
4. **처음엔 안전하게 `--sandbox read-only`("보기만")** 로 감을 잡고, 파일을 만들/고칠 때만 기본 설정(workspace-write)으로. Codex는 위험한 작업 전에 항상 먼저 물어본다(follow-through). 자세한 샌드박스 3모드(read-only / workspace-write / danger-full-access)는 codex-05에서.
5. **AI는 비서·짝꿍이지 시험 감독이 아니다.** 첫 답이 70점이어도 정상. "다시 해줘"로 다듬으면 된다. Part 1에서 배운 좋은 프롬프트 습관(`@파일`로 콕 집기·결과 모양 못 박기·안 맞으면 다시 시키기)이 Codex에도 그대로 통한다.

> 🎯 가장 중요한 한 줄: **"CC를 익혔으면 Codex는 90% 전이된다."** 오늘 새로 외운 건 사실상 "계정은 ChatGPT, 폴더는 `--cd`로 지정" 정도뿐입니다.

## 📊 이번 레슨 사이클 (텍스트 다이어그램)

오늘 지우가 한 바퀴 돈 흐름을 그림으로 봅니다.

```text
[ 맨바닥 새 컴퓨터 ]
        │  node --version  (토대 확인)
        ▼
[ npm install -g @openai/codex ]  ← 두 번째 비서 채용
        │  codex --version  (잘 깔렸나, 숫자 뜨면 성공)
        ▼
[ codex login ]  ← ChatGPT 계정으로 "나 누구야" 알려주기
        ▼
[ codex --cd practice-data/creator --sandbox read-only "둘러봐 줘" ]  ← 안전하게 첫 실행 (와우!)
        │
        ├─→ @course_sales_monthly.csv  → 효자 강의 TOP 3 (실제 숫자: 187명·1,477만원)
        ├─→ 클로드코드 강의 2~3월 비교 → 46명 → 118명 급성장
        ├─→ @student_inquiries.csv     → 반복 문의 = '설치/터미널'에서 막힘 (24건 중)
        └─→ 매출 + research_memo.md     → "빈 시장 + 비개발자 포지션" 이유 연결
        ▼
[ (선택) codex_first_summary.md 파일로 남기기 ]  ← 첫 결과물
        ▼
[ 같은 질문 CC에도 던져 비교 ]  ← 두 비서를 다 부리는 감각
```

설치 → 로그인 → 폴더 가리키기 → 분석 → (결과물) → 비교. **cc-01에서 한 흐름과 거의 똑같죠?** 그래서 오늘이 사실상 복습 + α였던 겁니다 — **그게 정상이고, 좋은 신호예요.** 뼈대가 같으니 하나를 익히면 나머지가 금방 따라옵니다.
