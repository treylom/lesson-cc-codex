# codex-11: CC ↔ Codex 같은 것·다른 것 — "CC를 익혔으면 Codex는 90%가 따라온다"

> **레슨 목표**: Claude Code(CC)와 Codex를 나란히 놓고 "같은 것"과 "다른 것"을 직접 체험으로 정리한다. 두 도구의 우열을 가리는 게 아니라, 언제 어느 도구를 어떻게 쓸지 스스로 판단하는 감각을 만드는 것이 목표.
>
> **예상 시간**: 60분 (개념 도입 10분 + 실습 1 CC 분석 25분 + 실습 2 Codex 비교 20분 + 실습 3 비교 보고서 초안 15분)
>
> **사전 준비**: Claude Code + Codex 둘 다 설치·로그인 완료 / `practice-data/creator/` 폴더 접근 가능 / 터미널 1개
>
> **진입 커맨드**: `/codex-11`
>
> 🔎 **검증 배너**: 이 레슨에 나오는 명령·모델명(`gpt-5.x`)·플래그·샌드박스 모드명은 codex-101 2026-06-18 스냅샷 기준입니다.

## 📚 학습 목표 — 4가지 와우 모먼트

1. **"같다"는 걸 직접 본다** — 규칙서·샌드박스·MCP·effort, 이 네 개가 CC와 Codex에서 어떻게 나타나는지 나란히 확인한다.
2. **"다르다"는 걸 직접 만진다** — 스킬 호출 기호(`$` vs `/`), 규칙 파일 이름(`AGENTS.md` vs `CLAUDE.md`), 모델 이름(gpt-5.x vs Claude)이 바뀌는 순간을 눈으로 본다.
3. **같은 데이터, 두 도구** — `course_sales_monthly.csv`·`student_inquiries.csv`·`newsletter_stats.csv`를 CC로도 Codex로도 분석해 보고, "어, 비슷하게 해주네"라는 감각을 얻는다.
4. **"CC 익혔으면 Codex는 90% 전이"를 내 입으로 말한다** — 이 코스의 마지막 퍼즐 조각을 맞추고, 두 도구를 진짜 내 사업 파트너로 부를 준비를 마친다.

## 🎯 개념 도입 — 두 도구는 형제다 (10분)

### 비유: 브랜드가 다른 자판기 두 대

공항에 자판기 두 대가 있다고 생각해 보세요.

- **왼쪽 자판기(CC)**: Anthropic(앤스로픽)이 만들었고, 안에는 Claude가 들어 있습니다.
- **오른쪽 자판기(Codex)**: OpenAI(오픈에이아이, ChatGPT를 만든 곳)가 만들었고, 안에는 GPT가 들어 있습니다.

두 자판기 모두 버튼 위치, 동전 넣는 곳, 음료 나오는 통 — 구조가 거의 같습니다. 브랜드 로고와 음료 이름만 다릅니다. 한 자판기 쓸 줄 알면 다른 쪽도 10분이면 적응됩니다.

이게 CC와 Codex의 관계입니다. **뼈대가 같고, 브랜드·모델·몇 가지 세부 표기만 다릅니다.**

지우도 처음엔 의아했습니다.

> "CC를 이미 배웠는데, 또 새로 다 외워야 해?"

결론부터 말하면 — **아니요, 90%는 그대로 옮겨옵니다.**

스마트폰 비유로 설명해 볼게요. 삼성 스마트폰을 오래 써서 익숙해졌다고 합시다. 어느 날 LG 스마트폰으로 바꿨습니다. 전화, 문자, 카카오톡, 인터넷 — 다 됩니다. 앱 배치가 약간 다르고, 설정 메뉴 위치가 조금 달라서 처음엔 헷갈리지만, "스마트폰 쓰는 법" 자체는 그대로예요. 한 달 지나면 완전히 익숙해집니다.

CC와 Codex가 딱 그 관계입니다.

### 한눈에 — 공통과 차이

| 축 | CC (이미 배운 것) | Codex (지금 비교) | 같다/다르다 |
|---|---|---|---|
| 만든 곳 | Anthropic | OpenAI | **다르다** — 브랜드만 |
| 설치 | `npm install -g @anthropic-ai/claude-code` | `npm install -g @openai/codex` | **같다** — npm 한 줄 |
| 로그인 | `claude login` (Anthropic 계정) | `codex login` (ChatGPT 계정) | **같다** — 계정만 다름 |
| 로컬 파일 접근 | 내 폴더 직접 읽기·쓰기 | 내 폴더 직접 읽기·쓰기 | **완전히 같다** |
| 규칙서 | **`CLAUDE.md`** | **`AGENTS.md`** | **같은 역할, 이름만 다름** |
| 스킬 호출 기호 | `/스킬이름` | **`$스킬이름`** | **다르다** — 슬래시 vs 달러 |
| 세션 명령 | `/compact` `/model` `/plan` | `/compact` `/model` `/plan` | **완전히 같다** |
| 모델 | Claude (sonnet, opus…) | **gpt-5.5 / gpt-5.4-mini** | **다르다** — 회사 따라 |
| 안전 모드 | 승인 게이트 (허용/거부) | **샌드박스 3모드** (read-only / workspace-write / danger-full-access) | **같은 정신, 표기 다름** |
| 외부 도구 연결 | **MCP** | **MCP** | **완전히 같다** |
| 긴 대화 정리 | `/compact` | `/compact` | **동일** |
| 세션 상태 확인 | `/context` | `/status` | **개념 같음, 명령 다름** |
| 오케스트레이션 강조 | 단일 확장 컨텍스트 (한 호흡 깊이) | **멀티 worktree** (여러 창구 병렬) | **다르다** — 강조점 차이 |

**핵심 한 줄**:
> "CC를 익혔으면 Codex는 90% 전이된다. 규칙서·샌드박스·MCP·effort는 같고, 회사·모델·호출 기호(`$` vs `/`)·규칙 파일 이름만 다르다."

### 왜 두 가지를 배우나 — 지우의 상황

지우는 이런 상황에 있습니다.

- **CC**: 이미 손에 익었다. 강의 기획, 뉴스레터 초안, 매출 분석을 CC로 끝낸다.
- **Codex**: 같은 일을 다른 엔진(GPT)으로 교차 확인하고 싶을 때, 또는 CC 안에서 codex plugin으로 Codex를 같이 부르고 싶을 때 쓴다.

지우의 목표는 "CC가 낫냐, Codex가 낫냐"를 가리는 게 아닙니다. **두 파트너를 상황에 맞게 쓰는 것**입니다. 매출 분석은 CC, 특정 부분을 교차 확인할 때는 Codex. 마치 화가가 붓과 팔레트 나이프를 상황에 따라 고르듯.

## 🔧 실습 1 — CC로 지우의 데이터 전체 분석하기 (25분)

> 이미 배운 CC로 세 개의 실제 데이터 파일을 분석합니다. 다음 실습에서 Codex로 같은 일을 해보고 비교할 기반을 만드는 단계입니다.

### 1단계: 작업 폴더로 이동하고 CC 켜기

터미널을 열고 아래를 입력해 보세요.

```text
cd ~/Documents/cc-codex-101/practice-data/creator
claude
```

**무슨 일이 벌어지나**: `cd`로 지우의 사업 데이터 폴더로 들어갑니다. `claude`를 치면 CC가 이 폴더를 무대로 켜집니다. 이 폴더에는 `course_sales_monthly.csv`(강의별 월 매출), `student_inquiries.csv`(수강생 문의), `newsletter_stats.csv`(뉴스레터 성과) 등 지우의 실제 사업 데이터가 있습니다.

**기대 결과**: 입력창(`>`)이 뜨면 CC가 준비된 것입니다.

**관찰/체크포인트**: 폴더 이동 없이 `claude`를 켜면 그 자리 폴더가 무대가 됩니다. "CC는 내가 서 있는 폴더 안에서 일한다"는 감각을 다시 확인하세요.

---

### 2단계: 강의 매출 분석 요청

아래 프롬프트를 그대로 입력해 보세요.

```text
나는 온라인 강의를 운영하는 1인 사업자 지우야.
@course_sales_monthly.csv 를 읽고 다음 3가지를 알려줘:
1) 2025-07부터 2026-03까지 순매출이 가장 높았던 강의 TOP 3
2) 가장 빠르게 성장한 강의와 그 추이 (월별 수치 포함)
3) 환불율이 가장 높은 강의 (환불수 ÷ 수강신청수로 계산)
결과는 표로 정리하고, 마지막에 "다음 달에 집중할 강의 하나"를 추천해줘.
```

**기대 결과 (예시)**:

CC가 CSV를 읽고 다음과 같이 분석합니다.

```
📊 강의별 순매출 TOP 3 (2025-07 ~ 2026-03 합산)

| 순위 | 강의명 | 총 순매출 |
|------|-------|---------|
| 1 | 챗GPT로 콘텐츠 10배 빠르게 | 약 7,260만원 |
| 2 | 1인 지식사업 수익화 로드맵 | 약 5,312만원 |
| 3 | 클로드코드로 업무 자동화 입문 | 약 1,750만원 |

📈 가장 빠르게 성장한 강의: 클로드코드로 업무 자동화 입문
- 2026-02: 순매출 508만원 (46명 수강)
- 2026-03: 순매출 1,242만원 (118명 수강) → 한 달 만에 +144.5%

🔴 환불율 최고: 챗GPT로 콘텐츠 10배 빠르게 (약 8.5%)

💡 추천: 클로드코드 강의 — 출시 두 달 만에 순매출 2.4배 성장 중이고
아직 두 달치 데이터만 있어 추가 홍보 여지가 큽니다.
```

**관찰/체크포인트**:
- CC가 CSV를 직접 읽어 실제 수치(`2026-03: 1,242만원`, `+144.5%`)로 답했나요?
- 추측이 아니라 파일 안의 숫자를 근거로 답했는지 확인하세요. 만약 막연한 말만 한다면 "CSV의 실제 수치로 근거를 보여줘"라고 다시 시키면 됩니다.
- 이렇게 파일 안의 실제 숫자로 답하는 것이 채팅 AI와 CC의 결정적 차이입니다.

### 3단계: 문의 데이터에서 패턴 찾기

같은 대화 안에서 이어서 입력합니다.

```text
이번엔 @student_inquiries.csv 를 읽어줘.
클로드코드 강의 관련 문의만 모아서:
- 어떤 카테고리의 문의가 제일 많았는지 (실습/결제/수료증/수강방법 등)
- 처리에 오래 걸린 문의 유형은 무엇인지 (소요시간 기준)
- 진행중(미완료) 상태로 남은 문의는 몇 건인지
이것도 표로 정리해줘.
```

**기대 결과 (예시)**:

```
📬 클로드코드 강의 문의 분석 (2026-03-01 ~ 2026-03-09)

카테고리별 문의 수:
| 카테고리 | 건수 |
|---------|-----|
| 실습    |  6건 |
| 결제    |  2건 |
| 환불    |  1건 |
| 수강방법 |  1건 |

평균 소요시간이 가장 긴 유형: 실습 문의 (평균 약 1.7시간)
→ WSL 설치(S013, 3.0시간), 터미널 에러(S004, 2.0시간), Codex 로그인(S022, 2.0시간) 상위

미완료 상태: 4건
→ 터미널 에러(S004), WSL 설치(S013), CLAUDE.md 적용(S015), Codex 로그인(S022)
```

**관찰/체크포인트**:
- "실습" 카테고리가 압도적으로 많고, 소요시간도 가장 깁니다. 이게 지우에게 무슨 뜻일까요? → 클로드코드 강의 수강생이 설치·실습에서 가장 많이 막힌다는 뜻. 강의에 "설치 트러블슈팅" 섹션을 보강할 근거가 됩니다.
- 이렇게 "데이터에서 다음 행동 힌트를 뽑는 것"이 CC 분석의 핵심입니다.

### 4단계: 뉴스레터 데이터도 읽기

이번에는 `newsletter_stats.csv`를 분석합니다. 매출과 문의 말고, 뉴스레터 구독자 반응도 사업의 중요한 지표입니다.

```text
@newsletter_stats.csv 를 읽어줘.
최근 6개월(2025-10 ~ 2026-03)의 오픈율·클릭율 추이를 정리하고:
1) 오픈율이 가장 높았던 호와 낮았던 호
2) 클릭율이 역대 최고였던 호와 그 제목·클릭율
3) 구독자 수 성장 흐름 (2025-10 vs 2026-03 비교)
를 알려줘.
```

**기대 결과 (예시)**:

```
📧 뉴스레터 최근 6개월 성과 분석 (2025-10 ~ 2026-03)

오픈율:
- 최고: 30호 「2026년 1인 사업 트렌드 5가지」 — 62.0% (2026-01-05)
- 최저: 27호 「내년 콘텐츠 캘린더 미리 짜기」 — 42.0% (2025-12-08)

클릭율:
- 역대 최고: 40호 「사전등록 마감 임박 (오늘 밤 12시)」 — 30.3% (3,231명 오픈 중 1,687명 클릭)
- 직전 호(39호)도 26.0%로 높음 — 클로드코드 사전등록 오픈 관련

구독자 성장:
- 2025-10 초 누적: 3,192명
- 2026-03 말 누적: 5,755명
→ 약 6개월 만에 +2,563명(+80.3%) 성장
```

**관찰/체크포인트**:
- 40호의 클릭율 30.3%는 뉴스레터 업계에서 매우 높은 수치입니다. 클로드코드 강의 사전등록 마감 긴박감이 작동했다는 신호입니다.
- 뉴스레터와 강의 매출 데이터를 같이 보면 어떤 패턴이 보이나요? → 다음 단계에서 연결합니다.

### 5단계: 세 데이터를 연결해 인사이트 만들기

```text
방금 분석한 세 결과(매출 + 문의 + 뉴스레터)를 연결해서 생각해줘.
클로드코드 강의가 한 달 만에 순매출 144% 성장했고,
뉴스레터 39·40호(클로드코드 사전등록 관련)의 클릭율이 26~30%로 역대 최고였어.
그런데 문의는 실습·설치 관련이 55%를 차지해.
이 세 데이터를 연결해서 지우가 가장 먼저 해야 할 운영 개선 하나만 뽑아줘.
근거는 데이터를 인용해서 설명해줘.
```

**기대 결과 (예시)**:

```

근거:
- 매출: 2026-03 수강신청 118명, 순매출 1,242만원으로 급성장 (+144.5%)
- 뉴스레터: 39호 클릭율 26.0%, 40호 30.3% — 클로드코드 관련 콘텐츠에 구독자 반응 최고
  (누적 구독자 5,572명 기준, 40호 클릭 1,687건 = 관심이 실제 행동으로 이어짐)
- 문의: 클로드코드 강의 문의 11건 중 실습(설치·터미널) 6건(55%), 평균 1.7시간 소요
  미완료 4건 중 3건이 설치 관련(WSL·터미널·Codex 로그인)

수강생이 늘어날수록 같은 문의가 반복됩니다.
"맥 설치", "윈도우 WSL", "Codex 로그인" 3개 케이스를 5~7분 영상으로 미리 만들어두면
지우의 응대 시간을 크게 줄일 수 있습니다.
```

- 세 파일의 숫자가 연결되어 "다음에 뭘 해야 할지"가 나왔습니다. 이게 단순 정리와 분석의 차이입니다.
- 실습 1이 끝났습니다. `/compact`로 이 대화를 정리해두세요.

```text
/compact
```

> 💡 **왜 `/compact`를 하나요?** 긴 대화가 쌓이면 AI가 처음 내용을 잊기 시작합니다(기억 공간 = 컨텍스트 창이 가득 차기 때문). `/compact`는 지금까지 한 대화를 요약해 AI의 기억 공간을 가볍게 만들어 줍니다. 작업 하나가 끝났을 때 써주면 좋습니다. 다음 실습에서 Codex와 비교할 기준도 이 정리로 남겨둡니다.

### 6단계: 분석 결과 파일로 저장하기

이제 CC에게 방금까지의 분석 결과를 파일로 남기게 합니다.

```text
방금 분석한 세 가지 결과(강의매출 TOP 3 · 문의 카테고리 · 뉴스레터 성과 · 핵심 제안)를
"cc분석결과_20260320.md" 파일로 저장해줘.
마크다운 형식으로, 각 섹션에 날짜와 데이터 출처 파일명을 기록해줘.
```

**기대 결과**: 폴더에 `cc분석결과_20260320.md`가 생성됩니다.

**관찰/체크포인트**:
- CC가 직접 파일을 만들어 저장합니다. 복사·붙여넣기 없이.
- 파일이 생겼는지 확인하려면: `ls` 명령으로 폴더 목록을 보거나, 파인더(Mac)/탐색기(Windows)에서 그 폴더를 열어보세요.
- 이 파일이 실습 1의 산출물입니다. 다음 실습에서 Codex로도 같은 분석을 해보고 결과를 나란히 비교할 기반이 됩니다.

## 🎬 실습 2 — Codex로 같은 질문 던지기 (20분)

> 이번에는 Codex를 켜서 실습 1과 같은 데이터에 같은 질문을 던집니다. 무엇이 같고 무엇이 다른지 직접 보는 게 목표입니다.

### 1단계: 터미널에서 Codex 켜기

터미널 새 창을 하나 더 엽니다. (기존 CC 창은 그대로 둬도 됩니다.) 새 창에서 아래를 입력합니다.

```text
codex --cd ~/Documents/cc-codex-101/practice-data/creator "이 폴더에 어떤 파일이 있는지 한눈에 보여줘"
```

**무슨 일이 벌어지나**: `codex --cd 폴더경로`는 CC에서 `cd 폴더 → claude`와 같은 역할입니다. 폴더를 지정해서 Codex를 켜는 방법입니다. 이게 Codex의 첫 번째 차이점 — CC는 "폴더로 이동 후 claude" 패턴, Codex는 `--cd` 옵션으로 한 줄에 지정합니다.

**기대 결과**: Codex가 `creator/` 폴더를 스캔해서 CSV 3개(`course_sales_monthly.csv`, `newsletter_stats.csv`, `student_inquiries.csv`), 문서 3개(`content_plan.md`, `research_memo.md`, `consulting_schedule.md`), `README.md`를 나열합니다.

**관찰/체크포인트**:
- CC와 Codex 둘 다 "폴더를 지정해서 일한다"는 구조는 같습니다. 방법만 약간 다릅니다.

---

### 2단계: 실습 1과 같은 질문 던지기 (샌드박스 확인 포함)

먼저 샌드박스(안전 울타리) 모드를 안전하게 설정합니다.

```text
codex --cd ~/Documents/cc-codex-101/practice-data/creator --sandbox read-only "course_sales_monthly.csv 를 읽고 순매출 TOP 3 강의를 알려줘. 2025-07부터 2026-03까지 합산 기준으로."
```

> 💡 **샌드박스(sandbox)란**: "모래놀이판"입니다. 아이가 모래판 안에서만 노는 것처럼, Codex가 내 컴퓨터를 함부로 건드리지 않도록 울타리를 정해주는 안전장치입니다. 세 단계가 있습니다.
>
> | 모드 | 읽기 | 파일 수정 | 인터넷 | 언제 쓰나 |
> |------|:---:|:--------:|:-----:|---------|
> | `read-only` | ✅ | ❌ | ❌ | 처음 볼 때, 보기만 할 때 |
> | `workspace-write` | ✅ | ✅ | ❌ | 일반 사용 (파일 수정 필요할 때) |
> | `danger-full-access` | ✅ | ✅ | ✅ | 거의 안 씀 — 인터넷까지 필요할 때만 |
>
> 지금은 `read-only`로 시작합니다. Codex가 데이터를 **보기만** 하고 아무것도 바꾸지 않는 가장 안전한 상태입니다.

**CC와 비교**:
- CC에서 배운 승인 게이트 = "이 파일 수정해도 될까요?" 하고 물어보는 것
- Codex 샌드박스 = "어디까지 허락할게?" 를 미리 모드로 정해두는 것

두 개념은 **같은 정신**입니다. "AI에게 단계적으로 권한을 준다." 비유하자면 — 새 비서에게 처음엔 "서류 보기만 해(read-only)" → 익숙해지면 "정리도 해(workspace-write)" → 완전히 믿으면 "알아서 다 해"입니다.

**기대 결과 (예시)**:

```
📊 순매출 TOP 3 (2025-07 ~ 2026-03 합산)

1. 챗GPT로 콘텐츠 10배 빠르게 — 합산 순매출 약 7,260만원
2. 1인 지식사업 수익화 로드맵 — 합산 순매출 약 5,312만원
3. 클로드코드로 업무 자동화 입문 — 합산 순매출 약 1,750만원
   (단, 2026-02~03 두 달치만 있음 — 추세 주목)
```

**관찰/체크포인트**:
- CC와 Codex의 답이 비슷한가요? 같은 CSV 파일을 같은 로직으로 읽으면 숫자는 같게 나와야 정상입니다.
- 그렇다면 두 도구의 차이는 어디서 나올까요? → 다음 단계에서 확인합니다.

---

### 3단계: `/status`로 샌드박스 모드 확인

Codex 세션 안에서 아래를 입력해 봅니다.

```text
/status
```

**기대 결과 (예시)**:

```
현재 세션 상태:
- 모델: gpt-5.5
- effort: medium
- 샌드박스 모드: read-only
- 네트워크: 차단됨
```

이걸 보는 것만으로 "지금 Codex가 어떤 안전 설정으로 돌고 있는지"를 한눈에 알 수 있습니다.

**CC와 비교**:
- CC에서는 `/context` 명령으로 세션 상태와 컨텍스트 사용량을 확인했습니다.
- Codex에서는 `/status`가 해당 역할을 합니다.

**관찰/체크포인트**:
- `/status` 명령은 Codex에서도 슬래시(`/`)로 시작합니다. `$`(달러)가 아닙니다.
- 이 차이가 중요합니다. Codex에서 `$`는 "스킬(반복 작업 묶음)을 부를 때"만 씁니다. `/model`, `/compact`, `/plan`, `/status` 같은 **세션 관리 명령은 Codex에서도 `/`를 그대로** 씁니다.

---

### 4단계: 모델과 effort(추론 강도) 바꿔보기

이번에는 같은 질문을 가벼운 모델로 시도합니다. `-m gpt-5.4-mini`는 **모델을 바꾸는 것**이지 effort(추론 강도)를 바꾸는 게 아닙니다. effort는 `/model` 명령이나 `config.toml`의 `model_reasoning_effort` 항목으로 조절합니다.

```text
codex --cd ~/Documents/cc-codex-101/practice-data/creator -m gpt-5.4-mini "course_sales_monthly.csv 에서 2026-03 한 달 데이터만 보고, 강의별 순매출을 정리해줘. 빠르게만."
```

> 💡 **effort와 모델이란**: Codex에는 두 개의 손잡이가 있습니다.
>
> | 손잡이 | 낮을 때 | 높을 때 |
> |--------|---------|---------|
> | **모델** | `gpt-5.4-mini` — 빠르고 가벼움 | `gpt-5.5` — 깊고 정확 (느리고 비쌈) |
> | **effort** | `low` — 빠른 답 | `high`/`xhigh` — 신중하게 검증 후 답 |
>
> 비유: 급한 메모는 빠르게 적고, 중요한 제안서는 꼼꼼하게 씁니다. 지우도 AI에게 "단순한 일이면 빨리 / 중요한 분석이면 꼼꼼하게"를 골라 시킬 수 있습니다.

**기대 결과**: `gpt-5.4-mini`로 쓰면 더 빠르고 간결하게 답합니다. 복잡한 분석 없이 숫자만 정리한 짧은 표가 나올 가능성이 높습니다.

**CC와 비교**:
- CC에서도 `/model`로 모델을 바꾸고, 가볍거나 깊은 응답을 선택할 수 있었습니다.
- "작업 난이도에 맞춰 도구의 힘을 조절한다"는 원리는 **두 도구가 완전히 같습니다.**
- 이름(gpt-5.x vs Claude)과 정확한 옵션만 다릅니다.

**관찰/체크포인트**:
- `gpt-5.4-mini`(빠름) vs `gpt-5.5`(깊음)의 답 품질이 다르게 느껴지나요?
- 세션 안에서 `/model`을 입력하면 모델과 effort(`low` / `medium` / `high` / `xhigh`)를 함께 바꿀 수 있습니다. effort를 높이면 더 꼼꼼하고 제안이 풍부해집니다.

---

### 5단계: AGENTS.md와 CLAUDE.md 나란히 비교

이번에는 Codex 세션(TUI) **안**에서 `/init`을 입력해 AGENTS.md 뼈대를 만들어 봅니다. (`/init`은 터미널 쉘 명령이 아니라, Codex가 켜진 상태에서 입력창에 치는 슬래시 명령입니다.)

```text
/init
```

**기대 결과**: Codex가 `AGENTS.md` 뼈대를 자동으로 만들어 줍니다. "이 프로젝트가 뭔지 / 어떤 규칙을 지켜야 하는지 / 완료 기준이 뭔지"를 적는 칸이 있습니다.

이제 다른 터미널 창에서 CC를 쓰던 폴더로 가서 `CLAUDE.md`를 열어봅니다.

```text
cat CLAUDE.md
```

> 💡 **보이나요?** `AGENTS.md`와 `CLAUDE.md`는 적는 내용과 구조가 거의 같습니다. 프로젝트 소개, 지켜야 할 규칙, 완료 기준, 파일 구조 설명 — 항목이 같습니다. 이름만 달라요.

**CLAUDE.md vs AGENTS.md 비교표**:

| 항목 | CLAUDE.md (CC) | AGENTS.md (Codex) |
|------|----------------|-------------------|
| 파일 이름 | `CLAUDE.md` | `AGENTS.md` |
| 역할 | 프로젝트 규칙서 | 프로젝트 규칙서 |
| 만드는 법 | CC 세션 중 `/init` | Codex 세션 중 `/init` |
| 내용 | 사업 맥락, 규칙, 완료 기준 | 사업 맥락, 규칙, 완료 기준 |
| 적용 시점 | 다음에 CC 켤 때 자동 적용 | 다음에 Codex 켤 때 자동 적용 |
| 우선순위 | 가까운 폴더 > 상위 폴더 > 전체 기본 | 가까운 폴더 > 상위 폴더 > 전체 기본 |

**핵심 관찰**: 이름이 다를 뿐, 쓰는 이유와 쓰는 방법이 거의 같습니다. CC에서 CLAUDE.md를 만드는 법을 배웠다면, AGENTS.md는 "이름만 다르게 저장"하면 됩니다.

> 💡 **비유**: 새 알바생에게 주는 "업무 매뉴얼"이 가게마다 이름이 다른 것과 같습니다. 어느 가게는 "운영 가이드", 어느 가게는 "직원 핸드북"이라고 부르지만 — 안에 쓰인 내용은 같습니다.

---

### 6단계: `$`(달러) 스킬 호출 vs `/`(슬래시) 스킬 호출 체험

CC에서는 스킬을 `/스킬이름`으로 불렀습니다. Codex에서는 `$스킬이름`으로 부릅니다.

> 💡 **스킬(skill)이란**: "이 순서대로 해줘"를 미리 적어둔 작업 묶음입니다. 매달 똑같이 하는 일(매출 정리 → 채널 비교 → 이슈 요약 → 한 줄 코멘트)을 스킬로 만들어두면 `$매출정리` 한 번으로 끝낼 수 있습니다.

Codex에서 아래를 입력해 보세요.

```text
$매출요약
```

스킬이 아직 등록되어 있지 않으면 "해당 스킬을 찾을 수 없습니다"라는 에러가 납니다. 이게 정상입니다. `$`는 "미리 만들어 둔 작업 묶음"을 부르는 기호이기 때문입니다. 등록하지 않은 스킬은 에러로 응답하는 게 맞습니다.

**CC와 Codex의 슬래시 명령 비교**:

| 하고 싶은 것 | CC | Codex |
|------------|-----|-------|
| 긴 대화 요약 | `/compact` | `/compact` |
| 모델·추론 강도 바꾸기 | `/model` | `/model` |
| 세션 상태 확인 | `/context` | `/status` |
| 계획 먼저 보기 | Shift+Tab → Plan 모드 | `/plan` |
| 규칙서 뼈대 만들기 | `/init` (CLAUDE.md) | `/init` (AGENTS.md) |
| **스킬 부르기** | `/스킬이름` | **`$스킬이름`** |

**관찰/체크포인트**:
- 대부분의 슬래시 명령은 두 도구에서 **같습니다.**
- 딱 하나, 스킬(=반복 작업 묶음)을 부르는 기호가 CC는 `/`, Codex는 `$`로 다릅니다.
- 이것만 기억하면 헷갈릴 때 "아, 지금 스킬 호출이면 Codex는 달러구나" 하고 바로 잡힙니다.

### 1단계: CC를 다시 켜고 통합 보고서 초안 요청

```text
cd ~/Documents/cc-codex-101/practice-data/creator
claude
```

CC 대화창에서:

나는 Claude Code와 Codex를 모두 써봤어.
두 도구로 같은 데이터(course_sales_monthly.csv, student_inquiries.csv, newsletter_stats.csv)를 분석했는데,
결과는 비슷했어. 이제 내 사업에서 두 도구를 어떻게 나눠 쓰면 좋을지
"지우의 두 도구 활용 기준" 한 장을 초안으로 만들어줘.

**기대 결과 (예시)**:

```markdown
# 지우의 두 도구 활용 기준 (초안 · 2026-03 기준)

## 1. Claude Code를 쓸 때
- 매출·문의·뉴스레터 데이터를 한 번에 불러 분석할 때

- 강의 스크립트·뉴스레터 초안을 만들 때 (한 호흡으로 긴 맥락 유지)

## 2. Codex를 쓸 때

## 3. 둘을 같이 쓸 때 (codex plugin)

  → 예: CC로 강의 커리큘럼을 짜다가 "이 부분 GPT 시각으로 한 번만 봐줘" 할 때

## 4. 이번 분석에서 배운 것
CC와 Codex는 같은 데이터를 비슷하게 읽는다.
차이는 "어떤 엔진으로, 어떤 모드로" — 규칙서·MCP·샌드박스 정신은 같다.
```

**관찰/체크포인트**:
- 이 초안은 지우 자신이 쓴 게 아닙니다. CC가 만든 초안입니다.
- "초안이라서 완벽하지 않아도 돼"라고 했기 때문에 CC가 덜 다듬어진 형태로 빠르게 냈습니다.
- 이게 AI 비서를 쓰는 방법입니다 — 내가 처음부터 완벽하게 쓰는 게 아니라, 초안을 받아 다듬는 것.

**관찰/체크포인트**:
- CC가 파일을 직접 만들어 폴더에 저장합니다. 복붙 없이.
- 이 파일이 지금까지 배운 것의 작은 산출물입니다.

**관찰/체크포인트**:
- 파일에 내용이 제대로 들어가 있나요?
- CC가 저장한 내용과 터미널에서 출력된 내용이 일치하나요?

**관찰/체크포인트**:

- "덮어쓰지 말고"라는 지시를 CC가 잘 따랐나요?

## 💡 핵심 규칙 / 개념 정리

### 🔑 CC ↔ Codex 핵심 차이 3가지 (외울 것)

**① 규칙서 이름**
- CC → `CLAUDE.md`
- Codex → `AGENTS.md`
- 역할은 완전히 같습니다. 이름만 다릅니다. 프로젝트에 한 번 만들어두면 그 도구를 켤 때마다 자동으로 읽힙니다.

**② 스킬 호출 기호**
- CC → `/스킬이름`
- Codex → `$스킬이름`
- 단, `/compact`·`/model`·`/plan`·`/status` 같은 세션 명령은 Codex도 `/`를 씁니다. `$`는 "반복 작업 묶음(스킬)"을 부를 때만입니다.

**③ 샌드박스(안전 모드) 표기**
- Codex는 3단계를 명확하게 이름으로 부릅니다: `read-only` / `workspace-write` / `danger-full-access`
- CC는 "허용/거부" 승인 게이트 방식
- 정신은 같습니다: "AI에게 단계적으로 권한을 준다."

### 🔑 "같다"는 것의 의미

이 레슨에서 가장 중요한 한 가지: **뼈대가 같다는 것이 우열을 가리는 기준이 아니라, "하나를 익히면 나머지가 따라온다"는 좋은 소식**입니다.

지우가 CC를 써서 매출·문의·뉴스레터를 분석하고, Codex에서 같은 분석을 했을 때 숫자가 비슷하게 나왔다면? 그 말은 "두 도구 중 어느 걸 써도 같은 일을 할 수 있다"는 뜻입니다. 선택의 여지가 생긴 것입니다.

### 🔑 언제 어느 도구를?

정해진 정답은 없지만, 지우 같은 1인 사업자에게 실용적인 기준을 제안합니다.

| 상황 | 추천 |
|------|------|
| 이미 CC에 익숙하고, 평소 업무 분석·초안 | CC 메인으로 |
| 같은 분석을 다른 엔진(GPT)으로 교차 확인하고 싶을 때 | Codex로 한 번 더 |
| 두 도구를 한 화면에서 쓰고 싶을 때 | codex plugin (cc-08에서 배운 것) |
| 처음 배울 때 | 둘 중 하나만 — 익숙해진 뒤 비교 |
