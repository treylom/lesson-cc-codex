## 📚 학습 목표 — 4가지 도달 지점

1. **"AI도 업무 매뉴얼이 있어야 한다"** — CLAUDE.md 없이 일 시키던 것과 있을 때의 차이를 눈으로 직접 본다
2. **나만의 규칙서를 직접 만든다** — 지우의 사업 맥락(강의·뉴스레터·컨설팅)에 맞는 CLAUDE.md를 손으로 작성해 본다
3. **"전과 후"를 비교한다** — 규칙서 없이 물었을 때와 있을 때 AI 응답이 얼마나 달라지는지 본다
4. **이 규칙서가 앞으로 어디에 쓰이는지 안다** — CLAUDE.md가 Codex AGENTS.md · 지식관리 규칙 · 봇 성격으로 자라나는 흐름을 머릿속에 그린다

## 🎯 개념 도입 — CLAUDE.md란 무엇인가

### AI의 기억 문제 — "어제 했던 얘기를 오늘 또 해야 한다"

Claude Code(이하 CC)와 대화할 때마다 지우는 이걸 반복했습니다.

> "저는 1인 사업자고, 뉴스레터를 운영해요. 요약은 3줄 이내로, 말투는 친근하게 해주세요."

어제 말했는데 오늘 또 말해야 합니다. CC는 **세션(대화 한 회)이 끝나면 그 내용을 잊어버리기** 때문입니다. 이건 결함이 아니라 원래 그런 구조입니다.

> **비유 — 기억 초기화 비서**
>
> 아주 능력 있는 비서가 있는데, 출근할 때마다 기억이 초기화됩니다. 그래서 매일 아침 "저는 지우 대표예요. 저희는 온라인 강의를 만들어요. 보고서는 세 줄 요약 먼저, 전문 용어 쓰지 말고, 수치는 반드시 파일에서 인용해 주세요." 처음부터 전부 다시 알려줘야 합니다.
>
> **CLAUDE.md = 비서 책상 위에 올려둔 업무 매뉴얼.** 출근하자마자 자동으로 읽어서 오늘도 어제처럼 일할 수 있게 하는 파일입니다.

CC는 새 세션을 시작할 때마다 **CLAUDE.md를 자동으로 읽습니다.** 이 파일에 "나는 이런 사람이고, 이렇게 일해줬으면 해"를 적어두면, 매번 새로 알려줄 필요가 없어집니다.

---

### CLAUDE.md vs 그냥 말하기 — 어떻게 다른가

```
CLAUDE.md 없을 때                 CLAUDE.md 있을 때
──────────────────────────         ──────────────────────────
매 대화마다 "저는 1인 사업자고…"   파일 한 번 작성 → 이후 자동 적용
말투가 매번 달라짐                 항상 내가 정한 친근한 말투
형식이 제각각                      항상 일관된 구조 (3줄 요약→상세)
AI가 내 파일 이름을 모름           내 파일 역할을 이미 앎
실수 방지 장치 없음                금지 사항을 미리 박아둠
```

### 파일 위치에 따라 적용 범위가 다릅니다

| 파일 위치 | 적용 범위 | 언제 쓰나 |
|-----------|-----------|-----------|
| `./CLAUDE.md` (현재 폴더) | 이 폴더에서 CC 켤 때만 | 특정 사업 프로젝트 전용 |
| `~/.claude/CLAUDE.md` (홈 폴더) | 내 컴퓨터 모든 폴더 | 전역 공통 규칙 |
| `./CLAUDE.local.md` | 이 폴더 · 나만 | 팀에 공유 안 할 개인 메모 |

오늘은 **작업 폴더 안에 `CLAUDE.md`를 만드는 것**부터 시작합니다. 가장 빠르게 효과를 느낄 수 있기 때문입니다.

### 4단계: CLAUDE.md 파일 만들기

이제 규칙서를 만들 차례입니다. 텍스트 에디터(메모장, VS Code 등)를 열고 `practice-data/creator/CLAUDE.md` 파일을 새로 만들어 아래 내용을 붙여 넣습니다.

```text
# 지우의 사업 운영 규칙서 (CLAUDE.md)

## 나는 누구인가
저는 1인 콘텐츠·지식 사업자 지우입니다. 온라인 강의 제작·판매,
주간 뉴스레터 발송, 1:1 컨설팅을 혼자 운영합니다.
직원이 없어서 기획·글쓰기·고객 응대·매출 정리·리서치를 전부 혼자 합니다.
항상 시간이 부족합니다.

현재 운영 중인 강의:
- 챗GPT로 콘텐츠 10배 빠르게 (AI활용, 79,000원)
- 1인 지식사업 수익화 로드맵 (비즈니스, 149,000원)
- 노션으로 1인 사업 시스템 만들기 (생산성, 89,000원)
- 클로드코드로 업무 자동화 입문 (AI활용, 119,000원, 신규)
- 초보 뉴스레터 7일 완성 (글쓰기, 59,000원)

## 말투·스타일
- 친근하고 따뜻한 말투로 이야기하듯 써주세요 ("~해요" 체)
- 전문용어가 나오면 바로 뒤에 괄호로 쉬운 풀이를 붙여주세요
- 영어·한국어 혼용, 딱딱한 문어체는 피해주세요
- 비개발자인 1인 사업자가 읽는다고 생각하고 써주세요

## 보고서·분석 형식
모든 분석 결과는 아래 순서로 정리해 주세요:
1. **세 줄 요약** — 바쁜 사람이 30초에 핵심을 잡을 수 있게
2. **관찰** — 파일에서 실제로 보이는 것 (수치 직접 인용 필수)
3. **해석** — 왜 그렇게 됐을지 가설
4. **제안** — 다음에 뭘 해볼지 (1~3가지)

수치는 반드시 파일에서 직접 읽은 실제 값을 써주세요. 지어내지 마세요.

## 절대 하지 말 것
- 데이터에 없는 수치를 지어내거나 추정값을 사실처럼 쓰지 마세요
- 확인 안 된 내용은 반드시 "추정" 또는 "가설"이라고 표시해 주세요
- 개발자 용어(API, 컴파일, 배포 등)를 그냥 쓰지 마세요 — 쉬운 말로 풀어주세요
- 수강생 이름 등 개인정보를 외부 공유용 파일 제목에 노출하지 마세요

## 자주 쓰는 파일
- `course_sales_monthly.csv` — 강의별·월별 매출 (2025-07~2026-03)
- `newsletter_stats.csv` — 뉴스레터 18호~41호 성과 (오픈율·클릭율·구독자)
- `student_inquiries.csv` — 3월 수강생·고객 문의 내역
- `content_plan.md` — 2026년 2분기 콘텐츠 기획 노트
- `research_memo.md` — AI 도구·시장 리서치 메모
- `consulting_schedule.md` — 3월 컨설팅·운영 일정
```

파일을 저장합니다.

**기대 결과**: 터미널에서 아래를 실행해 파일이 생겼는지 확인합니다.


## 🎬 실습 2 — 실제 데이터로 규칙서 효과 깊이 확인 (25분)

이제 규칙서가 살아있는 상태에서 지우의 사업 데이터를 본격적으로 파고들어 봅니다.

### 6단계: 뉴스레터 성과 분석 — "어떤 주제가 잘 통하나?"

지우는 매주 뉴스레터를 보냅니다. 18호(2025-10)부터 41호(2026-03)까지 `newsletter_stats.csv`에 데이터가 있습니다. 어떤 호가 잘 됐고 공통점이 뭔지 찾아봅니다.

```text
newsletter_stats.csv 파일을 읽어주세요.
오픈율이 58% 이상이었던 호들의 공통점을 찾아주세요.
제목·내용 특징 위주로요.
```

**기대 결과 (실제 데이터 기반)**:

```
세 줄 요약
• 오픈율 58% 이상인 호는 모두 "AI·자동화" 또는 "내 실전 경험 공개" 주제였어요.
• 강의 사전등록 프로모션(39·40호)은 클릭율도 26~30%로 역대 최고치를 기록했어요.
• 반면 "감성 에세이"(34·38호) 주제는 오픈율 42~44%로 평균 이하였습니다.

관찰 — 오픈율 58% 이상인 호들
- 21호 (61.1%): "챗GPT 프롬프트 모음집 무료 배포"
- 30호 (62.0%): "2026년 1인 사업 트렌드 5가지"
- 32호 (60.0%): "클로드코드라는 게 뭔가요?"
- 33호 (60.0%): "AI에게 일을 시키는 사람 vs 못 시키는 사람"
```

```
✅ CLAUDE.md에 한 번: 이후 모든 대화에서 자동 적용
```

### 규칙 2: 처음부터 완벽할 필요 없다 — 쓰면서 키운다

```
시작: "나는 이런 사람이에요" 3줄 + "이렇게 말해줘" 2줄
         ↓ 쓰다가 불편한 점 발견
추가: "이건 하지 말아줘" / "이 형식으로 해줘"
         ↓ 계속 키움
나중: 지우만의 완성된 운영 매뉴얼
```

### 규칙 3: `/init` — AI가 초안을 대신 써주는 방법

직접 쓰기 막막하다면 CC에게 시킬 수도 있습니다.

```bash
/init
```

이 명령을 실행하면 CC가 현재 폴더를 분석해 CLAUDE.md 초안을 자동 생성합니다. 그걸 읽고 내 상황에 맞게 수정하면 됩니다. 단, 자동 생성된 내용은 일반적일 수 있으니 반드시 내 사업 맥락으로 다듬으세요.

### 규칙 4: `@파일명` — 다른 파일 참조 연결

CLAUDE.md 안에 `@파일명`을 쓰면 다른 파일 내용을 참조할 수 있습니다.

```markdown
## 운영 기준 참고
@consulting_schedule.md의 내용을 일정 관련 질문에 항상 참고할 것
```

CLAUDE.md 자체는 짧게 유지하면서, 필요한 맥락 파일을 연결할 수 있습니다.

### 규칙 5: CLAUDE.md는 자라나는 파일이다

오늘 만든 CLAUDE.md는 끝이 아니라 시작입니다.

```
cc-05 CLAUDE.md
    → codex-06 AGENTS.md (Codex용 규칙서 — 파일 이름만 다른 같은 원리)
    → Part 3 지식관리 규칙 (옵시디언 저장 방식)
    → Part 5~6 봇 성격 soul.md (AI 에이전트 팀의 뼈대)
```

지금 만든 규칙 한 줄이 나중엔 AI 에이전트 팀의 뼈대가 됩니다.
