# codex-04: Codex 멀티면 — App·IDE·Web·Mobile — "어느 창구로 들어와도 같은 비서가 기다린다"

> **레슨 목표**: CLI(터미널) 바깥에 있는 Codex의 네 창구(App·IDE·Web·Mobile)를 직접 구경하고, 각 창구가 지우의 어느 업무에 어울리는지 감을 잡는다. 실습보다 탐색·관찰이 중심인 **부록(appendix)** 레슨.
> **예상 시간**: 40분 (개념 도입 8분 + 창구 탐색 15분 + 실습 2 비교 체험 12분 + 마무리 5분)
> **사전 준비**: Codex CLI 설치·로그인 완료(codex-01 이수) · CLI 기본 명령 익힘(codex-02 이수) · `practice-data/creator/` 폴더 준비 · 브라우저 탭 하나 열어두기

## 📚 학습 목표 — 4가지 와우 모먼트

1. **"Codex는 터미널만 있는 게 아니다"** — App·IDE·Web·Mobile, 네 창구가 따로 있다는 걸 눈으로 확인한다.
2. **"창구마다 강점이 다르다"** — 터미널은 자동화·일괄처리, 앱은 간편한 시작, IDE는 편집기 안에서 바로, 웹은 어디서나. 상황별로 골라 쓸 수 있다는 감각을 얻는다.
3. **"같은 질문, 다른 창구 — 결과는 같다"** — CLI와 Web으로 똑같은 분석을 해보고, AI 엔진이 동일함을 직접 체감한다.
4. **"CLI가 기본, 나머지는 편의"** — 이미 배운 터미널 실력이 어떤 창구에서도 그대로 빛난다는 것을 확인하고, "지금은 CLI 하나로 충분하다"는 편안함을 가진다.

> ⚠️ **이번 레슨은 부록(appendix) 모듈**입니다. codex-01~03에서 배운 CLI·모델·effort를 쓸 줄 알면 충분합니다. 멀티면은 "이런 것도 있구나"를 구경하는 레슨이지, 모두 세팅해야 하는 필수 과제가 아닙니다. 부담 없이 따라오세요.

## 🎯 개념 도입 — 같은 비서, 다른 출입구

### 비유: 동네 카페에 들어가는 문이 네 개

동네에 새 카페가 생겼습니다. 안에서 파는 커피는 똑같고, 바리스타도 한 명인데, **들어가는 문이 네 개**입니다.

- **앱(데스크톱 앱)**: 스마트폰 앱처럼 아이콘을 눌러 들어가는 문. 설치하면 바탕화면에 아이콘이 생깁니다.
- **편집기 문(IDE 확장)**: 이미 열어둔 문서 편집기(VS Code 같은) 창 안에서 바로 들어가는 통로.
- **웹 문(`chatgpt.com/codex`)**: 인터넷 브라우저로 어디서든 — 다른 컴퓨터든, 카페든 — 접속하는 문.
- **모바일 문**: 스마트폰으로 진행 상황을 확인하고 승인·거부하는 창구.

Codex도 똑같습니다. 안에서 돌아가는 AI 엔진은 같지만, **접근하는 창구(면, surface)가 여러 가지**입니다. 어느 창구로 들어가도 같은 Codex가 기다리고 있습니다.

### 네 창구 한눈에 — "이미 아는 CLI와 비교"

> 💡 **CLI는 이미 배운 창구입니다.** 아래 표에서 CLI는 "비교 기준(이미 익숙한 것)"으로 별도 구분했습니다. 오늘 새로 구경할 창구는 App·IDE·Web·Mobile, 네 가지입니다.

| 창구 | 분류 | 어떻게 쓰나 | 지우에게 어울리는 상황 | 지금 당장? |
|---|---|---|---|---|
| 🖥️ **CLI** (터미널) | 이미 아는 것 | `codex --cd ~/폴더 "지시문"` | 파일 분석·자동화·일괄처리 — 가장 강력 | ✅ 이미 배움 |
| 💻 **App** (데스크톱) | 오늘 구경 | Mac/Windows 앱 설치 후 아이콘 클릭 | 터미널 없이 간편하게 시작 | 선택 |
| 🛠️ **IDE** (코드 편집기) | 오늘 구경 | VS Code 등 내부 확장 설치 후 사용 | 문서 파일을 편집기로 열고 작업할 때 | 선택 |
| 🌐 **Web** (`chatgpt.com/codex`) | 오늘 구경 | 브라우저로 접속 | 다른 컴퓨터에서, GitHub 연동할 때 | 바로 가능 |
| 📱 **Mobile** (스마트폰 앱) | 오늘 구경 | Mac/Windows 호스트 앱 원격 제어 | 외출 중 승인·거부, 결과 확인 | 나중에 |

> 💡 **핵심 안심**: 지우가 지금까지 배운 건 **CLI(터미널)** 창구입니다. 나머지 네 창구는 "이런 게 있구나" 하고 구경만 해도 됩니다. 어느 창구로 들어가든 Codex 엔진은 같고, codex-01~03에서 배운 명령·개념이 그대로 적용됩니다.

### Claude Code와 비교 — "CC는 창구 하나, Codex는 여러 개"

CC를 배울 때는 "터미널 한 창구"였습니다. Codex는 **멀티면(multi-surface, 여러 창구)**이라는 점이 가장 눈에 띄는 차이입니다.

| 항목 | Claude Code | Codex |
|---|---|---|
| 기본 창구 | CLI(터미널) | CLI(터미널) — 똑같음 |
| 추가 창구 | 데스크톱 앱(기능 유사) | **App + IDE + Web + Mobile** |
| 규칙서 이름 | `CLAUDE.md` | `AGENTS.md` |
| 비유 | 정문만 있는 가게 | 정문·뒷문·앱·웹까지 있는 가게 |

그렇다고 Codex가 "더 좋다"는 뜻이 아닙니다. CC는 단일 창구에서 더 깊이·길게 한 호흡으로 일하는 데 강점이 있고, Codex는 상황에 따라 창구를 바꿔 쓰는 데 강점이 있습니다. 둘을 각자 잘 쓰면 됩니다.

## 🔧 실습 1 — 네 창구 구경하기 (탐색·이해 중심)

> ⚠️ 이 실습은 **"설치부터 다 해야 한다"가 아니라 각 창구가 어떻게 다른지 눈으로 보고 이해하는 탐색 실습**입니다. 관심 있는 창구를 골라 체험하고, 내 상황에 맞는 것을 한두 개 고르는 것이 목표입니다.

### 1단계: CLI 창구 — 이미 아는 것, creator 폴더 탐색으로 확인

CLI는 codex-01~03에서 이미 배웠습니다. 여기서는 "어떤 상황에서 CLI를 선택하면 좋은가"를 다시 한 번 정리하며, 지우의 실습 데이터 폴더를 직접 가리켜 봅니다.

터미널을 열고 아래를 입력합니다.

```bash
codex --cd ~/Desktop/practice-data/creator "이 폴더에 있는 파일 목록을 간단히 알려줘. 각 파일이 무엇을 담고 있는지 한 줄씩."
```

**기대 결과**:
```
이 폴더에 있는 파일은 다음과 같습니다:

- course_sales_monthly.csv: 강의별·월별 매출 데이터 (2025-07 ~ 2026-03)
- newsletter_stats.csv: 뉴스레터 호별 발송·오픈율·클릭율·구독 증감 (18호~41호)
- student_inquiries.csv: 수강생 문의 내역과 처리 상태
- content_plan.md: 2분기 콘텐츠 기획 노트
- research_memo.md: AI 도구·1인 사업 시장 리서치 메모
- consulting_schedule.md: 3월 컨설팅·강의·발행 일정 노트
```

**관찰 포인트**:
- `--cd` 옵션 하나로 폴더를 지정하면 Codex가 알아서 파일을 읽습니다.
- 파일을 열거나 복사·붙여넣기 없이 바로 내용 파악이 됩니다.
- 결과가 터미널 안에 텍스트로 출력됩니다.

- [ ] "내 폴더를 AI가 직접 읽었다"는 느낌이 왔나요?

### 2단계: CLI 창구 — 두 번째 파일 `student_inquiries.csv` 조회

같은 CLI 창구에서 이번에는 수강생 문의 파일을 조회해봅니다. "창구는 같지만 다른 파일도 자유롭게 다룰 수 있다"는 것을 확인하는 단계입니다.

```bash
codex --cd ~/Desktop/practice-data/creator "student_inquiries.csv에서 처리 상태가 '진행중'인 문의만 골라서 날짜·수강생명·문의 내용을 간단히 정리해줘."
```

**기대 결과**:
```
현재 처리 중인(진행중) 문의 목록:

1. 2026-03-01 | 이도윤(S003) | 1인 지식사업 수익화 로드맵
   → 내용 달라 환불 가능한지 문의 (소요 1.0시간, 높음)

2. 2026-03-02 | 정유나(S004) | 클로드코드로 업무 자동화 입문
   → 터미널 설치 에러 (맥, 소요 2.0시간, 높음)

3. 2026-03-05 | 배준혁(S013) | 클로드코드로 업무 자동화 입문
   → 윈도우 WSL 설치 어려움 (소요 3.0시간, 높음)

4. 2026-03-06 | 조은서(S015) | 클로드코드로 업무 자동화 입문
   → CLAUDE.md 규칙 적용 안 됨 (소요 1.5시간, 일반)

5. 2026-03-08 | 백서윤(S022) | 클로드코드로 업무 자동화 입문
   → Codex 로그인 자꾸 풀림 (소요 2.0시간, 일반)

패턴: 미처리 5건 중 4건이 '클로드코드' 강의 설치·설정 관련.
```

**관찰 포인트**:
- 파일을 열지 않고 조건("진행중"만)으로 필터링해서 결과를 뽑았습니다.
- 엑셀 필터 기능과 같은 작업을, 말 한 마디로 해냈습니다.
- "미처리 문의 5건, 그 중 4건이 클로드코드 강의" — 이런 패턴이 바로 보입니다.

- [ ] 파일을 직접 열지 않았는데 AI가 필터링해줬다는 게 신기했나요?

### 3단계: App 창구 — 데스크톱 앱 구경

데스크톱 앱(macOS/Windows 전용 그래픽 화면)은 터미널 없이 마우스로 클릭해서 Codex를 쓸 수 있는 창구입니다.

> ⚠️ **〔확인 필요〕**: 데스크톱 앱의 정확한 다운로드 경로·UI 구성은 공식 docs(`developers.openai.com/codex`)와 실제 0.140.0 실측으로 사용 전 재확인합니다. 아래는 2026-06 스냅샷 기준 안내입니다.

**어떤 상황에서 유용한가**:
- 터미널 창이 열리기 귀찮고, 바탕화면 아이콘을 눌러 빠르게 시작하고 싶을 때
- Codex가 파일을 수정하기 전에 "변경 내용(diff, 달라진 부분)"을 시각적으로 확인하고 승인/거부하고 싶을 때

**지우 시나리오**:
> "오늘은 터미널 여는 게 귀찮다. 바탕화면 아이콘을 눌러서 빠르게 `content_plan.md` 내용을 정리하고 싶다."

**App 창구에서 파일 읽기 흐름 (예시)**:
```
[데스크톱 앱을 연 다음]
  → 파일 선택 또는 폴더 드래그
  → 입력창에 아래를 입력

content_plan.md의 '아이디어 백로그' 섹션에서
가장 빠르게 실행할 수 있는 아이디어 3개를 골라줘.
```

**관찰 포인트**: 데스크톱 앱은 CLI와 **같은 Codex**입니다. 차이는 "보는 방식"이 더 친숙하다는 것뿐입니다. 복잡한 자동화나 파일 여러 개를 한번에 다루는 건 여전히 CLI가 강합니다.

- [ ] "CLI와 결국 같은 Codex"라는 것을 이해했나요?

### 4단계: IDE 창구 — 편집기 안에서 바로 Codex 쓰기

IDE(통합 개발 환경, Integrated Development Environment)는 VS Code 같은 **문서·코드 편집기 프로그램**입니다. 비개발자도 Markdown 파일이나 CSV를 편집기로 여는 경우가 있는데, 그 편집기 안에서 바로 Codex와 대화하는 창구입니다.

> 💡 **비개발자 안심**: "IDE"라는 말이 낯설어도 걱정하지 마세요. **VS Code**(비주얼 스튜디오 코드)라는 무료 편집기가 대표적입니다. 지금 당장 쓸 필요는 없고, "이런 창구가 있구나" 정도만 알면 충분합니다.

**어떤 상황에서 유용한가**:
- 파일을 편집기에서 열어놓고 Codex에게 "이 파일의 이 부분을 고쳐줘"라고 할 때
- 파일 내용을 따로 복사해서 붙여넣지 않아도 Codex가 **자동으로 열린 파일을 인식**하는 기능(auto-context, 자동 맥락 인식) 덕분에 바로 대화 가능 〔확인 필요〕

**auto-context가 뭔가**:

```
[일반 CLI 방식이라면]
  → 터미널로 나가서 codex 실행
  → "@content_plan.md 이 파일에서..." 하고 파일 경로 직접 지정

[IDE 창구라면]
  → 이미 편집기에서 열려 있는 파일을 Codex가 자동 인식
  → "이 파일의 미해결 고민 섹션을 요약해줘" 바로 입력 가능
```

**IDE 창구에서 실제로 입력하는 예시**:

```text
[VS Code에서 content_plan.md를 열고, Codex 패널에 입력]

아이디어 백로그에서 가장 긴급한 3개를 골라서
긴급한 이유와 예상 제작 시간도 같이 알려줘.
```

**지우 시나리오**:
> "`content_plan.md`를 편집기에서 열고 수정하던 중에, 오른쪽 패널의 Codex에게 '아이디어 백로그에서 가장 긴급한 3개를 골라줘'라고 바로 말한다. 파일을 복사하지 않아도 된다."

**체크포인트** ✅

- [ ] auto-context(열린 파일 자동 인식)가 편리한 이유를 이해했나요?

### 5단계: Web 창구 — 브라우저로 어디서나

가장 접근하기 쉬운 창구입니다. 터미널도, 앱 설치도 필요 없습니다. **인터넷이 되는 곳이면 어디서나** 접속합니다.

**주소**: `chatgpt.com/codex`

**특징**:
- 브라우저에서 ChatGPT 계정으로 로그인하면 바로 사용
- GitHub 계정과 연동해 자신의 저장소를 바로 연결 가능 〔확인 필요〕
- 다른 컴퓨터에서도 같은 계정으로 접속

**지우 시나리오**:

| 상황 | 원인 | 해결 |
|---|---|---|

## 💡 핵심 규칙·개념 정리

**이 레슨에서 기억할 것 4가지**

**1. Codex는 CLI 하나가 아니다**

터미널(CLI)·데스크톱 앱(App)·코드 편집기(IDE)·브라우저(Web)·스마트폰(Mobile). 어느 창구를 쓰든 안에 있는 AI 엔진은 같습니다. 오늘 새로 구경한 것은 CLI 바깥의 네 창구(App·IDE·Web·Mobile)입니다.

**2. 창구마다 강점이 다르다**

| 상황 | 추천 창구 |
|---|---|
| 집중 분석·자동화·파일 여러 개 | CLI — 가장 강력 |
| 터미널 없이 간편하게 시작 | 데스크톱 앱 |
| 편집기로 파일 작업 중 | IDE 확장 |
| 이동 중·다른 컴퓨터·빠른 확인 | Web / Mobile |

**3. 지금 당장 다 쓸 필요 없다**

지금은 CLI 하나로 충분합니다. 나머지는 "필요할 때 꺼내는 도구"입니다. "다 설치해야 해" 부담을 가질 필요가 없습니다.

**4. CC와 뼈대는 같다**

Claude Code(CC)도 멀티면 개념이 비슷하게 적용됩니다. Codex의 멀티면을 이해하면 CC의 새 창구가 생길 때도 쉽게 적응할 수 있습니다.

**용어 정리**:
- **surface(면)**: Codex에 접근하는 창구·진입점. "멀티면"은 창구가 여러 개라는 뜻.
- **auto-context(자동 맥락 인식)**: IDE 확장에서 현재 열린 파일을 자동으로 Codex에게 보여주는 기능. "열린 파일 = 자동으로 알려준다"고 기억하면 됩니다.
- **diff(차이)**: 파일이 수정되기 전과 후의 달라진 부분. 데스크톱 앱이나 모바일에서 "이렇게 바꾸면 괜찮나요?"라고 색으로 보여주는 화면.
