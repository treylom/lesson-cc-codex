## 🎯 개념 도입 — 왜 트러블슈팅 레슨이 따로 있나?

**비유**: 요리 레시피 책에 "이렇게 됐다면?" 부록이 있듯이 — 설명대로 따라 했는데 결과가 다를 때 펼치는 지도다.

Codex는 기능이 강력한 만큼 "왜 안 되지?" 순간도 다양하게 생긴다. 특히 지우처럼 물어볼 동료가 없을 때, 이 레슨 하나로 막힘의 80%를 스스로 해결할 수 있다.

지금까지 배운 것들(설치·CLI·모델·effort·승인·AGENTS.md·MCP·비대화형·프롬프트)이 모두 연결되는 지점이기도 하다. 뭔가 안 된다면 — 대부분은 이 중 하나가 빠져있거나 설정이 어긋난 것이다.

**디버그 4단계 순서**:
```
① /status로 세션 상태 확인
    ↓
② /model로 모델·effort 전환
    ↓
③ codex --cd 로 폴더 재지정
    ↓
④ @파일명으로 파일 정확히 지목
```

이 순서를 외우면 된다. 막힐 때마다 ① → ② → ③ → ④ 순으로 한 단계씩 해결한다.

## 🔧 실습 1 — 내 환경 빠른 점검 (15분)

**상황**: Codex가 "이상하게 느리다", "응답이 짧다", "AGENTS.md가 안 먹힌다" — 지우가 `consulting_schedule.md`를 분석시키려는데 뭔가 이상하다. 먼저 현재 세션 상태를 한눈에 확인한다.

### 1단계: Codex 실행 + 버전 확인

터미널을 열고 Codex가 제대로 설치되어 있는지 먼저 확인한다.

```bash
# 설치 확인
codex --version

# Codex 실행 (대화형 세션 시작)
codex
```

**기대 결과**:
```
codex 0.140.0 (버전 번호 표시)
...
Welcome to Codex! Type a message to get started.
> ▌
```

**관찰 포인트**:
- 버전 번호가 표시되면 설치 정상 ✓
- `command not found` → 터미널을 완전히 닫고 다시 열기 → `npm install -g @openai/codex` 재실행
- 오래된 버전 → `npm update -g @openai/codex`로 업데이트

---

### 2단계: 세션 상태 한 번에 체크 (`/status`)

Codex 세션 안에서 아래 슬래시 명령을 입력한다. "뭔가 이상하다"는 느낌이 들 때 가장 먼저 해야 할 것이다.

```text
/status
```

**기대 결과**:
```
모델: gpt-5.5
Effort: medium
샌드박스: workspace-write
토큰 사용: 12,400 / 100,000
승인 정책: on-request
```

**관찰 포인트**:
- 모델이 `gpt-5.4-mini`로 되어 있으면 → 복잡한 분석엔 부족할 수 있음 (`/model gpt-5.5`로 전환)
- 토큰이 80,000을 넘어가면 → `/compact`로 정리 후 계속
- 샌드박스가 `read-only`인데 파일을 저장하려 한다면 → 당연히 실패

---

### 3단계: AGENTS.md 경로 확인

"AGENTS.md 규칙이 안 먹힌다"는 제일 흔한 문제다. 파일이 **어디에** 있느냐가 핵심이다.

```bash
# Codex 세션 밖 터미널에서 확인
ls ~/Desktop/jiu-business/AGENTS.md        # 작업 폴더 루트
ls ~/.codex/AGENTS.md                      # 전역 설정 (모든 세션에 적용)
```

우선순위(높은 것이 이긴다):
```
AGENTS.override.md (같은 폴더)
  > ./AGENTS.md (현재 작업 폴더)
    > ../AGENTS.md (한 단계 위)
      > ~/.codex/AGENTS.md (전역)
```

**지우의 케이스**: `consulting_schedule.md`가 `~/Desktop/jiu-business/`에 있다면, AGENTS.md도 그 폴더에 있어야 규칙이 적용된다.

**체크포인트** ✓:
- 규칙이 안 먹힌다 → 파일 위치부터 점검
- 파일명 오타(대소문자 포함) → `AGENTS.md` (정확한 철자 필수)

---

### 4단계: `--cd` 플래그로 정확한 폴더 지정

Codex가 엉뚱한 폴더를 보고 있다면 — 시작부터 폴더를 지정한다.

```bash
# 터미널에서 (대화형 세션 시작 명령)
codex --cd ~/Desktop/jiu-business
```

세션이 열리면 아래를 입력한다:

```text
consulting_schedule.md를 읽고 3월 컨설팅 예약 현황을 정리해줘.
표 형태로 날짜·고객명·주제·상태 4개 열로 보여줘.
```

**기대 결과**:
```
3월 컨설팅 예약 현황:

| 날짜   | 고객명 | 주제             | 상태 |
|--------|--------|------------------|------|
| 03-06  | 정유나 | 강의 라인업 정리  | 완료 |
| 03-13  | 한지호 | 구독자 늘리기     | 완료 |
| 03-13  | 강태민 | 수익화 구조       | 완료 |
| 03-20  | 서다은 | AI 콘텐츠 자동화  | 예약 |
| 03-20  | 문가영 | 부업→본업 전환    | 예약 |
| 03-27  | (미정) | —                | 오픈 |
```

**관찰 포인트**:
- "파일을 찾을 수 없습니다" → `--cd` 경로 재확인
- 빈 응답 → 파일이 비어있거나 경로 오타

---

### 5단계: `/permissions`로 샌드박스 모드 확인

파일을 저장하게 하려는데 안 된다면 — 현재 샌드박스 모드를 확인한다.

```text
/permissions
```

**기대 결과**:
```
샌드박스: workspace-write
승인 정책: on-request
읽기: 허용
쓰기: 허용 (승인 후)
명령 실행: 허용 (승인 후)
네트워크: 차단
```

**관찰 포인트**:
- `read-only`면 파일 저장 불가 → `codex -m gpt-5.5 --sandbox workspace-write`로 재시작
- 파일 삭제·덮어쓰기 전엔 항상 승인 요청이 뜨는 게 정상

---

### 6단계: config.toml 존재 여부 확인

매번 같은 설정을 반복해야 한다면 — 설정 파일로 기본값을 고정한다.

```bash
# 전역 설정 파일 확인
ls ~/.codex/config.toml

# 파일이 없으면 새로 만들기
cat > ~/.codex/config.toml << 'EOF'
model = "gpt-5.5"
model_reasoning_effort = "medium"
approval_policy = "on-request"
EOF
```

**기대 결과**: 이제 `codex`를 실행할 때마다 위 설정이 자동 적용된다. `/status`로 확인하면 `gpt-5.5 / medium`이 기본값으로 표시된다.

**체크포인트** ✓:
- 설정 저장 후 Codex를 재시작해야 적용됨
- 프로젝트별 설정은 `~/Desktop/jiu-business/.codex/config.toml`에 따로 두면 그 폴더에서만 적용

---

### 7단계: `/init`으로 AGENTS.md 뼈대 자동 생성

처음 AGENTS.md를 만들 때 — `/init`으로 뼈대를 자동으로 생성한다.

```bash
# jiu-business 폴더에서 Codex를 실행한 상태에서
codex --cd ~/Desktop/jiu-business
```

세션 안에서:

```text
/init
```

**기대 결과**: Codex가 현재 폴더 구조를 읽고 `AGENTS.md` 뼈대를 자동으로 생성한다. 폴더에 CSV·md 파일들이 있으면 그것을 참고해 "이 프로젝트는 강의 운영 데이터를 다루는 것 같다"는 내용이 포함된 초안이 만들어진다.

**체크포인트** ✓:
- 생성 후 파일을 열어 내 역할·규칙·완료 기준을 직접 수정
- 규칙이 5~10줄을 넘어가면 Codex가 중요한 것을 놓칠 수 있음

## 🎬 실습 2 — 매출 데이터로 인사이트 뽑기 (15분)

**상황**: 지우가 `course_sales_monthly.csv`로 분석을 돌리다 막혔다. "분석이 너무 느리다", "결과가 이상하다" — 두 가지 문제를 실제로 풀어보면서, 동시에 진짜 인사이트까지 뽑는다.

```bash
codex --cd ~/Desktop/jiu-business/practice-data/creator
```

세션 안에서:

```text
이 폴더에 어떤 파일이 있는지 읽어줘.
각 파일이 어떤 내용인지 한 줄씩 설명해줘.
```

**기대 결과**:
```
폴더 파일 목록:
- course_sales_monthly.csv: 강의별 월별 순매출 (2025-07~2026-03)
- student_inquiries.csv: 수강생 문의 내역 (2026년 3월)
```

**관찰 포인트**: Codex가 파일 목록을 탐색하고 각 파일의 성격을 파악한다. 이 단계가 이후 분석의 정확도를 높인다.

### 2단계: 느릴 때 — effort 낮추기

복잡한 분석이 아니라 단순 집계라면 effort를 낮춰 속도를 올린다.

```text
/model gpt-5.4-mini
```

그 다음 아래 프롬프트를 입력한다:

```text
@course_sales_monthly.csv를 읽고 강의별 순매출 합계를 내림차순으로 표로 정리해줘.
조건: 강의명 / 총 순매출(만원) / 최고 월 / 최고 월 순매출
```

**기대 결과** (실제 데이터 기준):

| 강의명 | 총 순매출(만원) | 최고 월 | 최고 월 순매출 |
|--------|----------------|---------|----------------|
| 챗GPT로 콘텐츠 10배 빠르게 | 7,258 | 2026-01 | 1,252만 |
| 1인 지식사업 수익화 로드맵 | 5,312 | 2026-01 | 1,167만 |
| 노션으로 1인 사업 시스템 만들기 | 1,920 | 2025-08 | 449만 |

| 초보 뉴스레터 7일 완성 | 1,268 | 2025-10 | 355만 |

**관찰 포인트**:
- gpt-5.4-mini는 단순 집계에서 gpt-5.5 대비 확연히 빠름
- 클로드코드 강의의 최고 월은 2026-03(1,242만) — 2026-02 출시(508만) 다음 달에 급성장한 패턴이 보임

### 3단계: 결과가 이상할 때 — `@파일명`으로 명확히 지목

Codex가 엉뚱한 파일을 참조하면 — `@파일명`으로 정확히 지목한다.

```text
@course_sales_monthly.csv에서 AI활용 카테고리 데이터만 뽑아줘.
2026년 월별 순매출 추이를 보여주고, 전월 대비 증감도 표시해줘.
```

**기대 결과**:
```
AI활용 카테고리 월별 순매출:

2026-01: 챗GPT 1,252만 (AI활용 합계: 1,252만 — 클로드코드 아직 출시 전)
2026-02: 챗GPT 913만 + 클로드코드 508만 = 1,421만 (+169만, +13%)
2026-03: 챗GPT 823만 + 클로드코드 1,242만 = 2,065만 (+644만, +45%)

→ 클로드코드 강의가 2월 출시 후 3월에 1,242만원으로 급성장.
   챗GPT 강의는 소폭 하락 중이나, 클로드코드 증가분이 이를 상쇄.
   AI활용 카테고리 전체는 전월 대비 +45% 성장.
```

**와우 모먼트**: "아, 클로드코드 강의 덕분에 AI 카테고리 전체가 3월에 급성장했구나." 숫자 하나가 아니라 흐름이 보인다.

### 5단계: 두 파일 교차 분석

## 📋 트러블슈팅 표 — "이럴 때 이렇게"

| 증상 | 원인 | 해결 |
|------|------|------|
| **"파일을 찾을 수 없습니다"** | 작업 폴더가 다름 | `codex --cd /정확한/경로` 재시작 |
| **AGENTS.md 규칙이 안 먹힌다** | 파일 위치 오류 | 작업 폴더 루트에 `AGENTS.md` 있는지 확인 (우선순위: override > ./ > ../ > ~/.codex/) |
| **응답이 너무 느리다** | effort·모델이 과함 | `/model gpt-5.4-mini` + effort 낮추기 |
| **응답이 중간에 끊긴다** | 토큰 한도 근접 | `/status`로 확인 → `/compact`로 압축 후 계속 |
| **결과가 이상하거나 파일을 엉뚱하게 읽는다** | 파일 특정 안 됨 | `@파일명`으로 정확히 지목 |
| **로그인이 계속 풀린다** | 세션 만료 or 환경변수 미설정 | `codex login` 재실행 / `~/.bashrc`에 `OPENAI_API_KEY` 확인 |
| **샌드박스 오류 — 파일 저장 안 됨** | `read-only` 모드 | `/permissions`로 모드 확인 → `workspace-write`로 재시작 |
| **gpt-5.5가 안 보인다** | 구독·계정 설정 문제 | 계정에서 사용 가능한 모델 확인 + `npm update -g @openai/codex` |
| **같은 질문을 계속 다시 물어본다** | 맥락 증발 | `/compact` 후 핵심 요청 재전달, 또는 AGENTS.md에 상황 설명 추가 |

| **`/compact` 후 맥락이 사라졌다** | 정상 동작 (요약으로 대체) | 다시 핵심 배경 1~2줄 재입력 → 계속 진행 가능 |

### CC↔Codex 비교 한 장 요약

| 축 | Claude Code (CC) | Codex |
|----|------------------|-------|
| 만든 곳 | Anthropic | OpenAI |
| 기본 모델 | Claude 3.x / 4.x | gpt-5.5 |

| 스킬 호출 기호 | `/스킬이름` | `$스킬이름` |
| 샌드박스 | 승인 게이트 | read-only / workspace-write / danger-full-access |
| 진입 명령 | `claude` | `codex` |
| 공통 | 로컬 파일 에이전트·승인 게이트·MCP·effort·outcome-first 프롬프트 |

**한 문장 정리**: "CC를 익혔으면 Codex는 90% 전이된다 — 규칙서·샌드박스·MCP·effort는 같고, 제품·모델·세부 기능만 다르다."

### 세션 슬래시 명령 모음

| 명령 | 기능 | 언제 쓰나 |
|------|------|----------|
| `/status` | 세션 상태 (모델·effort·토큰) | 뭔가 이상하다 싶을 때 첫 번째 |
| `/model` | 모델·effort 전환 | 빠르게 하려면 mini, 깊이 있게는 gpt-5.5 |
| `/permissions` | 샌드박스 모드 확인 | 파일 저장이 안 될 때 |
| `/plan` | 플랜 모드 켜기 (실행 전 계획만) | 먼저 계획을 보고 싶을 때 |
| `/compact` | 대화 요약·압축 | 토큰이 80% 넘어갈 때, 작업 완료 후 |
| `/init` | AGENTS.md 뼈대 자동 생성 | 새 프로젝트 폴더에서 처음 시작할 때 |

### 샌드박스 모드 3가지

| 모드 | 읽기 | 쓰기 | 명령 실행 | 언제 |
|------|------|------|----------|------|
| `read-only` | ✅ | ❌ | ❌ | 분석만 할 때 |
| `workspace-write` | ✅ | ✅ | ✅ | 일반 작업 (파일 저장·편집) |
| `danger-full-access` | ✅ | ✅ | ✅ + 네트워크 | CI 자동화·충분한 검증 후만 |

### effort 수준 선택 가이드

effort는 `/model` UI 또는 config.toml의 `model_reasoning_effort`로 설정한다. CLI에 `--effort` 플래그는 없다.

| 수준 | 적합한 작업 | 속도 |
|------|------------|------|
| `low` | 단순 변환·포맷·간단한 읽기·정리 | 빠름 |
| `medium` | 다파일 합산·중간 분석 | 보통 |
| `high` | 복잡한 리팩터·검증 | 느림·정확 |
| `xhigh` | 장기 에이전트·엄격 증명 | 가장 느림 |

**지우의 기준**: 매출 합계·문의 분류 → `low`~`medium`. 뉴스레터 초안·보고서 종합 → `medium`~`high`. (설정: config.toml의 `model_reasoning_effort = "medium"` 또는 세션 중 `/model`)

## 📚 FAQ 20선 — "이런 게 궁금했어요"

### 설치·로그인

**Q1. `codex login` 하면 뭐가 열리나요?**
브라우저에서 ChatGPT 계정 로그인 페이지가 뜬다. ChatGPT 계정(Plus/Pro)으로 로그인하면 터미널로 돌아온다. API 키 없이도 된다.

**Q2. ChatGPT Free 계정으로도 되나요?**
설치는 되지만, 구독·계정에 따라 사용 가능한 모델이 다를 수 있다. 자신의 계정에서 어떤 모델을 쓸 수 있는지는 공식 docs(`developers.openai.com/codex`) 또는 ChatGPT 계정 설정에서 확인한다.

**Q3. `npm install -g @openai/codex` 말고 다른 설치 방법이 있나요?**
Mac은 `brew install --cask codex`, 또는 `curl -fsSL https://chatgpt.com/codex/install.sh | sh`도 된다(이 주소는 OpenAI 공식이라 안전 — 단 일반적으로 출처가 불확실한 `| sh` 명령은 조심). 셋 다 같은 버전이 설치된다.

**Q4. 설치했는데 `codex: command not found`가 뜬다면?**
`npm`이 글로벌 경로를 PATH에 추가 안 했을 가능성이 높다. `npm bin -g` 명령으로 경로를 확인하고 `~/.bashrc` 또는 `~/.zshrc`에 추가한다. 터미널을 완전히 닫았다 다시 열기만 해도 해결되는 경우가 많다.

---

### 모델·effort

**Q5. gpt-5.5와 gpt-5.4-mini 중 뭘 써야 하나요?**
일반 분석·초안 작성엔 gpt-5.5로 충분하다. gpt-5.4-mini는 단순 정렬·요약처럼 빠른 처리가 필요할 때 선택한다.

**Q6. effort를 높이면 비용도 올라가나요?**
ChatGPT 구독 기반이면 사용량이 아닌 구독료가 기준이라 직접 비용 증가는 없다. 단 시간이 더 걸린다. API 키 방식이라면 토큰 사용이 늘어 비용이 올라간다.

**Q7. 세션 중간에 모델을 바꾸면 이전 대화는 유지되나요?**
대화 내용(맥락)은 유지된다. 모델만 바뀐다. 단 새 모델이 이전 대화를 어떻게 해석하는지는 약간 달라질 수 있다.

---

### AGENTS.md·config.toml

**Q8. AGENTS.md에 어떤 내용을 넣으면 좋나요?**
(1) 내 역할·사업 맥락, (2) 작업 방식(한국어로 답해줘, 표는 마크다운으로 등), (3) 금지 사항(파일 삭제 전 항상 확인), (4) 작업 완료 기준. 너무 길면 Codex가 중요한 규칙을 놓칠 수 있다 — 핵심 5~10줄이 적당하다.

**Q9. config.toml은 어디에 두나요?**
`~/.codex/config.toml`(전역) 또는 작업 폴더의 `.codex/config.toml`(로컬). 로컬이 전역을 덮어쓴다.

**Q10. AGENTS.md를 만들었는데 Codex가 안 읽는 것 같아요.**
파일명 오타(대소문자 확인: `AGENTS.md` — 다른 철자 불가), 파일 위치(작업 폴더 루트인지), 실행 디렉터리(`--cd`로 맞게 지정했는지) 세 가지를 순서대로 확인한다.

---

### 샌드박스·승인

**Q11. read-only 모드에서 분석만 하면 안전한가요?**
파일 쓰기·명령 실행이 차단되므로 분석 목적엔 가장 안전하다. 단 결과물을 파일로 저장하려면 `workspace-write` 모드가 필요하다.

**Q12. 승인 요청이 너무 자주 뜨면 어떻게 하나요?**
`config.toml`에 `approval_policy = "never"` 를 추가하면 자동 진행된다. 단 삭제·덮어쓰기 작업에서도 승인 없이 실행되므로 주의. 처음엔 `on-request`(기본값)를 유지하는 걸 권장한다.

---

### 파일·데이터 작업

**Q13. CSV 파일이 너무 크면 Codex가 다 읽지 못하나요?**
맥락 창(context window) 한도 내에서 읽는다. 큰 CSV는 "2026년 1~3월 데이터만" 같이 범위를 줄여서 요청하거나, 열 이름을 먼저 물어보고 필요한 열만 분석하게 한다.

**Q14. 여러 파일을 한 번에 분석하게 할 수 있나요?**
`@파일1.csv @파일2.csv`처럼 여러 파일을 한 번에 지목할 수 있다. `--add-dir /추가폴더` 플래그로 폴더 전체를 추가하는 방법도 있다.

**Q15. 결과를 파일로 저장하게 할 수 있나요?**
`workspace-write` 모드에서 "결과를 report.md 파일로 저장해줘"라고 요청하면 된다. 승인 창이 뜨면 확인하고 진행한다.

---

### 비대화형·CI

**Q16. `codex exec`은 언제 쓰나요?**
반복 작업을 사람 없이 자동으로 돌릴 때다. 예: 매주 자동으로 매출 요약 파일 생성, CI/CD 파이프라인에서 코드 리뷰 자동화. 결과는 `>> output.txt`로 파일에 저장한다.

**Q17. `--yolo`와 `--ask-for-approval untrusted`는 다른가요?**
`--yolo`(또는 `--dangerously-bypass-approvals-and-sandbox`)는 승인 요청과 샌드박스를 모두 건너뛴다. 외부 격리 없이 파일·명령을 자유롭게 실행하므로, 테스트 폴더 외 실제 데이터에는 쓰지 않는다. `--ask-for-approval untrusted`는 신뢰할 수 없는 명령에만 승인을 요청하는 중간 모드다.

### CC↔Codex 관계

**Q18. CC와 Codex를 동시에 쓸 수 있나요?**
CC의 codex plugin(`/codex` 또는 `$codex`)으로 CC 세션 안에서 Codex를 불러 쓸 수 있다. CC가 "기획·지시"를 하고 Codex가 "특정 작업"을 실행하는 역할 분담이 가능하다. 자세한 내용은 cc-08 플러그인·스킬 레슨 참고.

**Q19. CC에서 배운 CLAUDE.md 규칙을 Codex AGENTS.md로 옮길 수 있나요?**
구조와 내용은 거의 그대로 옮길 수 있다. 파일명만 `AGENTS.md`로 바꾸고, CC 전용 기능(hooks, 특정 플러그인 경로 등)을 제외하면 된다.

**Q20. 언제 CC를 쓰고 언제 Codex를 쓰면 좋나요?**
명확한 정답은 없지만 지우 기준으로: CC는 "길게 대화하며 글 초안·보고서 작업", Codex는 "파일 분석·비대화형 자동화·다양한 접근면(앱·IDE)". 두 도구를 번갈아 쓰면서 본인 사업에 맞는 패턴을 찾는 것이 정답이다. codex-11에서 같은 일을 두 도구로 해보며 직접 비교한다.

## ⚡ 실용 팁 15선 — 혼자 일하는 사람을 위한 요령

**설치·세팅**

1. **`~/.codex/AGENTS.md` 하나로 전역 규칙 세팅**: "한국어로 답해줘", "표는 마크다운으로", "파일 삭제 전 항상 물어봐줘"를 한 번 써두면 모든 프로젝트에 자동 적용된다.
2. **`config.toml` 기본값 고정**: `model = "gpt-5.5"`, `model_reasoning_effort = "medium"`, `approval_policy = "on-request"` 를 써두면 매번 설정할 필요 없다.
3. **폴더 구조 먼저 보여주기**: 처음 세션을 시작할 때 "이 폴더 구조를 읽고 파일이 어떻게 구성되어 있는지 설명해줘"로 시작하면 이후 분석이 훨씬 정확해진다.

**프롬프트**

4. **목표 + 성공 기준을 함께**: "매출 분석해줘" 대신 "course_sales_monthly.csv에서 AI활용 카테고리 3개월 추이를 표로 보여줘. 성공 = 월별 수치 + 증감율 포함."
5. **"한 번에 하나씩"**: 분석·초안·저장을 한 번에 시키면 중간에 실수가 생긴다. 분석 → 확인 → 저장 순으로 단계를 나눈다.
6. **길이 제한 걸기**: 보고서 초안을 뽑을 때 "500자 이내로" 처럼 분량 기준을 주면 과도하게 긴 결과가 안 나온다.
7. **"틀리면 다시 시키면 된다"**: AI가 틀린 값을 냈을 때 당황하지 말고 "방금 결과에서 2026-03 클로드코드 순매출이 1,242인데 다시 확인해줘"처럼 구체적으로 교정한다.

**세션 관리**

8. **`/status`를 루틴으로**: 세션 시작 직후와 토큰이 많아졌다 싶을 때 `/status`를 습관으로 만든다.
9. **`/compact`는 마일스톤 후**: 한 작업이 끝난 시점(분석 완료, 초안 완성)에 `/compact`를 쓴다. 작업 중간에 쓰면 중요한 맥락이 날아갈 수 있다.
10. **비대화형으로 반복 작업 자동화**: 매주 `student_inquiries.csv`를 읽고 "진행중" 항목만 요약하는 작업이라면 `codex exec`으로 자동화해두면 매번 수동으로 안 해도 된다.

**안전**

11. **중요한 파일은 Git 백업 먼저**: Codex에게 파일 수정을 맡기기 전 `git commit`을 한 번 해두면 되돌리기가 쉽다.
12. **`--yolo`는 연습 폴더에서만**: ⚠️ `--yolo`(외부 격리 없음)는 처음에는 빈 폴더나 복사본에서 테스트하고, 충분히 익숙해진 뒤 실제 데이터에 쓴다.
13. **`read-only`로 분석 후 `workspace-write`로 저장**: 분석 결과가 맞는지 먼저 `read-only`로 확인하고, 파일로 저장할 때만 `workspace-write` 모드로 재시작한다.

**지우만의 팁 (실제 사업 맥락)**

14. **뉴스레터 데이터는 오픈율·클릭율 기준으로 분석**: `newsletter_stats.csv`에서 오픈율 55%+(21·24·26·29·30·32·33·36·37·39·40호) 또는 클릭율 20%+(21·30·33·36·39·40호)인 호수의 공통점을 찾으면 다음 호 기획 방향이 보인다.
15. **반복 문의는 AGENTS.md 자동 안내에 추가**: `student_inquiries.csv`에서 같은 질문이 3회 이상 나오면(WSL 설치·AGENTS.md 경로·Codex 로그인 문제가 실제로 반복됨) 그 내용을 AGENTS.md의 "자동 안내" 섹션에 추가해두면 다음에 같은 질문이 와도 빠르게 답할 수 있다.
