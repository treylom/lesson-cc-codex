# cc-06: 세션 관리 + 맥락 오염 + 비용 — "AI의 메모지가 가득 차기 전에"

> **레슨 목표**: AI와의 대화가 길어질 때 왜 느려지고·헷갈리고·비용이 오르는지 이해하고, `/compact`·`/clear`·`/context`·`claude --resume`으로 **세션을 직접 정리**해 본다. "맥락 오염"과 "토큰·비용"을 비개발자 눈높이로 잡는다.
> **예상 시간**: 45분 (개념 도입 8분 + 실습 1 / 2 각 15분 + 실습 3·마무리 7분)
> **사전 준비**: 터미널 1개 + Claude Code 설치·로그인 완료(cc-01·cc-02) + `practice-data/creator/` 폴더 + cc-03~cc-05까지 한 번씩 따라와 본 상태
> **진입 커맨드**: `/cc-06`

지우는 어제 하루 종일 터미널을 켜 놓고 일했습니다. 아침엔 강의 매출을 분석하고, 점심엔 고객 문의를 정리하고, 오후엔 다음 강의를 기획했죠. 그런데 저녁이 되니 이상했습니다. **AI 답이 점점 느려지고**, 매출 얘기를 하는데 자꾸 아까 정리한 고객 문의 내용을 섞어서 엉뚱한 소리를 합니다. "분명 똑똑했는데 왜 갑자기 정신이 없지?" — 이게 바로 오늘 우리가 잡을 문제입니다.

> 💡 **다른 트랙이라면?** 본문은 creator(콘텐츠·지식) 트랙의 지우 데이터로 진행합니다. store(스마트스토어)·freelancer(외주)·local(카페·공방·학원) 트랙을 골랐다면, 똑같은 단계를 **자기 트랙 폴더(`practice-data/<내 트랙>/`)의 파일로** 그대로 따라 하면 됩니다. 명령(`/context`·`/compact`·`/clear`)은 트랙과 무관하게 똑같이 동작합니다.

## 📚 학습 목표 (4가지 도달·와우 모먼트)

1. **AI에게도 "메모지 한계"가 있다** — 대화가 길어지면 AI의 기억 그릇(컨텍스트)이 차고, 차면 느려지고·헷갈리고·비싸진다는 걸 몸으로 안다.
2. **`/compact`는 "요약해서 이어가기", `/clear`는 "백지에서 새 출발"** — 둘의 차이를 정확히 구분하고, 언제 무엇을 쓸지 스스로 판단한다.
3. **"맥락 오염"을 직접 본다** — 매출 분석 중에 전혀 다른 주제를 섞었을 때 답이 어떻게 흐려지는지 실습으로 체감하고, `/clear`로 깨끗이 끊는다.
4. **비용·토큰이 무섭지 않아진다** — `/context`로 기억이 몇 % 찼는지 배터리 보듯 확인하고, "구독제 vs 종량제"·절약 습관을 알아 마음 편히 쓴다.

## 🎯 개념 도입 (8분) — AI의 "기억 그릇" 이야기

### 1. 컨텍스트 = AI가 한 번에 들고 있는 메모지

지금까지 우리는 AI에게 파일을 읽히고(cc-03), 계획을 세우게 하고(cc-04), 규칙서(`CLAUDE.md`)를 줬습니다(cc-05). 그런데 한 가지 안 짚고 넘어간 게 있어요. **AI는 우리가 나눈 대화를 "전부 손에 들고" 일합니다.**

> 💡 **비유 — 책상 위 메모지 한 장**
> AI가 일하는 모습을 떠올려 보세요. 책상 위에 **메모지 한 장**이 있고, 우리가 하는 말, AI가 한 답, 읽은 파일 내용이 전부 그 메모지에 적혀 나갑니다. AI는 답할 때마다 **그 메모지를 처음부터 끝까지 다시 읽습니다.** 메모지가 깨끗할 땐 빠르고 정확하죠. 그런데 하루 종일 적다 보면 메모지가 빽빽해지고, AI는 매번 그 빽빽한 종이를 다 읽느라 **느려지고**, 옛날에 적힌 엉뚱한 줄을 보고 **헷갈립니다.**

이 "메모지"의 정식 이름이 **컨텍스트(context, 배경 정보 그릇)** 이고, 메모지의 최대 크기를 **컨텍스트 윈도우(context window, 한 번에 기억할 수 있는 분량)** 라고 부릅니다.

### 2. 메모지가 차면 생기는 세 가지 문제

| 문제 | 무슨 일이 일어나나 | 지우의 하루에선 |
|------|-------------------|----------------|
| ① **느려짐** | 매번 긴 메모지를 다 읽어야 하니 답이 굼떠진다 | "아침엔 빨랐는데 저녁엔 한참 기다려야 답이 나와요" |
| ② **맥락 오염** | 옛날 다른 주제가 섞여 엉뚱한 답을 한다 | "매출 물었는데 자꾸 아까 본 고객 문의 얘기를 섞어요" |
| ③ **비용 증가** | 메모지가 길수록 한 번 답할 때 드는 '글자값(토큰)'이 커진다 | "쓴 만큼 내는 요금제면 대화가 길수록 돈이 더 나가요" |

여기서 **맥락 오염(context pollution)** 이라는 말이 처음 나왔죠. 어려운 말이 아닙니다. **"한 메모지에 서로 다른 일을 너무 많이 적어서, AI가 지금 무슨 일을 하는 중인지 헷갈리는 상태"** 예요. 책상 한 장에 장보기 목록·회의 메모·일기를 다 적어두면 나중에 뭐가 뭔지 모르게 되는 것과 똑같습니다.

### 3. 정리 도구 4총사 — 오늘 배울 것

다행히 메모지를 정리하는 도구가 있습니다. 오늘 네 개를 배웁니다. 앞의 셋(`/context`·`/compact`·`/clear`)은 **대화방 안에서 치는 슬래시 명령**이고, 마지막 `claude --resume`은 **터미널에서 대화방을 켤 때 쓰는 옵션**이라는 차이가 있어요.

| 도구 | 어디서 치나 | 한 줄 비유 | 언제 쓰나 |
|------|------------|-----------|----------|
| `/context` | 대화방 안 | **배터리 잔량 보기** — 메모지가 몇 % 찼나 확인 | 수시로, 정리하기 전에 |
| `/compact` | 대화방 안 | **메모지를 요약해 깨끗한 새 장으로 옮겨 적기** (핵심은 유지) | 한 작업 끝, 이어서 계속할 때 |
| `/clear` | 대화방 안 | **메모지를 버리고 완전 백지** (다 잊음) | 전혀 다른 일로 넘어갈 때 |
| `claude --resume` | 터미널(켤 때) | **어제 쓰던 메모지를 골라서 다시 꺼내기** | 며칠짜리 일을 이어서 할 때 |

> 핵심 한 줄: **`/compact`는 "정리하되 기억 유지", `/clear`는 "기억 삭제하고 새 출발".** 이 둘의 차이만 확실히 잡으면 오늘 절반은 끝납니다.

### 4. 새 세션 vs 이어가기 — 판단표 (출처: CC101 가이드 Sec11)

| 상황 | 권장 |
|------|------|
| 완전히 다른 주제로 넘어감 | **`/clear`** (또는 새 세션) |
| 같은 오류·같은 막힘이 계속 반복됨 | **`/clear`** (맥락이 오염돼 꼬인 것) |
| 대화가 길어졌지만 같은 작업을 이어감 | **`/compact`** 후 계속 |
| 오늘 끝낸 일을 내일 이어서 함 | **`claude --continue`**(직전 세션 바로 이어받기) 또는 **`claude --resume`**(여러 세션 중 골라 재개) |

> 📌 **표기 한 줄 정리**: `claude --continue`는 짧게 `claude -c`로도 칩니다(둘은 같은 명령). `--resume`은 "여러 과거 대화 중 골라서" 이어갈 때 쓰는 **터미널 옵션**입니다(대화방 안 슬래시 명령이 아니라, 대화방을 *켤 때* 붙이는 옵션이에요).

이제 직접 해 봅시다.

## 🔧 실습 1 — `/context`로 잔량 보고, `/compact`로 정리하기 (15분)

지우가 되어서, 강의 매출을 분석하며 메모지가 차는 걸 직접 보겠습니다. 천천히, 한 단계씩.

### 1단계: 사업 폴더에서 Claude Code 켜기

터미널을 열고 실습 데이터 폴더로 이동한 뒤 AI를 깨웁니다.

```text
cd ~/Documents/내사업/practice-data/creator
claude
```

> 폴더 경로는 여러분이 실습 데이터를 둔 위치에 맞춰 바꾸세요. 경로가 헷갈리면 `pwd`(현 위치)·`ls`(폴더 내용)로 확인하면 됩니다(cc-03 복습).

**기대 결과**: 입력창(`>`)이 뜨고, 화면 아래쪽에 작은 컨텍스트 게이지(예: `Context: 3%`)가 보입니다. 이게 우리의 "메모지 잔량"이에요.

**관찰 포인트**: 지금은 갓 켰으니 메모지가 거의 비어 있습니다(보통 한 자릿수 %). 여기서 출발해 점점 차오르는 걸 지켜볼 거예요.

### 2단계: `/context`로 지금 잔량 확인하기

대화방 안에서 슬래시 명령을 칩니다.

```text
/context
```

**기대 결과** (예시 — 환경마다 숫자는 다릅니다):

```
컨텍스트 사용량
─────────────────────────────
System prompt        2.1k tokens
CLAUDE.md            1.4k tokens
대화                  0.3k tokens
─────────────────────────────
합계  약 3.8k / 200k  (2%)
```

**관찰 포인트**:
- `CLAUDE.md`가 이미 메모지의 일부를 차지하고 있죠? cc-05에서 만든 규칙서도 AI가 매번 읽는 "기억"의 일부라서 그렇습니다. (그래서 규칙서를 너무 길게 쓰면 비용·속도에 영향이 있다는 걸 여기서 알 수 있어요 — 뒤 💡 비용 절약 팁에서 다시.)
- `%` 숫자를 기억해 두세요. 작업하면서 이게 오르는 걸 볼 겁니다.

### 3단계: 진짜 일 시켜서 메모지 채우기

이제 실제 매출 파일을 읽혀 분석시켜 메모지를 한가득 채워 봅니다. **검산이 가능하도록, 어떤 컬럼을 합산할지 프롬프트에 콕 집어줍니다.**

```text
@course_sales_monthly.csv 이 파일은 내 온라인 강의 월별 매출이야.
'매출(만원)' 컬럼을 강의별로 전부 더해서(2025-07~2026-03 누적),
누적 매출 상위 3개를 표로 정리하고,
각 강의가 어떤 흐름(상승/하락/등장)인지 한 줄씩 붙여줘.
```

> 💡 왜 컬럼을 지정하나? 이 파일에는 `매출(만원)`과 `순매출(만원)`(환불·쿠폰 뺀 값) 두 컬럼이 있습니다. **무엇을 합산하는지 말해주지 않으면 AI마다 다른 답**이 나올 수 있어요. "매출 컬럼을 더해줘"라고 못 박으면 우리가 직접 검산할 수 있습니다.

**기대 결과** (이 실습 데이터의 실제 값 기준 — `매출(만원)` 컬럼 합산):

```
강의별 누적 매출 (2025-07~2026-03, '매출(만원)' 컬럼 합산)
┌────────────────────────────────┬──────────────┬──────────────────────────┐
│ 강의명                          │ 누적 매출    │ 흐름                     │
├────────────────────────────────┼──────────────┼──────────────────────────┤
│ 챗GPT로 콘텐츠 10배 빠르게      │ 8,445만원    │ 8월 등장 후 폭발적 성장,  │
│                                 │              │ 2026-01 1,477만원 최고치  │
│ 1인 지식사업 수익화 로드맵      │ 5,975만원    │ 10월 등장, 객단가 149,000 │
│                                 │              │ 원으로 꾸준한 효자        │
│ 노션으로 1인 사업 시스템 만들기 │ 2,101만원    │ 초반 주력→점차 비중 축소  │
└────────────────────────────────┴──────────────┴──────────────────────────┘
```

> 🔎 **직접 검산해 보기**: 1등 챗GPT는 814+1011+1232+1122+774+1477+1059+956 = **8,445**(8개월). 2등 로드맵은 507+909+1147+1326+1073+1013 = **5,975**(6개월). 3등 노션은 374+490+427+347+463 = **2,101**(5개월). AI 답과 내 계산이 맞는지 직접 맞춰 보세요 — 이게 "AI 답을 그냥 믿지 않고 확인하는" 1인 사업자의 핵심 습관입니다.

**관찰 포인트**: 답이 길게 주르륵 떨어졌죠? 이만큼이 다 메모지에 적힌 겁니다. 한 번 더 일을 시켜 메모지를 더 채워 봅시다.

### 4단계: 한 번 더 — 환불·신규 강의까지

같은 세션에서 이어서 물어봅니다(맥락이 살아 있으니 "그 데이터"로 통합니다).

```text
방금 그 데이터에서 환불수가 가장 많았던 달과 강의를 찾아주고,
2026년에 새로 등장한 '클로드코드로 업무 자동화 입문' 강의의 2~3월 매출 흐름도 짚어줘.
```

**기대 결과** (실제 값 기준):

```
- 환불 최다: 2026-01 '챗GPT로 콘텐츠 10배 빠르게' — 환불 16건
  (수강신청 187건 중, 신청이 많은 달에 환불도 함께 늘어남)
- 신규 '클로드코드로 업무 자동화 입문':
  2026-02 매출 547만원(신청 46) → 2026-03 1,404만원(신청 118)
  → 출시 두 달 만에 2.5배 이상 급성장, 라인업 확장 신호
```

**관찰 포인트**: 두 번의 분석으로 메모지가 꽤 찼습니다. 다시 `/context`를 쳐서 확인해 보세요 — 아까 2~3%였던 게 눈에 띄게 올라 있을 겁니다(예: 12~20%대). **일을 할수록 메모지가 찬다**는 걸 숫자로 본 거예요.

### 5단계: `/compact`로 메모지 정리하기

매출 분석이라는 "한 덩어리 작업"이 끝났습니다. 같은 흐름(매출)을 이어갈 거라면, 여기서 메모지를 요약해 깨끗하게 만들 차례입니다.

```text
/compact
```

**기대 결과**:

```
✓ 대화를 요약했습니다.
  (이전 분석 결과의 핵심은 유지됩니다)
```

화면이 한결 깔끔해지고, `/context`를 다시 쳐 보면 **% 가 확 줄어 있습니다.** 그런데 중요한 건—

### 6단계: 기억이 유지됐는지 확인 (핵심 체험!)

`/compact` 직후, 방금 한 분석을 AI가 아직 기억하는지 떠봅니다.

```text
방금 정리한 매출 상위 3개 강의 중 1등이 뭐였지? 한 줄로만 답해줘.
```

**기대 결과**:

```
'챗GPT로 콘텐츠 10배 빠르게'(누적 8,445만원)가 1위였습니다.
```

**관찰 포인트** ⭐: 메모지를 정리(`/compact`)했는데도 **결론은 그대로 기억**하고 있죠? 이게 `/compact`의 핵심입니다 — **자잘한 대화 과정은 버리고 핵심만 요약해 이어가기.** "정리했지만 잊지는 않았다" — 이 느낌을 꼭 기억하세요.

> 💡 **언제 `/compact`를 누르면 좋나** (체크리스트)
> - [ ] 큰 작업 하나가 끝났을 때 ("매출 분석 끝! 이제 정리하고 다음으로")
> - [ ] **30~45분마다** 한 번씩, 또는 중간 목표를 달성한 직후
> - [ ] 답이 눈에 띄게 느려졌다고 느낄 때
> - [ ] 게이지가 가득 차기 **전에** 미리 (차고 나서가 아니라!)
> - ⚠️ **한창 생각·작업하는 중에는 누르지 마세요** — 흐름이 끊깁니다. "작업이 끝난 매듭"에서만.

## 🎬 실습 2 — "맥락 오염"을 직접 만들고 `/clear`로 끊기 (15분)

이번엔 일부러 **메모지를 더럽혀서** 무슨 일이 벌어지는지 본 뒤, `/clear`로 깨끗하게 끊는 걸 배웁니다. "오염을 직접 본다"가 이 실습의 목표예요.

### 1단계: 새 작업 시작 — 고객 문의 분석

실습 1의 매출 세션을 이어서, 같은 대화방에 **전혀 다른 주제**를 들이밉니다(일부러 오염을 만드는 중!).

```text
@student_inquiries.csv 이건 수강생 문의 내역이야.
'카테고리' 컬럼을 기준으로 몇 건씩 들어왔는지 세어주고, 가장 많은 3개를 알려줘.
```

**기대 결과** (실제 데이터 기준 — 총 24건):

```
문의 카테고리 집계 (총 24건)
1. 실습     — 7건  (터미널 설치, WSL, AGENTS.md, CLAUDE.md 등 따라하다 막힘)
2. 강의문의 — 5건  (비개발자도 되나요, 평생소장인가요 등)
3. 결제     — 4건  (강의가 안 열려요, 세금계산서, 무이자 할부 등)
   그 외: 수강방법 3건 · 환불 3건 · 수료증 2건
```

> 🔎 **직접 검산해 보기**: 24건 = 실습 7 + 강의문의 5 + 결제 4 + 수강방법 3 + 환불 3 + 수료증 2. 합이 딱 24가 되는지 확인해 보세요. ('환불'이 의외로 결제·수강방법과 같거나 적다는 점도 눈여겨볼 만합니다 — 막연한 인상과 실제 데이터가 다른 거죠.)

**관찰 포인트**: 좋은 답이 나왔습니다. 그런데 지금 이 메모지에는 **아까 매출 분석 내용 + 지금 문의 분석 내용**이 함께 쌓여 있어요. 슬슬 오염이 시작된 겁니다.

### 2단계: 또 다른 주제 — 뉴스레터까지 섞기

오염을 한 단계 더 강하게 만들기 위해, 같은 세션에 세 번째 주제를 넣습니다.

```text
@newsletter_stats.csv 이건 뉴스레터 성과야.
'오픈율(%)' 컬럼이 가장 높았던 호 3개만 알려줘.
```

**기대 결과** (실제 값 기준 — `오픈율(%)` 내림차순):

```
오픈율 상위 3개 호
1. 30호(2026-01-05) '2026년 1인 사업 트렌드 5가지' — 62.0%
2. 21호(2025-10-27) '챗GPT 프롬프트 모음집 무료 배포' — 61.1%
3. 36호(2026-02-16) '터미널이 무서웠던 내가 AI 비서를...' — 61.0%
```

> 🔎 **여기서 함정 하나**: 21호(61.1%)와 36호(61.0%)는 소수점 한 자리 차이라 헷갈리기 쉽습니다. **61.1 > 61.0**이니 21호가 2위, 36호가 3위예요. AI가 순서를 바꿔 답하면 바로잡아 줄 수 있어야 합니다 — 데이터를 직접 본 사람만 잡아낼 수 있는 오류죠.

이제 한 메모지에 **매출 + 문의 + 뉴스레터** 세 가지가 뒤섞였습니다.

### 3단계: 오염 확인 — 일부러 모호하게 물어보기

이제 오염된 메모지가 어떻게 헷갈리는지 봅니다. **일부러 두루뭉술하게** 물어보세요.

```text
그래서 환불이 제일 많은 거 뭐였지?
```

**기대 결과** (오염된 세션의 전형 — 둘 중 하나가 나옵니다):

```
어떤 환불을 말씀하시는지 다시 확인할게요.
- 매출 데이터(course_sales_monthly.csv) 기준: 2026-01 '챗GPT로 콘텐츠 10배...' 환불 16건
- 문의 데이터(student_inquiries.csv) 기준: '환불' 카테고리 문의 3건
어느 쪽을 말씀하신 건가요?
```

(또는 더 나쁘면, 묻지도 않고 둘 중 엉뚱한 걸 골라 답하기도 합니다.)

**관찰 포인트** ⭐: 이게 바로 **맥락 오염**입니다! 한 메모지에 서로 다른 일(매출 환불 16건 vs 문의 환불 3건)이 섞여서 AI가 "지금 무슨 환불 얘기지?" 하고 헷갈리는 거예요. 깨끗한 메모지였다면 바로 답했을 질문인데, 섞여 있으니 되묻거나 헤맵니다.

> 💡 두 가지 깨달음:
> 1. 한 세션에 너무 많은 주제를 섞으면 답 품질이 떨어진다.
> 2. 질문을 또렷하게 하는 것(cc-04 좋은 프롬프트)도 오염을 줄이지만, **주제 자체가 바뀌면 메모지를 비우는 게 근본 해결**이다.

### 4단계: 중요한 결과 먼저 저장하기 (`/clear` 전 안전장치!)

`/clear`는 메모지를 **통째로 버리는** 명령이라 되돌릴 수 없습니다. 그러니 지우고 싶은 게 있어도, **남기고 싶은 결과는 먼저 파일로 저장**하는 습관을 들이세요.

```text
지금까지 정리한 '문의 카테고리 상위 3개'와 '뉴스레터 오픈율 상위 3개'를
inquiry_newsletter_summary.md 라는 파일로 저장해줘.
```

**기대 결과**:

```
✓ inquiry_newsletter_summary.md 파일을 만들었습니다.
  (문의 카테고리 집계 + 뉴스레터 오픈율 Top3 포함)
```

**관찰 포인트**: 파일로 빠져나온 결과는 메모지를 비워도 안 사라집니다. `ls`로 파일이 생겼는지 확인해 보세요. **"`/clear` 전엔 저장부터"** — 이 순서를 몸에 익히는 게 이번 단계의 진짜 목적입니다.

### 5단계: `/clear`로 백지 만들기

이제 전혀 다른 일(예: 다음 강의 기획)로 넘어간다고 가정하고, 메모지를 완전히 비웁니다.

```text
/clear
```

**기대 결과**:

```
✓ 대화 기록을 모두 지웠습니다. 새 대화를 시작합니다.
```

`/context`를 쳐 보면 **거의 처음처럼 % 가 바닥**으로 떨어져 있습니다(`CLAUDE.md`·시스템 분량만 남음).

### 6단계: 정말 비워졌는지 확인 (`/compact`와의 결정적 차이!)

`/clear` 직후, 아까 한 분석을 기억하는지 떠봅니다.

```text
방금 정리한 문의 카테고리 1등이 뭐였지?
```

**기대 결과**:

```
앞선 대화 내용이 없어서 어떤 문의를 말씀하시는지 모르겠습니다.
어떤 파일을 분석할까요? (@파일명 으로 지정해 주세요)
```

**관찰 포인트** ⭐⭐: 실습 1의 `/compact` 뒤엔 "1등 기억해?"에 척척 답했는데, `/clear` 뒤엔 **"모르겠다"**고 하죠? 이게 두 명령의 **결정적 차이**입니다.

| | `/compact` | `/clear` |
|---|---|---|
| 메모지를 | **요약해서 이어 적음** | **버리고 백지** |
| 핵심 기억은 | **유지** | **삭제** |
| 언제 | 같은 작업 이어갈 때 | 다른 일로 넘어갈 때 |
| 되돌리기 | (요약이라 안전) | **불가 — 저장부터!** |
| 헷갈리면 | ✅ 이게 더 안전 | 확실히 끝났을 때만 |

> 💡 그래서 4번 판단표가 이렇게 정리됩니다: **"같은 일 계속 = `/compact` / 다른 일로 전환·꼬임 리셋 = `/clear` / 내일 이어서 = `claude --continue` 또는 `claude --resume`."**

## 📝 실습 3 (선택·시간 남으면) — 어제 일 이어가기 + 비용 감각 (7분)

### A. `claude --continue`·`claude --resume` — 어제 메모지 다시 꺼내기

지우는 며칠에 걸쳐 분기 보고서를 씁니다. 어제 하던 대화를 오늘 이어 하려면 두 가지 길이 있어요. 둘 다 **터미널에서 대화방을 켤 때** 붙이는 옵션입니다(대화방 안 슬래시 명령이 아니에요).

**길 1 — 가장 최근 대화 바로 이어 켜기**:

```text
cd ~/Documents/내사업/practice-data/creator
claude --continue
```

`--continue`(짧게 `-c`)는 "방금/어제 그 대화방 그대로 다시 열기"입니다. 고를 것 없이 직전 세션이 바로 열립니다.

**길 2 — 여러 과거 대화 중 골라 이어가기**:

```text
cd ~/Documents/내사업/practice-data/creator
claude --resume
```

`--resume`은 과거 대화가 여러 개일 때 **목록에서 골라** 이어가는 옵션입니다.

**기대 결과**: 과거 대화 목록이 뜨고, 화살표로 골라 엔터하면 그 메모지를 다시 펼칩니다.

```
이어갈 대화를 고르세요:
1) 2026-03-18  매출 분석 + 환불 정리      (12 메시지)
2) 2026-03-17  뉴스레터 오픈율 분석        (8 메시지)
3) 2026-03-15  콘텐츠 기획 브레인스토밍    (21 메시지)
```

**관찰 포인트**: "어제 하던 거 어디 갔지?" 할 필요가 없습니다. 다만 **너무 오래된 대화를 이어가면 그만큼 메모지가 무거운 상태로 시작**하니, 이어 켠 직후 대화방 안에서 `/compact` 한 번 해 주면 가뿐합니다.

> 📌 **헷갈리지 않기**: `--continue`·`--resume`은 **터미널에서 `claude` 옆에 붙이는 옵션**(대화방을 *켤 때*)이고, `/compact`·`/clear`·`/context`는 **이미 켜진 대화방 안에서 치는 슬래시 명령**입니다. 둘은 사는 곳이 달라요.

### B. 비용·토큰 — 무서워하지 않기 (출처: CC101 가이드 Sec19)

"대화가 길면 비싸진다"는 게 무슨 뜻인지, 1인 사업자 눈높이로 짚고 갑니다.

**토큰(token)이 뭔가요?** AI가 글자를 세는 단위입니다. 대략 **한국어 2~3글자 ≈ 1토큰**, 영어 4글자 ≈ 1토큰. 메모지에 적힌 글자 수라고 보면 됩니다. 메모지가 길수록 토큰이 많고, 종량제(쓴 만큼 내기) 요금이면 그만큼 비용이 오릅니다.

**두 가지 요금 방식**:

| 방식 | 어떻게 내나 | 1인 사업자에게 |
|------|------------|---------------|
| **구독제 (Pro/Max)** | 매달 정액, 토큰 비용 따로 없음 | 마음 편함 — 비용 걱정 없이 실습 가능 ⭐ |
| **종량제 (API 키)** | 쓴 토큰만큼 과금 | 자동화·대량 작업에 유리하나 비용 관리 필요 |

> 💡 지우 같은 1인 사업자는 보통 **구독제**가 마음 편합니다(cc-01 로그인에서 다룬 그 계정). 종량제는 뒤 파트의 자동화·대량 처리에서 다시 만납니다.

**비용·속도 절약 습관 5가지** (오늘 배운 것 + 가이드 정리):

1. **`/compact`를 습관화** — 가장 효과적. 긴 메모지를 줄여 속도·비용 둘 다 잡는다.
2. **파일·범위를 콕 집어주기** — "그 폴더 다 봐" 대신 `@course_sales_monthly.csv`처럼 지정하면 불필요한 읽기가 줄어 메모지가 덜 찬다(cc-03·cc-04 복습).
3. **단순 작업엔 가벼운 모델** — "회의록 3줄 요약" 같은 단순 일은 `/model`로 가벼운 모델(예: Haiku/Sonnet). 깊은 추론만 무거운 모델(Opus).
4. **주제가 바뀌면 `/clear`** — 옛 메모지를 끌고 다니지 않기.
5. **`CLAUDE.md` 다이어트** — 규칙서가 길면 매번 읽는 메모지도 무거워진다. 핵심만 간결하게(cc-05 연결). `/context`에서 `CLAUDE.md`가 차지하는 분량을 직접 보면 실감 납니다.

> 💡 비용 확인은 `/context`(지금 메모지 잔량)로 평소 감을 잡고, 환경에 따라 `/cost` 같은 명령이 있으면 누적 사용량도 볼 수 있어요(버전마다 다르니 `/help`로 확인).

## 🔍 흔한 오류와 해결법 — 트러블슈팅

| 상황 | 원인 | 해결 |
|------|------|------|
| 답이 점점 느려지고 빙빙 돈다 | 메모지(컨텍스트)가 가득 참 | `/compact`로 정리. 그래도 무거우면 저장 후 `/clear` |
| 매출 물었는데 자꾸 다른 주제를 섞는다 | 한 세션에 여러 주제 = 맥락 오염 | 주제별로 `/clear`해서 메모지 분리. 질문도 또렷하게 |
| `/compact` 했는데 하던 내용을 잊은 것 같다 | 작업 *도중*에 눌러 흐름이 끊김 | 매듭(작업 끝)에서만 누르기. 핵심은 다시 한 줄로 요약 요청 |
| `/clear` 했더니 중요한 결과가 사라졌다 | `/clear`는 되돌릴 수 없음 | **다음부터는 `/clear` 전에 파일로 저장**(실습2-4단계). 이번 건 파일 분석부터 다시 |
| 누적 매출 숫자가 AI 답마다 다르게 나온다 | `매출`과 `순매출` 중 무엇을 합산할지 안 정해줌 | 프롬프트에 **합산할 컬럼명**을 콕 집기("'매출(만원)' 컬럼을 더해줘") |
| `/context`가 늘 높게 시작한다 | `CLAUDE.md`가 너무 길다 | 규칙서를 핵심만 남겨 다이어트(cc-05). 불필요한 설명 줄이기 |
| 어제 대화가 안 보인다 | 다른 폴더에서 켰거나 새 세션 | 같은 폴더로 `cd` 후 `claude --continue`, 또는 `claude --resume`로 목록에서 찾기 |
| `/compact`·`/clear`가 안 먹는다 | 터미널에서 `claude /compact`로 친 것 | 먼저 `claude`로 대화방을 켠 **뒤** 그 안에서 `/compact` (슬래시 명령은 대화방 안에서만, cc-01 복습) |
| 비용이 걱정돼 실습이 망설여진다 | 종량제 오해 | 구독제(Pro/Max)면 정액이라 마음껏 실습해도 추가 비용 없음 |

> **마음가짐**: 메모지를 정리하다 실수해도 컴퓨터는 안 망가집니다. `/clear`만 "저장 먼저" 챙기면, 나머지는 마음껏 눌러 보며 감을 잡으세요. AI는 시험감독이 아니라 짝꿍입니다.

## 💡 핵심 규칙·개념 정리

1. **컨텍스트 = AI의 메모지.** 대화·답·읽은 파일이 다 적히고, AI는 매번 처음부터 다시 읽는다. 차면 ① 느려지고 ② 헷갈리고(맥락 오염) ③ 비싸진다.
2. **`/context` = 배터리 잔량 보기.** 정리하기 전에 수시로 확인하는 습관. (대화방 안 슬래시 명령)
3. **`/compact` = 요약해서 이어가기.** 핵심 기억은 유지. **작업이 끝난 매듭**에서, 30~45분마다. 작업 도중엔 ❌. (대화방 안 슬래시 명령)
4. **`/clear` = 백지에서 새 출발.** 기억을 통째로 삭제·되돌리기 불가 → **반드시 저장부터.** 주제가 완전히 바뀔 때·꼬임을 리셋할 때. (대화방 안 슬래시 명령)
5. **이어가기 = `claude --continue`(직전 바로) / `claude --resume`(골라서).** 며칠짜리 일에 쓰는 **터미널 옵션**(대화방을 켤 때). 이어 켠 직후 `/compact` 권장.
6. **비용은 토큰(메모지 글자값)에서 나온다.** 구독제면 정액이라 마음 편하고, 절약 5습관(`/compact`·파일 콕 집기·가벼운 모델·주제별 `/clear`·`CLAUDE.md` 다이어트)으로 속도·비용을 함께 잡는다.
7. **AI 답은 직접 검산한다.** 누적 매출·문의 건수·오픈율 순위처럼 숫자가 걸린 답은, 합산할 컬럼을 정해주고 내가 한 번 더 맞춰 본다. (8,445 / 5,975 / 2,101처럼 실제 데이터로 확인하는 습관)

> 한 문장 요약: **"같은 일은 `/compact`로 이어가고, 다른 일은 (저장 후) `/clear`로 끊는다. 메모지를 깨끗이 관리하면 AI는 더 빠르고·정확하고·저렴하게 일한다."**
