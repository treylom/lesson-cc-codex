## 🎯 개념 도입 (8분) — '내 작업실'이란 무엇인가

### 비유 먼저: 잘 정돈된 사무실 책상

지우가 처음 사업을 시작했을 때를 떠올려 봅시다. 매출 표는 노트북 바탕화면에, 고객 문의는 메일함에, 강의 기획은 메모 앱에, 리서치는 카톡 '나에게 보내기'에… **다 흩어져** 있었죠. 뭐 하나 찾으려면 여기저기 뒤져야 했습니다.

지금까지 우리도 도구를 하나씩 배웠습니다. `/cc-03`에서 파일 읽히는 법, `/cc-05`에서 `CLAUDE.md` 규칙서, `/cc-06`에서 `/compact`로 대화 정리하기. 그런데 이것들이 **따로 노는 도구 모음**처럼 느껴졌을 거예요.

이번 시간엔 이 조각들을 **하나로 묶습니다.** 그 묶음을 **워크스페이스(workspace, 우리말로 '작업실')**라고 부릅니다. 한 폴더 안에 — **내 자료**가 정리되어 있고, **AI가 지킬 규칙서**(`CLAUDE.md`)가 놓여 있고, **내가 자주 시키는 일**이 손에 익은 흐름으로 굴러갑니다.

> 💡 **비유**: 워크스페이스 = **잘 정돈된 내 책상**입니다.
> - 책상 서랍에 자료가 종류별로 정리돼 있고(→ **폴더 구조**),
> - 책상 위에 "이 일은 이렇게 하라"는 업무 수첩이 놓여 있고(→ **`CLAUDE.md` 규칙서**),
> - 자주 하는 일은 미리 만들어 둔 양식으로 빠르게 처리하고(→ **반복 작업 묶기**),
> - 일이 끝나면 결과물을 정해진 자리에 둡니다(→ **결과물 위치**).
>
> 옆자리엔 내가 일하는 방식을 다 아는 비서(AI)가 있어서, 매번 "나는 이런 사람이고, 이렇게 일해요"를 설명할 필요가 없어요. 책상에 앉기만 하면 일이 시작됩니다. 그게 워크스페이스의 핵심입니다.

### 작업실의 4가지 살림살이

복잡하게 외울 것 없습니다. 원본 가이드(CC101 Sec18)는 작업실을 네 가지로 나누는데, 우리가 이미 배운 것들이 그대로 작업실의 살림살이예요.

| 살림살이 | 쉬운 말로 | 어디서 배웠나 | 무슨 역할 |
|---|---|---|---|
| **폴더와 파일** | 내 자료가 쌓이는 곳 | `/cc-03` | AI가 읽을 매출·문의·기획 자료 |
| **CLAUDE.md** | AI에게 주는 업무 매뉴얼 | `/cc-05` | "나는 이런 사람, 이렇게 일해줘"를 적어 둔 규칙서 |
| **반복 작업 묶기** | 매번 똑같이 하는 일을 한 번에 부르기 | `/cc-08`(플러그인·스킬의 씨앗) | "매출 정리 → 요약 → 코멘트" 같은 단골 작업 |
| **세션 관리** | 대화 정리 습관 | `/cc-06` | `/compact`·`/clear`로 대화가 길어져도 가뿐하게 |

> 📌 **설계 철학 한 줄** (Sec18 원문): *"모든 규칙을 `CLAUDE.md`에 다 욱여넣지 마세요."* 1인 사업자에게 쉽게 풀면 — **규칙서는 짧고 또렷하게, 자주 쓰는 일은 손에 익혀 두고, 자료는 폴더에 정리.** 이 세 가지 균형이 좋은 작업실이에요. 한 곳에 다 몰아넣으면 AI도 헷갈리고 나도 헷갈립니다. 지금은 "한 폴더에 자료 + 규칙서 하나만 잘 둬도 충분하다"만 기억하면 됩니다.

### 왜 지금(코스 끝물에) 이걸 하나요?

`/cc-01`부터 `/cc-10`까지는 **도구 하나하나**를 배웠어요. 망치 쓰는 법, 톱 쓰는 법을 따로 배운 셈이죠. cc-11은 **그 도구들을 한 작업대 위에 올려놓고 "이제 같이 써 보자"** 하는 시간입니다. 그래서 새로운 게 거의 없고, **있던 걸 묶기만** 합니다. 처음 보는 명령어가 거의 없으니 부담 가지지 마세요.

조립이 끝나면 이게 그대로 **Part 2 Codex의 작업실**(규칙서 이름만 `AGENTS.md`로 바뀜)이 되고, **Part 3에서 '사업 두뇌(LLM Wiki)'의 씨앗**이 됩니다. 지금 정리하는 `creator/` 폴더가 코스 전체의 뿌리예요.

## 🔧 실습 1 — 흩어진 것을 '내 작업실' 하나로 합치기 (17분)

지우가 되어, `practice-data/creator/` 폴더를 "AI가 일하는 작업실"로 바꿔 봅니다. 자기 트랙(store/freelancer/local)이면 자기 폴더에서 **똑같이** 하면 됩니다 — 파일 이름이 같으니 그대로 따라오면 돼요. 한 단계씩, 천천히.

### 1단계: 지금 작업실에 뭐가 있는지 둘러보기

먼저 "지금 내가 어디 있는지" 확인하고, 자료가 있는 폴더로 들어간 뒤 안에 뭐가 있는지 봅니다. `/cc-03`에서 배운 `pwd`·`cd`·`ls`를 씁니다.

```text
pwd
```

```text
cd practice-data/creator
```

```text
ls
```

**기대 결과**: `pwd`는 지금 위치(예: `/Users/jiwoo/cc-codex-101`)를 보여주고, `cd`로 `creator` 폴더에 들어간 뒤, `ls`를 치면 파일들이 보입니다.

```
README.md                content_plan.md          course_sales_monthly.csv
consulting_schedule.md   newsletter_stats.csv     research_memo.md
student_inquiries.csv
```

**관찰/체크포인트**:
- 숫자 자료(CSV) 3개 + 글 자료(`.md`) 3개 + 안내(README) 1개 = 파일 7개. 이게 지우의 사업 자료 전부예요. 아직 "정리"는 안 된 상태 — 그냥 한 폴더에 다 들어 있죠.
- `cd` 했더니 "그런 폴더 없어요"가 뜬다면? → `ls`를 먼저 쳐서 `practice-data`가 보이는 폴더에 있는지 확인하세요. 길을 잃었으면 `cd ~`(홈으로 리셋)이 안전합니다. (망가지는 게 아니라 **그냥 안내문일 뿐**이에요.)

### 2단계: AI를 켜고 "이 책상에 뭐가 있는지" 진단시키기

이제 `claude`를 켜고, AI에게 폴더를 진단시킵니다. `/cc-03`에서 배운 "폴더 통째로 읽히기"를 작업실 정리 목적으로 씁니다.

```text
claude
```

대화창(`>`)이 뜨면 그대로 입력하세요.

```text
나는 온라인 강의·뉴스레터·컨설팅을 혼자 하는 1인 사업자 지우야. 비개발자고.
이 폴더(creator)를 "AI가 매번 같은 방식으로 일하는 작업실"로 정리하려고 해.
README.md 부터 읽어서 어떤 파일이 있고 각각 무슨 자료인지 한 줄씩 정리한 다음,
1) 자료를 어떻게 그룹으로 묶으면 좋을지 (매출 / 고객 / 기획 같은 식으로)
2) 결과물(분석 보고서 등)을 어디에 저장하면 깔끔할지
딱 이 두 가지만, 비개발자가 따라할 수 있게 짧게 제안해줘.
```

**기대 결과** (예시):

```
이 폴더(creator)의 파일들이에요:
- course_sales_monthly.csv : 강의별·월별 매출 (2025-07 ~ 2026-03)
- newsletter_stats.csv     : 뉴스레터 18~41호 오픈율·클릭율·구독자 수
- student_inquiries.csv    : 수강생 문의 내역 (채널·카테고리·처리상태)
- content_plan.md          : 2분기 콘텐츠 기획 노트
- research_memo.md         : AI 도구·시장 리서치 메모
- consulting_schedule.md   : 3월 컨설팅·발행 일정

이렇게 묶으면 좋겠어요:
- 📊 매출·성과 : course_sales_monthly.csv, newsletter_stats.csv
- 💬 고객      : student_inquiries.csv
- 📝 기획·운영 : content_plan.md, research_memo.md, consulting_schedule.md

결과물 저장:
- creator/reports/ 폴더를 새로 만들어 분석 보고서를 모으면 깔끔합니다.
  (예: reports/2026-03-월간성과.md)
```

**관찰/체크포인트**:
- 한국어로 물었더니 한국어로, 게다가 **내 파일 이름을 실제로 읽어서** 그룹을 제안했죠. "매출 표 2개", "고객 문의 1개"처럼 *내 자료에 맞춘* 정리예요. 이게 '내 책상을 아는 비서'의 첫 느낌이고, 작업실 설계의 첫걸음입니다.
- 파일 이름이 위와 달라 보여도 괜찮습니다 — AI가 실제 폴더를 읽은 그대로니까요.

### 3단계: 규칙서(`CLAUDE.md`) 초안 자동으로 만들기

이제 작업실의 핵심, **AI 업무 매뉴얼**을 만듭니다. `/cc-05`에서 배운 `/init`을 작업실 규칙서로 씁니다. 대화창 안에서:

```text
/init
```

**기대 결과**: Claude Code가 폴더를 슥 둘러보고 `CLAUDE.md`라는 파일의 **초안**을 만들어 줍니다. 화면에 "CLAUDE.md를 만들었어요" 비슷한 안내와 함께, 그 폴더에 파일이 하나 생깁니다.

**관찰/체크포인트**:
- `/init`은 **터미널의 `$`에서 `claude /init`이 아니라**, `claude`를 켜서 대화 중인 `>` 상태에서 `/`로 칩니다. (이거 하나만 구분하면 절반은 끝!)
- 파일을 만들기 전에 AI가 "만들어도 될까요?" 물어볼 수 있어요. 내용 보고 `yes` 하면 됩니다. 이런 **'고치기 전에 묻는'** 안전장치가 기본으로 깔려 있어요.

`/init`이 만든 초안은 일반적이에요. 여기에 **지우의 색깔**을 입힙니다. 이어서 입력하세요.

```text
- 분석에는 CSV의 실제 숫자를 근거로 인용한다 (지어내기 금지).
```

**기대 결과**: `creator/CLAUDE.md`가 위 항목들이 들어간 모습으로 저장됩니다. AI가 "저장했습니다"라고 알려줘요.

**관찰/체크포인트**:
- "통째로 갈아엎지 말고 이 항목들만"이라고 **범위를 못 박은** 데 주목하세요. 이게 `/cc-04`에서 배운 좋은 프롬프트(또렷한 요청)예요. 막연히 "잘 고쳐줘" 하면 엉뚱하게 다 바꿀 수 있어요.

### 5단계: 규칙서가 진짜 먹히는지 시험해 보기 (와우 모먼트)

규칙서를 만들었으니 **효과**를 봅시다. 일부러 짧고 막연하게 시켜 봅니다.

```text
course_sales_monthly.csv 보고 이번 분기 매출 흐름 알려줘.
```

**기대 결과**: "비개발자야, 쉽게 해줘, 표로 해줘, 끝에 다음 할 일 한 줄"을 **다시 말하지 않았는데도** 규칙서대로 답합니다. 예:

```
결론: AI활용 강의가 매출을 끌고 있고, 2026-01이 정점이었습니다.

| 강의 | 2026-01 매출 | 2026-03 매출 | 흐름 |
|------|-----------|-----------|------|
| 챗GPT로 콘텐츠 10배 빠르게 | 1,477만원 | 956만원 | 정점 후 하락 |
| 1인 지식사업 수익화 로드맵 | 1,326만원 | 1,013만원 | 완만한 하락 |
| 클로드코드로 업무 자동화 입문 | (없음) | 1,404만원 | 2월 출시→3월 급성장 |

다음 할 일: 새로 출시한 '클로드코드' 강의가 3월 단숨에 1,404만원으로 올라왔으니,
이 강의 마케팅에 힘을 싣는 걸 검토하세요.
```

**관찰/체크포인트**:
- 시키지도 않았는데 **결론이 맨 위**에 나왔죠? → 규칙서가 먹힌 증거.
- "다음 할 일" 한 줄이 자동으로 붙었죠? → '완료 기준' 규칙이 작동.
- **이게 작업실의 힘이에요.** 매번 "나는 비개발자야"를 반복하지 않아도, 규칙서가 대신 말해 줍니다. 같은 말을 두 번 안 하게 해 주는 비서 수첩.
- 숫자가 위 예시와 조금 달라도 정상입니다 — 진짜 CSV를 읽고 답하니까요. 다만 **"실제 CSV 값으로 근거를 보여줘"** 라고 한 번 확인하면 더 안심돼요. (AI가 지어내는 걸 막는 습관.)

### 6단계: 자주 하는 일을 '단골 요청'으로 묶기 (반복 작업의 씨앗)

지우는 매달 똑같이 "이번 달 매출 정리 → 전월 대비 → 한 줄 코멘트"를 합니다. `consulting_schedule.md`에도 "03-31 3월 마감 정산 + 분기 결산 시작"이 고정 일정으로 잡혀 있죠. 이런 반복은 **요청 문장 자체를 메모로 저장**해두면 매번 새로 안 짜도 돼요. (`/cc-08`에서 이걸 '스킬'로 정식화합니다 — 지금은 그 씨앗.)

대화창에서:

```text
내가 매달 반복하는 '월간 매출 정리' 요청을 재사용할 수 있게,
아래 형식의 메모를 creator/reports/_월간정리_템플릿.md 파일로 만들어줘:

# 월간 매출 정리 (매달 복붙용)
course_sales_monthly.csv에서 [이번 달]을 읽고:
1) 강의별 매출 표 (전월 대비 증감 화살표)
2) 환불수가 가장 많은 강의
3) 다음 달 한 줄 제안
이 셋만 표로 짧게.
```

**기대 결과**: `creator/reports/_월간정리_템플릿.md`가 생성됩니다. 다음 달엔 이 파일을 열어 `[이번 달]`만 `2026-04`로 바꿔 복붙하면 끝.

**관찰/체크포인트**: "반복 작업을 한 번 적어두고 재사용한다" — 이게 `/cc-08` 플러그인·스킬, 그리고 Part 4 자동화로 자라는 첫 단추예요. 지금은 단순 메모지만, 원리는 똑같습니다.

### 7단계: 작업실 한 바퀴 — 폴더 통째로 분석 + 저장 + 정리

작업실이 정리됐으니, 이제 **폴더 전체를 한 번에** 일 시킬 수 있습니다. `/cc-03`의 "폴더 통째 맡기기"를 작업실 위에서 돌리고, 결과를 저장하고, `/cc-06`의 세션 관리로 마무리합니다.

```text
이 작업실(creator)의 매출·뉴스레터·문의 파일을 전부 읽고,
"이번 분기에 내 사업에서 가장 중요한 변화 3가지"를 실제 숫자와 함께 뽑아줘.
규칙서에 적은 대로 결론 먼저, 표로.
```

**기대 결과** (예시):

```
결론: 'AI' 키워드가 매출·뉴스레터·신규강의 셋 다를 끌어올렸습니다.

| 변화 | 근거 (실제 숫자) |
|------|----------------|
| 1. AI 신규강의 급성장 | '클로드코드 업무 자동화' 2026-02 출시 547만원 → 2026-03 1,404만원 |
| 2. AI 주제 뉴스레터 폭발 | 33호 'AI에게 일을 시키는 사람 vs 못 시키는 사람' 오픈율 60%·클릭율 24.7% |
| 3. 사전등록 긴급 카피 효과 | 40호 '오늘 밤 12시 마감' 클릭율 30.3%(최고치) |

다음 할 일: 'AI로 일하기' 포지션이 통하고 있으니, 후속 AI 강의 라인업을 검토하세요.
```

이어서 결과를 파일로 남기고, 대화를 정리합니다.

```text
방금 뽑은 '이번 분기 중요한 변화 3가지'를 creator/reports/2026-1분기-요약.md 로 저장해줘.
```

저장 확인 후, 다음 일로 넘어갈 거면:

```text
/compact
```

**관찰/체크포인트**:
- 여러 파일을 **사람이 하나씩 열어 비교하지 않고** AI가 한 번에 묶어줬죠. 작업실이 정리돼 있으니 "이 작업실 전체 봐줘"라는 한 줄이 통하는 겁니다. 이게 cc-11에서 도달하고 싶은 진짜 그림이에요.
- **분석 → 저장 → 정리**가 작업실의 한 사이클입니다. 다음에 같은 폴더에서 `claude`를 켜면, 규칙서가 자동 적용된 상태로 바로 일을 이어갈 수 있어요.

> 💡 자기 트랙(store/freelancer/local)이면: 위 1~7단계를 **자기 폴더의 같은 이름 파일**로 그대로 하면 됩니다. 묶는 그룹과 규칙서 문구만 자기 업종에 맞게 바꾸세요. 배우는 원리는 100% 같습니다.

## 🎬 실습 2 — 막혔을 때 펼칠 '나만의 문제 해결 사전' 만들기 (트러블슈팅, 16분)

작업실을 다 꾸려도, 쓰다 보면 막힐 때가 있어요. 중요한 건 **막혔을 때 당황하지 않는 것**입니다. 원본 가이드(CC101 Sec13)의 흔한 문제 14개 중, 비개발자 1인 사업자가 실제로 만나는 7가지를 골라 **직접 겪어보고**, 작업실 안에 **사전 파일**로 정리해 둡니다.

> 🌱 **마음가짐 먼저**: 어떤 명령을 잘못 쳐도 컴퓨터는 안 망가집니다. CC는 파일을 바꾸기 전에 보통 물어보는 안전장치가 있고, AI 변경은 거의 다 되돌릴 수 있어요. **에러는 '망가짐'이 아니라 '안내'**입니다. 틀려도 괜찮으니 일단 해보세요 — AI는 비서이지 시험감독이 아니에요.

### 1단계: 가장 흔한 첫 벽 — `command not found` 미리 겪어 보기

지우의 수강생 중에도 이 문의가 있었죠 — `student_inquiries.csv`의 **S004(정유나)**가 *"터미널 설치하다 막혔어요 맥인데 명령어 입력하면 에러가 나요"*라고 문의했고, 처리상태는 '진행중', 소요 2시간이었습니다. 미리 겪어 두면 안 무섭습니다. 터미널(대화창 말고, 검은 일반 명령줄 `$`)에서 일부러 오타를 쳐 봅니다.

```text
claudd --version
```

**기대 결과**: `command not found: claudd` 같은 안내가 뜹니다. (`claude`를 `claudd`로 오타 낸 거예요.)

**관찰/체크포인트**:
- 보세요 — **아무것도 안 망가졌어요.** "그런 명령 없는데요?"라는 안내만 떴습니다. 터미널의 에러는 대부분 이렇게 **그냥 알려줄 뿐**입니다.
- **진짜 해결 순서** (위에서부터 하나씩): ① 철자 확인 → 진짜 명령 `claude --version`을 다시 치면 버전 숫자(예: `2.1.183`)가 나옵니다. ② 그래도 안 되면 터미널을 **완전히 껐다 다시 켜기**(설치 직후 가장 흔한 일시 현상, 이거 하나로 80% 해결). ③ `command not found: npm`까지 뜨면 토대인 Node.js가 없는 거예요 → `/cc-01`의 설치 단계부터 다시.

### 2단계: 실수로 파일을 건드렸을 때 — 되돌리는 법 (가장 중요한 안심)

"AI가 내 파일을 잘못 고치면 어쩌지?"가 제일 큰 걱정이죠. 되돌리는 법을 알면 그 걱정이 사라집니다. (되돌리기·중단은 `/cc-06`에서 세션·안전을 다룰 때 배운 기본기예요.) 대화창(`>`)에서 일부러 작은 변경을 시켜 봅니다.

```text
content_plan.md 맨 아래에 "## 테스트 줄 - 나중에 지움" 이라는 한 줄을 추가해줘.
```

AI가 추가하고 나면, 곧바로 되돌립니다. **키보드에서 `Esc` 키를 두 번** 누르거나, 대화창에 이렇게 칩니다.

```text
/rewind
```

**기대 결과**: 방금 한 변경을 **취소하고 직전 상태로 되돌릴지** 물어보는 화면이 나옵니다. 되돌리기를 고르면 추가했던 "테스트 줄"이 사라집니다.

**관찰/체크포인트**:
- **되돌릴 수 있다 = 마음 놓고 시도해도 된다.** 이게 이 레슨에서 가장 중요한 깨달음이에요.
- 되돌리는 3가지 안전망: ① `Esc` 두 번(진행 중인 작업 멈추기) ② `/rewind`(직전 변경 되돌리기) ③ 정말 중요한 자료는 **고치기 전에 복사본을 따로** 두기.
- AI는 보통 파일을 **고치기 전에 먼저 묻습니다.** 내용 안 보고 무조건 `yes` 하지 말고, 한 번 읽고 승인하는 습관만 들이면 됩니다.

### 3단계: 답이 느려지고 엉뚱해질 때 — 대화 정리하기

지우처럼 하루에 "매출 분석 → 문의 정리 → 기획"을 한 대화창에서 길게 다루면 답이 느려지고 갑자기 옛날 얘기를 섞어요. `/cc-06`에서 배운 정리 명령을 씁니다.

```text
/context
```

```text
/compact
```

**기대 결과**: `/context`는 지금 AI의 기억이 몇 % 찼는지(배터리 잔량 보듯) 보여 주고, `/compact`는 긴 대화를 **핵심만 남기고 요약**해 다시 가뿐하게 만듭니다. 하던 작업 맥락은 유지돼요.

**관찰/체크포인트**:
- `/compact`(정리하되 기억 유지) vs `/clear`(기억 통째로 삭제)를 헷갈리지 마세요. **헷갈리면 `/compact`가 더 안전**합니다(맥락을 안 잃으니까).
- `/clear`는 "매출 분석 끝 → 이제 완전히 다른 강의 원고 교정"처럼 **주제가 싹 바뀔 때만**. ⚠️ 되돌릴 수 없으니 중요한 결과는 **파일로 저장한 뒤**에 쓰세요(실습 1의 7단계처럼).

### 4단계: AI가 내가 원한 대로 안 고쳐요

`content_plan.md`의 '5. 미해결 고민' 항목만 다듬어 달라고 했는데, 엉뚱한 데를 손댑니다. 원인은 대개 **지시가 막연**해서예요. `/cc-04`의 '좋은 프롬프트'를 다시 씁니다 — ① 대상을 콕 집고(`@파일명`), ② 범위를 좁히고, ③ "먼저 제안만" 안전장치를 겁니다:

```text
@content_plan.md 의 '5. 미해결 고민' 섹션만 봐줘.
다른 부분은 절대 건드리지 말고, 그 섹션 문장만 더 명확하게 다듬는 안을
'먼저 제안'으로 보여줘. 내가 OK 하면 그때 고쳐.
```

**관찰/체크포인트**: "먼저 제안으로 보여줘" 한 마디가 **실수를 막는 안전벨트**예요. 우리 작업실 규칙서(`CLAUDE.md`)에 "확인 없이 통째로 덮어쓰지 말 것"을 이미 넣어뒀으니, 이게 이중으로 지켜줍니다.

### 5단계: 큰 변경이 예상될 땐 — Plan(계획) 모드

"전체 기획 노트를 다음 분기 버전으로 새로 써줘" 같은 큰 일은, 곧장 시키면 무섭습니다. `/cc-04`에서 배운 Plan 모드가 **여기서 안전장치로 다시 등장**합니다.

```text
(Shift+Tab → plan mode 확인 후)
content_plan.md를 2026년 3분기 버전으로 새로 구성해줘.
먼저 어떤 섹션을 어떻게 바꿀지 계획만 보여주고, 내가 OK 하면 진행해.
```

**관찰/체크포인트**: 대화창에서 키보드 **`Shift + Tab`**을 눌러 `plan mode`로 바꾸면(화면에 `plan mode` 표시 확인), AI가 **"①이렇게 ②저렇게 하겠습니다" 계획만 먼저** 보여주고, 내가 "진행해" 해야 실행합니다. 인테리어를 "알아서 다 해주세요"가 아니라 **도면 먼저 보고 OK** 하는 것과 같아요.

### 6단계: AI가 숫자나 프로그램을 '지어낼' 때 — 한 번 더 확인

AI는 똑똑하지만 가끔 "그럴듯하게 지어내요"(이걸 할루시네이션이라고 해요). 두 장면을 조심합니다.

**(가) 매출 숫자가 자꾸 다르게 나올 때** — 추측으로 채우는 중입니다. 실제 값으로 다시 시킵니다:

```text
course_sales_monthly.csv의 실제 값만 인용해서 다시 정리해줘. 추측으로 채우지 말고.
```

**(나) 처음 보는 프로그램을 깔라고 할 때** — `/cc-08`에서 플러그인을 배웠는데, AI가 **존재하지 않는 가짜 패키지**를 추천하고 나쁜 사람이 그 이름으로 악성 패키지를 올려두는 일이 드물게 있어요(이걸 '슬롭스쿼팅'이라 부릅니다). 깔기 전에 출처를 물으세요:

```text
방금 추천한 그 도구, 공식 출처(GitHub 주소나 만든 곳)랑
실제로 많이 쓰이는지(별점·다운로드)를 알려줘. 믿을 만한지 먼저 확인하고 싶어.
```

**관찰/체크포인트**: 1인 사업자는 IT 담당자가 따로 없으니 **내가 곧 보안 담당**입니다. "중요한 숫자·프로그램은 한 번 더 확인" — 이 습관 하나면 충분해요.

### 7단계: 7가지 막힘을 한 파일로 모아 두기 (내 사전 완성)

이제 위에서 겪은 것 + 자주 만날 것들을 **작업실 안 사전 파일**로 만들어 둡니다. 막히면 인터넷 뒤질 필요 없이 이 파일만 펼치면 돼요.

```text
"내문제해결사전.md" 라는 파일을 만들어줘. 1인 사업자인 내가 Claude Code 쓰다
자주 막히는 상황 7개를 "증상 / 왜 그런가 / 어떻게 푸나" 표로 정리해줘.
어려운 말 없이, 비개발자가 당황 안 하게 차분한 말투로.
아래 7개를 꼭 포함해줘:
1) command not found 가 떠요
2) 로그인이 자꾸 풀려요
3) 답이 느리고 이상해져요
4) AI가 내 파일을 엉뚱하게 고쳤어요
5) AI가 숫자를 지어낸 것 같아요
6) 한글이 깨져 보여요
7) AI가 추천한 프로그램, 깔아도 안전한가요?
```

**기대 결과**: `creator/내문제해결사전.md` 파일이 생기고, 이런 표가 들어 있습니다.

```
| 증상 | 왜 그런가 | 어떻게 푸나 |
|------|----------|-------------|
| command not found | 철자 오타이거나 설치 직후 터미널이 못 찾는 중 | 철자 확인 → 터미널 껐다 켜기 → 안 되면 재설치(/cc-01) |
| 로그인이 자꾸 풀려요 | 인증이 만료됐거나 꼬임 (S022 수강생 사례) | 대화창에서 /login 다시, 안 되면 Ctrl+C로 껐다 claude 재시작 |
| 답이 느리고 이상해요 | 대화가 너무 길어 기억 그릇이 꽉 참 | /context 확인 → /compact 정리, 주제 바뀌면 /clear |
| 파일을 엉뚱하게 고침 | 요청이 막연해 범위가 넓었음 | Esc 두 번 / /rewind 로 되돌리기, 다시 시킬 땐 @파일명+범위 좁혀서 |
| 숫자를 지어낸 것 같아요 | AI가 추측으로 채움 (할루시네이션) | "CSV의 실제 값으로 근거 보여줘" 라고 다시 요청 |
| 한글이 깨져 보여요 | 글자 표기 설정(UTF-8) 문제 | LANG/LC_ALL을 ko_KR.UTF-8로, 안 되면 코스 안내/검색 |
| 추천 프로그램 안전한가요? | AI가 없는 프로그램 이름을 댈 수 있음(슬롭스쿼팅) | 깔기 전에 "출처·사용량 확인해줘" 요청, 유명한 것만 |
```

**관찰/체크포인트**:
- 2번(로그인 풀림)은 지우의 수강생 **S022(백서윤)**가 *"Codex 설치할 때 로그인이 자꾸 풀려요"*라고 실제로 문의한 케이스예요(진행중). 5번(숫자 지어내기)과 7번(없는 프로그램 추천)은 **AI를 쓸 때 늘 의식할 두 가지**고요.
- 이 사전이 **작업실 안에** 있다는 게 핵심입니다. `student_inquiries.csv`를 보면 S004(터미널 에러)·S013(WSL 설치)·S015(CLAUDE.md 안 먹힘) 같은 진짜 막힘들이 줄줄이 있죠 — 지우가 수강생에게 답할 때도, 자기가 막힐 때도 이 한 파일이 든든한 안내서가 됩니다.

> 💡 우리 작업실 규칙서(`CLAUDE.md`)에 미리 규칙을 넣어두면 위 문제의 절반(원치 않는 수정·실명 노출·장황함)은 **애초에 안 생깁니다.** 트러블슈팅은 결국 "좋은 작업실 + 좋은 프롬프트"로 줄어들어요. 그리고 어떤 상황도 **치명적이지 않습니다** — 터미널 에러는 거의 다 '안내문', AI 변경은 거의 다 '되돌리기' 가능.

## 💡 핵심 규칙·개념 정리

1. **워크스페이스(작업실) = 규칙서(CLAUDE.md) + 폴더 구조 + 반복작업 묶기 + 세션 관리.** 새로 배우는 게 아니라 cc-01~cc-10을 **조립**한 것.
2. **규칙서는 같은 말을 두 번 안 하게 해 준다.** "나는 비개발자야, 쉽게, 표로"를 한 번 적어 두면 매번 자동 적용 — 단, 다 욱여넣지 말고 핵심만 짧게(Sec18 설계 철학).
3. **막힘은 사전으로, 실수는 되돌리기로.** 흔한 7가지는 `내문제해결사전.md`로, 실수는 `Esc` 두 번·`/rewind`로. 90%는 "터미널 껐다 켜기 / `/compact` / 범위 좁히기 / 되돌리기" 안에서 해결.
4. **AI는 가끔 지어낸다 — 중요한 숫자·프로그램은 한 번 더 확인.** "실제 값으로 근거 보여줘", "이 프로그램 진짜 있는지 확인해줘." 안전은 막연한 불안이 아니라 **규칙서로 통제하는 것**.
5. **이 코스는 끝이 아니라 시작.** 지금 만든 작업실이 Part 2(Codex, `AGENTS.md`) → Part 3(지식 위키) → Part 5~6(AI 에이전트 팀)으로 자란다.
