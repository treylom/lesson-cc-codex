# codex-02: CLI 사용법 + 슬래시·단축키·$스킬 호출 — "버튼 하나로 반복 작업을 끝낸다"

> 🔎 **이 레슨에서 나오는 명령·모델명(gpt-5.x)·플래그는 codex 0.140.0 실측 기준입니다. 실제 Codex 0.140.0 실측 + 공식 문서(`developers.openai.com/codex`) 검증 완료.**

> **레슨 목표**: Codex CLI의 핵심 조작 패턴을 손에 익히고, 슬래시(`/`) 명령과 `$` 스킬 호출의 차이를 이해한 뒤, 지우의 실제 사업 데이터를 Codex와 함께 분석한다.
> **예상 시간**: 55분 (도입 5분 + 실습 1 20분 + 실습 2 25분 + 실습 3 선택 5분)
> **사전 준비**: codex-01 완료 (Codex 설치·`codex login` 완료), `practice-data/creator/` 폴더 접근 가능, 터미널 창 1개
> **진입 커맨드**: `/codex-02`

## 📚 학습 목표 — 4가지 도달/와우 모먼트

1. **`codex --cd`로 내 폴더를 건네는 순간 AI가 직접 일한다** — "이 폴더 봐줘" 한 마디로 파일을 읽고 분석하는 체감이 생긴다.
2. **슬래시(`/`) 명령으로 세션 중 방향을 내가 조정한다** — `/plan`·`/status`·`/compact`가 어떤 순간에 필요한지 손에 익는다.
3. **Codex의 `/skills 또는 $스킬이름`이 CC의 `/스킬이름`과 기호만 다른 이유를 이해한다** — "반복 작업 묶음"이라는 개념까지.
4. **`@파일명`으로 특정 파일을 정확히 집어내면 답이 달라진다** — `course_sales_monthly.csv`·`newsletter_stats.csv`·`student_inquiries.csv` 실제 수치로 인사이트를 뽑아낸다.

## 🎯 개념 도입

### Codex CLI, 일상 조작은 딱 세 가지 패턴

codex-01에서 Codex를 설치하고 첫 실행을 해봤다. 이제 "어떻게 조작하나"를 배울 차례다.

결론부터 말하면, Codex CLI의 일상 조작은 세 가지 패턴으로 돌아간다.

```
① 시작할 때    →  codex --cd ~/폴더이름  "지시문"
② 대화 중에    →  /슬래시명령             (세션 관리)
③ 스킬 부를 때 →  /skills 또는 $스킬이름 (묶음 작업 호출)
```

이 세 패턴만 손에 익히면 이번 레슨은 성공이다.

---

### 비유: Codex는 "작업실에 상주하는 비서"

예전 방식: 파일을 열고 → 내용을 복사하고 → 채팅창에 붙여넣고 → 답을 받고 → 다시 파일에 옮기고 → 다음 파일 또 복사…

Codex 방식: 터미널에서 "이 폴더 봐줘"라고 한 번 말하면, Codex가 **직접 폴더에 들어가 파일을 열어 읽고**, 분석하고, 결과물까지 만들어 준다.

비서를 고용했는데, 그 비서가 내 작업실(터미널)에 **상주**하게 된 것. 매번 서류를 출력해서 갖다 주는 대신, "저쪽 서랍 두 번째 칸 파일 꺼내봐"라고 말하면 알아서 꺼내 보는 비서.

이때 `--cd 폴더이름`이 "이 서랍에서 일해"라는 지시다. CD는 "change directory(폴더 이동)"의 줄임말로, 터미널 세계에서 "지금부터 이 폴더를 기준으로 일해"를 뜻한다.

---

### CC vs Codex: 조작 기호 비교

Claude Code(CC)를 먼저 배웠으니, 나란히 보자.

| 하고 싶은 것 | Claude Code (CC) | Codex | 핵심 차이 |
|---|---|---|---|
| 시작할 때 | `claude` | `codex` | 도구 이름만 다름 |
| 폴더 지정 | CC 실행 후 폴더 이동 | `--cd ~/폴더` | Codex는 시작부터 지정 가능 |
| 세션 중 명령 | `/명령어` | `/명령어` | **동일** — 슬래시는 같다 |
| 스킬 호출 | `/스킬이름` | **`/skills 또는 $스킬이름`** | ← 오늘의 핵심 차이 |
| 특정 파일 지목 | `@파일명` | `@파일명` | **동일** |
| 대화 정리 | `/compact` | `/compact` | **동일** |
| 계획 먼저 보기 | `/plan` | `/plan` | **동일** |

**한 줄 요약**: "슬래시 세션 명령은 같고, 스킬 호출은 `/skills` 또는 달러(`$`)를 쓴다."

---

### `$스킬` vs `/슬래시명령` — 왜 다른가

Codex에서 기호를 두 가지 맥락으로 쓴다.

- **슬래시(`/`)**: 세션 *도중에 쓰는 설정·관리 명령.* `/plan`·`/status`·`/compact`·`/model` 같은 것. 리모컨 버튼처럼 — TV를 보다가 채널 바꾸고, 볼륨 줄이는 것처럼 대화를 멈추지 않고 설정을 바꾸는 도구다.
- **달러(`$`)**: 미리 만들어 둔 *스킬(반복 작업 묶음)을 부르는 기호.* CC에서는 `/스킬이름`이었는데, Codex에서는 `/skills 또는 $스킬이름`으로 부른다.

**비유**: 슬래시(`/`)가 "메뉴판에서 그때그때 하나를 고르는 것"이라면, 달러(`$`)는 "항상 시키는 단골 세트"를 한 번에 부르는 것이다.

지우가 매달 똑같이 하는 일이 있다고 생각해보자.

```
매달 해야 하는 일:
1. course_sales_monthly.csv 읽기
2. 강의별 순매출 정리
3. 전월 대비 증감 계산
4. "다음 달 제안" 한 줄 코멘트 추가
```

이 4단계를 `$매월매출정리`라는 스킬로 한 번 만들어두면, 다음 달부터는 `/skills 또는 $매월매출정리` 한 줄로 끝난다. 4단계를 매번 하나씩 시킬 필요가 없다.

> ⚠️ **스킬 등록·관리의 상세 방법은 codex-09에서 배운다.** 이번 레슨에서는 "/skills 또는 $기호로 스킬을 부른다"는 개념에 집중한다.

---

### 슬래시(`/`) 명령 한눈에 보기

세션 안에서 자주 쓸 명령들만 추려두었다.

| 명령 | 언제 쓰나 | CC와 같은가 |
|---|---|---|
| `/plan` | "일 시작 전에 계획 먼저 보여줘" | ✅ 같음 |
| `/status` | "지금 모델·토큰·샌드박스 상태 확인" | ✅ 개념 같음 |
| `/model` | "모델·생각 강도 바꿔" (codex-03에서 상세히 배운다) | ✅ 개념 같음 |
| `/permissions` | "내 권한 확인·조정" (codex-05에서 상세히 배운다) | ✅ 개념 같음 |
| `/compact` | "대화가 너무 길어졌으니 요약해" | ✅ 같음 |
| `/init` | "이 폴더에 AGENTS.md 뼈대 만들어줘" | CC 초기화와 짝 |
| `/review` | "방금 수정한 코드·파일 변경을 리뷰해줘" (코드/diff 리뷰 전용) | 개념 같음 |

> 💡 **`/compact` 주의**: CC와 똑같이, **작업 하나가 완전히 끝난 다음에만** 쓴다. 한창 분석하는 중에 누르면 흐름이 끊긴다. "오늘 할 일 다 끝냈는데, 대화가 너무 길어졌네" 싶을 때 쓰는 것이다.

## 🔧 실습 1 — CLI 기본 조작 익히기 (20분)

### 1단계: Codex 실행 + 폴더 지정

터미널을 열고, `practice-data/creator` 폴더를 Codex에게 건네준다.

```bash
codex --cd ~/practice-data/creator "이 폴더에 어떤 파일이 있고, 각 파일이 무엇인지 한 줄씩 설명해 줘"
```

> 📌 **`~/practice-data/creator`는 실제 폴더 경로로 바꿔야 한다.** 경로를 모르겠으면 터미널에서 `pwd` 입력 → 현재 위치 확인 → 거기서 `creator` 폴더까지 경로를 이어 붙이면 된다.

**기대 결과 (예시)**:

```
이 폴더에는 7개 파일이 있습니다.

CSV 데이터:
- course_sales_monthly.csv — 강의별·월별 매출 데이터 (2025-07 ~ 2026-03)
- newsletter_stats.csv — 뉴스레터 호별 발송·오픈율·클릭율 성과 (18호~41호)
- student_inquiries.csv — 수강생·고객 문의 내역 및 처리 현황

문서:
- content_plan.md — 2분기 콘텐츠 기획 노트
- research_memo.md — AI 도구·1인 사업 시장 리서치 메모
- consulting_schedule.md — 3월 컨설팅·강의 운영 일정 노트
- README.md — 폴더 안내 문서

추천 분석 순서:
1. course_sales_monthly.csv → 어느 강의가 얼마나 팔렸는지 큰 그림 파악
2. newsletter_stats.csv → 어떤 주제가 독자에게 통했는지
3. student_inquiries.csv → 수강생이 어디서 막히는지
→ 세 파일을 연결하면 "무엇을 더 만들까"라는 다음 강의 기획 질문에 답할 수 있습니다.
```

**관찰/체크포인트**:
- Codex가 직접 폴더를 열어 파일을 나열했는가? 파일명이 실제와 일치하는지 대조해 보자 ✓
- 각 파일의 역할 설명이 있는가? README.md를 읽은 결과가 반영됐는가? ✓
- 단순 나열이 아니라 "이 순서가 좋다"는 제안이 있는가? — 이게 있으면 Codex가 에이전트처럼 일하고 있다는 신호 ✓

### 2단계: `/plan` — 일 시작 전에 계획 먼저 보기

Codex에게 복잡한 작업을 시킬 때, 바로 결과를 내기 전에 계획을 먼저 보여달라고 할 수 있다. 낯선 AI에게 일을 맡기기 전에 "어떻게 할 건지"를 확인하는 습관이 좋다.

```text
/plan

course_sales_monthly.csv 분석해서 매출이 가장 많이 오른 강의와 가장 많이 떨어진 강의를 찾고, 이유를 한 줄씩 붙인 요약 파일 만들어줘
```

**기대 결과 (예시)**:

```
계획을 먼저 보여드릴게요:

1단계: course_sales_monthly.csv 파일 읽기 (읽기 전용, 수정 없음)
2단계: 강의별·월별 순매출 집계
3단계: 상승/하락 트렌드 계산 (첫 달 대비 마지막 달)
4단계: 상위 1개, 하위 1개 선택 + 이유 가설 작성
5단계: 요약 결과를 sales_summary.md 파일로 저장

계속할까요? (yes/no)
```

**"yes"를 입력하면 Codex가 5단계를 순서대로 실행한다.**

**관찰/체크포인트**:
- Codex가 결과를 바로 내지 않고 계획부터 보여줬는가? ✓
- "파일 읽기(수정 없음)"처럼 안전한 단계와 "파일 저장"처럼 주의가 필요한 단계를 구분하는가? ✓
- "잠깐, 4단계는 이렇게 바꿔줘"라고 말하면 계획을 고쳐주는가? ✓

> 💡 **왜 `/plan`이 좋은가?** 처음부터 결과물을 만들게 했다가 "이게 아닌데…"라고 하면 다 되돌려야 한다. 계획을 먼저 보고 수정하면, 작업 방향이 틀어지는 걸 초입에 잡을 수 있다. CC의 Plan 모드(cc-04)와 같은 정신이다.

### 3단계: `@파일명` — 특정 파일을 정확히 집어내기

"이 폴더 전체를 봐"가 아니라, 딱 하나의 파일만 정확히 지목하고 싶을 때 `@`를 쓴다.

```text
@newsletter_stats.csv 이 파일에서 오픈율이 가장 높았던 호 Top 3와, 가장 낮았던 호 Bottom 2를 뽑아줘. 각각 어떤 제목이었는지도 함께 알려줘.
```

**기대 결과 (예시)**:

```
newsletter_stats.csv 분석 결과:

오픈율 Top 3:
1. 30호 (2026-01-05): "2026년 1인 사업 트렌드 5가지" — 오픈율 62.0%
2. 21호 (2025-10-27): "챗GPT 프롬프트 모음집 무료 배포" — 오픈율 61.1%
3위 동률 (두 호 모두 60.0%):
   - 32호 (2026-01-19): "클로드코드라는 게 뭔가요?" — 오픈율 60.0%
   - 33호 (2026-01-26): "AI에게 일을 시키는 사람 vs 못 시키는 사람" — 오픈율 60.0%

오픈율 Bottom 2:
1. 27호 (2025-12-08): "내년 콘텐츠 캘린더 미리 짜기" — 오픈율 42.0%
2. 34호 (2026-02-02): "설 연휴에 읽기 좋은 글 모음" — 오픈율 42.0%
```

> 📌 **AI 응답 확인 포인트**: 32호(클로드코드라는 게 뭔가요?)와 33호(AI에게 일을 시키는 사람 vs 못 시키는 사람)는 오픈율이 모두 60.0%로 동률 3위다. AI가 이 두 호를 함께 3위로 언급하거나, 또는 한 호만 보여줄 수도 있다. 실제 파일을 열어 비교해보면 두 호가 동률임을 직접 확인할 수 있다. "AI 답과 파일이 조금 다르네?"라고 느끼면 — 그게 바로 검증하는 습관의 시작이다.

**관찰/체크포인트**:
- `@파일명`을 붙이자 정확히 그 파일만 집중해서 읽는가? ✓
- 실제 파일의 숫자(62.0%·61.1%·60.0% 등)를 인용하는가? — 지어낸 숫자가 아닌지 파일을 직접 열어 확인해 보자 ✓
- 동률 호(32호·33호)를 AI가 어떻게 처리하는지 관찰하자 — 두 호를 모두 언급하면 완전한 답, 한 호만 보여주면 "혹시 같은 오픈율인 호가 더 있나요?"라고 물어봐도 좋다 ✓
- "파일 전체를 막연히 봐줘"보다 답이 훨씬 정확한가? ✓

---

### 4단계: `/status` — 세션 상태 확인하기

작업을 몇 개 하고 나면 "지금 어떤 모델이 돌고 있지?", "얼마나 썼지?" 궁금해진다.

```text
/status
```

**기대 결과 (예시)**:

```
현재 세션 상태:
- 모델: gpt-5.5
- 추론 강도(effort): medium
- 토큰 사용량: 약 2,800 tokens
- 샌드박스 모드: workspace-write (내 폴더 읽기·쓰기 허용, 인터넷 차단)
- 승인 정책: on-request (모델이 필요하다 판단할 때만 확인 요청)
```

> 📌 **gpt-5.5와 effort에 대해**: `/status`에 모델 이름과 추론 강도(effort)가 표시된다. "저걸 바꿀 수 있나?"라는 궁금증이 생기는 게 자연스럽다 — 다음 레슨(`/codex-03`)에서 이 두 가지를 직접 바꿔보고 응답이 어떻게 달라지는지 체험한다. 지금은 "현재 설정을 확인하는 명령"이라는 것만 기억하면 된다.

**관찰/체크포인트**:
- 현재 어떤 모델이 돌고 있는지 확인됐는가? ✓
- 토큰(AI가 한 번에 기억하는 분량)이 얼마나 쓰였는지 ✓
- 샌드박스 모드가 어느 단계인지 (read-only / workspace-write / danger-full-access 중 하나 — codex-05에서 상세히 배운다) ✓

## 🔍 흔한 오류와 해결법

| 상황 | 원인 | 해결 |
|---|---|---|
| `codex: command not found` | 설치됐지만 터미널이 인식 못함 | 터미널을 완전히 닫고 다시 열기 → `codex --version` 재시도 |
| `--cd` 뒤 경로 오류 | 경로가 실제 폴더 위치와 다름 | `pwd`로 현재 위치 확인 → `ls` 로 폴더 실제 내용 확인 → 경로 수정 |
| `/plan` 입력해도 반응 없음 | `/`로 시작하는 명령은 세션 **안**에서만 작동 | `codex --cd 경로` 실행해서 세션을 먼저 열고, 그 안에서 입력 |
| `@파일명` 했는데 파일을 못 찾음 | 파일이 `--cd`로 지정한 폴더 안에 없거나 철자 오류 | 파일 이름 철자 확인 + `ls`로 폴더 내용 재확인 |
| AI가 지어낸 숫자를 말함 | 파일을 못 읽고 상상으로 답함 | `@파일명`으로 명확히 지목 후 "CSV에 있는 실제 값만 인용해줘"라고 다시 요청 |
| AI가 Top 3를 한 호만 보여줌 | 동률인 호들 중 하나만 선택해서 보여줌 | "같은 오픈율인 호가 더 있나요?"라고 확인 질문 → AI가 전체 목록을 보여줌 |
| 파일 저장 전에 물어보지 않음 | 샌드박스 승인 정책에 따라 다름 | `/status`로 현재 모드 확인, 다음 실행 시 `--ask-for-approval untrusted` 옵션 추가 (신뢰하지 않는 명령은 반드시 물어봄) |
| `/compact` 후 이전 분석 내용을 모름 | 대화가 압축되면 상세 내용 일부 소실 | 중요한 결과는 파일로 먼저 저장하고 `/compact` 실행 |
| 답이 너무 길고 장황함 | 기본 effort 설정이 높거나 질문이 모호함 | "핵심만 5줄로" 추가 요청, 또는 질문을 더 구체적으로 |
| 로그인이 자꾸 풀림 | ChatGPT 계정 세션 만료 | `codex login`으로 재로그인 |

## 💡 핵심 규칙 정리

### 규칙 1: `--cd`로 폴더를 건네면 일이 시작된다

```bash
# ❌ 그냥 codex만 치면 어느 폴더인지 모른다
codex "course_sales_monthly.csv 분석해줘"

# ✅ --cd로 폴더를 먼저 알려주면 AI가 직접 파일을 연다
codex --cd ~/practice-data/creator "course_sales_monthly.csv 분석해줘"
```

한 번 지정하면 그 세션 동안은 "내 파일들"을 계속 쓸 수 있다.

---

### 규칙 2: 세션 중 조작은 슬래시, 스킬은 달러

```
# 세션 중 조작 (대화 관리·설정 변경)
/plan      — 계획 먼저 보기
/status    — 현재 상태 확인
/compact   — 대화 정리 (작업 끝난 뒤에만!)
/model     — 모델 바꾸기 (codex-03에서 배움)

# 스킬 호출 (반복 작업 묶음)
/skills 또는 $스킬이름  — 미리 만든 작업 묶음 실행
```

CC에서 `/스킬이름`이었던 것이 Codex에서는 `/skills 또는 $스킬이름`으로 부른다. 슬래시 세션 명령은 두 도구가 같다.

---

### 규칙 3: `@파일명`을 쓰면 답 품질이 올라간다

```text
# ❌ 막연하게
"이 폴더 파일 분석해줘"  →  AI가 어느 파일을 봐야 할지 추측

# ✅ 정확하게
@newsletter_stats.csv "오픈율 Top 3 뽑아줘"  →  그 파일만 정확히 읽고 답
```

파일을 여러 개 엮을 때도 `@파일1 @파일2`처럼 함께 쓸 수 있다.

---

### 규칙 4: `/plan`으로 먼저 방향을 잡는다

복잡한 작업을 시킬 때, 결과를 바로 내게 하지 말고 `/plan`으로 계획을 먼저 보자. 방향이 틀렸으면 실행 전에 고칠 수 있다. 이미 만들어진 결과물을 바꾸는 것보다 훨씬 빠르다.

---

### 규칙 5: AI 답과 파일이 다를 때는 파일이 정답이다

AI가 Top 3를 보여줄 때 동률인 항목을 빠뜨릴 수 있다. 이번 실습에서 본 것처럼, 32호와 33호가 모두 60.0%로 동률 3위인데 AI가 한 호만 말한다면 — 파일을 직접 열어 확인하는 것이 정답이다. AI가 틀렸다면 "혹시 같은 수치인 항목이 더 있나요?"라고 물어보면 된다. AI는 비서이지 시험감독이 아니다 — 다시 물으면 된다.

```
① codex --cd ~/폴더  "지시문"
        ↓

        ↓

        ↓

        ↓

```

이 사이클이 Codex CLI 일상 사용의 기본 흐름이다. 오늘은 ①~⑦까지 해봤다. ⑧의 스킬 등록은 더 깊이 쓰게 되면 자연스럽게 필요해진다.
