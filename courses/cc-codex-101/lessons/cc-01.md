# cc-01: 설치와 첫 실행 (맨바닥 터미널) — "검은 화면, 하나도 안 무섭습니다"

> **레슨 목표**: 아무것도 안 깔린 컴퓨터에서 Claude Code(클로드코드)를 직접 설치·로그인하고, 한국어로 첫 대화를 성공시킨다. 오늘의 진짜 목표는 "잘하기"가 아니라 **"무사히 켜졌다, 안 망가졌다"는 안심**을 얻는 것.
> **예상 시간**: 45분 (개념 도입 8분 + 실습 1 설치·로그인 15분 + 실습 2 첫 대화 15분 + 실습 3 첫 메모 5분 + 마무리 2분)
> **사전 준비**: 인터넷이 되는 컴퓨터 1대 + Claude.ai 계정(Pro/Max 구독) + 이 코스 실습 폴더(`practice-data/creator/`)
> **진입 커맨드**: 강의를 따라오다 막히면 덱에 뜬 `/cc-01` 을 입력하세요. 이 레슨 내용을 처음부터 다시 안내합니다.

## 📚 학습 목표 (도달하면 성공)

1. **"검은 화면(터미널)은 채팅방이다"** — 터미널이 코딩 도구가 아니라 *AI에게 말 거는 창구*임을 몸으로 느낀다. (두려움 해소 와우 모먼트)
2. **명령어 한 줄로 설치된다** — 복잡한 세팅 없이 복사-붙여넣기 한 줄로 Claude Code가 깔리는 경험.
3. **브라우저 로그인 한 번이면 끝** — 로그인은 "내가 이 계정 주인이에요"를 알려주는 것뿐, 내 파일이 어디 올라가지 않는다는 안심.
4. **한국어로 말을 걸면 AI가 내 실제 파일을 읽고 답한다** — 개발자처럼 영어로 안 쳐도, 완벽한 문법이 아니어도 '지우'의 첫 질문에 AI가 우리 폴더의 진짜 숫자(예: 매출 1,477만원)를 꺼내 대답하는 첫 성공.

## 🎯 개념 도입 (8분)

### 먼저, 우리가 누구인지 다시 떠올려요

당신은 **1인 사업자 '지우'**입니다. 온라인 강의를 만들어 팔고, 매주 일요일 뉴스레터를 보내고, 가끔 1:1 컨설팅도 합니다. 직원은 없습니다 — 기획·글쓰기·고객응대·매출정리·리서치를 **혼자 다 합니다.** 늘 시간에 쫓기죠.

그래서 우리는 **Claude Code(클로드코드)** 라는 AI 도구를 '내 사업의 운영 파트너'로 들이기로 했습니다. 이번 레슨은 그 파트너를 **내 컴퓨터에 처음 들이는 날**입니다. 새 직원 첫 출근날이라고 생각하면 돼요.

> 이 코스에서 우리가 끝낼 한 사이클 = **자료 읽기 → 분석 → 보고서/콘텐츠 초안**. 오늘 cc-01은 그 사이클의 *시동을 거는 날*입니다. 분석이나 보고서는 다음 레슨들의 몫이고, 오늘은 "켜고, 말 걸고, 답을 받는다"까지만 합니다.

### Claude Code가 대체 뭐죠? — "내 폴더 안에서 일하는 AI 비서"

이미 ChatGPT나 Claude.ai 채팅창은 써봤을 수 있어요. 거기서는 파일을 하나씩 끌어다 올리고, 답을 복사해서 다시 내 문서에 붙여넣어야 했죠. 왕복이 번거롭습니다.

**비유**: Claude Code는 *내 컴퓨터 폴더 안으로 직접 출근한 비서*입니다.

```
[ 당신(지우) ]  ←→  [ Claude Code ]  ←→  [ 내 컴퓨터의 폴더·파일 ]
   한국어로 지시        읽고·분석하고·만든다     강의 매출 CSV, 뉴스레터 기록, 문의 모음...
```

- 채팅창 AI = 손님처럼 밖에서 대화. 파일은 내가 하나씩 갖다줘야 함.
- Claude Code = 사무실 안으로 들어와 *내 책상 서랍(폴더)을 직접 열어보는* 비서. 복사-붙여넣기 왕복이 없습니다.

| 항목 | Claude.ai 채팅창 | Claude Code (오늘 배울 것) |
|------|------------------|----------------------------|
| 파일 접근 | 한 개씩 끌어다 올림 | 폴더째로 직접 읽음 |
| 결과물 | 답을 복사해서 내가 붙여넣기 | 새 파일로 알아서 저장 |
| 반복 작업 | 매번 수동 | "이 폴더 전부 같은 식으로" 가능 |
| 추천 대상 | 가벼운 질문 | ⭐ 자료를 다루는 1인 사업자 |

> 첫 등장 용어 풀이 — **CSV(씨에스브이)**: 엑셀처럼 표 데이터를 쉼표로 저장한 파일. 우리 실습의 매출표(`course_sales_monthly.csv`)·문의표(`student_inquiries.csv`)가 이 형식입니다. 엑셀로도 그대로 열려요.

### 터미널 = 무서운 게 아니라 "AI에게 말 거는 채팅방"

오늘 우리는 **터미널(terminal)** 이라는 검은 화면을 씁니다. 여기서 많은 분이 멈칫합니다. "검은 화면에 글자 치는 거… 해커들이 쓰는 거 아닌가?"

> 첫 등장 용어 풀이 — **터미널**: 컴퓨터에게 글자로 명령을 주고받는 창. Mac은 `터미널`이라는 앱, Windows는 `PowerShell`(파워셸)이라는 앱입니다. 검게 생겼지만 *그냥 카카오톡 채팅창의 글자 버전*이라고 보면 됩니다.

실제로 지우가 받은 수강생 인터뷰에서도 가장 많이 나온 말이 이거였어요(`research_memo.md`):

> "엑셀은 하는데 코드는 무서워요."
> "검은 화면에 글자 치는 게 무섭다."

그래서 이 강의는 맨 처음 30분을 *맨바닥에서 설치부터* 천천히 갑니다. 무서운 걸 인정하고, 한 칸씩 같이 갈게요. (지우의 메모에도 적혀 있죠 — "강의 초반 30분을 '맨바닥 새 컴퓨터에 설치부터'로 잡은 게 주효했음. 완주율이 올랐다.")

### 🛡️ 시작 전 4가지 안심 (이거 먼저 읽으세요)

1. **명령어를 잘못 쳐도 컴퓨터는 안 망가집니다.** 대부분 "그런 명령 없는데요?" 같은 안내만 뜨고 끝이에요. 깨지지 않아요.
2. **로그인해도 내 파일이 인터넷에 올라가지 않습니다.** 로그인은 "이 사람이 쓸 권한이 있구나"만 확인하는 절차예요.
3. **AI가 엉뚱한 답을 해도 괜찮습니다.** 다시 시키면 돼요. AI는 비서이지 시험 감독이 아닙니다.
4. **한 번에 하나씩만** 따라오면 됩니다. 이 레슨도 위에서 아래로 한 칸씩만 내려오세요.

### 입력은 딱 두 종류뿐 (이거 하나만 구분하면 절반 끝)

```
① 터미널 명령어  → AI를 "켜고 끄는" 명령.   터미널 일반 프롬프트($)에서 친다.   예: claude
② 슬래시(/) 명령 → AI를 "켜둔 다음" 대화 중에 주는 지시.   AI 채팅창 안에서 친다.   예: /login
```

> 💡 자주 헷갈리는 지점 미리: `/login`을 터미널에서 `claude /login`처럼 치는 게 **아니라**, `claude`를 먼저 켜서 대화하는 상태에서 `/login`을 칩니다. 이거 하나만 구분해도 절반은 끝입니다.

오늘은 거의 ①번(설치하고 켜기)만 합니다. 슬래시 명령은 다음 레슨부터 본격적으로 만나요. 지금은 "두 종류가 있구나"만 알아두면 충분합니다.

## 🔧 실습 1 — 맨바닥에서 설치 → 로그인 (15분)

> 새 컴퓨터를 가정합니다. 아무것도 안 깔려 있어도 괜찮아요. 순서대로만 따라오세요. 한 단계 끝낼 때마다 잠깐 숨 한 번 쉬고 다음으로.

### 1단계: 터미널 열기 — "채팅방 켜기"

먼저 명령을 입력할 창을 엽니다.

- **Mac**: `Command(⌘) + Space` → 검색창에 `터미널` 입력 → 엔터.
- **Windows**: 시작 버튼 → `PowerShell` 검색 → 클릭.

```text
(아직 아무것도 칠 필요 없어요. 창만 띄웁니다.)
```

**기대 결과**: 검은(또는 흰) 창이 뜨고, 끝에 `$` 또는 `%`(Mac), `>`(Windows) 기호가 깜빡입니다. 이 기호가 "말씀하세요, 듣고 있어요"라는 뜻이에요.

**관찰 포인트**: 이 깜빡이는 커서가 보이면 성공입니다. 아무것도 안 쳐도 돼요. *채팅방이 열린 것*뿐입니다.

---

### 2단계: 내가 어디 서 있는지 확인하기 (`pwd`)

비서가 일하려면 *어느 폴더에서 일할지*가 중요합니다. 지금 내가 컴퓨터 안 어느 위치에 서 있는지부터 봅니다. 아래를 그대로 칩니다(복사-붙여넣기 OK).

```text
pwd
```

> 용어 풀이 — **pwd**: "print working directory(지금 일하는 폴더 위치를 알려줘)"의 약자. 지도에서 '현재 위치' 핀을 찍는 것과 같아요.

**기대 결과** (예시):

```
/Users/jiwoo
```

또는 Windows라면 `C:\Users\jiwoo` 같은 줄이 뜹니다.

**관찰 포인트**: 줄 하나가 떴다면 성공. *무슨 명령을 쳤더니 컴퓨터가 글자로 대답을 했다* — 이게 터미널의 전부입니다. 무섭지 않죠?

---

### 3단계: 토대 엔진(Node.js)이 있는지 확인하기

Claude Code는 **Node.js(노드제이에스)** 라는 토대 위에서 돕니다. 자동차로 치면 엔진 같은 거예요. 깔려 있는지 먼저 봅니다.

```text
node --version
```

> 용어 풀이 — **Node.js**: Claude Code 같은 프로그램이 돌아가게 해주는 '엔진'. 한 번만 깔면 됩니다.

**기대 결과 (두 갈래)**:

```
v20.11.0
```
- 위처럼 `v` + 숫자가 뜨면 → 엔진 있음. **4단계로 바로 가세요.**
- `command not found` 또는 `'node'은(는) ... 명령이 아닙니다`가 뜨면 → 엔진을 먼저 깔아야 합니다:
  - **Mac/Windows 공통**: 브라우저에서 [nodejs.org](https://nodejs.org) 접속 → 큰 버튼(LTS 버전) 다운로드 → 설치 파일 실행(다음·다음·완료). 설치 후 **터미널을 껐다 다시 켜고** 이 단계를 한 번 더 확인하세요.

**관찰 포인트**: "없으면 깔고, 있으면 넘어간다." 이게 전부입니다. 없다고 잘못한 게 아니에요 — 새 컴퓨터엔 원래 없는 경우가 많아요.

---

### 4단계: Claude Code 설치하기 (한 번만)

이제 AI 비서 본체를 내 컴퓨터에 내려받습니다. 앱스토어에서 앱 하나 까는 것과 똑같아요.

```text
npm install -g @anthropic-ai/claude-code
```

Mac에서 Homebrew(홈브루, 맥용 설치 도구)를 이미 쓰는 분이라면 이 방법도 됩니다(둘 중 하나만):

```text
brew install --cask claude-code
```

> 용어 풀이 — **npm install -g**: "이 프로그램을 컴퓨터 어디서나 쓸 수 있게(-g = global, 전역) 깔아줘"라는 뜻. `-g`는 '집 안 어디서나 쓰는 공용 도구함에 넣어라' 정도로 이해하면 돼요.

**기대 결과**: 글자가 주르륵 흐르며 진행 막대 비슷한 게 지나가고, 1~2분 뒤 다시 `$`(또는 `%`/`>`) 커서로 돌아옵니다. 중간에 노란 경고(warning) 줄이 몇 개 보여도 **정상**이에요 — 빨간 에러로 멈추지만 않으면 됩니다.

**관찰 포인트**: 설치 중 화면이 멈춘 듯 보여도 기다리세요. 인터넷에서 파일을 받아오는 시간입니다. 절대 창을 닫지 마세요.

### 5단계: 잘 깔렸는지 확인하기

```text
claude --version
```

**기대 결과** (예시):

```
2.1.183 (Claude Code)
```

- 위처럼 **버전 숫자**가 뜨면 → 설치 성공! 🎉
- `command not found`가 뜨면 → 당황하지 말고: **터미널을 완전히 껐다 다시 켜세요.** 새로 깐 프로그램은 새 창에서 인식되는 경우가 많습니다. 그래도 안 되면 3단계(Node.js)가 제대로 깔렸는지 다시 확인.

**관찰 포인트**: 버전 숫자 = "비서가 출근 도장 찍었다"는 신호입니다. (숫자는 사람마다 다를 수 있어요. `2.1.` 같은 형태면 됩니다. 버전·설치법은 자주 바뀌니 화면이 살짝 달라도 괜찮아요 — 막히면 `claude --help`가 항상 최신 정답입니다.)

### 6단계: 로그인 — "나 이 계정 주인이에요"

비서에게 *내가 누구인지* 알려줍니다. Claude.ai에 가입할 때 쓰던 그 계정으로 로그인해요.

```text
claude
```

**기대 결과**:
- 처음 켜면 로그인 안내가 뜨고, 보통 **브라우저 창이 자동으로 열립니다.**
- 브라우저에서 Claude 계정으로 로그인하고 "승인(Authorize)" 버튼을 누르면, "터미널로 돌아가세요(You can return to your terminal)" 같은 안내가 나옵니다.
- 터미널로 돌아오면 입력창(`>`)이 떠 있습니다.

> 만약 브라우저가 자동으로 안 열리면: 화면에 뜬 긴 주소(링크)를 복사해 브라우저 주소창에 붙여넣어 로그인하세요. 또는 채팅창 안에서 `/login` 을 쳐서 다시 시도할 수 있어요.

**🛡️ 안심 포인트**: 로그인한다고 내 파일(매출 CSV, 강의 원고)이 어디 올라가는 게 절대 아닙니다. "이 사람 쓸 권한 있네" 확인만 하는 거예요.

> 💡 **요금제 한마디**: Claude Code는 ① **구독제(Pro $20 / Max $100·$200)** 와 ② **API 키(쓴 만큼 과금)** 두 방식이 있어요. 1인 사업자는 매달 정액인 **구독제**가 마음 편합니다. 실습을 넉넉히 돌리려면 사용량이 더 큰 **Max** 쪽이 덜 막혀요(Pro는 한도가 빨리 찰 수 있음). 정확한 비용 관리는 cc-06에서 다룹니다 — 지금은 *로그인만* 되면 충분합니다.

**관찰 포인트**: 입력창 `>` 가 떴다면 — 비서가 책상에 앉았습니다. 이제 말을 걸 수 있어요.

## 🎬 실습 2 — 한국어로 첫 대화 성공시키기 (15분)

> 이제 진짜 와우 모먼트. *말을 걸면 일이 시작된다*를 직접 봅니다. 우리 실습 폴더(`practice-data/creator/`)를 무대로 삼아 볼게요. 이 폴더에는 지우의 강의 매출표, 뉴스레터 성과표, 수강생 문의표, 그리고 기획·리서치·일정 노트가 들어 있습니다.

### 7단계: 사업 폴더로 이동한 뒤 다시 켜기

비서는 *지금 서 있는 폴더*를 무대로 삼습니다. 그래서 우리 실습 자료가 있는 폴더로 먼저 이동한 다음 켜는 게 좋아요. 일단 채팅창에서 한 번 나옵니다.

```text
/exit
```

그다음 실습 폴더로 이동합니다. (폴더 경로는 강의에서 안내한 위치로 바꿔 입력하세요. 아래는 예시입니다.)

```text
cd practice-data/creator
```

> 용어 풀이 — **cd**: "change directory(폴더를 옮겨라)". 비서를 데리고 *자료가 있는 방으로 함께 걸어 들어가는* 것과 같아요. **/exit**: 채팅방에서 나오기.

이동했는지 확인하고 그 안에 뭐가 있는지 봅니다.

```text
pwd
ls
```

> 용어 풀이 — **ls**: "list(이 폴더에 뭐가 있나 목록 보기)". 파인더·탐색기로 폴더를 더블클릭해 연 것과 같아요.

**기대 결과** (예시):

```
/Users/jiwoo/.../practice-data/creator
README.md                  course_sales_monthly.csv   newsletter_stats.csv
student_inquiries.csv      content_plan.md            research_memo.md
consulting_schedule.md
```

이제 이 폴더를 무대로 다시 비서를 켭니다.

```text
claude
```

**관찰 포인트**: `ls` 결과에 우리 실습 파일 7개가 보이면, 비서가 이 방에 있는 자료를 전부 들여다볼 수 있는 상태가 된 겁니다.

### 8단계: '지우' 1인칭으로 첫 자기소개 + 현황 묻기

입력창(`>`)에 아래를 **그대로** 붙여넣습니다. 영어도, 완벽한 문법도 필요 없어요. (좋은 첫 질문 = **자기소개(역할) + 상황 + 요청 방향**. 이 구조 하나만 기억하면 됩니다.)

```text
안녕! 나는 1인 콘텐츠 사업자 지우야. 온라인 강의·뉴스레터·컨설팅을 혼자 운영해.
지금 이 폴더(practice-data/creator)에 있는 파일들을 보고,
각 파일이 뭔지 한 줄씩, 그리고 내가 지금 할 수 있는 가장 현실적인 일 3가지를 알려줄래?
README.md부터 읽고 정리해줘.
```

**기대 결과** (예시):

```
이 폴더에는 이런 파일들이 있어요:
- course_sales_monthly.csv : 강의별·월별 매출 (2025-07 ~ 2026-03)
- newsletter_stats.csv : 뉴스레터 호별 오픈율·클릭율·구독 증감 (18호~41호)
- student_inquiries.csv : 수강생 문의 내역 (채널·카테고리·처리상태)
- content_plan.md / research_memo.md / consulting_schedule.md : 기획·리서치·일정 노트

지금 할 수 있는 현실적인 일 3가지:
1. 매출이 가장 잘 나온 강의 찾기 (course_sales_monthly.csv)
2. 반응 좋았던 뉴스레터의 공통점 찾기 (newsletter_stats.csv)
3. 가장 자주 들어온 고객 문의 유형 정리 (student_inquiries.csv)
```

**관찰 포인트**:
- 한국어로 물었는데 한국어로 답한다 ✓
- 폴더 안 *실제 파일 이름*을 읽고 답한다(지어내지 않음) ✓
- "뭐 할 수 있어?"라는 막연한 질문에도, 우리 자료에 맞춘 구체적 제안을 준다 ✓

💡 **이 순간이 와우 모먼트입니다.** 검은 화면이 *내 사업 자료를 읽고 대답하는 똑똑한 동료*로 바뀌었어요.

### 9단계: AI가 '실제 숫자'를 읽어오는 걸 관찰하기

CC가 답하는 동안 화면을 가만히 보세요. **글자가 한 줄씩 주르륵 떨어집니다.** 또 어떤 답은 "파일을 읽는 중…" 같은 표시가 잠깐 지나가요. 이게 *AI가 직접 파일을 열어보며 일하는 모습*입니다.

체험을 위해 이어서 한 문장만 더 던져봅니다. (방금 1번으로 제안받은 일을 바로 시켜보는 거예요.)

```text
방금 1번 말한 거 좋아. course_sales_monthly.csv 를 실제로 열어서,
매출(만원)이 가장 컸던 한 달과 그 강의 이름을 딱 한 줄로 알려줘.
```

**기대 결과** (예시 — 실제 데이터 기준):

```
가장 큰 단일 매출은 2026-01 "챗GPT로 콘텐츠 10배 빠르게"로, 매출 1,477만원(수강신청 187건)이었어요.
```

**관찰 포인트**: 비서가 *상상으로 답한 게 아니라* 파일을 열어 실제 숫자(1,477만원, 187건)를 꺼내왔습니다. 이것이 채팅창 AI와 가장 다른 점이에요 — **내 실제 파일을 근거로** 답합니다.

> 💡 **할루시네이션(AI가 지어내기) 방지 한 줄**: 답이 미심쩍으면 "**파일의 실제 값**으로 다시 확인해서 알려줘"라고 한 번 더 시키면 됩니다. 다음 레슨들에서 계속 연습할 핵심 습관이에요.

---

### 10단계: 다른 파일도 한 번 — 뉴스레터 살짝 들춰보기

같은 방식이 다른 파일에도 그대로 통한다는 걸 한 번 더 확인해봅니다. (분석은 다음 레슨 몫이니, 오늘은 "읽어 온다"만 확인하는 가벼운 질문으로.)

```text
이번엔 newsletter_stats.csv 를 열어서, 클릭율(%)이 가장 높았던 호 하나만,
호수랑 제목이랑 클릭율을 한 줄로 알려줘.
```

**기대 결과** (예시 — 실제 데이터 기준):

```
클릭율이 가장 높았던 건 40호 "사전등록 마감 임박 (오늘 밤 12시)"로, 클릭율 30.3%였어요.
```

**관찰 포인트**: 파일이 바뀌어도(매출 → 뉴스레터) 비서는 똑같이 *그 파일을 열어 실제 값*을 꺼냅니다. "이 폴더 안 자료라면 뭐든 읽힐 수 있구나"를 손으로 확인하는 단계예요. (왜 이 호가 잘됐는지 같은 *해석*은 다음 레슨에서 `content_plan.md` 같은 맥락 노트와 연결해 깊게 다룹니다 — 오늘은 여기까지.)

---

### 11단계: "다시 해줘"가 된다는 걸 체험하기

답이 어려우면 *다시 시키면 됩니다.* 부담 갖지 마세요.

```text
방금 매출 답, 우리 어머니도 이해하실 만큼 쉽게 한 문장으로 다시 말해줄래?
```

**기대 결과** (예시):

```
작년 1월에 '챗GPT로 콘텐츠 10배 빠르게' 강의가 한 달 동안 제일 많이 팔려서,
그 한 강의로만 약 1,477만원을 벌었어요.
```

**관찰 포인트**: 같은 사실을 더 쉬운 말로 다시 줬죠. *한 번에 완벽하지 않아도 된다, 대화하면 된다* — 이게 AI 짝꿍과 일하는 방식입니다. (지우가 받은 수강생 피드백과도 똑같아요: "한 번에 다 알려주면 머리 아파요." 그래서 *한 번에 하나씩, 다시 시키면서* 갑니다.)

### 12단계: 채팅방 깔끔하게 닫기

오늘 첫 출근은 여기까지. 비서를 퇴근시킵니다.

```text
/exit
```

또는 키보드로 `Ctrl + C` 를 두 번 눌러도 나와집니다.

**기대 결과**: 다시 일반 터미널 커서(`$`/`%`/`>`)로 돌아옵니다.

**관찰 포인트**: 닫아도 *설치와 로그인은 그대로 남습니다.* 내일 다시 `cd practice-data/creator` → `claude` 두 줄이면 곧장 켜져요. 오늘 한 설치·로그인은 **이 코스 전체에서 딱 한 번**만 하면 됩니다.

| `claude : command not found` (설치 직후인데) | 새 프로그램이 현재 창에 아직 안 잡힘 | 터미널을 **완전히 껐다가** 새로 열어 `claude --version` 다시 |
| 설치 중 노란 `warning` 줄이 잔뜩 | 정상적인 안내 메시지 | 빨간 `error`로 멈추지만 않으면 무시하고 진행 |

| "파일을 못 읽겠다"고 함 | 비서가 다른 폴더에 서 있음 | `/exit` 후 `cd practice-data/creator` 로 이동 → `pwd`로 확인 → 다시 `claude` |
| AI가 숫자를 지어낸 것 같음 | 할루시네이션(모델 한계) | "**파일의 실제 값**으로 확인해서 다시 알려줘" 요청 |
| 답이 너무 길고 어렵다 | AI가 과하게 설명 | "핵심만 3줄로" / "비개발자 눈높이로 다시" 요청 |
| 화면이 멈춘 것 같다 | 설치/응답 진행 중 | 기다리기 — 창을 닫지 말 것. 정 안 되면 `Ctrl + C` 두 번으로 빠져나오기 |

## 💡 핵심 규칙·개념 정리

### 규칙 1: 터미널은 채팅방이다
검은 화면 = AI에게 말 거는 창구. 잘못 쳐도 안 망가집니다. *겁먹지 말고 한 줄씩.*

### 규칙 2: 입력은 두 종류뿐
- **터미널 명령어**(`claude`, `claude --version`) = AI를 켜고 끄는 명령(터미널 `$`에서).
- **슬래시 명령**(`/login`, `/exit`) = AI를 켜둔 *상태에서* 주는 지시(채팅창 안에서).

### 규칙 3: 설치·로그인은 평생 한 번
오늘 한 설치(4단계)와 로그인(6단계)은 이 코스 내내 **딱 한 번**. 내일부턴 `cd 폴더` → `claude` 두 줄이면 출근 끝.

### 규칙 4: 한국어 OK, 다시 시키기 OK
```
❌ "Read the csv and summarize"
✅ "이 매출 CSV에서 제일 잘 팔린 강의를 실제 숫자로 알려줘"
```
답이 별로면 "더 쉽게", "다시" 하면 됩니다. AI는 시험 감독이 아니라 짝꿍입니다. 첫 답이 70점이어도 정상이에요.
