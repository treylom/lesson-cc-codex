> **예상 시간**: 55분 (개념 도입 10분 + 실습 1 서브에이전트 18분 + 실습 2 Hooks 17분 + 실습 3 자동화 7분 + 마무리 3분)
> **사전 준비**: 터미널 1개 + Claude Code 설치·로그인 완료(`/cc-01`) + `practice-data/creator/` 폴더(자기 트랙이면 그 폴더) / cc-05(CLAUDE.md)·cc-06(세션·비용)을 한 번씩 해본 상태면 더 좋음
> **진입 커맨드**: 터미널에 `/cc-10` 입력 → 이 실습 시작

> 🟢 **이건 부록(appendix, 본편 뒤에 붙는 보충 레슨)이에요.** 강의 본편(cc-01~cc-06)을 끝낸 분이 "조금 더 욕심내볼까?" 할 때 보는, 한 단계 위의 재미난 기능들입니다. **꼭 지금 다 익히지 않아도 됩니다.** 오늘은 "아, 이런 게 있구나, 나중에 내 일에 이렇게 쓰겠네" 하고 **구경하고 한 번 따라해보는 것만으로 100% 성공**이에요. 어렵게 느껴지는 부분은 그냥 넘기고, 마음에 드는 것 하나만 손에 익혀도 충분합니다.

## 📚 학습 목표 (4가지 와우 모먼트)

1. **"비서가 한 명에서 여러 명으로 늘어난다"** — 큰 일을 잘게 나눠 **여러 AI에게 동시에** 맡기는 서브에이전트(sub-agent, '보조 일꾼')를 직접 한 번 부려본다. 지우가 매출·문의·뉴스레터를 세 갈래로 동시에 점검시키는 장면. 지우의 "혼자 다 한다"가 처음으로 "여럿이 나눠 한다"로 바뀌는 순간.
2. **"실수해도 AI가 먼저 막아준다"** — 위험한 작업(파일 삭제 등)을 *시작도 못 하게* 가로막아 주는 안전벨트(Hook, '훅')의 원리를 이해하고, "내 고객 명단·매출 파일은 안 망가진다"는 안심을 한 단계 굳힌다.
3. **"긴 작업이 끝나면 소리로 알려준다"** — 일 시켜놓고 자리를 비워도 놓치는 게 없게, **완료 알림 Hook**(가장 쉽고 바로 와닿는 자동화)을 한 번 켜본다.
4. **"내가 자는 동안에도 한 번씩 일해준다"** — 사람이 옆에 없어도 AI가 알아서 돌아가는 **자동 실행(headless, '헤드리스')**의 입구를 맛본다. 매주 일요일 뉴스레터 인사이트를 알아서 뽑아두는 미래를 그려본다.

> 이 넷은 따로 노는 기능이 아니라 **한 줄로 이어집니다** — **"AI 비서를 한 명에서 → 여러 명으로 → 안전하게 → 알아서 돌아가게."** 서브에이전트(일을 나눈다) → Hooks(나눈 일을 안전하게 한다) → 자동화(그 일을 사람 없이 돌린다). 1인 사업자가 진짜 "AI 팀과 일하는" 단계의 첫 발입니다.

## 🎯 개념 도입 (10분) — 천천히, 비유부터

지금까지 지우는 Claude Code를 **한 명의 비서**처럼 썼어요. "이 파일 봐줘", "이거 정리해줘" 하고 한 번에 하나씩 시켰죠(cc-01~cc-08). 지우의 기획 노트(`content_plan.md`)에도 이런 고민이 적혀 있습니다 — *"콘텐츠 제작에 주 25시간 이상 쓰는 중. AI로 초안·리서치를 더 자동화해서 시간을 줄여야 함."* 오늘은 그 비서를 **더 영리하게 부리는 세 가지 방법**을 구경합니다. 셋 다 어려운 말처럼 들리지만, 사실 우리가 일상에서 늘 하는 일이에요.

### 한 비유로 보는 세 개념 — "작은 가게에 알바생 들이기"

지우가 작은 가게에 **알바생을 들인다**고 상상해 봅시다.

| 개념 | 비유 | 한 줄 설명 |
|------|------|-----------|
| **서브에이전트**(sub-agent, 보조 일꾼) | 알바생을 *여러 명* 두고 일을 나눠 줌 | 메인 AI가 "너는 매출만 봐, 너는 문의만 정리해" 하고 **동시에** 여러 AI에게 시킨다. 각자 따로 일하고 결과 요약만 보고한다. |
| **Hooks**(훅, 안전장치) | 가게 규칙을 *벽에 붙여 둠* — "금고는 절대 열지 마" | AI가 일하기 *직전*에 자동으로 검사하는 안전장치. 위험한 행동은 시작도 못 하게 막는다. |
| **자동화**(headless 모드) | 사장이 없어도 *예약 취사처럼 알아서* | 사람이 매번 시키지 않아도, 정해진 시각·조건에 AI가 스스로 일을 돌린다. |

### ① 서브에이전트 = "일을 나눠 맡는 보조 일꾼들"

지우의 사업 폴더(`practice-data/creator/`)엔 파일이 6개나 있어요(매출 표·뉴스레터 표·문의 표·기획 노트·리서치 노트·일정 노트). 이걸 **전부 한 사람**에게 "다 읽고 정리해줘" 하면, 그 한 명이 1번부터 6번까지 차례차례 읽느라 시간이 걸립니다.

그런데 **보조 일꾼 여러 명**에게 "너는 매출 표만, 너는 문의 표만" 하고 나눠주면? 여러 명이 **동시에** 읽고, 각자 "제 담당은 이렇습니다" 하고 짧게 보고합니다. 그걸 대표 비서가 모아 정리하죠. 훨씬 빠릅니다.

> 💡 **비유**: 식당에서 주문이 밀리면 주방장 혼자 다 만들지 않고 "너는 면, 너는 고명, 너는 국물" 하고 나눠 동시에 만듭니다. **서브에이전트 = 그 보조 요리사들**이에요. 한 명(메인 Claude)이 지휘하고, 여러 명이 나눠 일합니다.

- **서브에이전트(sub-agent)** = 메인 Claude가 **"이 부분만 맡아"** 하고 떼어주는 별도의 AI 일꾼. (sub = 보조, agent = 일꾼) 각자 따로 일하고, **결과 요약만** 대표에게 돌려줍니다.
- 좋은 점: **빠르다**(동시에 일하니까), **깔끔하다**(각자 자기 것만 보니 메인 비서의 기억이 안 헷갈림).
- 주의: 일꾼이 많아질수록 **비용(돈)도 늘어납니다** — 사람을 더 쓰면 인건비가 느는 것과 같아요. 그래서 "정말 나눌 만큼 큰 일일 때만" 씁니다. (비용 얘기는 cc-06에서 배운 그대로예요.)

#### 비교표 — 서브에이전트 vs 그냥 메인에서 시키기

| | 메인 AI 혼자 (cc-03~cc-08 방식) | 서브에이전트 (이번 레슨) |
|---|---|---|
| 일하는 방식 | 한 가지씩 순서대로 | 여러 작업을 나눠서 동시에 |
| 기억(컨텍스트) | 한 대화에 다 쌓임 → 길어지면 무거워짐 | 각 보조 일꾼이 *따로* 기억 → 메인은 가벼움 |
| 적합한 일 | 짧은 단일 작업 | "여러 파일을 각각 분석" 같은 분담 가능한 일 |
| 비용 | 보통 | **여럿이 동시에 = 토큰(비용) 더 듦** ⚠️ |

### ② Hooks = "AI에게 채우는 안전벨트·자동 알림"

지우가 가장 걱정하는 게 뭐였죠? **"내가 잘못 시켜서 소중한 파일이 지워지면 어쩌지?"** 였습니다. 고객 문의 명단(`student_inquiries.csv`)이나 매출 표(`course_sales_monthly.csv`)는 한 번 날리면 끝이에요. 게다가 지우는 혼자라 검산해 줄 사람이 없습니다.

**Hook(훅)** 은 그 걱정을 시스템이 대신 막아주는 장치예요. "AI가 이런 위험한 짓을 하려고 하면 → 자동으로 멈춰!", "AI가 일을 다 끝내면 → 소리로 알려줘!" 같은 **자동 규칙**을 미리 걸어두는 겁니다.

> 💡 **비유**: 자동차 안전벨트와 도착 알람이에요. 운전(작업)은 내가 하지만, **위험하면 자동으로 잡아주고**(삭제 차단), **도착하면 띵동 알려주는**(완료 알림) 장치죠. 한 번 채워두면 매번 신경 안 써도 알아서 작동합니다.
> - **Hook(훅)** = "걸어 두는 갈고리". 특정 순간(예: AI가 명령을 실행하기 직전)에 자동으로 *걸려서* 정해진 검사를 돌립니다. 문에 달린 자동 잠금장치처럼요.

Hook은 종류가 여럿이지만, 비개발자 지우에게 **딱 두 개만** 와닿습니다(나머지는 "그런 것도 있구나"):

| Hook | 언제 작동하나 | 무엇을 해주나 | 지우에게 |
|---|---|---|---|
| **안전 훅** (위험 차단) | AI가 **위험한 명령**(파일 삭제 등)을 하려는 **직전** | 자동으로 막고 "이건 위험해요" 알림 | 😌 "실수로 파일 날릴 일 없겠네" |
| **완료 알림 훅** | AI가 **일을 다 끝냈을 때** | 띵! 소리로 알려줌 | 🔔 "딴짓하다 놓칠 일 없겠네" |

> 🟢 **안심 포인트**: Hook은 **켜지 않아도 됩니다.** 그리고 Claude Code는 Hook을 따로 안 걸어도, 원래부터 **위험한 작업은 하기 전에 "이거 해도 될까요?" 하고 물어보는** 기본 안전장치가 있어요(cc-01·cc-05에서 봤죠). Hook은 그 위에 **한 겹 더** 단단히 채우는 옵션입니다.

### ③ 자동화(headless) = "사람 없이도 한 번씩 알아서"

지금까지는 지우가 터미널을 열고, 말을 걸고, 답을 받았어요. **항상 사람이 옆에 있어야** 했죠.

**자동화(headless, '헤드리스' = 머리(화면) 없는 실행)** 는 이걸 한 줄로 미리 적어두고, **정해진 시각에 알아서 실행**되게 하는 겁니다. 지우의 일정 노트(`consulting_schedule.md`)를 보면 *"일 09:00 뉴스레터 발송"*, *"03-31 월 마감 정산"*처럼 매주·매월 반복되는 일이 가득합니다. 이런 반복 정리는 자동화의 1순위 후보예요.

> 💡 **비유**: 밥솥 예약 취사예요. 저녁에 쌀 안치고 "아침 7시"로 맞춰두면, 내가 자는 동안 알아서 밥이 됩니다. **자동화 = AI에게 거는 예약 취사**입니다.

- 핵심 명령은 이번 모듈에서 처음 보는 옵션, `-p`(print, 부탁 한 줄을 주고 한 번에 실행)예요. `claude -p "지시문"` = **"한 번 시키고 답만 받고 끝"**. 사람이 대화를 이어가지 않아도 되니, 스크립트나 예약 실행에 딱 맞습니다. (지금까지는 `claude`를 켜고 *대화*했지만, `-p`는 *사람 없이 한 번에* 돌리는 새 방식이에요 — 오늘 처음 만나는 도구입니다.)
- 이게 가능한 이유가 바로 cc-03에서 배운 "AI가 파일을 직접 읽고 직접 저장한다"는 능력이에요.

### 왜 1인 사업자에게 이 셋이 필요한가

- **서브에이전트**: 직원이 없는 지우가 "매출 분석 + 문의 정리 + 뉴스레터 점검"을 *한 번에* 시키면 메인 AI 혼자서 순서대로 합니다. 서브에이전트를 쓰면 이 셋을 **나눠 동시에** 돌릴 수 있어요 — 직원 없이도 "팀처럼".
- **Hooks**: 혼자라 검산해 줄 사람이 없습니다(시나리오 §2의 진짜 문제). Hook은 *사람 대신 지켜보는 안전장치*입니다.
- **자동화**: 지우의 화요일은 *"종일 유튜브 촬영·편집"*입니다. 반복되는 정리·요약을 자동으로 돌려두면 그 시간을 기획과 고객에게 쓸 수 있어요.

> ⚠️ **오늘의 범위**: 서브에이전트·Hook·자동화는 **부록**이라, 오늘은 "개념 이해 + 한 번 따라하기"가 목표예요. 진짜 매일 돌리는 예약 설정(cron 같은 컴퓨터 깊은 설정)이나 GitHub 자동 연동은 더 깊은 주제라 **"이렇게 하면 되는구나"까지만** 봅니다. 무리해서 다 세팅하려 하지 마세요. 🙂 그리고 cc-06에서 배운 비용 감각을 여기서도 씁니다 — 단순한 일은 그냥 메인에서, "분담이 진짜 이득인 일"에만 서브에이전트를.

## 🔧 실습 1 — 여러 손으로 늘리기: 서브에이전트로 일 나눠 주기 (18분)

지우의 오늘 목표: **"3월 한 달치 데이터를 세 갈래로 동시에 점검"** — 매출(`course_sales_monthly.csv`), 문의(`student_inquiries.csv`), 뉴스레터(`newsletter_stats.csv`)를 각각 다른 보조 일꾼에게 맡기고, 메인은 결과만 모읍니다.

> 🟢 **마음 편하게**: 아래는 전부 **"읽고 정리해줘"** 부탁뿐이라 파일이 바뀌거나 지워지지 않아요. 안심하고 따라오세요.

### 1단계 — 내 사업 폴더로 이동해 Claude Code 켜기

먼저 자료가 있는 폴더로 들어가서 AI를 깨웁니다(cc-03에서 배운 길 찾기예요).

```text
# 먼저 자료 폴더가 어디 있는지 찾기 (cc-03에서 배운 길 찾기)
find ~ -name "course_sales_monthly.csv" 2>/dev/null
```

위에서 나온 경로의 폴더가 자료 폴더입니다. 아래 경로는 **예시**이니 찾은 실제 경로로 바꿔 입력한 뒤 AI를 켭니다.

```text
cd <위에서 찾은 practice-data/creator 폴더 경로>
claude
```

**왜 이렇게 하나요?**: `cd`는 "그 폴더로 이동", `claude`는 "그 폴더를 무대로 AI 켜기"예요. AI는 **지금 있는 폴더**를 무대로 삼으니, 자료 폴더로 먼저 들어간 뒤 켜는 게 좋습니다.

**기대 결과(예시 화면)**:
```text
✻ Claude Code  (지금 폴더: .../practice-data/creator)
>
```

**관찰 포인트**: 화면 위쪽 폴더 경로가 `creator`로 끝나는지 확인하세요. 엉뚱한 폴더면 `cd`로 다시 이동하면 됩니다. (헷갈리면 대화창에서 "여기 무슨 폴더야? 파일 목록 보여줘"라고 물어봐도 돼요.)

### 2단계 — 내장 서브에이전트가 뭔지 물어보기

Claude Code가 **기본으로** 가진 보조 일꾼들을 먼저 확인합니다.

```text
저는 1인 콘텐츠 사업자 지우예요. 비개발자입니다.
Claude Code에는 기본으로 어떤 "서브에이전트(보조 일꾼)"가 들어 있나요?
각각 무슨 일에 쓰는지 비개발자도 알게 한 줄씩 쉽게 설명해 주세요.
```

**기대 결과(예시)**:
```text
기본으로 들어 있는 보조 일꾼 3종:
1. Explore         — 폴더·파일을 빠르게 둘러보는 "탐색 담당" (읽기만, 가벼운 모델)
2. Plan            — 일을 시작하기 전 자료를 모아 계획을 세우는 "준비 담당" (읽기만)
3. general-purpose — 여러 단계가 얽힌 복잡한 일을 끝까지 맡는 "만능 담당" (모든 도구)
```

**관찰 포인트**:
- "탐색은 가벼운 모델(읽기 전용)"이라는 점 — *단순한 둘러보기는 싼 일꾼에게* 라는 비용 감각이 여기 녹아 있습니다.
- 보조 일꾼은 **결과 요약만** 메인에게 돌려줍니다. 자기가 본 파일 내용을 통째로 메인 기억에 쏟지 않아요 → 메인이 가벼워집니다(cc-06의 "기억 아끼기").

### 3단계 — 탐색(Explore) 담당에게 폴더 정찰 시키기

직접 "보조 일꾼을 써 줘"라고 부탁해 봅니다.

```text
이 폴더(practice-data/creator)를 "탐색 담당(서브에이전트)"에게 맡겨서
어떤 파일이 있고 각각 뭔지 한 줄씩 정리해 오게 해 주세요.
메인 대화는 가볍게 유지하고, 요약만 받고 싶어요.
```

**기대 결과(예시)**:
```text
탐색 담당이 정찰을 마쳤습니다. 요약:
- course_sales_monthly.csv : 강의별·월별 매출(2025-07~2026-03)
- newsletter_stats.csv     : 뉴스레터 18~41호 발송·오픈·클릭 성과
- student_inquiries.csv    : 수강생 문의 내역(채널·카테고리·처리상태)
- content_plan.md / research_memo.md / consulting_schedule.md : 기획·리서치·일정 노트
```

**관찰 포인트**:
- 메인 AI가 직접 6개 파일을 다 읽은 게 아니라, **보조 일꾼이 읽고 요약만** 가져왔습니다. 파일 원문이 메인 기억에 안 쌓여요.

### 4단계 — 세 갈래로 동시에 분담시키기 (와우 모먼트)

이제 핵심입니다. **세 작업을 한 번에** 나눠 주세요.

```text
3월 데이터를 세 갈래로 나눠서, 각각 다른 보조 일꾼에게 동시에 맡겨 주세요.

1) 매출 담당: course_sales_monthly.csv에서 2026-03 순매출 상위 강의 3개와 순매출 금액
2) 문의 담당: student_inquiries.csv에서 2026-03 문의 카테고리별 건수 Top 3
3) 뉴스레터 담당: newsletter_stats.csv에서 38~41호 중 클릭율이 가장 높은 호

각자 결과만 짧게 요약해서 가져오면, 마지막에 한 화면으로 합쳐 주세요.
```

**기대 결과(실제 데이터 기준 예시)** — 화면에 `Task(...)` 같은 줄이 여러 개 동시에 뜹니다:
```text
● Task(매출 담당)        ⎿ 실행 중…
● Task(문의 담당)        ⎿ 실행 중…
● Task(뉴스레터 담당)    ⎿ 실행 중…

[매출 담당] 2026-03 순매출 상위:
  1. 클로드코드로 업무 자동화 입문 — 1,242만원 (수강 118명)
  2. 1인 지식사업 수익화 로드맵   — 895만원 (수강 68명)
  3. 챗GPT로 콘텐츠 10배 빠르게    — 823만원 (수강 121명)

  1. 실습    7건 (터미널·WSL·CLAUDE.md·AGENTS.md 설치/설정)
  2. 강의문의 5건 (인스타DM 사전 질문)

[뉴스레터 담당] 38~41호 중 클릭율 1위:
  40호 "사전등록 마감 임박 (오늘 밤 12시)" — 클릭율 30.3%

[합본] 3월엔 신규 '클로드코드' 강의가 순매출 1위로 올라섰고,
관련 '실습' 문의가 가장 많았으며, 마감 임박 카피(40호)가 가장 잘 눌렸습니다.
```

**관찰 포인트**:
- 화면에 `Task(...)` 줄이 **여러 개 동시에** 뜨면, 그게 보조 일꾼들이 나눠 일하는 모습이에요(버전에 따라 표시가 조금 달라도 핵심은 "여러 개가 같이 돈다").
- 셋이 **각자 따로** 일하고 결과만 모입니다 — 지우가 직원 셋에게 일을 나눠 준 것과 같죠.
- 문의 Top 3는 건수 순(실습 7건 > 강의문의 5건 > 결제 4건)으로 정렬됩니다. 4위는 환불 3건 — 한 끗 차이라, 다음 달에 순위가 바뀔 수도 있어요.
- 실제 숫자(1,242만원·30.3%)가 인용돼 나오는지 확인하세요. 추측만 하면 5단계 체크포인트로.

### 5단계 — "실제 숫자로 근거 보여줘" 체크포인트

보조 일꾼이 많아질수록, 누가 어디서 숫자를 가져왔는지 흐려지기 쉽습니다. 한 번 검산을 요청하세요.

```text
매출 담당이 말한 "클로드코드 1,242만원"이 진짜 CSV에 있는 값인지,
course_sales_monthly.csv의 해당 줄을 그대로 인용해서 보여 주세요.
```

**기대 결과**:
```text
course_sales_monthly.csv 26번째 줄 그대로:
2026-03,클로드코드로 업무 자동화 입문,AI활용,자사사이트,118,9,119000,1404,19,1242
→ 매출 1,404만원, 순매출 1,242만원이 맞습니다.
```

> 💡 **왜 이 단계가 중요한가**: 일을 여러 일꾼에게 나누면 빨라지지만, "지어낸 숫자(할루시네이션 = AI가 그럴듯하게 꾸며내기)"도 *여러 곳에서* 생길 수 있습니다. AI는 비서이지 시험감독이 아니니, 의심되면 *원본 줄을 인용해 달라*고 다시 시키면 됩니다(cc-03의 할루시네이션 방지 팁 그대로).

### 6단계 — 나만의 보조 일꾼(커스텀 에이전트) 개념 맛보기

지우가 매주 하는 "문의 분류"를 *전담하는* 보조 일꾼을 만들 수도 있습니다. 만드는 법만 물어봅니다(이번엔 개념만, 실제 파일 생성은 선택).

```text

전담하는 나만의 보조 일꾼을 만들고 싶어요.
어디에 어떤 식으로 정의하는지, 비개발자 눈높이로 알려 주세요.
(지금은 설명만, 실제로 만들진 마세요.)
```

**기대 결과(예시)**:

```text
나만의 보조 일꾼은 .claude/agents/ 폴더에 "이름.md" 파일로 만듭니다.
맨 위에 간단한 설정(이름/설명/쓸 도구/모델)을 적고,
그 아래에 "너는 문의 분류 담당이야. 결제·실습·환불·강의문의 4개로만 나눠"
같은 역할 지시를 우리말로 써 두면 됩니다.
```

**관찰 포인트**:
- 이건 cc-05에서 만든 **CLAUDE.md(규칙서)와 똑같은 원리**입니다 — "규칙을 파일로 박아 두면 매번 자동 적용". 보조 일꾼은 *그 규칙서를 가진 작은 AI*예요.
- 🎯 이 "역할을 파일로 박는다"는 감각이 **Part 5~6의 봇 성격(soul.md)**으로 그대로 자랍니다.

### 7단계 — 나눠 보니 뭐가 좋은지, 비용은 어떤지 점검 (cc-06 복습)

방금 한 일을 "한 명한테 차례차례" 시킨 것과 비교해보게 물어봅니다.

## 🎬 실습 2 — 안전벨트와 알림: Hook 개념 잡고 완료 알림 켜보기 (17분)

지우의 걱정: *"AI가 실수로 내 고객 명단(`student_inquiries.csv`)이나 매출 표를 지우면 어쩌지?"* 이번엔 "내 파일은 안 망가진다"는 안심을 한 단계 굳히고, **가장 쉬운 자동화인 완료 알림**을 직접 켜봅니다. Hook은 두 개만 알면 충분합니다 — **① 위험 차단 Hook**과 **② 완료 알림 Hook**.

> 🟢 **마음 편하게**: 이번 실습의 핵심은 **개념 이해**예요. 설정 파일을 깊이 건드리지 않고, AI에게 "이런 안전장치가 뭔지 / 어떻게 켜는지" 물어보는 방식으로 안전하게 진행합니다.

### 1단계 — Hook이 언제, 어떻게 작동하는지 물어보기

먼저 Claude Code가 **기본으로** 어떤 안전장치를 갖고 있고, Hook은 언제 작동하는지 AI에게 직접 물어봅니다.

```text
나는 비개발자 1인 사업자예요. 실수로 소중한 파일을 지울까 봐 걱정돼요.
Claude Code가 기본적으로 위험한 작업(파일 삭제 같은)을 어떻게 막아주는지,
그리고 'Hook(훅)'이라는 추가 안전장치는 언제 자동으로 작동하는지
비유로 아주 쉽게, 표로 설명해 주세요.
```

**기대 결과(예시)**:
```text
기본 안전장치: Claude Code는 파일을 지우거나 덮어쓰는 위험한 작업을 하기 전에 항상
"이거 해도 될까요?" 하고 먼저 물어봐요. 당신이 yes 하지 않으면 실행되지 않습니다.

Hook(훅)은 그 위에 한 겹 더 채우는 자동 규칙이에요. 작동 시점은:
| 언제 작동? | 무슨 일에 쓰나 |
|-----------|---------------|
| 도구 실행 "직전" (PreToolUse)  | 위험한 명령 미리 차단 |
| 도구 실행 "직후" (PostToolUse) | 자동 정리·기록 남기기 |
| 응답이 끝났을 때 (Stop)        | "완료!" 알림 소리 |
| 알림이 필요할 때 (Notification)| "확인 필요" 알림 |
| 세션이 시작될 때 (SessionStart)| 배경 정보 자동 넣기 |
```

**관찰 포인트**:
- 핵심은 **"직전(PreToolUse)"** — AI가 위험한 일을 *하기 전에* 막는다는 점. 사고가 난 뒤가 아니라 *나기 전에* 잡습니다.
- 작동 원리는 한 줄: **통과(안전)면 그냥 진행, 차단(위험)이면 멈추고 AI에게 이유를 알려 줌.**

### 2단계 — 위험 차단 Hook의 작동 그림 보기

```text
제가 가장 무서운 건 AI가 실수로 파일을 지우는 거예요.
"파일 삭제 같은 위험한 명령을 막는 안전 Hook"이 어떻게 동작하는지,
실제로 막히면 화면에 뭐가 뜨는지 예시로 보여 주세요. (설정 코드는 짧게)
```

**기대 결과(예시)**:
```text
원리: 위험한 패턴(파일 삭제 rm, 강제 삭제, 폴더 통째 삭제 등)을 미리 목록으로 정해 두면,
AI가 그 명령을 실행하려는 "직전"에 Hook이 가로채 차단합니다.

막혔을 때 화면 예시:
  ⛔ 차단됨: 'rm'(삭제) 명령은 금지돼 있습니다.
  → 대안: 파일을 지우는 대신 휴지통(trash)으로 옮기세요.

이렇게 AI는 멈추고, 왜 막혔는지와 대안을 받아 다른 길을 찾습니다.
```

**관찰 포인트**:
- 차단되면 끝이 아니라 **대안을 제시**받습니다("삭제 대신 휴지통으로"). AI가 막다른 길에 안 갇혀요.
- 막을 패턴은 *내가 정합니다* — 지우라면 "고객 명단 CSV 덮어쓰기"를 목록에 추가할 수 있습니다.
- 원리는 단순해요: **"위험하면 멈춤, 안전하면 통과."** 이 한 줄만 기억하면 됩니다.

### 3단계 — 내 사업에 맞는 "지키고 싶은 것" 목록 만들기

기능을 *내 일*로 가져옵니다.

```text
저는 1인 콘텐츠 사업자라 고객 문의 파일(student_inquiries.csv)과
매출 파일(course_sales_monthly.csv)이 제일 소중해요.
이 파일들을 AI가 실수로 지우거나 통째로 덮어쓰지 못하게 하려면
안전 Hook으로 무엇을 막아 두면 좋을지, 막을 목록을 표로 제안해 주세요.
```

**기대 결과(예시)**:
```text
| 막을 것 | 왜 |
|---------|----|
| 파일 삭제(rm) 명령 전반          | 실수 한 번에 데이터가 사라짐 |
| *.csv 파일 통째 덮어쓰기          | 고객·매출 원본 손실 |
| practice-data/ 폴더 통째 이동·삭제 | 실습 자료 보호 |
권장: "삭제 대신 백업본을 먼저 만들기"를 대안으로 안내.
```

**관찰 포인트**:
- Hook은 "AI를 불신해서"가 아니라 **"사람도 실수하니까 거는 안전벨트"**입니다. 운전 잘해도 벨트는 매잖아요.

### 4단계 — 가장 쉬운 자동화: 완료 알림 Hook 켜는 법 물어보기

이제 지우에게 가장 와닿는 것 — **"일 다 끝나면 띵! 소리로 알려주기"** 를 어떻게 켜는지 물어봅니다.

```text
긴 분석을 돌려놓고 커피 한 잔 하고 와도 되게,
AI가 작업을 끝냈을 때 소리로 알려주는 '완료 알림'을 켜고 싶어요.
내 컴퓨터(맥/윈도우)에서 켜는 방법을, 비개발자도 따라할 수 있게
한 단계씩, 가장 안전한 방법으로 알려 주세요.
```

**기대 결과(예시 — 맥 기준)**:
```text
'Stop Hook'(응답이 끝났을 때 작동하는 훅)에 소리 재생 명령을 한 줄 넣으면 됩니다.
- 맥: 작업이 끝날 때 시스템 소리를 울리게 (예: afplay /System/Library/Sounds/Glass.aiff)
- 윈도우/리눅스: 화면 알림(notify)을 띄우는 방식
"끝남" 소리와 "확인 필요" 소리를 다르게 두면, 멀리서도 구분돼요.
설정 파일을 직접 손대기 부담되면 "그 설정을 대신 해줄 수 있어?"라고 부탁해도 됩니다
(AI가 설정을 만들 땐 역시 "해도 될까요?" 하고 물어봅니다).
```

**관찰 포인트**:
- 지우의 화요일은 *종일 유튜브 촬영·편집*이에요. 분석을 걸어 두고 촬영하다가, "땡!" 소리에 와서 결과만 받으면 됩니다.
- "끝남"과 "확인 필요" 소리를 다르게 두면, 화면을 안 봐도 *결과 보면 되는지 / 내가 결정해줘야 하는지*를 소리로 구분할 수 있어요.
- 🎯 이 "끝나면 알림"은 Part 4 자동화·Part 6 봇 팀에서 **"작업 끝났어요" 보고**로 자랍니다.

### 5단계 — 안전한 상태에서 실제로 한 번 시켜 보기

Hook이 걸린(걸렸다고 가정한) 상태에서, *안전한* 작업을 시켜 차이를 체감합니다.

```text
student_inquiries.csv를 "읽어서" 2026-03 환불 문의만 뽑아
요약해 주세요. 원본 파일은 절대 고치지 말고요.
```

**기대 결과(실제 데이터 기준 예시)**:
```text
2026-03 환불 문의(원본 변경 없음, 읽기만):
- 이도윤 / 1인 지식사업 수익화 로드맵 / "생각과 달라서" / 진행중
- 오현우 / 클로드코드로 업무 자동화 입문 / "결제 두 번, 한 건 환불" / 완료
- 장시우 / 초보 뉴스레터 7일 완성 / "효과가 없어서" / 완료
관찰: 환불 사유에 "생각과 다름"이 반복 → 상세페이지 보강 검토 (content_plan §5와 일치)
```

**관찰 포인트**:
- "읽기만, 고치지 마"가 지켜졌는지 확인 — 이게 Hook이 지키려는 바로 그 경계입니다.
- 환불 사유 "생각과 다름"이 `content_plan.md` §5의 고민("환불 사유 중 '생각과 다름'이 반복됨")과 정확히 맞물립니다. 데이터가 서로 말을 거는 순간이죠.

### 6단계 — Hook은 "내가 정한 규칙" — CLAUDE.md와 한 식구

```text
방금 배운 Hook(안전장치)이 cc-05에서 만든 CLAUDE.md 규칙서와
어떻게 다르고, 어떻게 짝이 되는지 한 문단으로 정리해 주세요.
```

**기대 결과(예시)**:
```text
CLAUDE.md = "이렇게 일해 줘"라는 부탁(말투·형식·우선순위) — AI가 참고하는 안내문.
Hook      = "이건 절대 안 돼"라는 강제 장치 — AI가 어겨도 시스템이 막음.
둘은 짝: CLAUDE.md로 방향을 주고, Hook으로 안전선을 친다.
```

**관찰 포인트**: 규칙서가 "이렇게 일해줘"라면, 안전 훅은 "이건 시스템이 자동으로 막아"예요. 둘이 짝을 이루면 더 든든합니다. 마지막으로 `/compact`로 대화를 정리하고 다음으로 갑니다.

> ✅ **실습 2 한 줄 정리**: "Claude Code는 원래 위험한 일 전에 물어본다. **Hook**은 그 위에 ① 위험 자동 차단(안전벨트) ② 완료 알림(띵!)을 더 채우는 옵션이다."

## 📝 실습 3 (선택·시간 남으면) — 자동화 한 줄 맛보기: 주간 인사이트 뽑기 (7분)

"사람 없이 한 번에 끝내기(`-p`)"가 어떤 느낌인지 **딱 한 줄**로 체험합니다. 이게 나중에 "예약 실행"의 씨앗이에요.

> 🟢 **마음 편하게**: 진짜 매주 자동 예약까지 거는 건 컴퓨터 깊은 설정이라 오늘은 안 해요. 오늘은 **"한 줄로 시키고 결과 파일까지 받는"** 자동화의 *맛만* 봅니다.

### 1단계 — `claude`를 켜지 않고, 터미널에서 바로 한 줄 시키기

지금까지는 `claude`를 켜고 대화했죠. 이번엔 **켜지 않고**, 터미널에서 바로 한 번에 시킵니다. (대화 중이면 먼저 `/exit`로 나와 터미널 `$` 상태로 가세요.)

```text
claude -p "creator 폴더의 newsletter_stats.csv를 읽고, 이번 주 뉴스레터에 쓸 인사이트 3개를 한국어 불릿으로 뽑아줘"
```

**왜 이렇게 하나요?**: `-p`는 이번 모듈에서 처음 만나는 옵션이에요 — 사람 없이 **"한 번 답하고 끝"**. 지금까지는 `claude`를 켜서 *대화*했지만, 이건 대화를 이어가지 않으니 나중에 "매주 일요일 8시에 이 한 줄을 자동 실행"으로 확장할 수 있어요.

**기대 결과(실제 데이터 기준 예시)** — 답이 화면에 주르륵 나오고 **바로 끝납니다**(입력창이 안 남아요):
```text
- AI 주제 호가 압도적 — 33호 "AI에게 일을 시키는 사람 vs 못 시키는 사람" 클릭율 24.7%,
  36호 "터미널이 무서웠던 내가..." 25.8%. 이번 호도 'AI로 일하기' 각도 권장.
- 마감 임박 카피가 최고 — 40호 "사전등록 마감 임박(오늘 밤 12시)" 클릭율 30.3%로 1위.
  긴급성 카피가 가장 잘 눌림.
- 누적 구독 5,755명까지 성장(18호 3,192명 → 41호) — 5,000명 돌파 후 후기 콘텐츠(41호)로 안정화.
```

**관찰 포인트**: 답이 한 번 나오고 바로 끝나죠? 이게 "사람 없이 돌리는" 방식의 느낌이에요.

### 2단계 — 결과를 파일로 저장하는 형태로 바꿔보기

자동화는 보통 결과를 **파일로 남깁니다**(아침에 와서 파일만 열어보게). 한 줄을 살짝 바꿔봅니다.

```text
claude -p "creator 폴더의 newsletter_stats.csv를 읽고, 이번 주 뉴스레터 인사이트 3개를 '주간인사이트.md' 파일로 이 폴더에 저장해줘" --allowedTools "Read,Write"
```

**무엇이 다른가요?**: 끝의 `--allowedTools "Read,Write"`는 "이번 자동 실행에서는 **읽기와 쓰기만** 허락한다"는 뜻이에요. 사람이 옆에서 yes를 못 누르는 자동 실행이니, **할 수 있는 일을 미리 좁혀** 안전하게 만드는 거죠 — **실습 2의 Hook(안전선) 정신이 자동화에도 그대로** 들어갑니다. 사람이 안 볼수록 안전선은 더 중요하니까요.

**기대 결과**: `주간인사이트.md` 파일이 생성됩니다.

**관찰 포인트**: "이 한 줄을 매주 자동으로 돌리면, 일요일 아침에 인사이트가 이미 파일로 준비돼 있겠구나"를 상상하면 자동화의 그림이 완성돼요. (실제 "매주 자동 실행"하는 예약·cron 설정과 GitHub 자동 연동은 더 깊은 주제라 부록에선 "이런 게 가능하다"까지만.)

### 3단계 — "내 일에 자동화할 만한 것" 하나 떠올리기

마지막으로, 지우 맥락에서 자동화하면 좋을 반복 작업을 AI에게 추천받습니다.

```text
제 일정 노트(consulting_schedule.md)와 기획 노트(content_plan.md)를 참고해서,
제가 매주·매월 손으로 반복하는 일 중에 '한 줄 자동 실행'으로 미래에 자동화하면
시간을 아낄 만한 일 3가지를, 이유와 함께 골라 주세요.
```

**기대 결과(실제 노트 기준 예시)**:
```text
1. 일요일 뉴스레터 발송 전 "지난주 성과 3줄 요약" 자동 생성
   → 매주 반복 + 데이터(newsletter_stats.csv)가 이미 있음.
2. 월말(03-31 '월 마감 정산') "강의별 매출 정산 요약" 초안 자동 작성
   → consulting_schedule의 정산 일정과 직결, 매월 반복.
3. 반복되는 WSL/터미널 설치 문의용 "설치 안내 FAQ 초안" 자동 갱신
   → student_inquiries에 설치 문의 반복(정유나·배준혁 등), content_plan §5와도 일치.
```

**관찰 포인트**: 자동화 1순위 = **"반복 + 데이터가 이미 있음"**. 지우의 일은 이 조건에 딱 맞는 게 많습니다.

> 🎯 **세 실습이 만나는 곳**: 자동화 한 줄(`-p`) 안에 *서브에이전트로 나눠 분석*하고, *Hook/allowedTools로 안전선*을 치는 게 다 들어갑니다. 그리고 이 전체 그림이 **Part 4 자동화·Part 5~6 AI 에이전트 팀**의 씨앗입니다 — "매월 1일, 봇이 알아서 매출·문의 리포트를 만들어 둔다"가 바로 여기서 출발합니다.

> ✅ **실습 3 한 줄 정리**: "`claude -p \"...\"` = 사람 없이 한 번에 시키기. 결과를 **파일로 저장**하게 만들면, 나중에 **예약 실행**으로 키울 수 있다."

## 🔍 흔한 오류와 해결법 (트러블슈팅)

| 상황 | 원인 | 해결책 |
|------|------|--------|
| `Task(...)` 같은 일꾼 줄이 안 보임 / 일반 답만 옴 | 일이 작아 AI가 "나눌 필요 없다"고 판단했거나, "나눠줘"라고 명시 안 함 | 정상이에요. 파일이 적으면 한 명이 더 효율적. "이 작업을 서브에이전트(보조 일꾼)에게 나눠 맡겨줘"라고 분명히 요청 |
| 비용·시간이 갑자기 확 늘었다 | 작은 일에 보조 일꾼을 너무 많이 부름 | 단순 작업은 메인에서. cc-06의 `/compact`로 기억 정리 + "꼭 나눌 일만" 나누기 |
| 보조 일꾼이 숫자를 지어낸 것 같다 | 여러 갈래라 근거가 흐려짐 | "원본 CSV 줄을 그대로 인용해 줘"로 검산 (실습1 5단계) |
| 안전 Hook이 안 막는다 | 막을 패턴 목록에 그 명령이 없음 | "이 명령도 차단 목록에 넣어 줘"라고 추가 요청 / 패턴 철자 확인 |
| Hook이 정상 작업까지 막는다 | 차단 패턴이 너무 넓음 | "읽기·요약은 허용하고 삭제만 막아 줘"로 범위 좁히기 |
| 완료 알림 소리가 안 난다 | OS에 안 맞는 명령이거나 컴퓨터 음소거 | 음량 확인 → 맥은 시스템 사운드, 윈도우는 화면 알림 → "내 OS에 맞게 다시" 요청 |
| AI가 위험 작업을 막고 멈췄다 | 안전장치(또는 안전 훅)가 정상 작동한 것 | 오류가 아니라 **잘 막은 거예요.** 정말 해도 되면 다시 명확히 지시하거나 yes로 승인 |
| `claude -p "..."` 가 파일을 못 만듦 | 자동 실행이라 쓰기 권한이 막혀 있음 | 명령 끝에 `--allowedTools "Read,Write"`를 붙여 읽기·쓰기 허락 |
| `claude -p` 결과가 화면에만 나오고 사라짐 | `-p`는 원래 "한 번 답하고 끝" | 결과를 남기려면 "~를 파일로 저장해줘"를 지시문에 넣기 |
| 파일을 못 찾는다 | 폴더 위치가 틀림 | `pwd`로 현재 위치 확인 후 `practice-data/creator/파일명` 정확히 |

> **마음가짐**: 어떤 명령을 잘못 쳐도 컴퓨터는 안 망가집니다. Hook과 기본 안전장치가 위험한 일을 먼저 막아줘요. **틀려도 괜찮으니 일단 해보세요.**

## 💡 핵심 규칙/개념 정리

### 규칙 1 — 나눌 일만 나눈다 (서브에이전트)
여러 명을 고용하면 빠르지만 비쌉니다. "서로 무관한 작업을 각각 깊게"일 때만 분담하세요. 짧은 단일 작업은 메인이 낫습니다. 메인이 지휘, 일꾼은 *요약만* 보고 → 그래서 깔끔합니다.

### 규칙 2 — 방향은 CLAUDE.md, 안전선은 Hook
`CLAUDE.md`는 "이렇게 일해 줘"(부탁), Hook은 "이건 절대 안 돼"(강제). 둘은 짝입니다. 소중한 파일(고객·매출)은 Hook으로 못 박으세요.

### 규칙 3 — 두 Hook이면 충분하다
부록 수준에선 **① 위험 차단 + ② 완료 알림** 두 개만 알면 됩니다. 그리고 Claude Code는 Hook 없이도 위험 작업 전에 물어봅니다 — Hook은 그 위에 한 겹 더.

### 규칙 4 — 자동화 1순위 = "반복 + 데이터 있음"
매주·매월 똑같이 하는 일 중, 데이터가 이미 있는 것부터 자동화 후보입니다. 사람이 안 볼수록 `--allowedTools`로 안전선을 더 단단히.

### 규칙 5 — AI는 비서지 시험감독이 아니다
보조 일꾼이 늘어도 원칙은 같습니다. 의심되면 "원본 인용해 줘", 마음에 안 들면 "다시 해 줘". 몇 번이고 괜찮습니다.

### 규칙 6 — 전부 cc-05·cc-06과 한 가족
규칙서(`CLAUDE.md`)·세션 관리·비용 감각이 여기서 그대로 다시 쓰입니다. 새 기능이지만 뿌리는 이미 배운 것이에요.

> 🟢 **부록 레슨다운 한마디**: 오늘 셋을 다 못 익혀도 괜찮아요. **"비서를 여럿으로 / 안전하게 / 알아서 돌게 할 수 있다"**는 그림 하나만 가져가면 대성공입니다.

```
  매출/문의/뉴스           위험 작업 차단              사람 없이
  세 갈래 동시 분석         + 완료 띵! 알림              한 줄 실행
        │                        │                         │
        └──── 결과만 모음 ──── 안전선 유지 ──── 파일로 저장 → 다듬어 사용 ────┐
                                                                          ▼
                              (미래: 매주 일요일 9시 예약 = 밥솥 예약취사)
                                                                          ▼
                                       [ Part 4 자동화 · Part 5~6 AI 봇 팀의 씨앗 ]
```

핵심 한 줄: **나눈다(서브에이전트) → 안전하게(Hooks) → 사람 없이(자동화).** 한 사람 비서가 → **여러 명**이 되고 → **안전벨트**를 차고 → **예약 취사**처럼 알아서 돌아가는 흐름입니다.
