# cc-09: MCP 외부도구 연결 — "AI 비서에게 열쇠 꾸러미를 쥐여주는 날"

> **레슨 목표**: Claude Code에 MCP(Model Context Protocol, 외부 도구 연결 표준)를 연결해, 파일 폴더 너머의 Notion·Google Sheets·웹사이트까지 AI가 직접 들어가 읽고 정리하게 만든다.
> **예상 시간**: 50분 (개념 도입 10분 + 실습 1 기본기 15분 + 실습 2 실전 활용 20분 + 실습 3 선택 5분)
> **사전 준비**: Claude Code 설치·로그인 완료(cc-01~cc-06 이수) / 인터넷 연결 / 터미널 1개 열어두기
> **진입 커맨드**: `/cc-09`
> **부록 위치**: 이 레슨은 self-study 부록입니다. 강의 본편(cc-01~cc-08)을 마친 뒤 "AI 비서에게 더 많은 손을 달아주고 싶다"는 분이 선택해서 보세요.

## 📚 학습 목표 — 4가지 와우 모먼트

1. **"복붙이 사라진다"** — Notion·Google Sheets·웹사이트에 AI가 직접 들어가 읽고 정리하는 걸 직접 체험한다.
2. **"USB 꽂듯 쉽다"** — 명령어 한 줄로 외부 도구를 Claude Code에 연결하는 흐름을 직접 따라한다.
3. **"내 사업 데이터가 달라진다"** — course_sales_monthly.csv·student_inquiries.csv 같은 실제 파일을 분석하고 Notion으로 보내는 흐름을 경험한다.
4. **"언제 MCP가 필요한지 스스로 판단할 수 있다"** — 지금 배운 것으로 충분한 경우와 MCP가 시간을 절약해 주는 경우를 구분하는 기준을 내면화한다.

## 🎯 개념 도입 — "AI 비서에게 열쇠 꾸러미를 쥐여주는 것"

### 지금까지 지우의 Claude Code는 어디까지 들어갔나

cc-01~cc-08 실습을 통해 지우는 Claude Code에게 practice-data/creator 폴더를 열어 줬습니다. AI가 `course_sales_monthly.csv`를 읽고 강의별 매출 패턴을 찾고, `student_inquiries.csv`에서 반복 문의를 뽑고, `content_plan.md`의 기획 노트를 정리해줬습니다. 이건 전부 **"내 컴퓨터 안에 있는 파일"**을 AI가 읽어서 처리하는 것이었습니다.

그런데 한 가지 벽이 남아 있습니다.

> **"정리한 결과를 Notion에 넣으려면 직접 복사해서 붙여 넣어야 한다."**
> **"경쟁사 강의 페이지를 리서치하려면 내가 직접 읽고 옮겨야 한다."**
> **"뉴스레터 구독자 통계를 Google Sheets에 자동으로 올리려면 손이 간다."**

이 마지막 벽을 허무는 것이 바로 **MCP**입니다.

---

### MCP = AI 비서에게 열쇠 꾸러미를 쥐여주는 것

지우의 Claude Code 비서는 지금 "내 컴퓨터 폴더"라는 방 하나에만 들어갈 수 있습니다. MCP는 이 비서에게 "Notion 사무실 열쇠", "Google Sheets 서랍 열쇠", "웹사이트 도서관 열쇠"를 하나씩 추가로 쥐여주는 것입니다. 열쇠가 많을수록 비서가 갈 수 있는 곳이 늘어납니다.

또 다른 비유로 보면 — USB 포트와 같습니다. 컴퓨터 본체는 그대로인데 USB에 마우스·키보드·외장 하드를 꽂으면 할 수 있는 일이 늘어납니다. MCP(Model Context Protocol, 모델 맥락 연결 규약)가 Claude Code에 달린 USB 포트이고, 각 외부 도구(Notion·Slack·Google Sheets)가 꽂는 주변기기입니다. 꽂으면 작동하고, 뽑으면 연결이 끊깁니다.

```
기존 Claude Code:
  지우  ←→  Claude Code  ←→  내 컴퓨터 파일만

MCP를 추가한 Claude Code:
  지우  ←→  Claude Code  ←→  내 컴퓨터 파일
                          ←→  Notion (페이지 읽기·쓰기)
                          ←→  Google Sheets (스프레드시트)
                          ←→  웹 브라우저 (URL 읽기)
                          ←→  Slack (채널 요약·공지)
```

MCP는 Anthropic(클로드를 만든 회사)이 2024년에 발표한 **오픈 표준**입니다. "어떤 외부 도구든 이 규격으로 만들면 Claude Code에 꽂을 수 있다"는 약속입니다. 덕분에 세계 각지의 개발자들이 Notion MCP, Google Drive MCP, Slack MCP 등을 만들어 무료로 공개하고 있습니다.

---

### MCP가 생기면 무엇이 달라지나 — 지우의 실제 상황 비교

| 작업 | MCP 없이 | MCP 있을 때 |
|------|---------|------------|
| 수강생 문의를 Notion에 정리 | 터미널에서 분석 → 결과 복사 → Notion 열기 → 붙여넣기 | "student_inquiries.csv 읽고 Notion에 정리해줘" 한 마디 |
| 뉴스레터 오픈율 Google Sheets에 기록 | newsletter_stats.csv → 복사 → 시트 열기 → 직접 입력 | "newsletter_stats.csv 읽고 시트에 채워줘" |
| 경쟁사 강의 페이지 분석 | 웹사이트 방문 → 복사 → 터미널에 붙여넣기 | "이 URL 읽고 경쟁 강의 차별점 정리해줘" |
| Slack에 수강생 Q&A 요약 발송 | 분석 결과 복사 → Slack 열기 → 채널에 붙여넣기 | "weekly Q&A 요약 Slack 채널에 올려줘" |

"복붙(복사 붙여넣기)"이 사라집니다. 도구 간 이동이 사라집니다. 지우가 집중해야 할 "다음 강의 기획"에 그 시간을 쓸 수 있게 됩니다.

---

### MCP의 구조 — 딱 세 가지만 알면 된다

**① MCP 서버(server)** — "열쇠를 발급하는 곳"

Notion, Google, Slack 같은 외부 도구 회사(또는 오픈소스 커뮤니티)가 "우리 서비스에 Claude Code가 들어올 수 있도록" 만들어 둔 연결 통로입니다. 이미 만들어진 것을 가져다 쓰면 되고, 직접 만들 필요는 없습니다.

**② 연결 방식(transport)** — "열쇠를 어떻게 건네주나"

- **원격 HTTP(http)**: 인터넷 어딘가에 있는 서버에 연결. `claude mcp add --transport http 이름 URL` 형식. 설치가 간단하고 권장.
- **로컬 stdio**: 내 컴퓨터 안에 직접 설치된 서버에 연결. 더 세밀하게 제어 가능하나 설정이 복잡.

비개발자인 지우는 처음엔 **원격 HTTP 방식**을 씁니다.

**③ 스코프(scope)** — "이 열쇠를 어디까지 쓸 수 있나"

| 스코프 | 의미 | 언제 쓰나 |
|--------|------|----------|
| 유저(user) | 내 컴퓨터 전체에서 이 MCP 사용 | Notion처럼 모든 폴더에서 쓸 때 |
| 로컬(local) | 지금 폴더에서만 사용 | 특정 프로젝트에서만 연결할 때 |
| 프로젝트(project) | 팀원과 공유(.mcp.json에 저장) | 팀 공동 작업 |

혼자 일하는 지우는 **유저 스코프**가 가장 편합니다. 한 번 등록하면 어느 폴더에서 `claude`를 켜도 그 MCP가 쓰입니다. `claude mcp add` 명령의 기본 스코프는 **유저(user)**입니다. 명시적으로 지정하고 싶으면 `--scope user`(또는 `--user`) 플래그를 추가할 수 있습니다.

> 💡 **중요한 구분**: MCP는 강력한 기능이지만, 지금까지 익힌 것(파일 읽기·CLAUDE.md·Plan 모드)과 **완전히 별개**입니다. MCP가 없어도 지우의 실무 대부분은 잘 됩니다. MCP는 "더 하고 싶을 때 추가하는 옵션"입니다.

> ⚠️ **MCP 서버 URL은 자주 바뀝니다**: 이 레슨의 URL과 설치 방식은 개념과 흐름을 익히기 위한 예시입니다. 실제 프로덕션 사용 전에 반드시 `docs.anthropic.com/mcp` 또는 각 도구 공식 문서에서 최신 서버 주소와 인증 방식을 확인하세요. 이 주의사항은 실습 1·2·3 전체에 적용됩니다.

> 💡 **다른 트랙(store·freelancer·local) 수강생에게**: 본 레슨 예시는 creator 트랙으로 진행합니다. store 트랙이면 상품 리뷰 데이터를 Notion에 보내는 식으로, freelancer 트랙이면 견적서를 Google Sheets에 정리하는 식으로 자기 트랙 데이터로 동일하게 적용할 수 있습니다.

## 🔧 실습 1 — MCP 서버 추가·확인·제거 (기본기 15분)

### 1단계: 터미널에서 현재 MCP 목록 확인하기

Claude Code 대화방(`claude`)을 켜기 **전에**, 터미널에서 직접 아래 명령을 입력합니다. 이건 대화방 안이 아니라 터미널 명령어입니다.

```bash
claude mcp list
```

**기대 결과 (아무것도 없을 경우):**
```
No MCP servers configured.
```

**기대 결과 (무언가 있을 경우):**
```
notion: http - https://mcp.notion.com/mcp
```

**관찰·체크포인트:**
- 지금 아무것도 없어도 정상입니다. 이제 추가해볼 거예요.
- `claude mcp 무엇`의 형태가 MCP를 관리하는 명령어입니다. 대화방 없이 터미널에서 직접 씁니다.

---

### 2단계: Notion MCP 연결하기 (첫 번째 열쇠)

Notion은 지우가 강의 운영 메모, 콘텐츠 캘린더, 수강생 현황을 정리해두는 도구입니다. Claude Code와 연결하면 "Notion에 정리해줘"가 실제로 작동합니다.

```bash
claude mcp add --scope user --transport http notion https://mcp.notion.com/mcp
```

**명령어 해부:**
- `claude mcp add` — "MCP 하나를 추가해줘"
- `--scope user` — "내 컴퓨터 전체에서 쓸 수 있게 유저 스코프로 저장해줘" (기본값이기도 하므로 생략해도 결과 동일)
- `--transport http` — "인터넷 서버 방식으로 연결해줘"
- `notion` — "이 연결의 이름은 'notion'이라고 부를게"
- `https://mcp.notion.com/mcp` — "연결할 서버 주소"

**기대 결과:**
```
Added MCP server "notion" (http) to user config
```

> ⚠️ **Notion 계정이 없다면**: 이 단계에서 Notion 계정과 Claude Code를 연동하는 인증 화면이 뜰 수 있습니다. 브라우저 창이 열리면 Notion 로그인 후 "허용(Allow)" 버튼을 누르면 됩니다. Notion 계정이 없으면 이 단계는 건너뛰고 3단계(fetch MCP)로 가도 됩니다.

---

### 3단계: Fetch MCP 연결하기 (웹사이트를 읽는 열쇠)

지우는 경쟁사 뉴스레터나 강의 페이지를 분석할 때 항상 손으로 복사했습니다. `fetch` MCP를 연결하면 URL만 주면 AI가 직접 웹사이트를 읽습니다.

```bash
claude mcp add --transport stdio fetch -- npx -y @modelcontextprotocol/server-fetch
```

**명령어 해부:**
- `--transport stdio` — "내 컴퓨터에서 직접 실행하는 방식"
- `fetch` — "이 연결의 이름은 'fetch'"
- `-- npx -y @modelcontextprotocol/server-fetch` — "이 패키지를 설치해서 서버로 써줘"

> 💡 **npx가 없다면**: Node.js가 설치돼 있어야 `npx`가 작동합니다. cc-01에서 Node.js를 설치했다면 이미 있습니다. `npx --version`을 쳐서 숫자가 나오면 OK.

**기대 결과:**
```
Added MCP server "fetch" (stdio) to user config
```

---

### 4단계: 전체 연결 상태 확인하기

```bash
claude mcp list
```

**기대 결과:**
```
notion: http  - https://mcp.notion.com/mcp
fetch:  stdio - npx -y @modelcontextprotocol/server-fetch
```

**관찰·체크포인트:**
- 이름과 방식, 주소가 보이면 성공입니다. 이 목록이 "지우의 AI 비서가 가진 열쇠 꾸러미"입니다.
- MCP를 연결해도 **Claude Code를 켜지 않으면 아무 일도 일어나지 않습니다.** 열쇠를 꾸러미에 넣어둔 것뿐이고, 실제로 문을 열지 않은 상태입니다. `claude`를 켜고 "Notion에 정리해줘"라고 시켜야 그때 비로소 열쇠를 씁니다.

---

### 5단계: MCP 설정 파일 직접 보기

MCP 설정은 JSON 파일(중괄호와 콜론으로 이루어진 데이터 형식)로도 관리됩니다.

```bash
cat ~/.claude.json
```

**기대 결과 (MCP가 추가된 경우):**
```json
{
  "mcpServers": {
    "notion": {
      "transport": "http",
      "url": "https://mcp.notion.com/mcp"
    },
    "fetch": {
      "transport": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-fetch"]
    }
  }
}
```

**관찰·체크포인트:**
- 이 파일은 CC의 "장비 목록"입니다. `CLAUDE.md`가 "어떻게 일할까 규칙서"라면, `.claude.json`은 "어떤 도구를 쓸까 장비 목록"입니다.
- 이 파일을 텍스트 편집기로 직접 편집해서 MCP를 추가할 수도 있습니다. 명령어 방식과 결과가 같습니다.

---

### 6단계: MCP 서버 제거하기

실습이 끝나면 필요 없는 서버를 깔끔하게 제거합니다.

```bash
# MCP 서버 제거
claude mcp remove fetch

# 제거 확인
claude mcp list
```

**기대 결과:**
```
notion: http - https://mcp.notion.com/mcp
```

**관찰·체크포인트:**
- 제거해도 실제 Notion 계정이나 데이터에는 아무 영향이 없습니다. CC의 "도구 목록"에서만 빠지는 것입니다.
- USB를 뽑는 것과 같아요 — 장치가 없어지는 게 아니라 연결이 끊기는 것입니다.
- 나중에 다시 쓰고 싶으면 `claude mcp add`로 다시 추가하면 됩니다.

## 🎬 실습 2 — 지우의 실제 데이터로 MCP 활용하기 (20분)

이 실습에서는 practice-data/creator의 실제 파일을 분석하고, MCP가 있으면 어떻게 달라지는지 단계별로 체험합니다.

### 1단계: practice-data 폴더로 이동하고 CC 열기

```bash
# 먼저 실습 폴더 위치를 찾는다
find ~ -name "course_sales_monthly.csv" 2>/dev/null
```

위에서 나온 경로의 폴더가 `practice-data/creator`입니다. 아래 경로는 **예시**이니 찾은 실제 경로로 바꿔 입력한 뒤 CC를 엽니다.

```bash
cd <위에서 찾은 practice-data/creator 폴더 경로>
claude
```

**기대 결과:** 대화방(입력창 `>`)이 열림. Claude Code가 현재 creator 폴더를 인식한 상태.

> 💡 **폴더가 어디 있는지 모르겠다면**: `find ~ -name "course_sales_monthly.csv" 2>/dev/null`을 치면 파일 경로가 나옵니다.

대화창이 열리면 먼저 어떤 MCP가 쓸 수 있는 상태인지 물어봅니다.

```text
지금 사용 가능한 외부 도구(MCP)가 있으면 목록으로 알려줘.
그리고 이 폴더에 어떤 파일들이 있는지도 함께 파악해줘.
나는 1인 콘텐츠 사업자 지우야. 온라인 강의·뉴스레터·컨설팅을 혼자 운영해.
```

**기대 결과:**
```
현재 연결된 MCP 도구:
- notion: Notion 페이지·데이터베이스 읽기·쓰기 가능

폴더에 다음 파일이 있습니다:
1. course_sales_monthly.csv — 강의별 월 매출 (2025년 7월~2026년 3월)
2. student_inquiries.csv — 수강생 문의 내역 (2026년 3월)
3. newsletter_stats.csv — 뉴스레터 성과 (18~41호)
```

### 2단계: 분석에서 나온 인사이트 정리

```text
방금 분석한 뉴스레터 데이터와 앞서 분석한 course_sales_monthly.csv를 함께 보면,
어떤 패턴이 보여?
"AI 관련 뉴스레터 발행 → 구독자 증가 → 강의 매출"의 연결고리가 실제로 성립하는지,
실제 수치를 근거로 설명해줘.
```

**기대 결과:**
```
세 데이터를 연결하면:

뉴스레터 AI 콘텐츠 집중 → 구독자 폭증 → 강의 매출 급등

- 2026년 1월: AI·클로드코드 뉴스레터 4편 집중 발행 → 신규 구독자 +787명(역대 최고)
  같은 시기 course_sales_monthly.csv: 챗GPT 강의 1,252만원(당월 최고)

- 2026년 2월: 36호 "터미널이 무서웠던 내가 AI 비서를 갖게 되기까지" (오픈율 61%, 클릭율 25.8%)
  클로드코드 강의 출시 첫 달(508만원) → 다음 달 1,242만원으로 2.4배 성장

- 21호 "챗GPT 프롬프트 모음집 무료 배포" (오픈율 61.1%, 클릭율 26.4%, 구독자 +210명)
  무료 제공이 신규 구독자를 끌어들이는 핵심 전략으로 확인.

결론: AI 주제 뉴스레터 → 구독자 증가 → 강의 매출 상승의 연결고리는 데이터로 성립.
      다음 달 AI 관련 뉴스레터를 주 1회 유지하면 구독자·매출 동시 효과 기대 가능.
```

**관찰·체크포인트:**
- newsletter_stats.csv, course_sales_monthly.csv의 분석을 연결하면 지우 사업의 핵심 성장 엔진이 보입니다.
- 이 연결 인사이트를 Notion에 자동 저장했습니다. 지우는 이 페이지를 매달 업데이트해서 사업 패턴을 추적할 수 있습니다.

## 🔍 흔한 오류와 해결법

| 상황 | 원인 | 해결 |
|------|------|------|
| `claude mcp list`에 추가한 서버가 안 보임 | 명령어 오타 또는 스코프 문제 | `--scope user` 플래그 추가: `claude mcp add --scope user --transport http ...` |
| MCP 서버 연결 후 대화방에서 "해당 도구 없음" 오류 | CC를 재시작하지 않음 | `/exit` 후 다시 `claude` 입력(재시작) |
| Notion MCP 연결했는데 "권한 없음" 오류 | Notion API 토큰 미설정 또는 인증 미완료 | Notion 설정 > 내 연결에서 Claude 연동 재허용 |
| fetch MCP에서 URL 오류 ("페이지를 읽을 수 없음") | 로그인이 필요한 페이지 또는 JS 전용 페이지 | 로그인 없이 공개된 URL로 시도. 유료 콘텐츠 페이지는 fetch 불가 |
| `npx` 명령어를 찾을 수 없음 | Node.js가 설치 안 됨 | cc-01 트러블슈팅 참고. `brew install node` 또는 nodejs.org에서 설치 |
| MCP 추가 후 CC가 평소보다 느림 | 외부 서버 응답 대기 시간 | 정상입니다. 외부 도구 연결은 파일 읽기보다 1~3초 더 걸림 |
| MCP 삭제하고 싶음 | 잘못 연결했거나 안 쓰게 됨 | `claude mcp remove 이름` (예: `claude mcp remove notion`) |
| 대화방에서 AI가 MCP 도구를 쓰지 않음 | AI가 MCP 사용 여부를 스스로 판단 | 명시적으로 "Notion MCP를 써서 저장해줘"라고 요청 |
| MCP 서버 주소가 작동하지 않음 | URL이 변경됐거나 서비스 정책 변경 | 공식 문서(docs.anthropic.com/mcp)에서 최신 주소 확인 |

## 💡 핵심 규칙·개념 정리

### 규칙 1: MCP는 "꽂고 쓰면 끝"

설정 이후로는 `claude mcp add`를 반복할 필요가 없습니다. 한 번 연결하면 Claude Code를 켤 때마다 자동으로 활성화됩니다. 유저 스코프로 연결했다면 어느 폴더에서 `claude`를 켜도 작동합니다.

### 규칙 2: 말로 시키면 된다

MCP가 연결돼 있으면 "Notion에 정리해줘", "이 URL 읽어줘"처럼 자연어로 지시하면 됩니다. 어느 MCP를 쓸지, API(외부 서비스 연결 통로)를 어떻게 호출할지는 Claude Code가 알아서 판단합니다.

### 규칙 3: CLAUDE.md에 MCP 활용 규칙도 넣을 수 있다

cc-05에서 만든 CLAUDE.md 규칙서에 MCP 관련 지침을 추가하면, 매번 "Notion에 저장해줘"라고 말하지 않아도 자동으로 저장됩니다.

```markdown
## MCP 활용 규칙
- 분석 결과를 정리할 때는 항상 Notion에 새 페이지로 저장해줘
- 웹 리서치가 필요하면 fetch로 실제 URL을 읽어줘 (기억 속 내용으로 지어내지 말 것)
- Notion 페이지 제목 형식: "YYYY-MM 분석 주제명"
- Notion 연결이 없으면 결과를 `output/YYYY-MM-DD-분석유형.md`로 저장
```

### MCP를 쓸 때 vs 안 써도 될 때

| MCP 없이도 충분한 경우 | MCP가 시간을 절약해주는 경우 |
|---|---|
| 내 컴퓨터 파일만 분석할 때 | 분석 결과를 Notion·Sheets에 **매번** 업데이트할 때 |
| 결과를 파일로 저장해도 될 때 | 인터넷 검색·사이트 내용을 **자동으로** 가져올 때 |
| 한 번만 하는 작업 | 여러 플랫폼을 **동시에** 다룰 때 |
| 반복성이 낮은 작업 | 같은 작업을 **주기적으로** 반복할 때 |

### 비개발자가 MCP를 쓸 때 주의할 점

**1. 인증 정보는 파일에 직접 쓰지 말기**: Notion 토큰, API 키 같은 비밀 정보는 `.mcp.json`에 넣지 마세요. 환경변수나 별도 비밀 관리 파일을 쓰는 것이 안전합니다.

**2. MCP가 하는 일을 한 번씩 확인하기**: "Notion에 저장해줘"라고 시키면 CC가 실제로 Notion 페이지를 만들거나 수정합니다. 중요한 데이터가 있는 워크스페이스에서는 먼저 테스트 페이지에 써보세요.

**3. 공식 서버 우선**: 가능하면 공식 서비스(Notion 공식 MCP 등)나 Anthropic이 검증한 서버를 우선 사용하세요. 출처 불명의 MCP 서버는 보안 위험이 있을 수 있습니다.

**4. 서버 URL은 최신 공식 문서로 확인**: MCP 생태계는 빠르게 변합니다. 이 레슨의 URL이 작동하지 않으면 `docs.anthropic.com/mcp`에서 최신 주소를 찾으세요.
