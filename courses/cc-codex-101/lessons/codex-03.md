# codex-03: 모델·effort·응답 차이 — "같은 부탁, 다른 온도로 시키기"

> **레슨 목표**: Codex에서 모델(model)과 effort(추론 강도)를 바꾸면 같은 부탁의 답이 얼마나 달라지는지 직접 눈으로 확인한다. 속도·깊이·비용 세 손잡이를 내 일의 난이도에 맞춰 고르는 감각을 익힌다.
> **예상 시간**: 50분 (개념 도입 8분 + 실습 1 · 15분 + 실습 2 · 20분 + 실습 3 선택 · 7분 + 마무리·체크리스트 5분)
> **사전 준비**: Codex 설치·로그인 완료 (`codex --version` 출력 확인) · CLI 기본 사용법 익힘(codex-02 완료) · `practice-data/creator/` 폴더 준비 · 터미널 1개 열기
> **진입 커맨드**: `/codex-03`

> 🔎 **명령·모델명·플래그 안내**: 아래 내용은 codex 0.140.0 기준으로 정리했습니다. 시연 전 현장에서 `/model` 명령으로 실제 목록을 한 번 확인하세요.

---

## 📚 학습 목표 — 3가지 와우 모먼트

1. **"같은 부탁을 더 꼼꼼하게 / 더 빠르게 시킬 수 있다"** — effort 다이얼의 존재를 처음 느끼는 순간. 빠른 확인과 깊은 분석이 같은 대화창에서 화면에 동시에 보인다.
2. **"모델을 바꾸면 답의 깊이·속도가 달라진다"** — AI에도 '엔진 등급'이 있다는 감각. 단순 작업엔 가벼운 엔진, 중요한 결정엔 무거운 엔진.
3. **"무조건 최고급이 아니라 일에 맞게 골라 쓰면 된다"** — 속도·비용·품질을 내가 조절할 수 있다는 자신감. 그리고 CC에서 익힌 감각이 Codex에서도 그대로 통한다.

## 🎯 개념 도입 (8분)

### 비유: 일꾼에게 "얼마나 공을 들일지" 설정하는 것

지우가 아르바이트생을 구했다고 상상해 보세요.

매일 아침 "오늘 매출 파일 좀 보고 요약해 줘"라고 시킬 수 있습니다.
- 어떤 날은 **빠르게 훑어보고 핵심만** 알려주면 됩니다. 급할 때.
- 어떤 날은 **꼼꼼히 읽고, 왜 그런 수치가 나왔는지, 다음엔 어떻게 하면 좋을지까지** 확인해 주면 좋습니다. 중요한 결정 앞에서.

Codex에는 이 설정이 있습니다. 바로 **effort(에포트, 추론 강도)**입니다.

**effort** = "Codex가 답하기 전에 얼마나 깊이 생각할지"를 결정하는 다이얼.

```
low  →  medium  →  high  →  xhigh
(빠름·약간 해석)   (균형·기본값)   (신중·검증)   (가장 꼼꼼)
```

| effort | 어떤 일에 쓰나 | 지우의 실제 예 |
|--------|--------------|--------------|
| `low` | 단순 변환, 빠른 숫자 확인 | "이 CSV 파일에 3월 매출 합계만 알려줘" |
| `medium` | 분석 요약, 일상 비교 | 강의별 월별 매출 흐름 정리, 문의 분류 |
| `high` / `xhigh` | 여러 파일 연결, 중요 결정 | "세 파일을 엮어 다음 분기 전략을 제안해줘" |

---

### 또 하나의 손잡이: 모델 (model)

effort가 "얼마나 깊이 생각할지"라면, **모델**은 "얼마나 능력 있는 일꾼을 부를지"입니다. 카페 비유로 하면, 동네 빠른 카페(저렴·신속)와 스페셜티 카페(깊은 맛·느림)의 차이입니다.

> ⚠️ 아래 모델명은 codex-101 2026-06 스냅샷 기준 예시입니다. 실제 세션 중 `/model`을 입력하면 사용 가능한 목록이 바로 나옵니다. 그 목록을 기준으로 진행하세요.

| 모델 (예시) | 성격 | 지우가 쓸 때 |
|------------|------|------------|
| `gpt-5.5` ⭐ | 권장 기본값. 깊이와 속도 균형 | 매출 정리, 문의 분류, 뉴스레터 전략, 멀티파일 분석 등 대부분 업무 |
| `gpt-5.4-mini` | 빠르고 가벼움 | 파일 구조 파악, 단순 수치 확인, 짧은 변환 |

---

### 두 손잡이를 함께 이해해야 하는 이유

모델과 effort는 독립적으로 조절할 수 있지만, 실제로는 쌍으로 생각합니다.

- 단순한 일 → 가벼운 모델 + 낮은 effort = **빠르고 저렴**
- 중요한 일 → 강한 모델 + 높은 effort = **신중하고 정확**

**비유**: 자동차의 기어와 엑셀 페달처럼. 어떤 도로(작업 종류)냐에 따라 조합을 바꾸는 겁니다.

지우에게 가장 좋은 소식은 이겁니다 — **모든 일에 최고급 모델과 최고 effort를 쓸 필요가 없습니다.** 일에 맞게 골라 쓰면 속도도 빠르고 비용도 절약됩니다.

### Claude Code(CC)와 비교: 뼈대는 같다

CC에서도 모델을 고르고 생각 강도를 조절합니다. 이름과 명령 모양이 살짝 다를 뿐, **"일의 난이도에 AI를 맞춘다"는 원리는 두 도구가 완전히 같습니다.** codex-03에서 익힌 감각이 CC에도, 앞으로 배울 멀티에이전트 실습에도 그대로 쓰입니다.

## 🔧 실습 1 — effort 차이를 눈으로 보기 (15분)

이번 실습에서는 **똑같은 부탁을 effort만 바꿔서** 두 번 던져봅니다. 답이 어떻게 달라지는지 화면에서 직접 비교해 보세요.

---

### 1단계: Codex 시작하기

터미널을 열고 `practice-data/creator/` 폴더를 지정하면서 Codex를 시작합니다.

```bash
codex --cd ~/Desktop/practice-data/creator
```

> 💡 `~/Desktop/practice-data/creator`는 실제 폴더가 있는 경로로 바꾸면 됩니다. codex-01에서 실습했던 그 방법 그대로입니다.

**기대 결과**: Codex가 실행되고 대화를 기다리는 상태가 됩니다.

**관찰 포인트**: 폴더를 지정했기 때문에 Codex가 이 폴더 안의 파일들을 읽을 수 있게 됩니다. "지우의 사업 폴더를 펼쳐놓고 앉아 있는" 상태라고 생각하면 됩니다. 지정 없이 시작하면 Codex가 어느 파일을 봐야 할지 몰라 엉뚱한 답이 나올 수 있습니다.

---

### 2단계: 현재 세션 상태 확인하기

Codex가 실행된 대화창 안에서 지금 어떤 설정으로 일하는지 확인합니다.

```text
/status
```

**기대 결과 (예시)**:
```
현재 모델: gpt-5.5
현재 effort: medium
세션 시작: 방금 전
토큰 사용량: 0
```

**관찰 포인트**: `/status`(슬래시 status)는 이번 레슨에서 다루는 명령입니다. 현재 어떤 모델이 켜져 있는지, 생각 강도는 어느 수준인지 한 눈에 볼 수 있습니다 — CC에서 상태 확인을 했던 것과 같은 개념입니다. 이 명령은 세션 도중 언제든 입력해 현재 설정을 점검할 수 있습니다.

---

### 3단계: 모델 목록 보기

```text
/model
```

**기대 결과 (예시)**:
```
사용 가능한 모델:
  gpt-5.5    ✓  — 권장 기본값 (현재 선택)
  gpt-5.4-mini  — 빠름, 단순 작업용
```

**체크포인트**: 목록이 나오면 성공. 지금 어떤 모델이 선택(✓)되어 있는지 확인하세요.

> 😟 **`gpt-5.5`가 목록에 없다면**: 구독·계정에 따라 사용 가능 모델이 다를 수 있습니다 → `npm update -g @openai/codex`로 최신 버전 업데이트 → 그래도 안 보이면 `gpt-5.4-mini`로 진행해도 실습 내용은 동일합니다.

---

### 4단계: effort를 낮게 — 빠른 답 받기

메뉴에서 모델을 가벼운 쪽으로, effort를 낮게 설정합니다. `/model` 명령 안에서 조정하거나 아래처럼 전환합니다.

```text
/model gpt-5.4-mini
```

전환 확인 후 이렇게 부탁해 보세요.

```text
course_sales_monthly.csv 파일에서 2026년 3월 강의별 순매출을 빠르게 알려줘.
```

**기대 결과 (gpt-5.4-mini 모드)**:
```
2026년 3월 강의별 순매출:
- 클로드코드로 업무 자동화 입문: 1,242만원 (수강 118명, 환불 9명)
- 챗GPT로 콘텐츠 10배 빠르게: 823만원 (수강 121명, 환불 10명)
- 1인 지식사업 수익화 로드맵: 895만원 (수강 68명, 환불 5명)

3월 합계 순매출: 약 2,960만원
```

**관찰 포인트**:
- 답이 빠르게 나옵니다 (몇 초 이내).
- 간결하고 사실 위주입니다. "왜 그런지", "앞으로 어떻게 해야 할지"는 없습니다.
- 이런 답이 나쁜 게 아닙니다. **"순매출만 빨리"라는 단순 질문에는 이 정도면 충분**합니다.

---

### 5단계: effort를 높게 — 같은 주제로 더 깊은 분석

이번엔 모델을 다시 올립니다.

```text
/model gpt-5.5
```

전환 후, 비슷한 주제로 더 넓은 요청을 합니다.

```text
course_sales_monthly.csv 에서 2026년 3월 강의별 순매출 흐름을 정리해 줘.
어떤 강의가 성장 중이고, 어떤 강의가 줄고 있는지도 봐줘.
특히 2월 출시한 클로드코드 강의가 기존 강의들과 비교해 어떤지 눈에 띄는 패턴이 있으면 알려줘.
```

**기대 결과 (gpt-5.5 + medium effort)**:
```
【 2026년 3월 강의 매출 흐름 분석 】

📈 빠른 성장 — 클로드코드로 업무 자동화 입문 (신규)
  - 2026-02 첫 출시: 순매출 508만원 (수강 46명)
  - 2026-03 단 한 달 만에: 순매출 1,242만원 (수강 118명)
  - 한 달 만에 +144% 성장. 3월 강의 순매출 1위 진입.

📊 완만한 하락 — 챗GPT로 콘텐츠 10배 빠르게
  - 2026-01 최고점: 순매출 1,252만원 (수강 187명)
  - 2026-03: 순매출 823만원 (수강 121명)
  - 신규 AI활용 강의(클로드코드) 출시 후 수강 집중 분산 가능성.

📊 안정 유지 — 1인 지식사업 수익화 로드맵
  - 2026-03: 순매출 895만원 (수강 68명)
  - 객단가(149,000원)가 가장 높아 수강 수 대비 매출 기여가 큼.
  - 하락세 없이 안정적.

[눈에 띄는 패턴]
클로드코드 강의가 출시 두 달 만에 기존 주력 강의(챗GPT 강의)를 매출에서 역전했습니다.
뉴스레터 39·40호의 사전등록·마감 임박 캠페인과 겹쳐 단기 런칭 효과가 집중된 것으로 보입니다.
```

**관찰 포인트**:
- 훨씬 길고 구체적입니다.
- 단순 수치 정리를 넘어 **흐름과 패턴을 해석**했습니다.
- 전월 대비 비교, 맥락 추론까지 해줬습니다.
- 대신 **시간이 더 걸렸습니다** (모델과 세션 상태에 따라 10~30초).

> 🔑 **핵심 비교 한 줄**: effort/모델을 높이면 → 더 깊이 생각하고, 더 많이 연결하고, 더 구체적으로 해석한다. 대신 느리고 비용이 조금 더 든다.

---

### 6단계: 비교 정리 — 나는 어떤 상황에서 뭘 써야 할까

직접 체험한 차이를 표로 정리해 봅니다.

| 상황 | 추천 조합 | 이유 |
|------|----------|------|
| "3월 매출 합계 얼마야?" (빠른 확인) | gpt-5.4-mini + low | 숫자만 알면 충분 |
| "이번 달 강의 성과 정리해줘" (일상 분석) | gpt-5.5 + medium | 적당한 깊이와 속도 |
| "여러 파일 연결해서 다음 분기 전략 제안해줘" (중요 결정) | gpt-5.5 + high | 교차 분석, 원인까지 |
| "전체 강의 라인업 전략 재설계" (복잡 전략) | gpt-5.5 + xhigh | 깊은 추론, 여러 관점 교차 |

> 💡 **안심 포인트**: effort를 잘못 설정해도 망가지지 않습니다. 답이 너무 짧으면 "더 자세히 봐줘"라고 하면 되고, 너무 길면 "핵심만 짧게"라고 하면 됩니다. AI는 비서이지 시험감독이 아닙니다.

## 🎬 실습 2 — 지우의 실제 사업 데이터로 모델·effort 함께 활용하기 (20분)

이번 실습에서는 `practice-data/creator/`의 실제 파일들을 가지고, 모델과 effort를 상황에 맞게 골라 쓰면서 분석 한 사이클을 완성해 봅니다.

**시나리오**: 3월이 끝난 뒤 지우는 이걸 알고 싶습니다 — "이번 달 어떤 강의가 잘됐고, 수강생들이 주로 무슨 문의를 했는지. 뉴스레터 데이터와 연결해서 다음 달엔 어떤 강의를 어떻게 홍보하면 좋을지."

---

### 1단계: 폴더 안에 무슨 파일이 있는지 빠르게 확인하기

먼저 가벼운 모드로 폴더 안의 파일 구성을 빠르게 파악합니다.

```text
/model gpt-5.4-mini
```

```text
이 폴더에 어떤 파일이 있고, 각 파일이 뭘 담고 있는지 짧게 소개해 줘.
```

**기대 결과 (예시)**:
```
폴더 안의 파일:
- course_sales_monthly.csv: 강의별·월별 매출 (2025-07 ~ 2026-03)
- newsletter_stats.csv: 뉴스레터 호별 오픈율·클릭율 (18호~41호)
- student_inquiries.csv: 수강생 문의 내역 (2026-03 기준)
- content_plan.md: 2분기 콘텐츠 기획 노트
- research_memo.md: AI 도구·시장 리서치 메모
- consulting_schedule.md: 3월 컨설팅·발행 일정

어느 파일부터 볼까요?
```

**관찰 포인트**: 이렇게 짧고 빠른 파악은 `gpt-5.4-mini` 모드면 충분합니다. 매번 gpt-5.5로 시킬 필요가 없습니다.

---

### 2단계: 1분기 강의 매출 흐름 분석하기

분석 작업으로 들어갑니다. 모델을 올립니다.

```text
/model gpt-5.5
```

```text
course_sales_monthly.csv 에서 2026년 1분기(1~3월) 강의별 순매출 흐름을 정리해 줘.
어떤 강의가 성장 중이고, 어떤 강의가 줄고 있는지도 봐줘.
```

**기대 결과 (예시)**:
```
【 2026년 1분기 강의별 순매출 흐름 】

📈 성장 중인 강의:
```

## 💡 핵심 규칙 정리

### 세 손잡이 기억법

```
모델   = 어떤 엔진을 쓸지   (경차 vs 대형차)
effort = 얼마나 깊이 생각할지  (빠르게 vs 꼼꼼하게)
```

**일의 종류에 맞춰 조합하면:**

| 일의 성격 | 추천 조합 |
|-----------|-----------|
| 빠른 수치 확인, 단순 변환 | `gpt-5.4-mini` + `low` |
| 일상적인 분석·요약·분류 | `gpt-5.5` + `medium` |
| 여러 파일 교차 분석, 중요 전략 | `gpt-5.5` + `high` |
| 사업 전체 방향 재설계 | `gpt-5.5` + `xhigh` |

### 이번 레슨에서 새로 배운 세션 명령 3총사

```
/model   → 모델·effort 전환 (대화 중 언제든)
/status  → 현재 설정·토큰 확인
/compact → 긴 대화 정리 (작업 끝난 뒤에. 작업 중간엔 ❌)
```

### `/compact`는 작업 경계에서만

```
작업 A 완료 → /compact → 작업 B 시작  ✅
작업 A 중간에 /compact → 맥락 손실    ❌
```

### CC와 Codex 비교 — "10%만 다르다"

| | Claude Code (CC) | Codex |
|-|-----------------|-------|
| 모델 조절 | 모델 선택 | `/model` |
| 추론 강도 | effort 개념 동일 | effort 개념 동일 |
| 대화 요약 | `/compact` | `/compact` |
| 상태 확인 | 유사 개념 | `/status` |
| 규칙서 | `CLAUDE.md` | `AGENTS.md` |
| 제작사 | Anthropic | OpenAI |
| 모델 라인업 | Claude | gpt-5.x |

한 문장 정리: **"CC를 익혔으면 Codex는 90% 전이된다. 규칙서·샌드박스·MCP·effort는 같고, 제품·모델만 다르다."**
