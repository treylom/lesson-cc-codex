# 이슈 대응 프로세스

고객문의나 내부에서 올라온 문제를 어떤 흐름으로 처리하는지 정리했습니다. 처음 오신 분은 "이슈가 들어오면 이렇게 흘러가는구나" 정도만 알면 됩니다.

## 전체 흐름
1. **접수** — 티켓 시스템에 등록 (고객문의·내부제보 모두 티켓으로)
2. **분류** — 긴급도 판단 (아래 등급표)
3. **담당 지정** — [[담당-체계]]에 따라 배정
4. **처리** — 답변 또는 수정
5. **확인** — 고객/요청자에게 해결됐는지 확인
6. **종료** — 티켓 닫고, 반복되는 문제면 회고에 기록

## 긴급도 등급
| 등급 | 기준 | 대응 시간 |
|---|---|---|
| 긴급 | 서비스 전체 영향, 결제 불가 | 즉시 ([[장애대응-체크리스트]]로) |
| 높음 | 일부 기능 안 됨, 다수 고객 영향 | 당일 |
| 보통 | 한두 명 불편, 우회 가능 | 2일 내 |
| 낮음 | 개선 요청, 단순 문의 | 일주일 내 |

## 자주 헷갈리는 점
- 긴급/높음은 혼자 안고 가지 말고 바로 공유하세요. 보통/낮음은 본인이 처리해도 됩니다.
- 같은 문의가 일주일에 세 번 이상 오면, 그건 개별 답변이 아니라 [[자주묻는질문-FAQ]]에 추가할 신호입니다.

> 정산·결제처럼 돈이 걸린 이슈는 등급을 한 단계 높여서 봅니다. 늦으면 신뢰가 깨지니까요.
