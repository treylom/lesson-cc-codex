# 업무 요청 주고받는 법

다른 팀(IT, 인사, 영업)에 뭔가 부탁하거나, 반대로 우리 팀에 요청이 들어올 때 쓰는 방식입니다. 말로만 부탁하면 까먹히기 쉬워서, 가능하면 글로 남깁니다.

## 우리가 요청할 때
티켓이나 채널에 아래 형식으로 적습니다. 빈칸 없이 채워야 빨리 처리돼요.

```
[요청] 무엇을
[기한] 언제까지
[이유] 왜 필요한지 (한 줄)
[연락] 막히면 누구에게
```

### 예시 — IT 계정 발급 (가장 자주 씀)
```
[요청] 신입 박지영 님 계정 4종 (메신저·대시보드·티켓·옵시디언)
[기한] 6/16 입사일 전날까지
[이유] 입사 당일 바로 업무 시작해야 함
[연락] 김민수
```
> 계정은 **입사 한 주 전**에 요청합니다. 당일 요청하면 늦어요. ([[온보딩-체크리스트]] 참고)

## 우리가 요청받을 때
- 티켓으로 들어오면 [[이슈대응-프로세스]] 흐름대로.
- 말로 부탁받았으면 "티켓으로 남겨주시면 빠뜨리지 않고 처리할게요"라고 안내.
- 우리 능력 밖이면 누구에게 가야 하는지 알려주기 ([[담당-체계]]).

## 흔한 문제
- "급해요"가 너무 많음 → 진짜 긴급도는 [[이슈대응-프로세스]] 등급표로 판단.
- 요청만 하고 기한이 없음 → 기한 없는 요청은 자꾸 밀립니다. 꼭 적기.
