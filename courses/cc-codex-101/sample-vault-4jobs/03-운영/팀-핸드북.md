# 운영팀 핸드북

새로 오신 분도, 오래 계신 분도 헷갈릴 때 여기 먼저 보세요.

## 우리 팀이 하는 일
서비스가 멈추지 않게 지키고, 새 기능을 안전하게 내보내고, 고객문의가 제때 처리되게 합니다. 한마디로 "굴러가게 만드는 팀".

## 근무 / 연락
- 코어타임: 오전 10시 ~ 오후 4시 (이 시간엔 자리 또는 온라인)
- 급한 일은 팀 채널 `#운영-알림`에 멘션. 채팅 DM은 답이 늦을 수 있음.
- 온콜(긴급 당번)은 당번표대로 돌아갑니다. 당번 주에는 휴대폰 알림 꼭 켜두기.

## 주요 도구
| 용도 | 도구 |
|---|---|
| 협업 메신저 | 사내 메신저 (`#운영-알림`, `#배포공지`) |
| 모니터링 | 대시보드 (서버 상태·에러율) |
| 문서 | 옵시디언 (이 폴더) |
| 이슈 관리 | 사내 티켓 시스템 |

## 우리만의 규칙
- 배포는 [[신규기능-배포-체크리스트]] 없이 하지 않습니다.
- 금요일 오후엔 큰 배포 금지 (주말에 문제 터지면 대응 인력 부족).
- 모르는 건 부끄러운 게 아니라 위험합니다. 막히면 30분 안에 물어보기.
- 매주 금요일 주간보고 작성 ([[주간운영보고-2026-W25]] 형식 참고).

## 비상 연락 체계
1. 장애 감지 → 온콜 당번이 1차 대응
2. 15분 내 해결 안 되면 팀장에게 보고
3. 고객 영향 크면 고객지원팀에 동시 공유
