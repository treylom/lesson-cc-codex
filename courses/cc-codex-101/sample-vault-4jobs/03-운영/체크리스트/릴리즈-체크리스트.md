# 릴리즈 / 롤백 체크리스트

[[신규기능-배포-체크리스트]]가 "배포할 때 빠짐없이 확인할 것"이라면, 이 문서는 **버전을 올리고 되돌리는 절차** 자체를 적어둔 곳입니다. 배포 담당자가 손이 떨릴 때 보고 따라 하라고 만들었어요.

## 릴리즈 절차
1. 릴리즈 노트 작성 (이번에 뭐가 바뀌는지 한 단락)
2. 버전 번호 올리기 (예: 1.4.2 → 1.4.3)
3. 테스트 서버에 먼저 올려 확인
4. 운영 서버 배포 ([[신규기능-배포-체크리스트]] 순서대로)
5. 배포 후 핵심 화면 점검 + 15분 모니터링

## 롤백 절차 (문제 생겼을 때)
1. 팀 채널에 "롤백 진행" 공지
2. 직전 버전으로 되돌리기 (이전 버전 번호 확인 후)
3. 되돌린 뒤 핵심 화면 다시 점검
4. 고객 영향 있었으면 고객지원팀 공유
5. 회고 메모 (무엇이·왜·다음엔 어떻게) → [[분기-회고-2026-Q2]]에 모음

## 꼭 지킬 것
- 롤백 방법은 **배포 전에** 글로 적어둔다. 닥쳐서 찾으면 늦는다.
- 금요일 오후·연휴 직전엔 큰 릴리즈를 하지 않는다. (문제 터지면 대응 인력 부족)
- 데이터가 바뀌는 변경은 백업을 먼저 받는다.

> 6/8 사고 때 롤백으로 잘 막았습니다. 그날 배운 점: 직전 버전 번호를 미리 메모해두면 롤백이 훨씬 빠릅니다.
