# 신규 기능 배포 체크리스트

배포 담당자는 매 배포마다 이 목록을 위에서부터 확인합니다. 하나라도 비면 배포 보류.

## 배포 전날
- [ ] 배포 일정·범위를 팀 채널에 공지했는가 (최소 하루 전)
- [ ] 변경 내용 요약과 영향 범위를 핸드오프 문서에 정리했는가
- [ ] 관련 부서(고객지원, 영업)에 영향 있는 변경이면 미리 알렸는가
- [ ] 롤백 방법을 확인했는가 (이전 버전으로 되돌리는 절차)

## 배포 직전
- [ ] 테스트 서버에서 정상 동작 확인했는가
- [ ] 백업이 최신인가 (데이터 변경 동반 시 필수)
- [ ] 배포 시간대가 트래픽 적은 시간인가 (가급적 오전 10~11시)
- [ ] 온콜 당번에게 배포 시작을 알렸는가

## 배포 중 / 직후
- [ ] 배포 후 핵심 화면 3개(로그인, 메인, 결제)를 실제로 눌러봤는가
- [ ] 모니터링 대시보드에 에러율 급증이 없는가 (15분 관찰)
- [ ] 고객문의 채널에 관련 문의가 들어오는지 확인했는가

## 마무리
- [ ] 배포 완료를 팀 채널에 공지했는가
- [ ] 주간보고에 배포 1건으로 기록했는가 ([[주간운영보고-2026-W25]] 형식)
- [ ] 문제 있었으면 회고 메모를 남겼는가

> 지난 사고 교훈: 결제 모듈 배포 때 롤백 절차를 안 적어둬서 문제 생겼을 때 우왕좌왕함. 롤백 방법은 반드시 배포 전에 글로 적어둘 것.
