# 장애 대응 체크리스트

장애가 의심되면 당황하지 말고 위에서부터 따라가세요. 혼자 다 하려 하지 말고, 알릴 사람부터 알립니다.

## 1단계 — 확인 (5분 안에)
- [ ] 진짜 장애인가, 아니면 알림 오탐인가 (대시보드에서 에러율·응답시간 확인)
- [ ] 영향 범위 파악 (전체인가 일부 화면인가, 고객이 실제로 못 쓰는가)
- [ ] 언제부터인지 시각 기록 (나중에 회고에 필요)

## 2단계 — 알림
- [ ] 팀 채널 `#운영-알림`에 "장애 발생" 공유 (시각·증상 한 줄)
- [ ] 온콜 당번 호출 ([[온콜-당번-안내]])
- [ ] 고객 영향이 크면 고객지원팀에 동시 공유

## 3단계 — 조치
- [ ] 최근 배포가 원인일 가능성부터 확인 (직전 배포 있었나)
- [ ] 원인이면 롤백 검토 → [[릴리즈-체크리스트]]의 롤백 절차
- [ ] 임시 우회라도 고객 영향부터 줄이기

## 4단계 — 마무리
- [ ] 복구 시각 기록, 팀 채널에 "복구 완료" 공지
- [ ] 회고 메모 작성 (원인·복구 시각·재발 방지 한 줄) → 분기 회고에 모음 [[분기-회고-2026-Q2]]
- [ ] 주간보고에 장애 1건으로 기록

> 가장 흔한 실수: 원인 분석부터 하느라 알림을 늦게 함. **알리는 게 먼저, 분석은 그다음.**
