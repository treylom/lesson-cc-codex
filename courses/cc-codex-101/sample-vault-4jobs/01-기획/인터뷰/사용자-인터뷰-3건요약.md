# 사용자 인터뷰 요약 (6월, 3건)

검색 개선 기획 전에 실제 사용자 세 명을 짧게 인터뷰한 기록. 각 30분 정도, 화상으로 진행. 자세한 현장 메모는 [[리서치-메모]]에 있고, 여기는 결론만 추린 요약.

## 한눈에
| 참여자 | 나이/직업 | 핵심 한마디 |
|--------|-----------|-------------|
| P1 | 30대 직장인 | "띄어쓰기 때문에 검색이 안 됐어요" |
| P2 | 20대 학생 | "검색은 안 될 걸 아니까 안 써요" |
| P3 | 40대 | "오타 좀 알아서 고쳐주면 좋겠어요" |

## 공통적으로 나온 이야기
1. **검색 실패 경험이 쌓이면 아예 안 쓴다.** P2가 가장 분명하게 말함. 한두 번 실패하면 검색 자체를 포기하고 카테고리로 돌아간다.
2. **띄어쓰기·오타가 주범.** 세 명 모두 언급. 기능 문제이지 사용자 잘못이 아니다.
3. **빈 화면이 제일 싫다.** "아무것도 안 나오면 그냥 닫는다"는 반응.

## 기획에 반영한 점
- 우선순위 A(부분일치+오타 보정)가 맞다는 확신. → [[2026-06-15-신규기능-기획회의]]
- 결과 없음 화면을 꼭 같이 바꾼다. → [[검색개선-PRD]]

## 다음 인터뷰 때 보완할 점
- 이번엔 3명뿐이라 표본이 작다. 다음엔 검색을 자주 쓰는 헤비 유저도 섞어서 5명 이상.
- 화면을 직접 보여주며 "여기서 막혔던 순간"을 짚게 하면 더 구체적일 듯.
