# 리서치 메모 (검색 개선)

[[제안서/검색개선-제안-초안]] 작성용으로 모은 메모.

## 경쟁 제품 살펴보기

### 마켓플레이스 A
- 오타를 쳐도 "혹시 이걸 찾으세요?"로 보정해서 결과를 보여줌.
- 검색창에 입력하는 동안 실시간으로 추천어가 뜸.

### 쇼핑앱 B
- 결과가 없을 때 빈 화면 대신 인기 상품을 추천.
- 검색 결과 상단에 '광고' 노출이 좀 많아서 사용자 평이 갈림. (우리는 참고만)

### 정리
- 공통점은 "결과를 0개로 안 보여준다"는 것. 어떻게든 대안을 제시함.
- 우리도 빈 화면을 줄이는 게 핵심 같다. → [[결정-로그]] 빈 화면 개선 방향과 연결됨.

## 사용자 인터뷰 메모 (6/16, 3명)

### P1 (30대, 직장인)
- "노트북 가방 검색했는데 아무것도 안 나와서 그냥 나갔어요. 띄어쓰기 때문인 것 같아요."

### P2 (20대, 학생)
- "검색은 거의 안 써요. 안 나올 걸 알아서 그냥 카테고리로 찾아요."
- → 검색 실패 경험이 누적되면 아예 안 쓰게 된다는 신호.

### P3 (40대)
- "오타 나면 다시 치기 귀찮아요. 알아서 좀 고쳐주면 좋겠어요."

## 시사점
- 띄어쓰기/오타 문제는 거의 모든 인터뷰에서 나옴. → 우선순위 A가 맞다.
- 검색을 한 번 실패하면 다시 안 쓰게 되는 경향. 첫 실패를 줄이는 게 중요.
