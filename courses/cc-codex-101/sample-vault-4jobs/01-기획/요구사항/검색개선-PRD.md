# 검색 개선 요구사항 정의서 (PRD)

> 제안서가 "왜·무엇을"이라면, 이건 "정확히 어떻게 동작해야 하나"를 적은 문서. 개발·디자인이 보고 만들 수 있는 수준으로. → [[검색개선-제안-초안]]

## 1. 목표
두 단어 이상 검색이나 오타가 있어도 결과가 나오게 한다. 검색 후 이탈률을 30% 이하로 낮춘다. → [[지표-정의]]

## 2. 범위
이번에 하는 것: 부분일치, 오타 보정, 결과 없음 화면 개선
이번에 안 하는 것: 연관 검색어 추천(다음 분기), 음성 검색

## 3. 상세 요구사항

### 3-1. 부분일치
- 입력한 단어가 상품명에 **포함만 되어도** 결과에 노출한다.
- 띄어쓰기는 무시하고 비교한다. ("노트북 가방" = "노트북가방")

### 3-2. 오타 보정
- 자주 틀리는 입력은 보정해서 검색한다. 보정했을 땐 화면에 "'OOO'(으)로 검색한 결과"라고 알려준다.
- 사용자가 원래 입력으로 다시 검색할 수 있는 링크도 같이 보여준다.

### 3-3. 결과 없음 화면
- 결과가 0개면 빈 화면 대신 **인기 상품 6개**를 추천한다.
- 상단에 "찾으시는 결과가 없어요. 이런 건 어때요?" 문구.

## 4. 화면 흐름
1. 검색어 입력 → 2. 부분일치/보정 적용해 결과 조회 → 3-a. 결과 있으면 목록 / 3-b. 없으면 추천 화면

## 5. 검증 기준 (이게 되면 완성)
- "노트북 가방"으로 검색 시 결과가 나온다.
- 오타("노투북")를 쳐도 보정되어 결과가 나온다.
- 존재하지 않는 단어 검색 시 빈 화면이 아니라 추천이 보인다.

## 열린 이슈
- 부분일치 시 결과가 너무 많이 나올 수 있음. 정렬 기준 필요. → 인기순? (박민호님 부하도 같이 확인)
- 오타 보정 사전을 어떻게 관리할지. 수동 등록 vs 자동.
