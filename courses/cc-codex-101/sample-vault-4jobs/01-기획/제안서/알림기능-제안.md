# 푸시 알림 재설계 제안

## 1. 배경
5월에 알림 기본값을 '꺼짐'으로 바꿨다. 앱 삭제 사유 1위가 '알림이 너무 많아서'였기 때문. → [[결정-로그]]
삭제율은 내려갔지만, 알림으로 들어오던 매출이 조금 줄었다. 이번엔 "적게, 그러나 도움 되게" 알림을 다시 설계한다.

## 2. 문제
- 예전 알림은 종류 구분 없이 다 보냈다. 사용자는 광고성·중요 알림을 구분할 수 없었다.
- 한 번 끄면 다시 켜는 사람이 거의 없다. 즉 기본값을 끄면 알림이 사실상 0이 된다.

## 3. 제안
- **알림을 종류별로 나눈다**: (1) 주문·배송 같은 꼭 필요한 알림 (2) 혜택·할인 알림
- (1)은 기본 켜짐, (2)는 기본 꺼짐. 사용자가 종류별로 켜고 끌 수 있게.
- 가입 직후가 아니라, **실제로 주문을 한 번 한 뒤** 알림 허용을 물어본다. (맥락이 있을 때 허용률이 높음)

## 4. 기대효과
- 꼭 필요한 알림은 닿으면서, 광고성 알림 피로는 줄임.
- 알림 매출 회복 + 삭제율은 유지.

## 5. 남은 질문
- 종류별 알림 설정 화면을 새로 만들어야 함. 디자인 공수 확인 필요. (이지원님)
- "주문 후에 묻기" 시점이 너무 늦으면 기회를 놓칠 수도. A/B로 테스트할지?
