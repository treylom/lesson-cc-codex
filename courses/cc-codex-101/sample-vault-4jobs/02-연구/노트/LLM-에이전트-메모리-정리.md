# LLM 에이전트 메모리 정리

요즘 'AI 에이전트'가 대화를 기억하는 방식을 자료 읽으면서 정리. 기반 개념은 → [[트랜스포머-한장정리]]

## 왜 메모리가 문제인가
- LLM은 기본적으로 **한 번에 볼 수 있는 분량(컨텍스트 윈도우)이 정해져 있다.** 대화가 길어지면 앞 내용이 잘려 나간다.
- 그래서 "어제 말한 걸 오늘도 기억하게" 하려면 따로 저장·검색하는 장치가 필요하다.

## 메모리 종류 (자료에서 본 분류)
1. **단기 메모리** — 지금 대화창 안에 들어있는 내용. 윈도우가 차면 사라짐.
2. **장기 메모리** — 중요한 내용을 외부 저장소(파일·DB)에 적어두고, 필요할 때 다시 꺼내옴.
3. **요약 메모리** — 긴 대화를 짧게 요약해서 보관. 통째로 안 들고 핵심만.

## 검색해서 가져오는 방식 (RAG)
- 저장해둔 자료 중 **지금 질문과 관련 있는 부분만 찾아 LLM에 같이 넣어주는** 방법.
- 관련도를 따질 때 문장을 벡터로 바꿔 비교한다. 이 벡터 표현이 → [[트랜스포머-한장정리]] 의 임베딩과 연결됨.
