# 연구 아이디어 메모

읽다가 떠오른 "해보면 재밌겠다" 목록. 검증 안 된 날것의 생각이라 가볍게. 발전시키면 따로 노트로 뺄 것.

## 1. 수강생 문의 자동 분류
- 강의 수강생 문의(`student_inquiries.csv`)를 임베딩으로 묶으면 "환불 / 결제오류 / 진도 / 자료" 같은 덩어리로 자동 분류될 듯. → [[임베딩-한장정리]]
- 비슷한 질문엔 비슷한 답을 미리 준비 → 응대 시간 절약. RAG로도 가능 → [[RAG란-무엇인가]]

## 2. 내 노트 전용 검색 봇
- 옵시디언 노트를 통째로 RAG에 넣고 "내가 예전에 어디서 임베딩 정리했더라?" 물으면 찾아주게.
- 이미 비슷한 걸 옵시디언이 해줌. 우리 강의에서 다루는 `/search`가 그 예.

## 3. 요약 품질 점검
- 요약 메모리가 핵심을 너무 깎는지 확인하는 작은 실험. 원문 vs 요약을 사람이 채점 → [[LLM-에이전트-메모리-정리]]의 "다음에 볼 것"과 연결.

## 4. "에이전트" 정의 갈래 정리
- 글마다 '에이전트' 뜻이 달라서, 정의들을 모아 갈래를 쳐보면 짧은 글 한 편 나올 듯 → [[용어집]]

## 우선순위
- 당장은 **1번**이 실무에 바로 쓸모 있음. 나머지는 시간 날 때.
