# 파인튜닝 vs RAG

"우리 자료를 AI가 답하게 하려면 어떻게 해야 하나?"라고 물으면 보통 이 둘이 나온다. 헷갈려서 한 번 정리.

## 한 줄 차이
- **파인튜닝(fine-tuning)** = 모델 자체를 우리 자료로 **추가 학습**시켜 '몸에 익히게' 하는 것.
- **RAG** = 모델은 그대로 두고, 답할 때마다 우리 자료를 **찾아서 옆에 붙여주는** 것. → [[RAG란-무엇인가]]

## 비유
- 파인튜닝 = 직원을 **연수 보내 교육**시키는 것. 한 번 배우면 알아서 함. 대신 시간·비용 듦, 새 내용 생기면 또 교육해야 함.
- RAG = 직원 책상에 **매뉴얼을 비치**해 그때그때 펼쳐보게 하는 것. 매뉴얼만 갈아끼우면 바로 최신화.

## 비교표
| 항목 | 파인튜닝 | RAG |
|------|----------|-----|
| 새 정보 반영 | 다시 학습해야 함 (느림) | 자료만 바꾸면 즉시 |
| 비용 | 학습 비용 큼 | 상대적으로 저렴 |
| 출처 제시 | 어려움 | 쉬움 (찾은 문서가 곧 출처) |
| 말투·스타일 학습 | 잘함 | 약함 |

## 결론 메모
- **자주 바뀌는 사실·내부 문서** → RAG.
- **고정된 말투·형식·전문 분야 감각** → 파인튜닝.
- 실무에선 둘을 섞어 쓰기도 함.
