# RAG (검색 증강 생성) 서베이 요약

- **제목**: Retrieval-Augmented Generation for Large Language Models: A Survey
- **저자**: Gao 외 (여러 명)
- **원문**: https://arxiv.org/abs/2312.10997 (접근일 2026-04-15)

## 한 줄 요약
LLM이 모르는 최신 정보나 회사 내부 자료를 답하게 하려면, **질문과 관련된 문서를 먼저 찾아서(retrieval) 같이 넣어준 뒤 답을 생성(generation)**하면 된다. 이 방식 전체를 정리한 서베이(여러 연구를 묶어 흐름을 보여주는 글) 논문.

## 왜 보게 됐나
- LLM은 학습한 시점 이후의 일을 모른다. 또 우리 회사 문서 같은 건 처음부터 모른다.
- "모델을 다시 학습시키지 않고도" 최신·내부 지식을 답하게 하는 가장 현실적인 방법이 RAG라서.
- 우리가 쓰는 옵시디언 노트 검색도 결국 RAG의 한 형태다 → [[RAG란-무엇인가]]

## 핵심 정리
1. **단순 RAG (Naive)** — 문서 쪼개기 → 벡터로 저장 → 질문과 비슷한 조각 찾아 붙이기. 기본형.
2. **고급 RAG (Advanced)** — 검색 전후로 다듬는 단계 추가. 질문을 고쳐 쓰거나, 찾은 결과를 다시 정렬(re-ranking).
3. **모듈형 RAG (Modular)** — 검색·생성을 레고처럼 조립. 상황별로 부품을 갈아끼움.
