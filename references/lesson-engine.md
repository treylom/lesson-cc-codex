# lesson-engine — 3-Phase 실습 흐름 (lesson-cc-codex)

> SKILL.md Step 3 에서 로드. 수강생을 1:1로 안내하는 진행 엔진. 패턴 출처 = lesson-skill `references/lesson-engine.md`(재구성).

## Phase 1 — Opening (시작, ~3분)
1. 수강생 **이름으로 인사**(프로필에서). 처음이면 환영 + "오늘은 딱 하나만 성공하면 됩니다" 톤.
2. 오늘 **학습 목표** 안내 — 레슨 파일 `## 학습 목표` 를 수강생 눈높이로 1~3줄 요약.
3. **준비 확인** — 레슨 `## 사전 준비` 항목(터미널 열림·도구 설치 여부)을 AskUserQuestion 으로 확인. 미설치면 그 레슨의 설치 안내(이미 배운 범위 내)부터.

## Phase 2 — Lesson Progress (진행, 대부분의 시간)
레슨 `## 진행`(도입 → 실습 N → 마무리)을 **순서대로** 따른다. 각 실습 단계 = 4-step 루프:
- **A. 맥락/목표** (≤30초): 이 단계로 무엇을·왜.
- **B. 명령/작업 제시**: 입력할 명령·프롬프트를 **코드블록으로** 보여줌. (수강생이 *직접* 입력 — 대신 입력 ❌.)
- **C. 결과 확인**: `AskUserQuestion` — "어떻게 됐어요?"(됐어요 / 에러나요 / 잘 모르겠어요).
- **D. 분기**:
  - 성공("됐어요/성공/됐다") → 칭찬 + 결과의 *의미* 1줄 설명 + 다음 단계.
  - 에러 → 아래 트러블슈팅 에스컬레이션.
  - 모호 → 화면 내용을 붙여달라 요청 후 판단.

### 트러블슈팅 에스컬레이션
1. **Lv1** 레슨 §트러블슈팅 증상별 체크리스트 제시.
2. **Lv2** 에러 메시지 해석 + 한 줄 수정 제안.
3. **Lv3** `Bash` 로 상태 확인(예: 설치·경로·git 상태) 후 안내.
4. **Lv4** 그래도 안 풀리면 강사/조교 에스컬레이션 안내(추측으로 밀어붙이지 말 것).

### 교시 전환
한 실습이 끝나면 1줄 전환 멘트 + 다음 실습 목표 예고. 수강생이 지치면 "여기까지 하고 다음에 이어서 해도 됩니다"(프로필 저장).

## Phase 3 — Closing (마무리, ~3분)
1. **오늘 배운 것 요약**(3줄 이내, 수강생 언어로).
2. (해당 레슨이면) **저장/커밋** 또는 산출물 정리 안내.
3. **다음 레슨 예고**(레슨 §다음 레슨).
4. **프로필 업데이트**: `completed_lessons` 에 이번 레슨 추가, 어려웠던 점→`struggles`, 잘한 점→`strengths`, `updated_at` 갱신.

## 수강생 응답 패턴 핸들링
- "됐어요/됐다/성공/잘됨" = 성공 → 칭찬 + 의미 설명 + 다음.
- "에러/안돼요/빨간 글씨" = 에러 → 트러블슈팅.
- "잘 모르겠어요/이게 맞아요?" = 확인 요청 → 화면/결과 붙여달라 + 함께 읽기.
- "다 했어요"(빠른 완료 주장) = AskUserQuestion 으로 실제 결과 확인 후 인정.

## ai_level 적응 (pedagogy.md 와 정합)
- **beginner**: 한 문장에 한 단계, 모든 용어 풀이, 매 성공마다 칭찬.
- **intermediate**: 핵심 + "왜", 2~3단계 묶음.
- **advanced**: 목표만 제시, 방법은 수강생 선택.
